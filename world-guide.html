QBS 世界作成ガイド

GM が舞台そのものを自作したいときのための本です。
既存の世界設定を読む本ではなく、QBS の宣言、裁定、シナリオ設計、キャンペーン設計と噛み合う舞台をどう作るかを扱います。
1本の事件の作り方は シナリオ設計ガイド、長い流れの組み方は キャンペーン設計ガイド に分け、本書はその前段で「そもそも、どんな世界なら QBS で遊びやすいか」を扱います。

1. はじめに

QBS で世界を作るとは、百科事典のような設定資料を厚くすることではありません。プレイヤーが「この PC なら今どう動くか」を自然に書けて、GM がその宣言に対して自然に世界を反応させられるよう、舞台の常識、通りやすい手段、起こりやすい代償、長く残る変化を先に答えられるようにすることです。

このゲームは、与えられたコマンドの中から選ぶことを中心にしていません。だから世界設定も、「何の職業なら何の行動ができるか」というメニュー表ではなく、「この世界では、誰が、何を、どんな代価で、どこまでできるか」が読める形で作る必要があります。

良い世界設定は、詳しい世界設定ではありません。読める、動ける、支払える、残せる 世界設定です。

この本で扱う範囲

この本で扱うのは、舞台の土台、世界常識、制度や共同体、移動と通信、異能や技術、代償の傾向、世界反応、Buildsの供給線、Storiesの主菜、公開情報と秘匿情報、そしてそれらをプレイヤーへどう見せるかです。

逆に、今回の事件の問いや期限や主要局面そのものを書くことは シナリオ設計ガイド、長期の節目と本数や難度曲線を決めることは キャンペーン設計ガイド に分けます。本書は、どのシナリオを書いても土台として使える世界の骨格 を作るための本です。

設計の基本線

世界は「読むもの」ではなく「宣言の足場」です。
先に作るべきなのは歴史年表ではなく、PC が知っていてよい常識、通りやすい手段、起こりやすい代償、世界がどう反応するか の四つです。

2. このガイドで使う言葉

この章では、本書でとくに使う語をまとめます。ルール全体の基本用語は必要に応じて GMガイド や各既存ガイドを参照してください。

世界常識

PC が初手から知っていてよいことです。歴史の全体像ではなく、「この町では夜警に逆らうとどうなるか」「この宗派の聖印はどこで効くか」「この港では何が禁制品か」のような、宣言に直結する常識を指します。

制度窓口

許可、紹介、認可、推薦、記録照会、取り調べ、検分、納品受理のように、社会の仕組みに触れる入口 です。QBS の世界では、扉、鍵、怪物だけでなく、この制度窓口が宣言の値段を大きく変えます。

越境コスト

階級、地域、宗派、言語、組織、身分、文化圏をまたぐときに発生しやすい価格です。旅費や危険だけでなく、不信、悪評、監視、誤解、借り、儀礼違反もここに入ります。

世界反応線

PC の成功や失敗を受けて、世界のどこが進むか をまとめた線です。治安、世論、怪異汚染、監査、補給、市場、外交、祭儀進行などが代表例です。

公開情報 / 秘匿情報

プレイヤーに最初から見せる情報と、GM だけが持つ裏側です。ただし QBS では、秘匿情報も完全後出しにしてはいけません。兆候、痕跡、噂、資料、言い回し、制度の歪みとして、読める材料か取得可能な情報源を先に置きます。

情景アンカー

その世界らしさを一行で思い出させる小さな具体物です。食べ物、挨拶、街路の匂い、通貨の呼び名、祈りの文句、制服の傷み方、夜の騒音、記録媒体、見張りの笛などがこれにあたります。

3. このガイドの使い方

この本は、次の三場面で使います。

  1. 卓を始める前に、舞台そのものを立てるとき
  2. PC 作成前に、プレイヤーへ見せる共有情報を整えるとき
  3. シナリオやキャンペーンを書き始める前に、世界の供給線と反応線を確認するとき

設計の基本順は次の通りです。

  1. 題材を 舞台 / 行動 / 脅威 / 代償 / 終了 の5枠へ落とす
  2. PC が最初に立つ位置を決める
  3. この世界で通りやすい手段と、高くつく行動を決める
  4. 制度窓口、共同体、移動、通信を決める
  5. Buildsの供給線と Storiesの主菜を決める
  6. プレイヤーへ見せる世界常識を作る
  7. シナリオやキャンペーンへ渡す

4. 世界作成の基本線

QBS の世界設定は、最低でも次の四問に答えられるようにしてください。

  • 誰が何をできるか
  • 何が高くつくか
  • 何が難しい / 不可能か
  • 何がどう反応するか

どんなに美しい世界観でも、この四つに答えられないなら、QBS では宣言の足場になりません。逆に、この四つに答えられるなら、細部が未完成でも十分に遊べます。

また、各設定要素は、最低でも次のどれか一つの役目を持つようにしてください。

  • 宣言を 許可 する
  • 宣言の値段を 高く / 安く する
  • ある方向を 難しい / 不可能 にする
  • 宣言のあとに 反応 を返す

たとえば「王立図書院」は、蔵書の多い施設という説明だけでは弱いです。QBS 的には、紹介状があれば禁書閲覧を許す、無断閲覧は監査を強める、夜間侵入は物理条件が重い、書記官との関係が Builds になる、学徒や異端審問官との Stories が立つ、ところまで決まっていて、はじめて機能します。

5. まず固定する九点

可動域

まず、世界全図ではなく PC が数回のシナリオで触れる範囲 を決めてください。一地方、一都市、一学園、一艦、一国境地帯でも十分です。外側の大陸や遠国は、必要になってから足して構いません。

例外の所在

魔法、霊能、超技術、祝福、呪い、異種族特性など、常識を破るものがどこにあるかを決めます。大事なのは「何ができるか」より、「誰にとって例外なのか」「何を支払うと使えるのか」です。

移動と通信

QBS は PBM なので、時間経過と距離 が特に重要です。徒歩、騎乗、列車、転移、通信塔、狼煙、使い魔、SNS、監査ログなど、移動と情報伝達の速さを先に決めておくと、期限や余波が自然になります。

権力と窓口

誰が検分し、誰が許可し、誰が逮捕し、誰が見逃し、誰が証人になれるかを決めてください。国家、教会、学校、企業、ギルド、部族会議、自治会、船長会、AI 管制など、制度窓口の所在 がない世界は、宣言が毎回ふわつきます。

生活と不足

食料、燃料、住居、衛生、学費、治療、仕事、埋葬、娯楽、通貨、贈答、身分証、ログ保全。世界に浸らせたいなら、大事件より先に 日常の主菜 を決めてください。人は世界観に、歴史年表よりも生活の手触りで入ります。

脅威の主系統

この世界で PC を止めるものは主に何かを決めます。武力、怪異、監査、階級、環境、露見、誤情報、資源枯渇、共同体圧、世論などです。脅威が曖昧だと、代償も余波も世界反応も曖昧になります。

代償の主菜

この世界では、何を支払わせると「らしい」かを決めます。剣と馬の世界なら負傷・借り・悪評、学園なら噂・停学危機・関係悪化、SF なら酸素・ログ・汚染・受取不合格のように、題材と噛み合う価格を先に選びます。これは GMガイド の「代償の傾向」を世界単位へ広げる作業です。

Buildsの供給線

この世界では、装備以外に何が長期資産になりうるかを決めます。工房、紹介網、制度資格、手順書、儀式場所、村落共同体、航路、隠れ家、研究許可、祭祀の役目などです。QBS の Builds は装備だけでなく、長期資産ノードや Build 素片として残るので、世界側にもその種が必要です。

Storiesの主菜

旅情、人情、激情、愛憎のどれを主菜にするかを決めます。世界そのものが、どの関係を壊しやすく、どの関係を深めやすいかです。同じ「王都」でも、旅情の王都と愛憎の王都では、見せる店、会話、夜景、噂、政治の切れ味が変わります。

6. プレイヤーに最初に見せるもの

世界観に浸らせたいなら、歴史を先に配るより、PC が最初から知っていてよいこと を先に渡してください。プレイヤーは、自分が何を知らないかより、自分の PC が何を知っていて、何を当然と思っていてよいか がわかったときに動きやすくなります。

物語傾向:

【通りやすい手段】
【起こりやすい代償】

ここで大切なのは、設定を減らすことではありません。最初に必要な設定を先に出す ことです。

たとえば「この地方では夜の橋を一人で渡らない」「港の検分補佐資格は下層街では強いが上流区では弱い」「遺体を焼くか沈めるかで宗派が割れる」といった情報は、年表より先に役立ちます。

7. 宣言を支える設定要素の書き方

世界の各要素は、次の書式で一度メモしておくと QBS と噛み合いやすくなります。

【設定要素メモ】
名称:
表の顔:
何を許すか:
何を高く請求するか:
何が難しい / 不可能か:
何が反応するか:
PCが最初から知っていてよいこと:
秘匿しておくこと:
Builds の種:
Stories の種:

場所

場所は、情景だけでなく できること / しにくいこと / 監視されること を持たせてください。「貧民街」なら、隠れるのは安いが公的手続きは弱い。「聖堂」なら、治療と保護は通りやすいが流血や窃盗は重く返る。そう書ける場所は、宣言の足場になります。

制度窓口

制度は、難しくするためだけに置かないでください。紹介、推薦、検分補佐、学籍、聴聞、戸籍照会、貨物受理、埋葬許可のように、通すための入口 と 失敗したときの反応 を両方持たせます。

集団と共同体

共同体は、単なる背景ではなく 関係の価格表 です。どの集団に属すると守られやすく、どの集団を怒らせると長く響くかを決めます。Builds なら紹介網、Stories なら恩義や偏見、Quiddities なら勢力図の変化になります。

異能と技術

魔法や超技術は、万能にしないでください。大事なのは派手さより、許可・制約・副作用・痕跡 です。使えたあとに何が残るかが曖昧だと、世界が宣言に追いつきません。

移動と通信

旅、護送、潜入、逃走、納品、救助を遊ぶなら、移動と通信は舞台装置ではなく中核です。どれくらいで人が来るか、どれくらいで噂が回るか、どれくらいで監査が届くかを決めてください。それだけで、期限と危険予告が自然になります。

8. Quiddities / Builds / Stories で世界を見る

世界設定は、三方向に返りが出るように作ると QBS と噛み合います。各シナリオでそうしているなら、世界作成はその「母体」を作る作業であるべきです。

Quiddities としての世界

勢力図、階級、宗派、法、季節、怪異汚染、開拓、物流、継承、戦線、外交。これらは「舞台説明」ではなく、宣言によって前進しうる世界状態 です。世界作成では、「何が変わると、この世界の答えが変わった感じになるか」を先に選んでください。

Builds としての世界

長期資産ノードは、武器と道具だけではありません。制度資格、共同体の信頼、工房、航路、研究手順、祈祷の型、検分補佐権、隠し倉庫、伝令網も Builds です。世界がそれを供給できないと、PC の成長が装備一辺倒になります。

Stories としての世界

家族、恩義、同門、偏見、婚約、破門、借り、贖罪、執着、土地への帰属。これらは個人設定ではなく、世界が強くしてくる関係 です。どんな世界なら義理が効き、どんな世界なら裏切りが重く、どんな世界なら和解が起きやすいかを決めてください。

世界課題タグ

世界課題タグは、「ちょっとした雑事」ではありません。Builds の足場を増やし、Stories の関係を動かし、Quiddities の世界状態を小刻みに前進させる供給線です。世界作成の時点で、1つの地域に 1〜4 本くらい、見える世界課題タグの候補を置いておくとシナリオが立ちやすくなります。

9. 世界観に浸らせる見せ方

PBM では、設定をどれだけ作ったかより、1投稿でどれだけその世界の空気を思い出させられるか のほうが重要です。世界の現在状態を毎回再掲しやすい形で作らせたほうが安定します。

1投稿に置く情景アンカーは、次の五つくらいで十分です。

  • その場らしい 具体物
  • その世界らしい 言い回しや習俗
  • いま効いている 制度や身分の圧
  • 放っておくと困る 代償の匂い
  • PC が見て取れる 世界の動き

たとえば「王都の西門」なら、ただ石壁と衛兵を書くのではなく、砂糖菓子を売る露店が検分列の外へ追いやられている、通行札の色で身分が見える、夜警の槍先より書記官の帳面のほうが怖い、噂は靴屋から兵舎へ流れる、雨で封蝋がにじんでいる、といった具体へ落とすと、世界設定が宣言の足場になります。

固有名詞の出しすぎに注意する

世界に浸らせたいときほど、固有名詞を増やしすぎないでください。一度の投稿で出す新語は少なめにして、役目のある名前だけを出す ほうが覚えられます。覚えられない固有名詞は、世界観ではなくノイズになりやすいです。

秘密は辞典ではなく兆候で見せる

「この世界には実はこういう真相がある」と辞典で抱え込むより、見張りの配置、伝承の穴、帳簿の抜け、祈りの言い淀み、遺体処理の不自然さのような、プレイヤーが宣言で触れられる形 で置いてください。

10. シナリオ設計・キャンペーン設計への接続

世界作成が終わったら、次はその世界から どんな事件が自然に生えるか を見ます。シナリオは世界の外から落ちてくるものではなく、世界の緊張、制度の歪み、共同体の欲望、異能の副作用、物流や季節の詰まりから切り出されるべきです。

世界からシナリオへ渡す最小メモは、次の形で十分です。

【世界 → シナリオ切り出しメモ】
今回よく効く制度窓口:
今回よく効く共同体:
今回の脅威主菜:
今回らしい代償:
今回らしい余波:
今回使える Build 素片候補:
今回動かしたい世界反応線:
今回深めたい関係 / 感情:

キャンペーンへ渡すときは、さらに次を持たせます。

【世界進行メモ】
長期で動く勢力:
長期で残る課題:
長期で伸びる Builds 供給線:
長期で壊れやすい関係:
節目で変わる常識:
次に重くなる地域 / 季節 / 制度:

キャンペーン設計ガイド が前進する回 / 回収する回 / 回復する回 / 反転する回 / 決着する回を分けるなら、世界作成の段階でも「この世界のどこが回復に向き、どこが反転を起こしやすいか」を持っておくと長編が自然になります。

11. よくある失敗

1) 世界が百科事典になる

詳しいが、宣言に必要な情報が見えません。まず可動域と PC 常識を削り出してください。

2) 世界がメニュー表になる

「この階級はこれだけ」「この職業はこの三択だけ」と決めすぎると、QBS の自由宣言とぶつかります。制限するなら、行動の種類ではなく 価格と反応 で制限してください。

3) 制度窓口がない

誰が許し、誰が止め、誰が受け取り、誰が記録するかが曖昧だと、社会的な宣言が毎回 GM の気分に見えます。入口と反応を先に置いてください。

4) 不可能ラインが後出しになる

「実はその魔法は効きません」「実は平民は一切入れません」が後から出ると、世界が閉じます。難しい / 不可能なことは、最初の公開情報に含めてください。

5) Builds が装備だけになる

長期資産ノードが武器と道具しかないと、関係や制度や共同体が世界から切れます。資格、工房、手順、紹介網、拠点を足してください。

6) 世界が反応しない

成功しても失敗しても景色だけが変わらない世界は、QBS では止まって見えます。最低でも一つ、世界反応線を前進させてください。

7) どこへ行っても同じ手触りになる

街も寺も学園も艦内も、通る手段・払う代償・返る余波が同じだと没入が薄れます。場所ごとに「安いこと」「高いこと」「怒るもの」を変えてください。

13. 最後に

世界を大きく作りすぎる必要はありません。QBS で本当に必要なのは、プレイヤーが自然に宣言できるだけの世界 と、GM が自然に反応を返せるだけの世界 です。

その世界で何を失い、何を守り、何を持ち帰れるのか。それが見えるなら、世界はもう十分に生きています。
