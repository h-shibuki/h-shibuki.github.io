QBS キャンペーン設計ガイド

複数本のシナリオを束ねて、PCの野望が長く変化していく流れを設計するGM向けの本です。
野望度終点、想定シナリオ数、節目、持ち越し資源、難度曲線、長期の世界反応、進行ログ、公平性チェックを扱います。単発1本そのものの構造作りは シナリオ設計ガイド に分けています。
モード（選択ルール）の配置基準は、別冊の 選択ルール集 に集約しています。

1. はじめに

キャンペーンとは、PCが野望へ近づいていく長い道のりを設計することです。
QBSでは、各回のシナリオを面白くするだけでなく、何を持ち越し、いつ節目を置き、どの問いへ向けて積み上げるか を文書として持っておくと安定します。

この本は、シナリオ設計ガイド で作った単発シナリオを、長い流れの中へ位置づけるための本です。
すでにある単発を並べるだけではなく、どの回が何を進め、どの回が回復・反転・決着の役目を持つかまで整理します。

この本で扱う範囲

この本で扱うのは、野望度終点と想定本数、シナリオ列の節目、持ち越し資源、回復ペースと難度曲線、長期のNPC / 勢力 / 世界反応、単発の組み込み、分岐・中断・打ち切り、進行ログ、状態管理表、公平性チェックです。

逆に、各回の問い・条件・期限・主要局面・構造難度・作者メモの細部 は シナリオ設計ガイド、卓等級と四軸の定義、および裁定運用 は GMガイド、各モードの体験差 は 選択ルール集、舞台そのものの土台 は 世界作成ガイド に分けています。既存資料を回すだけなら GMガイド を先に読んでください。

設計の基本線

長い流れでは、毎回全部を進めない
各回の役目を分けて、前進する回 / 回収する回 / 回復する回 / 反転する回 / 決着する回 を置きます。
各回の面白さはシナリオ設計で作り、全体の呼吸はキャンペーン設計で作ります。

旅と冒険の呼吸も節目へ織り込む

QBS のキャンペーンでは、前進だけでなく なぜそこへ行きたいのか、どこで自由を感じるのか、何を持ち帰ると物語になるのか も設計しておくと没入が強まります。長い流れでは、移動や滞在を単なるコストではなく、発見 / 文化接触 / 挑戦・越境 / 回復兼仕込み / 畏敬 のどれかとして扱うと、旅情と達成感が両立します。

各アークに少なくとも一つ、反復して訪れたくなる回復点、心を打つ畏敬点、安全ルート / 近道 / 地元ルートの選択 を置いてください。これでプレイヤーは「与えられた事件を解く」だけでなく、自分の意思で世界を歩いている感覚 を得やすくなります。

キャンペーンでは Q / B / S を積み上げる

キャンペーン設計では、各回の問いだけでなく、各回で何が新たに真になったか、そのうち何を次回以降も保持したか、それがだれとの関係の中でどう変わり始めたか を並行して管理します。

Quiddities は勢力図・秘密・世界状態・立場・次回の判断材料として積み上がります。Builds は保持タグ・保持ノード・素材だけでなく、関係、感情、悪評、人物履歴のような 保持された焦点 も含みます。Stories は、それらが約束、対立、赦し、執着、継承、断絶へどう変質していくかの連続です。

同じ一件が三つ全部に残ってよい と考えると、キャンペーンの手応えが強くなります。単発を連ねるだけではなく、前進する回 / 回復する回 / 反転する回 / 決着する回 のそれぞれに、三要素のどこが主に動く回なのかを持たせてください。

長編の欲求プロフィールを先に置く

長編では、何を主菜にするか を最初にそろえておくと、各アークの役目がぶれにくくなります。ここでいうプロフィールは、タスク / 物語 / 冒険 / 癒し / 絆 / 非常識 を同格で並べ、どう終わると満足しやすいか、何が残ると満足しやすいか、失敗をどう返すか を一言で固定するものです。

【長編の卓欲求プロフィール】
長編の主菜:
長編の副菜:
どう終わると満足しやすいか:
何が残ると満足しやすいか:
失敗をどう返すか:
避けたい返し方:

この欄があると、同じ 5 本キャンペーンでも「毎回事件を解く長編」か「関係の変化と余韻を追う長編」かを、アークの切り出し前にそろえやすくなります。

見本のシーズン・ログライン

長編でも、最初に「この季節はどんな物語か」を一文で言えると継続の引力が強くなります。見本は次のようなものです。

  • 「沈みゆく巡礼街道で、最後の渡し舟をめぐる旅人たちは、救う相手と捨てる約束を選び直していく。」
  • 「神託監査が厳しすぎる王都で、若い実務官たちは信仰を守るか、制度を守るかの板挟みになっていく。」
  • 「代償なしの奇跡が漏れ出す黒い泉を追って、辺境の記録官たちは世界の規則そのものに穴が開く瞬間へ近づいていく。」

2. このガイドで使う言葉

この章では、この本で多用する長期設計用語を、正式名称にそろえてまとめます。ルール全体の用語は必要に応じて GMガイド を参照してください。

アーク

キャンペーンの中で、ひとまとまりの転機を持つ区間です。立場の変化、世界反応の反転、保持の完成、関係の決着などで切ります。

節目

野望の進行や立場の変化が明確に起こる回です。情報が増えた回ではなく、以後の判断基準が変わる回として置きます。

持ち越し要素

次回以降の判断と世界反応を変え続ける要素の総称です。QBS では、いったん真になったことのうち 何を保持するか を、課題タグ、手札タグ、履歴、人物履歴、保持ノードなどの形式に分けて管理します。

手札タグ

使えば有利になる持ち越し資源です。複数回に効く道具、口利き、協力関係、抜け道、後ろ盾などを置きます。

履歴

タグではない事実記録です。用途未定のまま残し、あとで局面や人物や世界に返す余地を持たせます。

旅履歴

旅や探索の持ち帰りを記録する履歴です。景、人、風習、発見を短く残し、次回以降の世界や資産や物語へ返します。

人物履歴

人物がどう見えるようになったかを残す、人物像に関する 保持の表示名です。関係、印象、秘密、矛盾、傷、輪郭の更新を書きます。

保持ノード

Build のうち、設備、肩書、拠点、組織的後ろ盾、独自手順など、台帳で管理するもの を 1 件ずつ扱う単位です。

素材

将来の工作や昇格の種として読む Build です。多くはタグやノードで保持し、部材、手順、縁故、正当性、場、証左を含みます。

レギュラーNPC / 継続要素

キャンペーンをまたいで出入りする人物や要素です。立場、継続する利害、再登場時の圧力源をひと目で追える形にします。

キャンペーン要約

ここまでに何が起き、何が残り、次に何を問うかを短くまとめた総覧です。長くなった卓ほど効きます。

切り出しシート

長いキャンペーンから次のシナリオへ、何を切り出すかを整理するメモです。野望、残り方、持ち越し要素、圧の負債、次回の問いを並べます。

圧縮終了モード

卓を短く締めるために、残った線をまとめて決着へ寄せる運用です。未消化の線を増やすのではなく、答えの形を先に決めます。

高揚配線

終盤へ向けて、感情・期待・因縁・帰還線をどの順で持ち上げるかの配線図です。節目回の熱量を揃えるのに使います。

圧の負債帳

まだ請求していない圧を一覧で持つ帳面です。監査、悪評、義務、借り、追跡、未回収の約束などを、どこで返すかまで見ます。

QBS の三要素の長期管理で使う語

系統長期で見るもの正式語
Quiddities世界や立場として何が変わったか世界課題タグ / 世界常識 / 制度窓口 / 偶発フック / 秘密開示
Builds何を保持し続けるか素材 / 保持ノード / 保持台帳 / 関係表示 / 感情表示 / 火種表示 / 人物履歴

3. このガイドの使い方

この本は、キャンペーンの全体骨格を決めるとき、各回へ切り出すとき、進行中に破綻を直すときの3場面で使います。

やりたいこと先に読む章
最初に全体の長さと節目を決める4章 → 5章 → 22章
進行中の持ち越しと難度曲線を整える6章 → 7章 → 17章 → 19章 → 21章
次の1本へ切り出す22章 → 14章 → シナリオ設計ガイド
中断や打ち切りを畳む19章 → 21章 → 23章

設計の基本順

  1. 野望度終点と想定本数を決める
  2. 節目と最後の問いを置く
  3. 長期の持ち越し資源と世界反応を決める
  4. 難度曲線と回復ペースを決める
  5. 次の1本へ切り出すためのシートを書く
  6. 進行ログと公平性チェックで保守する

4. 野望度終点と想定シナリオ数の設計

まず各PCの野望を見て、どこまで行ったらその野望を一区切りとみなすか を決めます。QBSでは、ここを曖昧にしたまま各話を書き始めると、面白い単発はできても長い物語の進み具合が読めなくなります。

目安は 1シナリオで野望度が1変化する です。ただし、毎回必ず上がる必要はありません。回復回や足場作りの回では、野望度が動かずに 次回のための手札や立場だけが増える こともあります。

想定本数向いている題材野望度終点の目安特徴
1〜2本単発 / 導入卓 / テスト運用2直近の目標をかなえる短距離走
3本短編キャンペーン3〜4導入・反転・決着の三幕が作りやすい
5本長期キャンペーン4〜6回復回と中盤反転を置きやすい
8本以上長編 / 複数勢力戦6以上複数アークと持ち越し管理が必須

終点を決める基準

  • 立場の回復や獲得 が完了したとき
  • 最後に問いたい価値選択 へ到達したとき
  • 主要な敵対勢力との関係 が決着したとき
  • PCの野望の文言 自体を書き換える必要が出たとき

本数が伸びすぎるとき

終点が遠すぎると、毎回の達成感が薄くなります。
本数が伸びすぎると感じたら、野望を二段に分けて、前半の終点 を置いてください。たとえば「国を救う」では広すぎるなら、「国政に関わる立場を得る」を前半終点にします。

リラックス・モードでは本数ではなく見直し間隔を決める

リラックス・モードを使うなら、全体本数の固定 より 何本ごとに見直すか を決めてください。おすすめは 2〜3本ごと の小さな見直しです。

  • 見ること: まだ続けたいか、通常キャンペーンへ戻したいか、今の候補の重さが合っているか
  • 動かすこと: 野望度ではなく、次回候補の配分、回復回の頻度、持ち越し要素の整理
  • 決めすぎないこと: 最終局面の時期

このモードでは、終点の予約 より 節目ごとの温度確認 のほうが機能します。

5. シナリオ列の節目設計

各シナリオは、ただ並ぶのではなく、何が進む回か と 何が変わる回か を持つべきです。節目は「大事件の回」だけではありません。情報公開の意味が反転する、協力者が レギュラー化する、撤退不能になる、価値選択の前提が変わる、といった変化も節目です。

シーズン・ログラインと次回予告を先に置く

内部設計が強いほど、公開側では一文の見せ方が必要です。各アークやシーズンごとに、一文要約、未解決の問い、中盤反転の約束、終盤で感情的に回収するもの、関係フック、次回予告文 を先に作ってください。

【公開用シーズン・ログライン】
シーズンの一文要約:
今アークの未解決問い:
長期で揺さぶる倫理的ジレンマ:
中盤反転の約束:
終盤で感情的に回収するもの:
関係フック:
次回予告文:

次回予告は「何が起きるか」を全部言う欄ではなく、「何がまだ片付いていないか」を短く見せる欄です。予告文化があると、長編の入口と継続意欲が両方太くなります。

節目は「何が進むか」だけでなく「どんな旅になるか」でも分ける

節目設計では、事件の役目と同時に、その回がプレイヤーへ何の旅情を返すかも決めておくと波が出ます。

  • 発見回: 新しい土地・風景・常識を知り、Quiddities を増やす回
  • 文化接触回: 礼、作法、贈答、証人、埋葬、通行の違いが Stories と価格を変える回
  • 挑戦・越境回: ルート選択、越境条件、退路確保で自己効力を問う回
  • 回復兼仕込み回: 休息しつつ Build を増やし、次の自由選択肢を作る回
  • 畏敬回: 絶景、聖地、儀式、美しい瞬間が意味と方向を変える回

一つの回が複数の型を兼ねてもかまいませんが、同じ型だけが続かない ようにすると、長期キャンペーンでも「旅している感覚」が痩せにくくなります。

節目の種類

  • 導入節目: 題材と主軸を固定する
  • 足場節目: 支援者・設備・信用などの持ち越しを得る
  • 反転節目: これまでの勝ち筋がそのままでは通らなくなる
  • 代償節目: これまでの蓄積が重さを持って返る
  • 決着節目: 最後の問いへ答える

最後に問うこと

最後に問うことは、達成だけでなく 何の意味が反転するか、誰との関係が決定的に変わるか まで書いてください。終盤が「事件を解いた」で終わらず、「この旅で何者になったか」に届きます。

長期の倫理的ジレンマを通奏低音にする

単発では一回の価値選択でも強く機能しますが、キャンペーンでは同じ価値対立が別の顔で何度も返ってくると、プレイヤーの判断が物語そのものになります。たとえば「大義を優先するか、目の前の一人を守るか」「真実を公表するか、共同体の安定を守るか」「法的な正しさを取るか、感情的な救済を取るか」といった軸を、アークごとに違う事件へ埋め込んでください。

大事なのは、同じ問いを機械的に繰り返すことではありません。序盤では理念として語られていた対立が、中盤では大切な相手との関係に降りてきて、終盤では自分の野望そのものと衝突する、というふうに距離と痛みを変えながら再登場させることです。そうすると、キャンペーン全体に「今回は何を解くか」だけでなく、「この人は最後に何を選ぶ人だったのか」という芯が通ります。

アークの高揚配線を先に置く

【このアークの高揚配線】
- 開く: 世界拡張 / 関係変容
- ひっくり返す: 再文脈化
- 押し切る: 逼迫突破
- 最後に深まるもの:
- 崇高化ポイント:
- 蓄積からの解放ポイント:

同じ節目設計でも、開く → 反転する → 押し切る → 深まる の順を意識すると、長編の波が強くなります。

キャンペーン終盤でPCへ何を問うかを、できれば最初に一文で置いてください。
途中の各回は、最終局面そのものを先送りするためでなく、最後の問いに必要な視点や関係を積み上げるために存在します。

節目設計の短い型

導入: 題材・主軸・初期関係を固定する
中盤前: 最初の足場または最初の代償を置く
中盤: 勝ち筋の反転または別利益の重みを見せる
終盤前: これまでの履歴をまとめて返す
終盤: 最後の問いへ答える

各回でプレイヤーが火種表示から 開く / ひっくり返る / 押し切る / 深まる を要求できる卓では、この高揚配線はその受け口にもなります。アーク全体で どの要求をどこまで返せるか を先に決めておくと、単発の名場面が長編の波へ乗りやすくなります。

6. 保持ノード / 手札タグ / 履歴 / 大切なものの持ち越し管理

キャンペーンでは、毎回すべてを持ち越す必要はありません。何を持ち越し要素として残し、何を履歴として残し、何をその回だけで閉じるか を先に決めたほうが、管理が軽くなります。

項目長期へ残す基準閉じてもよい基準
持ち越し要素次回以降の判断や世界反応を変え続ける今回限りの軽い余波で終わる
手札タグまだ使っていない支援・資源・協力者使用済みで効果が終わった
履歴事実として参照され続ける出来事参照価値が薄く今後の裁定に効かない
大切なもの継続的に命運点や行動原理へ影響する意味が完全に変質し、差し替えが必要

持ち越し要素の粒度

持ち越し要素は、毎回細かく増やしすぎると管理不能になります。
似た方向の要素が増えてきたら、統合した1枚の持ち越し要素 へまとめてください。たとえば「監視が増えた」「噂が広がった」「身元照会が厳しくなった」を統合して「当局の警戒強化」にしてよいです。

手札と履歴の違い

まだ使えるもの は手札、もう起きた事実 は履歴です。協力者の信頼を得たなら、その協力者は手札であり、同時に「助けてもらった」という事実は履歴です。どちらも残すか、片方だけ残すかは次回の裁定でどちらを使うかで決めます。

大切なものの持ち越し

大切なものは、野望や命運点の読み方を変える核です。キャンペーンの途中で意味が変わったら、何が失われ、何が守られた結果その意味が変わったのか を履歴に残したうえで差し替えてください。

人物履歴の持ち越し

キャンペーンでは、「何をしたか」だけでなく、「その結果、その人物がどう見えるようになったか」も持ち越すと強くなります。人物履歴 は開始時に空欄でも、GMにおまかせ でも構いません。大事なのは、回をまたいで人物の輪郭が残ることです。

人物履歴は、雰囲気メモではなく 人物像に関する Build です。だれかを救うために何度も無理を押したなら 傷 / 欠落 や 主類型 がはっきりし、周囲を利用して自分だけ安全圏へ抜けたなら 裏面 や 秘密 や 矛盾 が濃くなります。善行も悪行も、その場の達成で消さず、履歴・関係・長期課題・秘密開示 のどこかへ返る線として残してください。

保持を台帳化するときの表示ラベル

キャンペーンでは、保持を台帳化するときに表示ラベルで整理すると管理しやすくなります。

  • 手順 / 制度: 宣言の射程と値段を変える

7. 回復ペースと難度曲線

キャンペーンでは、毎回同じ重さだと平坦になります。
回復しやすい回、別利益の競合が強い回、世界反応が重い回、複数PCの噛み合わせを問う回を散らして、何を難しくする回なのか を回ごとに分けてください。

難度曲線の考え方

  • 前半: 足場を作る回と軽い代償回を混ぜる
  • 中盤: 勝ち筋の反転や別利益の競合を強める
  • 終盤: 持ち越し要素と履歴の返りを重くする

回復回の役目

回復回は、ぬるい回ではありません。
大きな前進ではなくても、手札の整理、関係の修復、休息、再配置、次の仕込み をするための回です。回復回がないと、後半の代償がただの消耗戦になりやすくなります。

回復回は「次の仕込み」を主目的にしてよい

キャンペーン、とくに卓等級が上級以上では、回復回の本体は傷を消すことではなく、次の局面で役に立つ手札タグをどれだけ仕込めるかです。
休息、関係修復、設備点検、慰安旅行、祭礼参加、里帰り、取材旅行のような表向きの目的を置きつつ、実際には 協力者づくり / 補給線整備 / 許認可取得 / 土地勘獲得 / 監査逃れ / 外交線の確保 を進める回として設計してください。

例: キャンペーンのゴールが「国を救う」なら、終盤で外国から横やりが入らないよう、前もって友好国との関係を作っておく回を置けます。表向きは「疲れを癒すための慰安旅行」でも、裏では 現地の重鎮と知己を得る / 港湾利用の便宜を取る / 緊急時の亡命路を押さえる といった手札タグを仕込む回になります。

回復は結果、仕込みが本体 と考えると、後半で効いてくる回が作りやすくなります。回復回で何も仕込めないと、長編はただの消耗戦になりがちです。

難度ごとの「仕込み気づき」支援

難度帯GMが表に出してよい支援
初級「この回では外国の有力者と会える」「補給線を整えられる」など、仕込める手札タグの例を候補文や既知情報へ書いてよいです。
中級「交流行事がある」「現地の工房を見学できる」「監査担当が同席する」など、使えそうな接点は見せますが、どの手札になるかは言い切りません。
上級表向きの目的だけを主に見せ、仕込みに関わる手掛かりは 招待客名簿・地図・噂・既知情報 に散らします。裏の目的は説明しません。
熟練回復回であることだけ見せ、仕込みの方向はほぼ教えません。休息をどう手札タグへ変えるか自体がプレイ になります。

圧の負債帳を残す

各話の終わりに、まだ解放していない圧を記録してください。長編では「良い蓄積」がログの底へ沈みやすく、回収しないまま忘れると終盤が痩せます。

【圧の負債帳】
- 未回収の借り:
- 未処理の追加利益:
- まだ来ていない救援:
- 再訪で意味が変わる回復点:
- 再訪で意味が変わる畏敬点:
- まだ解放していない伏線:

予想外の方向から揺さぶる回を必ず混ぜる

QBSでは宣言は基本的に成功するため、PCの得意分野にだけ合わせた課題を出し続けると、勝ち筋が固定化して飽きやすくなります。キャンペーンには、全PCにとって不得手な方向から来る回 を意識的に混ぜてください。

たとえば、億万長者を目指して金融戦で勝ってきたPCたちに対して、世界恐慌による通貨価値の崩壊、国家破綻、国債の紙くず化、法制度の急変、外交圧力、災厄 のような「これまでの勝ち筋がそのままでは通らない」事件を入れる、という考え方です。重要なのは、過去の成功を無効にすること ではなく、成功の意味を変えること です。財産そのものは消えなくても、換金性、正当性、護衛、流通、同盟が新しい問題になります。

  • 初級でも、キャンペーン全体で 1本 はこうした回を入れてかまいません。
  • 中級以上では、長編なら 1〜2本 を目安に、得意分野外の回を置くと波が出ます。
  • 上級 / 熟練では、理不尽に見える事件でも、複数の局面と読める痕跡 を置いておけば成立します。答えを配らず、対応の余地だけを残してください。

理不尽イベントは、一文で全部を失わせる処理 にしてはいけません。必ず、観測できる兆候 / 取得可能な情報源 / 手札を仕込む余地 / 立て直しの局面 を持たせてください。

回復点と畏敬点を反復させる

回復回は消耗の穴埋めだけでなく、この世界へ戻ってきたくなる場所 を育てる回です。キャンペーン設計書には、反復して使う 回復点 と 畏敬点 をそれぞれ置いてください。

  • 回復点: 宿、工房、港、寺、温泉、夜市、隊商宿のように、関係修復・補給・再訓練・相談がしやすい場所
  • 畏敬点: 絶景、聖域、巨大建造物、祭礼、巡礼路、静かな高台のように、価値選択や決意を更新しやすい場所

回復点では 非戦闘の共同作業 を一つ置くと、共同性が太くなります。畏敬点では 次に何を持ち帰りたいか を言語化できるようにすると、承認や自己像の軸も育ちます。どちらの回でも、最後に 次回の自由選択肢を増やす Build を一つ仕込んでください。

難度曲線の簡易表

第1話: 題材提示 / 軽い代償
第2話: 足場獲得 / 小さな回復
第3話: 反転 / 別利益の競合
第4話: 回収 / 追い込み前の整理
第5話: 決着 / 長期の返りをまとめて受ける

劇的展開度の合わせ方

劇的展開度は独立した軸です。最初に迷ったときは、次の対応から決めると合わせやすくなります。

卓等級 / 裁定傾向合わせやすい劇的展開度

中級 / 適正人情
上級 / 峻烈激情

旅情なら回復回や発見回を厚くし、怨情なら対立回や関係破局回を厚くします。別軸指定もできますが、まずはこの対応から考えるとぶれません。

成長ループを野望度から切り離す

野望度は人生の到達度です。Builds の快感は、別に 成長点 で支えたほうが安定します。

  • 野望度: 節目、立場の変化、人生の到達度
  • 成長点: 小さな依頼、世界課題、工作、探索、地道な積み上げ

この二段構えにすると、野望は大きく、ビルドは細かく 育てられます。

Builds ループ を長期管理表へ書くときは、成長点の目安だけでなく、固定候補 / 開設候補 / 昇格候補 / 接続候補 / 圧縮候補 / 素材の供給線 を持たせると、やり込み欲が切れにくくなります。

8. 選択ルールの配置方針

各モードの配置基準と長編での扱いは、別冊の 選択ルール集 にまとめました。この章では、キャンペーン設計書へ最低限書いておきたい確認事項だけを置きます。

観点長編で先に決めること
どの回に置くか常時採用か、特定アーク限定か、節目回だけかを決めます。
何を強めるか物語の残り方、宣言タイミング、運、能力値、交渉、戦術、構造読解、競合、秘匿、旅情のどこを強めたいかを明確にします。
何を残すかQ / B / S のどこへ返りを積むか、長期台帳の欄を先に用意します。
参照先各モードの詳しい採用基準と配置例は 選択ルール集 を参照してください。

モードは「毎回全部入れるもの」ではありません。長編で機能させたい回の手触りに合わせて、必要なものだけを置くのが基本です。

9. レギュラーNPC / 勢力 / 世界反応の長期管理

再登場するNPC、勢力、場所、制度は、毎回シナリオ本文へ書き散らさず、キャンペーンの共通管理欄 にまとめておくと再利用しやすくなります。誰がPCに好意的か、誰が監視しているか、どの勢力がどの失敗を学習したかは、長期の歯ごたえに直結します。

旅の顔になるレギュラーを置く

再登場NPCは黒幕や協力者だけでなく、世界を歩く感触を作る顔 も重要です。宿主、案内人、検問係、船頭、工房主、商隊長、司祭、地図屋、巡礼仲介人のような「通す人 / 休ませる人 / 値段を変える人」を混ぜてください。

  • 通す人: 越境条件、礼、資格、紹介で移動の価格を変える
  • 休ませる人: 回復点の空気を作り、Stories を進める
  • 残す人: 名前、歌、記録、噂、称号として PC の旅を世界へ残す

管理欄に持つ最低項目

  • 現在の立場
  • PCへの態度
  • いま欲しているもの
  • 前回からの更新
  • 次に動く条件

重要NPC / 再登場予定NPCには、次章の推奨拡張欄も足すと、人物ドラマと長期反応を同時に管理しやすくなります。

世界反応を止めない

世界反応は、PCの行動を罰するためではなく、世界が生きていることを見せるために使います。
主目的が通った回でも、最低1系統は世界が動く と決めておくと、長期の流れが止まりません。

世界反応は旅路も動かす

長期の世界反応は、政争や敵対だけでなく、物流、祭礼、巡礼、検問、港の閉鎖、航路の変化、季節、風評 としても返してください。行き先の価値が変わり、同じ土地でも 安全ルート / 近道 / 地元ルート の意味が変わると、移動そのものが次回候補になります。

勢力線を増やしすぎない

長編では勢力が増えがちですが、同時に強く動かす線は2〜3本で十分です。増えすぎたら、脇線は「反応を返すだけ」に格下げし、主要勢力だけを毎回動かしてください。

10. 心に残るレギュラーNPCの作り方

QBS の再登場NPCは、設定資料の厚みで印象に残るのではありません。PC の宣言の経路・価格・余波を変える関係 として戻ってくるから残ります。背景設定だけを足すより、誰を通すと安くなるか / 誰に見つかると高く返るか / 誰を守る・利用する・捨てると別利益が動くか を先に決めてください。

再登場NPCは、背景設定ではなく、PC の宣言の経路・価格・余波を変える関係として作ります。
心に残る人物 は、普遍型を持ち、傷・欠落・秘密・矛盾のどれかを抱え、再登場のたびに 関係か立場を変える人物 です。

主類型 1つと裏面 1つで作る

類型は、人物を平板にするためではなく、初見のわかりやすさ と 再登場後の増幅点 を作るために使います。重要NPCは、主類型を1つ と、必要なら 副類型 / 裏面を1つ だけ持たせると扱いやすくなります。

世界観は人物を通して覚えさせる

没入は土地の説明だけでは足りません。重要NPCには、その人物らしい具体物 / 言い回し / 制度や身分の圧 / 代償の匂い / その人物を通して見える世界の動き を1つずつ持たせてください。これが 世界アンカー です。世界設定は、人を通した手触りで記憶されます。

旅の顔にする NPC なら、宿、橋、港、検問、夜市、工房、聖地のどれを通して覚えさせるかまで先に決めてください。人物と場所が結びつくと、「また会いたい」「また行きたい」が同時に生まれます。

再登場ごとの更新ルール

再登場させるたびに、同じ態度や役目のまま使い回さず、関係か立場のどちらかが必ず変わる ように管理してください。

再登場モードを先に決める

再登場NPCは「また出る」だけでは弱いです。戻るたびに 再会 / 和解 / 裏切り / 救援 / 継承 / 鏡像化 のどれで関係か立場が変わるかを先に決めると、同じ人物でも毎回違う熱が出ます。

同じNPCが同じ態度のまま三度出ると、便利でも印象には残りにくくなります。再登場させるなら、前回から次のどれか一つは必ず変えてください。

  • 態度 が変わる
  • 欲しいもの が変わる
  • 秘密の段階 が進む
  • 世界からの立場 が変わる

再登場時には、前回その人物が得たものと失ったもの が見えるように戻すと、PC の選択の重みも残ります。味方になったとしても、負い目や監視や別の板挟みまで消えたことにはしないでください。

11. 情報公開計画と長期伏線管理

単発シナリオに伏線管理表があるなら、キャンペーンではその長期版が必要です。何を早期に見せ、何を中盤で反転させ、何を終盤まで確定させないかを決めておきます。

三段階の公開

  • 早期公開: 題材と危険の種類を読む材料
  • 中盤公開: これまでの理解を反転させる材料
  • 終盤公開: 最後の問いへ答えるために必要な材料

再登場NPCの秘密を長期で扱うなら、初登場では兆候、中盤で輪郭と板挟み、節目で代償 / 当事者、終盤で真相と関係変質 を出す順にすると散りにくくなります。秘密は驚かせるためだけでなく、PC の判断基準を変えるため に公開してください。

景色の更新も情報公開に含める

情報公開は真相だけでなく、見える景色がどう変わるか でも設計します。初見で惚れさせる景色、二度目で意味が変わる景色、長期で通れなくなる道、名を残した結果として変わる看板や歌を用意すると、世界観への没入と「ここで生きた感」が強くなります。

  • 初見で惚れさせる景色: その世界でしか見られない美・聖性・生活
  • 二度目で意味が変わる景色: 前回の選択の結果が見える場所
  • 変わる旅路: 検問、祭礼、嵐、橋の修復、航路開通などで選択肢が変わる道
  • 可視化される名残: 称号、噂、地図、碑文、歌、借りの記録として残る PC の足跡

長期伏線の条件

長期伏線は、存在が見える、後から意味が変わる、先に消費しきらない の三条件を満たす形で置くと扱いやすくなります。

長期反転は三種類に分けて管理する

  • 伏線収束型: 断片が一本につながり、知的快感を返す。
  • 正体露見型: 人物・関係・歴史の本当の姿が見え、Stories を強く動かす。
  • 前提崩壊型: 世界観や判断基準が反転し、終盤の衝撃を作る。

三段階公開に加えて どの型の反転か を書くと、長編のどんでん返しが単調になりにくくなります。

長期伏線も、単発と同じく 存在が見える / 後から意味が変わる / 先に消費しきらない の三条件を満たしてください。
途中で意味を反転させたいなら、最初から複数の読み方ができる形で置くと扱いやすくなります。

共通情報と個別情報の往復

長編では、個別に見えた材料が後で共通情報になることがあります。誰の視点で最初に見せ、どの回で全員の判断基準へ上げるかを計画しておくと、秘密が散りにくくなります。

ウィスパー・モード を長編で使うなら、誰の視点で最初に見せ、どの回で共通情報へ上げるか をシートにしておくと安定します。秘密を抱えたまま引っぱること自体が目的ではありません。いつ共有世界へ変換するか まで決めておくと、露見が物語の推進力になります。

12. PCごとの野望競合とスポットライト配分

複数PCキャンペーンでは、全員の野望を毎回同じだけ進める必要はありません。
ただし、今回主進行でないPCにも、利益か立場の変化を必ず渡す ほうが卓が安定します。

主進行PCと従進行PC

各回ごとに、野望が主に進むPCと、従として関わるPCを決めます。従進行PCには、手札の獲得、関係の変化、履歴の追加など、次回へ効く利益を用意してください。

競合を対立だけにしない

野望同士がズレるときは、単純な対立よりも、相互干渉にすると扱いやすいです。たとえば、誰かの成功が別PCの信用を傷つける、急ぐと救出はできるが証拠は残らない、という形です。

ウィスパー・モード を使うときも、競合はまず 相互干渉 として置くのが基本です。たとえば「急げば救出はできるが証拠は傷む」「このNPCを守ると別PCの立場が揺らぐ」のように、終着点の差 を作ります。共有主目的の失敗そのものが勝ち になる設計は、半協力ではなく対立卓です。

配分の目安

5本キャンペーンなら、各PCに少なくとも1回は主進行回を渡し、残りの回で混成にするのが目安です。8本以上なら、主進行の短いアークを複数回作れます。

関係進展とテーマ進展を主進行と同じだけ重く見る

複数PCキャンペーンでは、毎回すべての野望を同じだけ進める必要はありません。
ただし、主進行でないPCにも、次のいずれかは必ず渡したほうが安定します。

  • 保持の進展: 成長点、素材、資産表示、制度表示、関係表示の保持化
  • 変質の進展: 関係の深化、感情の変質、火種表示、秘密開示

キャンペーン主題があるなら、テーマ進展 も別に管理してください。
たとえば「信頼」「帰属」「救済」「継承」のような 1〜3 個を置いておくと、各回が何を深めたかを見失いにくくなります。

13. 単発シナリオをキャンペーンに組み込む方法

単発シナリオをそのまま並べるだけでは、長い流れの中で浮くことがあります。組み込むときは、その単発が何を進める回なのか と 何を次回へ残す回なのか を明記してください。

組み込み前の三問

  1. この単発はどの野望をどこまで進めるか
  2. この単発で増える持ち越し要素 / 手札タグ / 履歴は何か
  3. この単発の結末は、次回の導入へどうつながるか

組み込みのやり方

  • 節目回として使う: 立場や関係が変わる回にする
  • 足場回として使う: 支援や手札を得る回にする
  • 代償回として使う: 持ち越し要素を返す回にする
  • 息継ぎ回として使う: 回復と整理の回にする

単発で完結したPCをそのままキャンペーン化する

単発だけのつもりだった PC でも、終わったあとに「この人たちの続きを見たい」と思うことは珍しくありません。その場合は、単発の成功をやり直さず、単発の結末を履歴として固定したうえで、その結果生まれた未解決を長期線へ上げる のが基本です。

単発の終了時に付いた +1 / 0 / -1 は、まずその 1 本の達成度採点 です。単発の最終採点としての 0 を、ただちに「野望が折れたので続投不可」とは読まないでください。続投させるなら、その単発で 何が守れ、何が失われ、何がまだ未解決か を見て、次回開始時点の現在野望度を改めて決めます。

  1. 単発で答えが出たことを履歴へ固定する
    まず「何を達成したか」「何を失ったか」「誰とどういう関係になったか」を履歴として残します。単発の結末そのものは、長編の第 1 話でひっくり返さないでください。
  2. 続けて見たい核を三つ抜く
    その PC について、守れたもの / 失ったもの / 新しく背負ったもの を 1 つずつ抜きます。ここが、そのまま次の持ち越し要素、手札タグ、継続要素の候補になります。
  3. 野望を「継ぐ / 広げる / 歪ませる」のどれかで置き直す
    単発の野望が未達ならそのまま継いでかまいません。達成済みなら、より大きい射程へ広げる、代償の返りで歪ませる、成功によって新しい責務が生まれた のいずれかで次の野望を立てるとつながりやすいです。
  4. 単発で得たものを長期資源へ振り分ける
    一時的な成功だったのか、今後も使える足場なのかを見て、持ち越し要素 / 手札タグ / 履歴 / レギュラー NPC / 継続勢力 に振り分けます。
  5. 次の第 1 話は「成功後のコスト」か「成功で生まれた呼び水」を扱う
    長編の第 1 話は、前回より大きい敵を即座にぶつける必要はありません。あの成功のせいで新しい依頼が来た / 守ったものを維持する負担が生まれた / 勝ったことで別勢力に見つかった といった橋渡し回のほうが自然です。

単発用に薄く作った PC だった場合も、すべてをあとから埋める必要はありません。卓が愛着を持った核だけを残し、残りは今後のシナリオで育てる ほうが、後付け感が薄くなります。

継承手順がある版では、敗着後の延命は既定にしません。 同じPCで意味の薄い足掻き局面を何本も続けるより、退場を確定し、新PCか継承PCで次の候補へ入ったほうが campaign の足場が整います。

18. PC交代・途中参加・離脱・引退・死亡の処理

長編では、メンバー変動は珍しくありません。変動が起きたときに長期設計をどう直すか を最初から決めておくと、卓が止まりにくくなります。

継承PCは新しい野望に合わせて再編してよい

継承PCは、旧来の状態を一字一句そのまま持ち込む必要はありません。新しい野望、残り本数、今の卓の役目に合わせて、GM 権限で大切なもの、長期課題、保持ノード、関係線を再編してかまいません。

大事なのは「何を消したか」ではなく、何を履歴として固定し、何を今の主人公線へ持ち直したか が見えることです。継承前の痕跡は履歴へ残し、現在の主役線は再編して始める、と考えると整いやすくなります。

途中参加

途中参加PCには、現在の主軸と最低限の履歴がわかる前回までのあらすじを渡し、最初の1本は足場獲得か関係形成を主目的にした回へ入れると安定します。

既定の合流タイミングは「次の主要局面の開始時」 としてください。いま解決中の宣言の途中へ新PCを割り込ませず、結果確定 → 現在状態の再掲 → 新PC導入 の順で入れると、初心者GMでも処理がぶれません。

状況既定処理理由
残り主要局面が2つ以上あり、1文で自然な導入が作れる今シナリオの次の主要局面頭から合流プレイヤーを長く見学にしないため
残りが最終局面だけ、または自然な導入に本文改造が要る新PCの本加入は次シナリオ第1局面から。今シナリオでは観測役・連絡役・同行NPCなどの仮役で関与させてよい不自然な乱入で構造を壊さないため
いま解決中の宣言の途中合流させない。現在の裁定を終えてから判断同一局面内の責任範囲を混ぜないため
 
【途中継承の合流判定メモ】
【退場が確定した局面】
【残り主要局面数】
【次の自然な合流点】
【今シナリオで合流する / 次シナリオから入る】
【今シナリオで渡す仮役】
【再掲する最低限の履歴】

離脱と引退

離脱・引退するPCは、野望を未回収のまま放置せず、何を残して去るのか を履歴へ残してください。支援者化、別陣営化、消息不明化など、後で使える形にしておくと長編に馴染みます。

死亡と再出発

死亡や再出発が起きたときは、どの持ち越し要素と履歴を残し、どの野望線を切るかを明記します。新PCを入れるなら、既存の勢力線やレギュラーNPCと接点を持たせると接続しやすくなります。

継承型の再出発

キャンペーンでは、単発と違って PC の退場をそのままゲームオーバーにしないでかまいません。死亡や再起不能が出たら、そのPCはいったん現キャンペーンの主役から降り、次のPCが野望線を継ぐ のを既定にすると saga らしさが出ます。

継承とは、能力や命運点を丸ごと譲ることではありません。未完の野望、残された履歴、未処理の借金、レギュラーNPCとの縁 を次の主人公へ渡すことです。新PCは初期値で入り、代わりに 継承物 を 2〜4 点だけ受け取ります。

継ぐもの継がないもの理由
履歴 1〜3 件先代の命運点命の余力は本人固有だから
手札タグ 1 件先代固有の短期傷や短期疲弊状態異常の持ち越しを簡潔にするため
長期課題 1 件失われた大切なものそのもの喪失は履歴として残し、同一物を復元しないため
レギュラーNPC / 勢力との接点 1 件先代の野望度新主人公の到達度は別に測るため

継承PCの最初の 1 本は、継承が成立する回 にしてください。遺志の受領、資料の引き継ぎ、協力者への顔つなぎ、先代の失敗処理の肩代わりなど、現行の長期線へ自然につながる内容にすると止まりにくくなります。

途中継承の既定 もここで固定しておくと安全です。先代が第2局面で退場したなら、新PCはその場に瞬間転移したように出すのではなく、第3局面の入口 で「先代の遺した資料を受け取った」「依頼人が後任を差し向けた」「現地協力者が主役化した」など、いま見えている状況に接続する1文 で入れてください。逆に、残りがほぼ決着だけなら無理に混ぜず、次シナリオの第1話から本加入に回したほうが自然です。

延命せず、退場を確定して継承へ渡す

継承手順がある以上、現キャンペーンで主役として復帰できないPCを、愛着だけで引っ張り続けない のが原則です。再起不能、任務達成・未帰還、野望度 0 / -1、主軸復帰不能のいずれかに入ったら、GMは「まだ少し動けるから」で局面を増やさず、退場処理に入ってかまいません。

状況既定処理
死亡確定即退場。継承PCか新PCを入れる
再起不能 / 長期療養現キャンペーンの主役から降板。支援者化するか、別キャンペーン候補へ送る
任務達成・未帰還 / 退出不能救出回をやるなら次の 1本 に限定。失敗したら継承へ移行する
野望度 0 / -1 で主軸喪失現行主軸には別PCを投入し、旧PCは離脱 / 引退 / 履歴化する
 
【継承への即時移行手順】
1. 退場理由を1文で確定する
2. 残す履歴 / 持ち越し要素 / 接点を選ぶ
3. 新PCの野望を、先代の失敗か遺志に接続する
4. 継承成立回を1本だけ挟む
5. 以後は新PCを主役として扱い、旧PC延命のための局面を増やさない

この切り替えは冷酷さのためではなく、saga として前へ進むため の処理です。負けを曖昧にしないからこそ、次の主人公が何を継ぐのかが鮮明になります。

生存離脱PCを別の野望へ移す

再起不能だが死んではいない PC にどうしても愛着があるなら、その PC を即座に消さなくても構いません。現キャンペーンでは離脱 / 引退として固定し、別の野望を立てた別キャンペーンの主人公にする 運用を許可してください。

大切なのは、現キャンペーンの主軸と、新しい個人キャンペーンを同じ線に混ぜない ことです。現行キャンペーンでは、その PC は支援者 / 保護対象 / 留守番 / 行方不明として履歴化し、主役枠は別PCへ渡します。後日、新しい野望を立てるときは、その時点の傷、喪失、関係、残っている持ち越し要素を前提に、野望度 1 から始めます。

【生存離脱PCの整理】
【このキャンペーンでここまでに確定した喪失】
【現キャンペーンではどの立場で残るか】
【別キャンペーンで立てる新しい野望】
【引き継ぐ持ち越し要素 / 引き継がない持ち越し要素】
【再登場させるなら、どの距離感で出すか】

この扱いにすると、「この物語では退場したが、この人物の人生は終わっていない」 を両立できます。saga らしさを保ちつつ、ひとりの人物への愛着も切り捨てずに済みます。

生きている元PCへの愛着を残す選択は、現キャンペーンでまだ主役として粘る ことを意味しません。愛着は別野望や別キャンペーンへ預け、現行キャンペーンでは退場を確定させたうえで継承や交代を進めます。

19. 分岐・中断・打ち切り・圧縮終了

キャンペーンは、理想どおりの本数で終わらないことがあります。だからこそ、途中で圧縮して終える方法 を最初から持っておくと安全です。

分岐の扱い

すべての分岐を独立アークに増やす必要はありません。許容する分岐と、次の節目で収束させる分岐を分けてください。

選ばれなかった候補の扱い

候補提示をしたのに選ばれなかったシナリオは、消えたわけではありません。放置された候補は世界反応として前進する、あるいは 別勢力が先に手を打つ と決めると、選択の重みが出ます。プレイヤーに「何を選ばなかったか」が残ると、自発的な判断が長期へつながりやすくなります。

中断時の最小処理

中断するときは、少なくとも 現在の持ち越し要素 / 手札タグ / 履歴 / 勢力状態 / 残っている問い を1枚にまとめて保存します。これがあれば、後日再開しやすくなります。

起きたことは履歴として進む

中断、打ち切り、圧縮終了、再出発のどれでも、すでに結果返答で確定したこと は、その世界で起きたこととして扱います。原則は巻き戻しではなく、履歴化・持ち越し要素化・圧縮した結末への接続です。

まだ結果返答で確定していない宣言や意図までは、自動では正史にしません。途中まで確定した成果と代償だけを履歴へ残し、未確定部分は次の候補や圧の負債帳へ戻します。

中断時に固定するもの

保存するときは、公開正本 / 個別秘匿正本 / 圧の負債帳 / 未解決線 を分けて残してください。差分ではなく、その時点の最新状態を丸ごと残すと再開が楽です。

【中断時の固定】
- 公開正本:
- 個別秘匿正本:
- すでに履歴として固定すること:
- まだ未確定で戻すこと:
- 圧の負債帳へ送ること:
- 次回最初に再掲すること:

再開・圧縮終了・再出発の再編

再開時や残り本数が少ないときは、未解決線を全部そのまま抱え込まない ことが大切です。主要線だけを現在の野望へ寄せ、脇線は履歴化、世界反応化、NPC の近況化のどれかへ畳みます。

野望が変わった継承PCでは、GM が新しい野望に合うように状態を整えて構いません。これも「書き換え」ではなく、起きたことを踏まえた再配置 と考えてください。

圧縮終了モード

残り1本で畳む:
- 最後の問いに必要な材料だけ残す
- 脇線は履歴化して回収を簡略化する
- レギュラーNPC は主要2本に絞る

残り2本で畳む:
- 1本目で持ち越し要素と勢力線を整理する
- 2本目で最後の問いと主要決着だけを扱う

20. 長期資源インフレ / デフレの補正

長編では、手札タグや協力者が増えすぎて易しくなったり、逆に持ち越し要素が重すぎて前へ出られなくなったりします。これを放置すると、後半が単調か停滞のどちらかになりがちです。

インフレ補正

  • 似た手札を統合する
  • 支援者に使用条件や反作用を付ける
  • 勢力側も学習して対抗する
  • 強い手札は一度使ったら履歴へ移す

デフレ補正

  • 回復回を1本挟む
  • 持ち越し要素を段階的に軽くする条件を置く
  • 協力者や設備を再獲得できる短い回を入れる
  • 脇線を切って主軸へ集中する
  • 大局的敗北直後は、失地回復 / 借金整理 / 迂回準備 / 継承導入の候補を少なくとも 1 本混ぜる

長期資源のインフレを Q / B / S の三面で見る

長編で強くなりすぎるのは、数だけではありません。世界側の前進量、保持タグの層の厚さ、関係線の多さ も同時に膨らみます。

したがって補正は、Quiddities では世界側の要求水準、Builds では保持の要求水準、Stories では関係の密度と秘密の返り を上げる三方向で考えると安定します。単純に敵を強くするだけだと、Builds は育っても Stories と Quiddities が薄くなります。

21. キャンペーン級の公平性チェックリスト

単発シナリオの公平性チェックに加えて、長編では次を確認してください。

  • 最後の問いに必要な材料が中盤までに見えているか
  • 主要分岐に事前の読める材料があるか
  • 各アークに回復の息継ぎがあるか
  • 同じ勝ち筋だけで全話を横滑りできないか
  • 単発差し込み回がつなぎの回で終わらず、野望や勢力線に接続しているか
  • 主進行でないPCにも利益か変化が渡っているか
  • 持ち越し要素と手札タグの量が管理可能な範囲に収まっているか
  • 回復回が「休むだけ」で終わらず、次の仕込みや手札タグの獲得につながっているか
  • 次回候補の提示や自発案の採用など、プレイヤーが向かう先を選べる余地があるか
  • PCの得意分野に合わせすぎず、少なくとも 1 本は全PCに不利な方向から揺さぶる回があるか
  • 理不尽イベントが即時無効化ではなく、複数の局面と読める痕跡を持つ構造になっているか
  • 単発から長編へ移った場合、単発の結末を崩さずに長期の問いへ変換できているか
  • 発見回 / 文化接触回 / 挑戦・越境回 / 回復兼仕込み回 / 畏敬回 のうち複数が混ざり、同じ型だけが続いていないか
  • 反復して訪れる回復点と畏敬点があり、再訪時に意味や関係が更新されるか
  • 安全ルート / 近道 / 地元ルート の差が実際に選択肢になっているか
  • 世界反応が政治や敵対だけでなく、物流・祭礼・巡礼・検問・航路変化としても返っているか
  • PC がこの世界で名を残す方法が定義され、看板・歌・地図・噂・称号などの形で可視化されているか
  • 各アークに 開く / ひっくり返す / 押し切る / 深まる の配線があり、同じ熱だけで長く引っ張っていないか
  • 「最後に問うこと」が、達成だけでなく 意味の反転 と 関係の決定的変化 まで届いているか
  • 圧の負債帳 を更新し、未回収の借り・追加利益・伏線・救援を忘れていないか
  • 長期反転が 伏線収束型 / 正体露見型 / 前提崩壊型 のどれかに偏りすぎていないか
  • 再登場NPCが、再会 / 和解 / 裏切り / 救援 / 継承 / 鏡像化 のどれで戻るか決まっているか

セルフチェックの順番

  1. 最後の問いを言い直す
  2. その問いに必要な材料がどの話で出るか確認する
  3. 各話の役目が重複しすぎていないか確認する
  4. 停滞しそうなら圧縮終了モードへ切り替える

一般用では、候補カードが未添付ならリラックスは条件付き対応 と書くのが親切です。逆に、候補カードまで添えてあるなら、リラックスは十分に事前設計できます。

25. アークごとのモード配分表

長編全体で同じモードを常時装備すると、便利な一方で、毎回の処理負荷が重くなりがちです。一般用では、どのアークで何のモードを太くするか を先に書いた モード配分表 を持つと、過不足が見えやすくなります。

モードは長編全体の常時装備にしない

たとえば旅立ちアークでは ゼノス と ヨーク を太くし、年代記アークでは ナラティブ と ヨーク を太くし、陰謀アークでは ソーシャル と ウィスパー を太くし、攻城アークでは タクティカル や アンロック を太くする、といった配分で構いません。全アークで全部を強く使う より、話数ごとに重心をずらしたほうが長編として読みやすくなります。
