QBS キャンペーン設計ガイド

複数本のシナリオを束ねて、PCの野望が長く変化していく流れを設計するGM向けの本です。
野望度終点、想定シナリオ数、節目、持ち越し資源、難度曲線、長期の世界反応、進行ログ、公平性チェックを扱います。単発1本そのものの構造作りは シナリオ設計ガイド に分けています。

1. はじめに

キャンペーンとは、PCが野望へ近づいていく長い道のりを設計することです。
QBSでは、各回のシナリオを面白くするだけでなく、何を持ち越し、いつ節目を置き、どの問いへ向けて積み上げるか を文書として持っておくと安定します。

この本は、シナリオ設計ガイド で作った単発シナリオを、長い流れの中へ位置づけるための本です。
すでにある単発を並べるだけではなく、どの回が何を進め、どの回が回復・反転・決着の役目を持つかまで整理します。

この本で扱う範囲

この本で扱うのは、野望度終点と想定本数、シナリオ列の節目、持ち越し資源、回復ペースと難度曲線、長期のNPC / 勢力 / 世界反応、単発の組み込み、分岐・中断・打ち切り、進行ログ、状態管理表、公平性チェックです。

逆に、各回の問い・条件・期限・主要局面・構造難度・作者メモの細部は シナリオ設計ガイド に分けています。既存資料を回すだけなら GMガイド を先に読んでください。

設計の基本線

長い流れでは、毎回全部を進めない
各回の役目を分けて、前進する回 / 回収する回 / 回復する回 / 反転する回 / 決着する回 を置きます。
各回の面白さはシナリオ設計で作り、全体の呼吸はキャンペーン設計で作ります。

旅と冒険の呼吸も節目へ織り込む

QBS のキャンペーンでは、前進だけでなく なぜそこへ行きたいのか、どこで自由を感じるのか、何を持ち帰ると物語になるのか も設計しておくと没入が強まります。長い流れでは、移動や滞在を単なるコストではなく、発見 / 文化接触 / 挑戦・越境 / 回復兼仕込み / 畏敬 のどれかとして扱うと、旅情と達成感が両立します。

各アークに少なくとも一つ、反復して訪れたくなる回復点、心を打つ畏敬点、安全ルート / 近道 / 地元ルートの選択 を置いてください。これでプレイヤーは「与えられた事件を解く」だけでなく、自分の意思で世界を歩いている感覚 を得やすくなります。

キャンペーンでは Quiddities / Builds / Stories を積み上げる

キャンペーン設計では、各回の問い だけでなく、各回で何が Quiddities として前進したか、何が Builds として残ったか、何が Stories として深まったか を並行して管理します。

Quiddities は勢力図・秘密・世界状態・立場・次回の判断材料として積み上がり、Builds は恒久タグ・装備・拠点・仲間・素材として積み上がり、Stories は絆・感情・約束・対立・テーマ進展として積み上がります。

単発を連ねるだけではなく、前進する回 / 回復する回 / 反転する回 / 決着する回 のそれぞれに、三要素のどこが動く回なのかを持たせてください。

2. このガイドで使う言葉

この章では、この本で多用する長期設計用語だけをまとめます。ルール全体の用語は必要に応じて GMガイド を参照してください。

アーク

キャンペーンの中で、ひとまとまりの転機を持つ区間です。3本なら1〜2本、8本以上なら複数のアークに分けると管理しやすくなります。

節目

野望の進行や立場の変化が明確に起こる回です。節目は「情報が増える回」ではなく、以後の判断基準が変わる回として置いてください。

長期タグ

次回以降の判断と世界反応を変え続けるタグです。怪我、信用、追跡、同盟、汚染、疑念、義務など、複数回に効くものだけを長期タグへ残します。

長期手札

まだ使っていない支援、コネ、設備、制度的立場など、将来の選択肢を増やす持ち越し資源です。使うたびに消費するのか、条件付きで再利用できるのかを明記します。

レギュラーNPC / 継続要素

再登場を前提に管理するNPC、勢力、場所、制度、怪異です。各回本文に散らさず、長期管理表にまとめて状態を更新します。

キャンペーン要約

プレイヤー向けに公開する短い要約です。前回までの要点、いま残っている長期課題、今回の導入だけを短くまとめます。

切り出しシート

キャンペーン全体の設計から、次の1本のシナリオへ情報を渡すための短い設計票です。どの野望をどこまで進めるか、今回動く勢力や持ち越しタグは何かをここで固定します。

圧縮終了モード

高揚配線

各アークで、何を開き、何をひっくり返し、何を押し切り、最後に何が深まるか を整理したメモです。劇的展開度が長期の味なら、高揚配線は長期の感情曲線です。

圧の負債帳

未回収の借り、まだ解放していない伏線、後で意味が変わる回復点や畏敬点など、蓄積した圧の残高 を記録する欄です。長編では「覚えているつもり」が一番漏れやすいため、毎話の終わりに更新します。

予定本数どおりに終われないときに、残り1〜2本で主要な問いだけを回収して畳むための終了設計です。長編では最初から用意しておくと安全です。

Quiddities / Builds / Stories の長期管理で使う語

3. このガイドの使い方

この本は、キャンペーンの全体骨格を決めるとき、各回へ切り出すとき、進行中に破綻を直すときの3場面で使います。

やりたいこと先に読む章
最初に全体の長さと節目を決める4章 → 5章 → 24章
進行中の持ち越しと難度曲線を整える6章 → 7章 → 18章 → 19章 → 26章
次の1本へ切り出す22章 → 23章 → シナリオ設計ガイド
中断や打ち切りを畳む27章 → 28章 → 30章

設計の基本順

  1. 野望度終点と想定本数を決める
  2. 節目と最後の問いを置く
  3. 長期の持ち越し資源と世界反応を決める
  4. 難度曲線と回復ペースを決める
  5. 次の1本へ切り出すためのシートを書く
  6. 進行ログと公平性チェックで保守する

4. 野望度終点と想定シナリオ数の設計

まず各PCの野望を見て、どこまで行ったらその野望を一区切りとみなすか を決めます。QBSでは、ここを曖昧にしたまま各話を書き始めると、面白い単発はできても長い物語の進み具合が読めなくなります。

目安は 1シナリオで野望度が1変化する です。ただし、毎回必ず上がる必要はありません。回復回や足場作りの回では、野望度が動かずに 次回のための手札や立場だけが増える こともあります。

想定本数向いている題材野望度終点の目安特徴
1〜2本単発 / 導入卓 / テスト運用2直近の目標をかなえる短距離走
3本短編キャンペーン3〜4導入・反転・決着の三幕が作りやすい
5本長期キャンペーン4〜6回復回と中盤反転を置きやすい
8本以上長編 / 複数勢力戦6以上複数アークと持ち越し管理が必須

終点を決める基準

  • 立場の回復や獲得 が完了したとき
  • 最後に問いたい価値選択 へ到達したとき
  • 主要な敵対勢力との関係 が決着したとき
  • PCの野望の文言 自体を書き換える必要が出たとき

本数が伸びすぎるとき

終点が遠すぎると、毎回の達成感が薄くなります。
本数が伸びすぎると感じたら、野望を二段に分けて、前半の終点 を置いてください。たとえば「国を救う」では広すぎるなら、「国政に関わる立場を得る」を前半終点にします。

リラックス・モードでは本数ではなく見直し間隔を決める

リラックス・モードを使うなら、全体本数の固定 より 何本ごとに見直すか を決めてください。おすすめは 2〜3本ごと の小さな見直しです。

  • 見ること: まだ続けたいか、通常キャンペーンへ戻したいか、今の候補の重さが合っているか
  • 動かすこと: 野望度ではなく、次回候補の配分、回復回の頻度、長期タグの整理
  • 決めすぎないこと: 最終局面の時期

このモードでは、終点の予約 より 節目ごとの温度確認 のほうが機能します。

5. シナリオ列の節目設計

各シナリオは、ただ並ぶのではなく、何が進む回か と 何が変わる回か を持つべきです。節目は「大事件の回」だけではありません。情報公開の意味が反転する、協力者が レギュラー化する、撤退不能になる、価値選択の前提が変わる、といった変化も節目です。

一つの回が複数の型を兼ねてもかまいませんが、同じ型だけが続かない ようにすると、長期キャンペーンでも「旅している感覚」が痩せにくくなります。

  • 発見回: 新しい土地・風景・常識を知り、Quiddities を増やす回
  • 文化接触回: 礼、作法、贈答、証人、埋葬、通行の違いが Stories と価格を変える回
  • 挑戦・越境回: ルート選択、越境条件、退路確保で自己効力を問う回
  • 回復兼仕込み回: 休息しつつ Build を増やし、次の自由選択肢を作る回
  • 畏敬回: 絶景、聖地、儀式、美しい瞬間が意味と方向を変える回

節目設計では、事件の役目と同時に、その回がプレイヤーへ何の旅情を返すかも決めておくと波が出ます。

節目は「何が進むか」だけでなく「どんな旅になるか」でも分ける

節目設計では、事件の役目と同時に、その回がプレイヤーへ何の旅情を返すかも決めておくと波が出ます。

  • 発見回: 新しい土地・風景・常識を知り、Quiddities を増やす回
  • 文化接触回: 礼、作法、贈答、証人、埋葬、通行の違いが Stories と価格を変える回
  • 挑戦・越境回: ルート選択、越境条件、退路確保で自己効力を問う回
  • 回復兼仕込み回: 休息しつつ Build を増やし、次の自由選択肢を作る回
  • 畏敬回: 絶景、聖地、儀式、美しい瞬間が意味と方向を変える回

一つの回が複数の型を兼ねてもかまいませんが、同じ型だけが続かない ようにすると、長期キャンペーンでも「旅している感覚」が痩せにくくなります。

節目の種類

  • 導入節目: 題材と主軸を固定する
  • 足場節目: 支援者・設備・信用などの持ち越しを得る
  • 反転節目: これまでの勝ち筋がそのままでは通らなくなる
  • 代償節目: これまでの蓄積が重さを持って返る
  • 決着節目: 最後の問いへ答える

最後に問うこと

最後に問うことは、達成だけでなく 何の意味が反転するか、誰との関係が決定的に変わるか まで書いてください。終盤が「事件を解いた」で終わらず、「この旅で何者になったか」に届きます。

- ひっくり返す: 再文脈化
- 押し切る: 逼迫突破
- 最後に深まるもの:
- 崇高化ポイント:
- 蓄積からの解放ポイント:

同じ節目設計でも、開く → 反転する → 押し切る → 深まる の順を意識すると、長編の波が強くなります。

キャンペーン終盤でPCへ何を問うかを、できれば最初に一文で置いてください。
途中の各回は、最終局面そのものを先送りするためでなく、最後の問いに必要な視点や関係を積み上げるために存在します。

節目設計の短い型

終盤前: これまでの履歴をまとめて返す
終盤: 最後の問いへ答える

6. 長期資産ノード / 手札 / 履歴 / 大切なものの持ち越し管理

キャンペーンでは、毎回すべてを持ち越す必要はありません。何を長期タグとして残し、何を履歴として残し、何をその回だけで閉じるか を先に決めたほうが、管理が軽くなります。

項目長期へ残す基準閉じてもよい基準
長期タグ次回以降の判断や世界反応を変え続ける今回限りの軽い余波で終わる
手札タグまだ使っていない支援・資源・協力者使用済みで効果が終わった
履歴事実として参照され続ける出来事参照価値が薄く今後の裁定に効かない
大切なもの継続的に命運点や行動原理へ影響する意味が完全に変質し、差し替えが必要

長期タグの粒度

長期タグは、毎回細かく増やしすぎると管理不能になります。
似た方向のタグが増えてきたら、統合した1枚の長期タグ へまとめてください。たとえば「監視が増えた」「噂が広がった」「身元照会が厳しくなった」を統合して「当局の警戒強化」にしてよいです。

手札と履歴の違い

まだ使えるもの は手札、もう起きた事実 は履歴です。協力者の信頼を得たなら、その協力者は手札であり、同時に「助けてもらった」という事実は履歴です。どちらも残すか、片方だけ残すかは次回の裁定でどちらを使うかで決めます。

大切なものの持ち越し

大切なものは、野望や命運点の読み方を変える核です。キャンペーンの途中で意味が変わったら、何が失われ、何が守られた結果その意味が変わったのか を履歴に残したうえで差し替えてください。

装備・拠点・仲間・素材を長期資源として持つ

キャンペーンでは、恒久タグを次の4群に分けて持つと管理しやすくなります。

  • 能力: 宣言の射程と値段を変える
  • 装備: スロットに入れて持ち歩く強み
  • 拠点 / 仲間: ダウンタイム、工作、補給、受取の基盤
  • 素材: 将来の工作や強化の種

素材は数値資源ではなく、名前つきタグ にすると物語との相性が良くなります。
例: 〔竜骨片〕 〔旧式封印器〕 〔港湾許可証〕

長期資産ノード方式で管理する

QBSでは、Builds で長く残るものをすべて 長期資産ノード として一枚ずつ管理します。
4群管理はそのまま残し、内部小分類を増やして扱います。

7. 回復ペースと難度曲線

キャンペーンでは、毎回同じ重さだと平坦になります。
回復しやすい回、別利益の競合が強い回、世界反応が重い回、複数PCの噛み合わせを問う回を散らして、何を難しくする回なのか を回ごとに分けてください。

難度曲線の考え方

  • 前半: 足場を作る回と軽い代償回を混ぜる
  • 中盤: 勝ち筋の反転や別利益の競合を強める
  • 終盤: 長期タグと履歴の返りを重くする

回復回の役目

回復回は、ぬるい回ではありません。
大きな前進ではなくても、手札の整理、関係の修復、休息、再配置、次の仕込み をするための回です。回復回がないと、後半の代償がただの消耗戦になりやすくなります。

回復回は「次の仕込み」を主目的にしてよい

キャンペーン、とくに上級者向け以上では、回復回の本体は傷を消すことではなく、次の局面で役に立つ長期手札をどれだけ仕込めるかです。
休息、関係修復、設備点検、慰安旅行、祭礼参加、里帰り、取材旅行のような表向きの目的を置きつつ、実際には 協力者づくり / 補給線整備 / 許認可取得 / 土地勘獲得 / 監査逃れ / 外交線の確保 を進める回として設計してください。

例: キャンペーンのゴールが「国を救う」なら、終盤で外国から横やりが入らないよう、前もって友好国との関係を作っておく回を置けます。表向きは「疲れを癒すための慰安旅行」でも、裏では 現地の重鎮と知己を得る / 港湾利用の便宜を取る / 緊急時の亡命路を押さえる といった長期手札を仕込む回になります。

回復は結果、仕込みが本体 と考えると、後半で効いてくる回が作りやすくなります。回復回で何も仕込めないと、長編はただの消耗戦になりがちです。

難度ごとの「仕込み気づき」支援

難度帯GMが表に出してよい支援
初級「この回では外国の有力者と会える」「補給線を整えられる」など、仕込める長期手札の例を候補文や既知情報へ書いてよいです。
中級「交流行事がある」「現地の工房を見学できる」「監査担当が同席する」など、使えそうな接点は見せますが、どの手札になるかは言い切りません。
上級表向きの目的だけを主に見せ、仕込みに関わる手掛かりは 招待客名簿・地図・噂・既知情報 に散らします。裏の目的は説明しません。
熟練回復回であることだけ見せ、仕込みの方向はほぼ教えません。休息をどう長期手札へ変えるか自体がプレイ になります。

各話の終わりに、まだ解放していない圧を記録してください。長編では「良い蓄積」がログの底へ沈みやすく、回収しないまま忘れると終盤が痩せます。

- 再訪で意味が変わる回復点:
- 再訪で意味が変わる畏敬点:
- まだ解放していない伏線:

QBSでは宣言は基本的に成功するため、PCの得意分野にだけ合わせた課題を出し続けると、勝ち筋が固定化して飽きやすくなります。キャンペーンには、全PCにとって不得手な方向から来る回 を意識的に混ぜてください。

たとえば、億万長者を目指して金融戦で勝ってきたPCたちに対して、世界恐慌による通貨価値の崩壊、国家破綻、国債の紙くず化、法制度の急変、外交圧力、災厄 のような「これまでの勝ち筋がそのままでは通らない」事件を入れる、という考え方です。重要なのは、過去の成功を無効にすること ではなく、成功の意味を変えること です。財産そのものは消えなくても、換金性、正当性、護衛、流通、同盟が新しい問題になります。

  • 初級でも、キャンペーン全体で 1本 はこうした回を入れてかまいません。
  • 中級以上では、長編なら 1〜2本 を目安に、得意分野外の回を置くと波が出ます。
  • 上級 / 熟練では、理不尽に見える事件でも、複数の局面と読める痕跡 を置いておけば成立します。答えを配らず、対応の余地だけを残してください。

理不尽イベントは、一文で全部を失わせる処理 にしてはいけません。必ず、観測できる兆候 / 取得可能な情報源 / 手札を仕込む余地 / 立て直しの局面 を持たせてください。

  • 畏敬点: 絶景、聖域、巨大建造物、祭礼、巡礼路、静かな高台のように、価値選択や決意を更新しやすい場所

成長ループを野望度から切り離す

野望度は人生の到達度です。Builds の快感は、別に 成長点 で支えたほうが安定します。

  • 野望度: 節目、立場の変化、人生の到達度
  • 成長点: 小さな依頼、世界課題、工作、探索、地道な積み上げ

8. オブザーバー・モードの配置

オブザーバー・モードは、各局面のひらめき より PC の一貫した行動原理 を見たい長編と相性が良いです。使うなら、どの PC が観測者か を最初に決め、観測方針の更新窓を 各シナリオの開始時と終了時 に置いてください。

キャンペーン側で大事なのは、観測者 PC だけが不利になるような「未取得情報がなければ選べない正解一手」に寄せすぎないことです。必要な読み筋は、既知情報、痕跡、取得可能な資料として出しておくと、長編でも破綻しにくくなります。

9. プロバビリティ・モードの配置

プロバビリティ・モードは、長編全体へ入れても、特定のアークだけ に入れても構いません。重要なのは、どの局面で運の揺らぎを使うのか を揃えることです。おすすめは、偶然・競合・不確実性が前面に出る回だけへ入れるやり方です。

難度は既存どおり 容易4 / 普通6 / 困難8 / 過酷10 / 絶望12 へ読み替えます。長編では、出目が悪かった回でも何かが残るよう、失敗の返り先 を Quiddities / Builds / Stories の三面で散らしてください。

10. クオンティティ・モードの配置

クオンティティ・モードは、キャラクターシートの手触り を強めたい長編で使います。能力値の基本形は 4,3,3,2,2 を維持し、成長の主役はあくまでタグ、履歴、関係、拠点、仲間に残してください。

長編で運用するなら、能力値は原則固定 とし、再訓練、後遺症、身体改造、身分変化、信念変化のように PC の基礎そのものが変わったときだけ、シナリオ終了時に 1 点入れ替える窓を用意すると安定します。

11. リラックス・モードの配置

基本原則

リラックス・モードでは、キャンペーン全体の最終決着を必須にしません。 ただし、各シナリオの問い、クリア条件、失敗条件、期限、終了処理 は通常どおり持ちます。

つまり、シナリオは終わる / キャンペーンは急がなくてよい、が原則です。QBS の自由宣言は「今回の答え」を変えることで強く働くため、各話の切れ目は残します。

既定値と採用基準

既定は 初級ではあり、中級・上級・熟練ではなし とします。ただし、これは機械的な強制ではありません。旅情、人情、日常、後始末、補給、調査、共同体の運営を重く見るキャンペーンでは、どの段階でも採用して構いません。

逆に、短本数で鋭く終点へ向かう設計、毎回大きな敗着線が動く設計、最終局面の圧を主軸にする設計では、切るほうが向きます。

進行の基本ループ

  1. 各シナリオで今回の問いに答えを出す
  2. 終了処理でタグ整理と成長処理を行う
  3. GM が次回候補を 2〜4 本出す
  4. プレイヤーが候補を選ぶか、同等の別案を出す

候補の役目は分けると安定します。主野望を押す回 / 回復兼仕込み回 / 後始末や修復の回 / 横道の発見回 を混ぜると、自由宣言の方向が生まれます。

野望度・成長点・命運点

このモードでは、普段の回では野望度を原則変動させません。 野望度が動くのは、立場の更新、大きな勝利や敗北、価値選択の固定など、人生の節目だと感じる回だけで十分です。

通常の達成感は、成長点、世界課題タグの解決、Build 素片、長期資産ノード、絆・感情・秘密の進展 で返します。これにより、「終わりを急がない」ことと「何も進まない」ことを切り分けられます。

命運点の回復を、このモードだけで厚くしてはいけません。 穏やかさを作りたいときは、命運点を無料で戻すのではなく、危険の薄い候補や回復兼仕込み回を増やして調整します。

回復回と長期タグの整理

回復回は、全快だけして元に戻る回 にしません。少なくとも一つは、次の宣言が軽くなるものを残します。隠れ家、航路、紹介状、工房、協力者、謝罪、信用回復、記録整理のような軽い足場が向きます。

また、長く続くほど 長期タグの増えすぎ が最大の敵になります。終了ごとに、残す / 閉じる / 履歴へ送る の三択で整理し、似た向きのものは一枚へ圧縮してください。

通常キャンペーンへ戻す / ここで締める

リラックス・モードは、最終局面を禁止するものではありません。大きな敵や終わりの問いが見えてきたら、通常のキャンペーン進行へ戻して終点へ向かってよい です。

逆に、プレイヤーが「この辺りで十分満足した」と感じたなら、その節目で静かなエピローグへ入って終えてもかまいません。終点を必須にしない ことが、このモードの核です。

12. ソーシャル・モードの配置

ソーシャル・モードは、尋問、密談、交渉、審問、評議、派閥調整のように、表層サインと共有圧が主軸 の回へ置きます。毎回入れるより、人間関係の節目、情報開示の節目、立場が動く節目 に置くほうが呼吸が良くなります。

長編では、ソーシャルの成果を 情報開示 / 条件付き協力 / 関係タグや感情タグの変化 として持ち越すと、Stories 線が太くなります。本心の断定より、次の局面で何が言えるようになったか を残してください。

13. タクティカル・モードの配置

タクティカル・モードは、戦闘や難所にだけ 位置、共有圧、敵意図 を足す任意追加です。既定は 初級 / 中級ではなし、上級 / 熟練ではあり ですが、これは出発点にすぎません。シナリオの性質 と プレイヤーの希望 を優先してください。

キャンペーンでは、毎回付けるよりも、アークの山場、役割分担が主軸の救出、退路が主題の脱出 のような回だけへ置くほうが呼吸が良くなります。自由宣言を細切れにするためではなく、ここだけは位置と圧を読ませたい 回へ置いてください。

14. アンロック・モードの配置

アンロック・モードは、爆弾処理、封印解除、機械再起動、隔離工程、儀式停止、医療工程のように、構造の読解そのもの を主軸にする回へ置きます。長編では、一つのアンロック局面が別アークの鍵 になるように置くと、Quiddities 線と Builds 線の両方が立ちやすくなります。

置く回を決めたら、可視構成要素 / 既知の物理条件 / 構造圧 / 次の構造反応 / 共有安全状態 の公開欄と、核心障害 / 真のボトルネック / C1 / C2 / 最速敗着線 の作者メモを1セットで持ってください。ここを分けて管理すると、即死級処理が GM の気分になりにくくなります。

15. バーサス・モードの配置

キャンペーン設計書への追記欄

重要なのは、バーサスを置いても 世界反応を止めない ことです。教師、制度、警察、世論、怪異、締切のどれかを毎ラウンド最低 1 つは進め、PC同士だけで世界を閉じさせないでください。

  • 中盤の反転回 — 味方だったはずの派閥が別利益を優先する
  • 学園 / 都市の対立回 — 役職、証拠、世論、放送、後援を取り合う
  • 終盤前の離脱 / 護送回 — 同じ当事者や同じ退路の最終状態を奪い合う

どの回に置くか

既定は 初級 / 中級ではなし、上級 / 熟練ではあり です。ただし、シナリオの性質 と プレイヤーの希望 を優先してください。長編で常時バーサスにすると、関係管理と情報管理が遊びを食いやすいので、使う回を選ぶ ほうが安定します。

既定と卓宣誓

キャンペーンでバーサス・モードを使うときは、毎回入れる よりも、派閥対立が主軸の回、証拠や窓口を取り合う回、離脱や護送の成否を奪い合う回 に絞るほうが呼吸が良くなります。QBS のバーサスは、対立を別ゲームへ切り離すためではなく、自由宣言どうしの競合を整列するためのモードです。

16. ウィスパー・モードの配置

ウィスパー・モードは、複数PCキャンペーンへ 秘匿情報 と 個人別利益 を足す任意追加です。既定は 初級 / 中級ではなし、上級 / 熟練ではあり ですが、半協力をしたくない卓へ無理に入れる必要はありません。

どの回に置くか

毎回入れるよりも、中盤の反転回、関係が揺れる回、最終局面の着地点が割れる回 に絞るほうが扱いやすくなります。長編で常時秘匿にすると、情報管理が遊びを食いやすいからです。

長編で崩さないための基準

  • 共有主目的は一つ に固定する
  • 個人別利益 は別利益として置く
  • 主進行でないPCにも、Builds か Stories の進展 を必ず返す
  • 露見した秘密は、長期タグ / 関係 / 履歴 / 世界課題タグ へ変換する

秘密が回をまたいで残るなら、誰が知っていて、どの回で共通情報へ上がるか をキャンペーン側で先に持っておくと、長編でも散りにくくなります。

成長点の返り

個人別利益の達成は、既存の 追加で +1（最大2回まで） の中へ吸収します。条件は、共有主目的が達成されていること と そのPCの個人別利益が達成されていること です。共有主目的が未達なら、関係進展や秘密開示段階の前進で返すとぶれません。

21. PCごとの野望競合とスポットライト配分

複数PCキャンペーンでは、全員の野望を毎回同じだけ進める必要はありません。
ただし、今回主進行でないPCにも、利益か立場の変化を必ず渡す ほうが卓が安定します。

主進行PCと従進行PC

各回ごとに、野望が主に進むPCと、従として関わるPCを決めます。従進行PCには、手札の獲得、関係の変化、履歴の追加など、次回へ効く利益を用意してください。

競合を対立だけにしない

野望同士がズレるときは、単純な対立よりも、相互干渉にすると扱いやすいです。たとえば、誰かの成功が別PCの信用を傷つける、急ぐと救出はできるが証拠は残らない、という形です。

ウィスパー・モード を使うときも、競合はまず 相互干渉 として置くのが基本です。たとえば「急げば救出はできるが証拠は傷む」「このNPCを守ると別PCの立場が揺らぐ」のように、終着点の差 を作ります。共有主目的の失敗そのものが勝ち になる設計は、半協力ではなく対立卓です。

配分の目安

5本キャンペーンなら、各PCに少なくとも1回は主進行回を渡し、残りの回で混成にするのが目安です。8本以上なら、主進行の短いアークを複数回作れます。

関係進展とテーマ進展を主進行と同じだけ重く見る

複数PCキャンペーンでは、毎回すべての野望を同じだけ進める必要はありません。
ただし、主進行でないPCにも、次のいずれかは必ず渡したほうが安定します。

  • Build の進展: 成長点、素材、装備、拠点
  • Story の進展: 絆の深化、感情の変質、ドラマフラグ、秘密開示

キャンペーン主題があるなら、テーマ進展 も別に管理してください。
たとえば「信頼」「帰属」「救済」「継承」のような 1〜3 個を置いておくと、各回が何を深めたかを見失いにくくなります。

22. 単発シナリオをキャンペーンに組み込む方法

単発シナリオをそのまま並べるだけでは、長い流れの中で浮くことがあります。組み込むときは、その単発が何を進める回なのか と 何を次回へ残す回なのか を明記してください。

組み込み前の三問

  1. この単発はどの野望をどこまで進めるか
  2. この単発で増える長期タグ / 手札 / 履歴は何か
  3. この単発の結末は、次回の導入へどうつながるか

組み込みのやり方

  • 節目回として使う: 立場や関係が変わる回にする
  • 足場回として使う: 支援や手札を得る回にする
  • 代償回として使う: 長期タグを返す回にする
  • 息継ぎ回として使う: 回復と整理の回にする

単発で完結したPCをそのままキャンペーン化する

単発だけのつもりだったPCでも、終わったあとに「この人たちの続きを見たい」と思うことは珍しくありません。その場合は、単発の成功をやり直さず、単発の結末を履歴として固定したうえで、その結果生まれた未解決を長期線へ上げる のが基本です。

  1. 単発で答えが出たことを履歴へ固定する
    まず「何を達成したか」「何を失ったか」「誰とどういう関係になったか」を履歴として残します。単発の結末そのものは、長編の第1話でひっくり返さないでください。
  2. 続けて見たい核を三つ抜く
    そのPCについて、守れたもの / 失ったもの / 新しく背負ったもの を一つずつ抜きます。ここが、そのまま次の長期タグ、長期手札、継続要素の候補になります。
  3. 野望を「継ぐ / 広げる / 歪ませる」のどれかで置き直す
    単発の野望が未達ならそのまま継いでかまいません。達成済みなら、より大きい射程へ広げる、代償の返りで歪ませる、成功によって新しい責務が生まれた のいずれかで次の野望を立てるとつながりやすいです。
  4. 単発で得たものを長期資源へ振り分ける
    一時的な成功だったのか、今後も使える足場なのかを見て、長期タグ / 長期手札 / 履歴 / レギュラーNPC / 継続勢力 に振り分けます。
  5. 次の第1話は「成功後のコスト」か「成功で生まれた呼び水」を扱う
    長編の第1話は、前回より大きい敵を即座にぶつける必要はありません。あの成功のせいで新しい依頼が来た / 守ったものを維持する負担が生まれた / 勝ったことで別勢力に見つかった といった橋渡し回のほうが自然です。

単発用に薄く作ったPCだった場合も、すべてをあとから埋める必要はありません。卓が愛着を持った核だけを残し、残りは今後のシナリオで育てる ほうが、後付け感が薄くなります。

27. PC交代・途中参加・離脱・引退・死亡の処理

長編では、メンバー変動は珍しくありません。変動が起きたときに長期設計をどう直すか を最初から決めておくと、卓が止まりにくくなります。

途中参加

途中参加PCには、現在の主軸と最低限の履歴がわかる前回までのあらすじを渡し、最初の1本は足場獲得か関係形成を主目的にした回へ入れると安定します。

既定の合流タイミングは「次の主要局面の開始時」 としてください。いま解決中の宣言の途中へ新PCを割り込ませず、結果確定 → 現在状態の再掲 → 新PC導入 の順で入れると、初心者GMでも処理がぶれません。

状況既定処理理由
残り主要局面が2つ以上あり、1文で自然な導入が作れる今シナリオの次の主要局面頭から合流プレイヤーを長く見学にしないため
残りが最終局面だけ、または自然な導入に本文改造が要る新PCの本加入は次シナリオ第1局面から。今シナリオでは観測役・連絡役・同行NPCなどの仮役で関与させてよい不自然な乱入で構造を壊さないため
いま解決中の宣言の途中合流させない。現在の裁定を終えてから判断同一局面内の責任範囲を混ぜないため

【退場が確定した局面】
【残り主要局面数】
【次の自然な合流点】
【今シナリオで合流する / 次シナリオから入る】
【今シナリオで渡す仮役】
【再掲する最低限の履歴】

離脱と引退

離脱・引退するPCは、野望を未回収のまま放置せず、何を残して去るのか を履歴へ残してください。支援者化、別陣営化、消息不明化など、後で使える形にしておくと長編に馴染みます。

死亡と再出発

死亡や再出発が起きたときは、どの長期タグと履歴を残し、どの野望線を切るかを明記します。新PCを入れるなら、既存の勢力線やレギュラーNPCと接点を持たせると接続しやすくなります。

継承型の再出発

キャンペーンでは、単発と違って PC の退場をそのままゲームオーバーにしないでかまいません。死亡や再起不能が出たら、そのPCはいったん現キャンペーンの主役から降り、次のPCが野望線を継ぐ のを既定にすると saga らしさが出ます。

継承とは、能力や命運点を丸ごと譲ることではありません。未完の野望、残された履歴、未処理の借金、レギュラーNPCとの縁 を次の主人公へ渡すことです。新PCは初期値で入り、代わりに 継承物 を 2〜4 点だけ受け取ります。

継ぐもの継がないもの理由
主要履歴 1〜3 件先代の命運点命の余力は本人固有だから
長期手札 1 件先代固有の短期傷や短期疲弊状態異常の持ち越しを簡潔にするため
長期課題 1 件失われた大切なものそのもの喪失は履歴として残し、同一物を復元しないため
レギュラーNPC / 勢力との接点 1 件先代の野望度新主人公の到達度は別に測るため

継承PCの最初の 1 本は、継承が成立する回 にしてください。遺志の受領、資料の引き継ぎ、協力者への顔つなぎ、先代の失敗処理の肩代わりなど、現行の長期線へ自然につながる内容にすると止まりにくくなります。

途中継承の既定 もここで固定しておくと安全です。先代が第2局面で退場したなら、新PCはその場に瞬間転移したように出すのではなく、第3局面の入口 で「先代の遺した資料を受け取った」「依頼人が後任を差し向けた」「現地協力者が主役化した」など、いま見えている状況に接続する1文 で入れてください。逆に、残りがほぼ決着だけなら無理に混ぜず、次シナリオの第1話から本加入に回したほうが自然です。

【継承再出発シート】
【先代PC】

【継ぐ側の新PC】
【継がれる野望の言い換え】

「復讐を継ぐ」「師の研究を継ぐ」「先代が守ろうとした町を守る」「失われた証拠の代わりを集める」のように、同じ野望をそのまま写すより、自分の言葉へ書き換える ほうが新PCが動きやすくなります。

延命せず、退場を確定して継承へ渡す

継承手順がある以上、現キャンペーンで主役として復帰できないPCを、愛着だけで引っ張り続けない のが原則です。再起不能、任務達成・未帰還、野望度 0 / -1、主軸復帰不能のいずれかに入ったら、GMは「まだ少し動けるから」で局面を増やさず、退場処理に入ってかまいません。

状況既定処理
死亡確定即退場。継承PCか新PCを入れる
再起不能 / 長期療養現キャンペーンの主役から降板。支援者化するか、別キャンペーン候補へ送る
任務達成・未帰還 / 退出不能救出回をやるなら次の 1本 に限定。失敗したら継承へ移行する
野望度 0 / -1 で主軸喪失現行主軸には別PCを投入し、旧PCは離脱 / 引退 / 履歴化する
 
【継承への即時移行手順】
1. 退場理由を1文で確定する
2. 残す履歴 / 長期タグ / 接点を選ぶ
3. 新PCの野望を、先代の失敗か遺志に接続する
4. 継承成立回を1本だけ挟む
5. 以後は新PCを主役として扱い、旧PC延命のための局面を増やさない

この切り替えは冷酷さのためではなく、saga として前へ進むため の処理です。負けを曖昧にしないからこそ、次の主人公が何を継ぐのかが鮮明になります。

生存離脱PCを別の野望へ移す

再起不能だが死んではいない PC にどうしても愛着があるなら、その PC を即座に消さなくても構いません。現キャンペーンでは離脱 / 引退として固定し、別の野望を立てた別キャンペーンの主人公にする 運用を許可してください。

大切なのは、現キャンペーンの主軸と、新しい個人キャンペーンを同じ線に混ぜない ことです。現行キャンペーンでは、その PC は支援者 / 保護対象 / 留守番 / 行方不明として履歴化し、主役枠は別PCへ渡します。後日、新しい野望を立てるときは、その時点の傷、喪失、関係、残っている長期タグを前提に、野望度 1 から始めます。

【生存離脱PCの整理】
【このキャンペーンでここまでに確定した喪失】
【現キャンペーンではどの立場で残るか】
【別キャンペーンで立てる新しい野望】
【引き継ぐ長期タグ / 引き継がない長期タグ】
【再登場させるなら、どの距離感で出すか】

この扱いにすると、「この物語では退場したが、この人物の人生は終わっていない」 を両立できます。saga らしさを保ちつつ、ひとりの人物への愛着も切り捨てずに済みます。

生きている元PCへの愛着を残す選択は、現キャンペーンでまだ主役として粘る ことを意味しません。愛着は別野望や別キャンペーンへ預け、現行キャンペーンでは退場を確定させたうえで継承や交代を進めます。

28. 分岐・中断・打ち切り・圧縮終了

キャンペーンは、理想どおりの本数で終わらないことがあります。だからこそ、途中で圧縮して終える方法 を最初から持っておくと安全です。

分岐の扱い

すべての分岐を独立アークに増やす必要はありません。許容する分岐と、次の節目で収束させる分岐を分けてください。

選ばれなかった候補の扱い

候補提示をしたのに選ばれなかったシナリオは、消えたわけではありません。放置された候補は世界反応として前進する、あるいは 別勢力が先に手を打つ と決めると、選択の重みが出ます。プレイヤーに「何を選ばなかったか」が残ると、自発的な判断が長期へつながりやすくなります。

中断時の最小処理

中断するときは、少なくとも 現在の長期タグ / 手札 / 履歴 / 勢力状態 / 残っている問い を1枚にまとめて保存します。これがあれば、後日再開しやすくなります。

圧縮終了モード

残り1本で畳む:
- 最後の問いに必要な材料だけ残す
- 脇線は履歴化して回収を簡略化する
- レギュラーNPC は主要2本に絞る

残り2本で畳む:
- 1本目で長期タグと勢力線を整理する
- 2本目で最後の問いと主要決着だけを扱う

29. 長期資源インフレ / デフレの補正

長編では、手札タグや協力者が増えすぎて易しくなったり、逆に長期タグが重すぎて前へ出られなくなったりします。これを放置すると、後半が単調か停滞のどちらかになりがちです。

インフレ補正

  • 似た手札を統合する
  • 支援者に使用条件や反作用を付ける
  • 勢力側も学習して対抗する
  • 強い手札は一度使ったら履歴へ移す

デフレ補正

  • 回復回を1本挟む
  • 長期タグを段階的に軽くする条件を置く
  • 協力者や設備を再獲得できる短い回を入れる
  • 脇線を切って主軸へ集中する
  • 大局的敗北直後は、失地回復 / 借金整理 / 迂回準備 / 継承導入の候補を少なくとも 1 本混ぜる

長期資源のインフレを Quiddities / Builds / Stories の三面で見る

長編で強くなりすぎるのは、数だけではありません。世界側の前進量、恒久タグの層の厚さ、関係線の多さ も同時に膨らみます。

したがって補正は、Quiddities では世界側の要求水準、Builds では恒久資源の要求水準、Stories では関係の密度と秘密の返り を上げる三方向で考えると安定します。単純に敵を強くするだけだと、Builds は育っても Stories と Quiddities が薄くなります。

30. キャンペーン級の公平性チェックリスト

単発シナリオの公平性チェックに加えて、長編では次を確認してください。

  • 最後の問いに必要な材料が中盤までに見えているか
  • 主要分岐に事前の読める材料があるか
  • 各アークに回復の息継ぎがあるか
  • 同じ勝ち筋だけで全話を横滑りできないか
  • 単発差し込み回がつなぎの回で終わらず、野望や勢力線に接続しているか
  • 主進行でないPCにも利益か変化が渡っているか
  • 長期タグと手札の量が管理可能な範囲に収まっているか
  • 回復回が「休むだけ」で終わらず、次の仕込みや長期手札の獲得につながっているか
  • 次回候補の提示や自発案の採用など、プレイヤーが向かう先を選べる余地があるか
  • PCの得意分野に合わせすぎず、少なくとも 1 本は全PCに不利な方向から揺さぶる回があるか
  • 理不尽イベントが即時無効化ではなく、複数の局面と読める痕跡を持つ構造になっているか
  • 単発から長編へ移った場合、単発の結末を崩さずに長期の問いへ変換できているか
  • 発見回 / 文化接触回 / 挑戦・越境回 / 回復兼仕込み回 / 畏敬回 のうち複数が混ざり、同じ型だけが続いていないか
  • 反復して訪れる回復点と畏敬点があり、再訪時に意味や関係が更新されるか
  • 安全ルート / 近道 / 地元ルート の差が実際に選択肢になっているか
  • 世界反応が政治や敵対だけでなく、物流・祭礼・巡礼・検問・航路変化としても返っているか
  • PC がこの世界で名を残す方法が定義され、看板・歌・地図・噂・称号などの形で可視化されているか
  • 各アークに 開く / ひっくり返す / 押し切る / 深まる の配線があり、同じ熱だけで長く引っ張っていないか
  • 「最後に問うこと」が、達成だけでなく 意味の反転 と 関係の決定的変化 まで届いているか
  • 圧の負債帳 を更新し、未回収の借り・追加利益・伏線・救援を忘れていないか
  • 長期反転が 伏線収束型 / 正体露見型 / 前提崩壊型 のどれかに偏りすぎていないか
  • 再登場NPCが、再会 / 和解 / 裏切り / 救援 / 継承 / 鏡像化 のどれで戻るか決まっているか

セルフチェックの順番

  1. 最後の問いを言い直す
  2. その問いに必要な材料がどの話で出るか確認する
  3. 各話の役目が重複しすぎていないか確認する
  4. 停滞しそうなら圧縮終了モードへ切り替える

32. 最後に

キャンペーン設計で大切なのは、すべてを先に決め切ること ではありません。
変化に合わせて更新できる文書を持ち、次の1本へ切り出し、終わらせる手段も最初から持っておくことです。

1本ごとの歯ごたえは シナリオ設計ガイド、既存資料を回す運用は GMガイド、プレイヤー側の導線はプレイヤー向け3冊へ分けています。
この本では、長い流れを保ち、変化させ、着地させるための設計に集中してください。
