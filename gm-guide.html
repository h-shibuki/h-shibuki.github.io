Quick Build Saga GMガイド

0. はじめに

Quick Build Saga は、サイコロを振らず、プレイヤーの宣言を基本成功させ、その代わりに代償を支払わせるTRPGです。
PBM形式、つまり文章の投稿を往復させる遊び方を前提にしています。

GMの役割は、プレイヤーを失敗で止めることではありません。
成功を通したうえで、何を支払わせれば物語が前へ進むかを決めること です。

このゲームでは、キャンペーン → シナリオ → 局面 の順で話が小さくなります。
キャンペーン はPCが野望を達成するまでの道のり全体、シナリオ はその道中で起こる一つの事件、局面 はその事件を解決するために積み重ねる大事な一歩です。

このガイドは、初心者GMが先頭から順に読めば、そのまま準備と進行ができるように書いてあります。
流れは、世界観の提示 → PC作成 → キャンペーン設計 → シナリオ設計 → 導入と最初の局面提示 → 裁定 → シナリオ終了 → 次のシナリオ設計 の順です。

1. このゲームで使う言葉

TRPG

会話や文章のやり取りで物語を進めるゲームです。
参加者は、PCを動かすプレイヤーと、世界を担当するGMに分かれます。

PBM

「Play by Mail」の略です。
メール、掲示板、チャット、SNSのスレッドなどで、文章を投稿して進める形式です。

セッション

実際に遊ぶひとまとまりの時間です。
PBMでは一度に集まらず、投稿の往復で進むこともあります。
このガイドで「セッション開始前」と書くときは、最初の投稿を始める前 という意味です。

GM

ゲームマスターです。
舞台、NPC、脅威、時間経過、プレイヤーの行動結果を担当します。

GM気質

GMが、宣言の不足をどこまで補うか、同じ危険の再踏破をどれだけ厳しく扱うか、どこまで勝ち筋を見せるかを示す運用の傾向です。
このガイドでは、寛容 / 適正 / 峻烈 / 悪辣 の4段階で説明します。

PC死亡許容

その卓で、PCの死亡や退場がどれくらい起こりうるかの合意です。
低 / 中 / 高 のように、卓の開始前に目安を共有しておくと、命運点や後述する 致死級タグ の扱いがぶれにくくなります。

プレイヤー

PCを担当する参加者です。

PC

プレイヤーキャラクターです。
物語の主人公です。

NPC

GMが担当する人物です。
村人、兵士、商人、王、家族、協力者、敵、怪異などがここにあたります。

シナリオ

野望を達成するまでの道中で起こる、ひとまとまりの事件や問題です。
1本のシナリオは、複数の局面からできています。

局面

シナリオを解決するために積み重ねる、ひとまとまりの大事な場面です。
局面にはたいてい 期限・争点・脅威 があります。

宣言

プレイヤーが、その局面でPCがどう動くかを書く文章です。
このゲームでは、細かな一手ではなく、局面全体に対する大まかな方針 が書かれることが多いです。

裁定

GMが宣言を読んで、何が達成され、どんな代償や余波が起きたかを決めて返すことです。

主目的

その宣言で、プレイヤーがいちばん達成したいことです。
1宣言につき1つです。

手段

主目的を達成するために使う方法です。
力押し、調査、準備、交渉、祈り、魔術、変装、根回しなどがここに入ります。

タグ

このゲームで「タグ」とは、【】や〔〕のように書式が決まっている短い印の総称です。
PCや世界に残り、あとから参照しやすくするために使います。

タグには大きく2種類あります。課題タグ は、今すぐ、あるいは優先的に対処すべき強制的な状況です。手札タグ は、いつ・どう使うかをプレイヤーが選べる能動的なリソースです。用途をまだ固定したくない事実は、タグにせず 履歴 として残します。

書式の目安は次の通りです。

  • 課題タグ: 【名前｜期間: 今どう困るか / どうすれば軽減・解除できるか】
  • 手札タグ: 〔名前｜期間: 何に使えるか / どう失うか〕

期間の目安は、どちらも 短期 / 中期 / 長期 の3種類です。

タグはPCだけでなく、NPC、組織、町、噂、橋、調査の進み具合など、世界側に置いても構いません。

課題タグ

今すぐ、あるいは優先的に対処すべき「強制的」な状況です。
放置すると困り、宣言や処置で軽減・解除することが期待されます。

このゲームでは、成功の価格である代償タグも、その結果として広がる余波タグも、基本的には課題タグとして表現します。

例:

  • 【疲労｜短期: 次の局面では無理が利かない / 半日休めば軽減】
  • 【不信｜中期: 役人に疑われており協力を得にくい / 目に見える成果を示せば解除】
  • 【悪評｜長期: 街道筋で名が悪く広まっている / 次の土地で功績を立てれば解除】

手札タグ

PCや世界に残る、今後の宣言で使える「能動的」なリソースです。
支持、証拠、肩書き、協力者、安全地帯、貴重な所持品、使い切りの切り札などがここに入ります。

手札タグは、課題タグと違って今すぐ対処を迫るものではありません。 プレイヤーが「ここで使う」と宣言してはじめて力を持ちます。GMは、課題タグのように自動で軽減・解除させたり、勝手に消費したりしません。

書式の目安は次の通りです。

  • 〔名前｜期間: 何に使えるか / どう失うか〕

例:

  • 〔瞬間移動の巻物｜長期: 行ったことのある場所へ一瞬で行ける / 使用・破損・紛失で失う〕
  • 〔書記長の紹介状｜中期: 書庫や役所で顔が利く / 期限切れ・不正発覚で失う〕
  • 〔救国の名誉｜長期: 第一印象が良く、面会や事情聴取が通りやすい / 大失態・失脚で弱まる〕

履歴

用途をまだ決めない事実の記録です。
助けた人、退治した脅威、見つけた場所、交わした約束など、あとで意味を持つかもしれないが、今は効果を固定したくないものをここへ残します。

履歴はタグではありません。固定効果も、決まった書式も持ちません。プレイヤーが宣言で参照したときに、GMは既知情報、状況の変化、一時的な有利、継続する手札タグなどとして裁定して構いません。

履歴には、原則として用途を書きません。一行の事実記録で十分です。

  • 村人Aを助けた
  • 村近くの洞窟のゴブリンを退治した
  • 旧水路の抜け道を見つけた

代償タグ

成功のために直接支払った価格である課題タグです。
成功の値段そのものです。

余波タグ

代償や行動の結果として、あとから広がる面倒である課題タグです。
世界の反応や二次被害にあたります。

致死級タグ

解除できなければ、死亡・退場・生還失敗へ進む、効果の重い 課題タグ の総称です。
代償タグ か 余波タグ かは問いません。

致命傷 や 死線 は、その段階につける呼び名の例にすぎません。敵の一撃なら「致命傷」、飢餓や呪いなら「死線」「餓死寸前」「今が峠」などと呼んで構いません。

重要なのは名称ではなく、解除条件を満たせなければ死に進むことです。命運点が残っているだけでは防げません。

致命傷

致死級タグにつける代表的なラベルです。
敵の攻撃、事故、大出血など、身体損傷の文脈で使いやすい呼び名です。

機械的な扱いは他の致死級タグと同じで、解除できなければ死亡・退場に進みます。致命傷だけが特別な上位段階というわけではありません。

死線

致死級タグにつける代表的なラベルです。
呪い、病気、飢餓、寒波、極限環境、精神崩壊など、「今が峠」と示したい文脈で使いやすい呼び名です。

機械的な扱いは致命傷と同じで、解除できなければ死亡・退場に進みます。死線は致命傷より上位の別ルールではありません。

生還条件

主目的を達成したあとも、PCが生きて帰るために満たす必要のある条件です。
脱出、帰還、引き渡し、救命処置などがここに入ります。

命運点

PCの命は1つです。
命運点は、その1つの命に残った余力を何段階に分けて扱うかを示す値です。

  • 最大値（初期値）は GMが採用した「大切なもの」の数 + 1（開始時の標準はふつう 2〜3）
  • 0 で死亡、または冒険の主役としては再起不能
  • 通常は 課題タグが先、命運点は 最後の手段
イメージ
命運点 1 のPCは、一度の大きな無理で命そのものを賭けることになりやすく、命運点 2 や 3 のPCは、その同じ命を 1/2 や 1/3 ずつ削るように段階的に危険を受けられます。

ただし、命運点は残機ではありません。致死級タグや生還条件の不達成を、それだけで打ち消す回数ではありません。たとえば 【次のターンで死ぬ】 や、「致命傷」「死線」などの致死級タグが示されているなら、その課題タグを解除しないまま次へ進めば、命運点が残っていても死にます。

また、大切なものを永久に失ったときは、命運点の最大値が 1 減り、命運点も 1 減ります（下限 0）。

大切なもの

PCにとって、できれば失いたくないものです。
人、立場、持ち物、信念、居場所などから選びます。

  • 開始時の標準は 1つか2つ
  • 3つ以上 を候補にしてよいですが、命運点に数えるかどうかはGM確認で確定します
  • GMが採用した数が、命運点の最大値（採用数 + 1）に関わります

書かせるときは、名前 + その意味 + どう危険になりうるか を一言添えると運用しやすくなります。

  • 師の護符袋（師の継承者である証 / 奪われれば継承の正当性が揺らぐ）
  • 村の畑（家族の生活そのもの / 焼かれれば冬を越せない）
  • 妹の信頼（自分が人でいられる最後の支え / 裏切れば二度と戻らない）

野望

PCが長期的にかなえたい大きな目標です。

単発シナリオでは、長い人生目標ではなく、今回の話でかなえたい直近の目標 を野望として扱って構いません。
たとえば「迷い込んだ魔の森から脱出する」「ダンジョンの奥に眠る財宝を持ち帰る」などです。

野望度

PCが自分の野望にどれだけ近づいたかを表す値です。
戦闘力のレベルではなく、人生の到達度 です。

  • 初期値は 1
  • 標準ルールでは 5 は終点の目安
  • GMは開始時に、野望の大きさに応じて終点を 6 以上へ調整してよい
  • 単発シナリオでは、終点 2 を標準にすると扱いやすい
  • 0 で夢折れ。PCは引退するか、新しい野望を立てる

卓越宣言

目の前の障害に正面からぶつかるのではなく、障害の構造そのものを崩す宣言 です。
先回り、根回し、避難路の確保、証人の手配、敵の条件つぶしなどがここに入ります。

キャンペーン

PCが野望を達成するまでの道のり全体です。
複数のシナリオからできています。

長いキャンペーンでは、野望度や長期タグだけでなく、手札タグ や 履歴 も積み重なっていきます。

単発シナリオでは、その1本だけを短いキャンペーンとして扱って構いません。
このときは、終点を 2 にし、シナリオクリア（野望度が 2 になる）でキャンペーン終了です。

つまり、キャンペーンの中にシナリオがあり、シナリオの中に局面があります。
GMは、まずキャンペーンという長い道のりを設計し、その中の今回の事件をシナリオとして切り出し、さらにその中の一歩を局面として提示します。

2. GMは何をするか

GMの仕事は、世界を見せ、PCの野望に合う長い道のりを設計し、その都度の結果を返すこと です。
このゲームでは、まずキャンペーンを作り、その中の今回のシナリオを作り、その最初の局面を提示して進めます。

  1. 世界観を示す
    舞台、PCたちの立場、雰囲気を伝えます。事前に設定集があるなら、その要点を先に示します。
  2. PCを受け取る
    プレイヤーが作ったPCの野望、大切なもの、得意なやり方を読みます。
  3. キャンペーンを設計する
    PCが野望に届くまでの道のりを決め、何本くらいのシナリオで、どの節目を越えるかを文書として残します。
  4. 最初のシナリオを設計する
    その道のりの中で最初に起こる事件を決めます。問い、期限、失敗条件、主要局面の数を書き、これも文書として残します。
  5. 導入と最初の局面を提示する
    プレイヤーがすぐ宣言できるだけの状況と情報、そして選択肢例を示します。
  6. 宣言を裁定する
    主目的の達成、代償タグ、余波タグ、世界の変化を書いて返します。
  7. シナリオ終了後に更新する
    野望度、タグ、世界の変化を反映し、必要ならキャンペーン設計と次のシナリオ設計を書き直します。

重要なのは、キャンペーン設計とシナリオ設計を必ず文書として残し、裁定のたびに参照すること です。頭の中だけで回すと、野望、難度、代償の重さがぶれやすくなります。

3. セッション開始前に決めること

キャンペーン設計に入る前に、まず卓全体の土台を決めます。
ここで決めたことは、あとでキャンペーン設計書にも書き写してください。

設定がまだ決まっていない単発卓では、GMが最初から世界観を一人で確定しなくても構いません。まずプレイヤーから「どんな題材で遊びたいか」を短く集め、GMがそれを卓全体の共通前提に畳んでから、舞台、PCたちの立場、雰囲気へ落とし込んでください。

卓開始時に宣誓すること

厳しさの差そのものは問題ではありません。問題になるのは、その差が卓の途中で変わったり、説明なしに致死級裁定へ飛んだりすることです。

GMは卓の開始時に、少なくとも次の5項目を宣言してください。

  • GM気質: 寛容 / 適正 / 峻烈 / 悪辣 のどれで回すか
  • PC死亡許容: 低 / 中 / 高 のどれくらいを想定するか
  • 致死級タグ: 使う / 使わない と、主に使うラベル（致命傷 / 死線 / その他）
  • 生還条件の扱い: 主目的達成後も、生還や引き渡しを別に処理するかどうか
  • 卓運営保全: ルール無視の勝利主張・ジェイルブレイク・裁定拒否による進行破壊をどう扱うか

あわせて、開始時の大切なものの標準数、3つ以上はGM確認で確定すること、退場や生還失敗へつながる危険は危険予告として事前に示すことも共有しておくと、安全に回しやすくなります。

宣誓は卓の契約です。いちど宣言した厳しさは、原則として卓の途中で引き上げません。緊張感の高い卓をしたいなら、最初からそう言って始めてください。

告知テンプレート

GM気質: 適正
PC死亡許容: 中
致死級タグ: あり（主なラベルは致命傷。呪い・病気・飢餓では死線など別ラベルも使う）
生還条件: 主目的達成後も必要
大切なもの: 開始時の標準は1〜2、3つ以上はGM確認で可
危険予告: 退場や生還失敗につながる危険は事前に示す
卓運営保全: ルール無視の勝利主張・ジェイルブレイクは一度警告し、再発で即敗北・卓終了

設定未定の単発卓では先に題材を合わせる

個別希望をそのまま全部採用すると、卓の軸とPC作成の前提が散りやすくなります。GMは、遊びたいことと遊びたくないことを先に聞き、2〜3案のピッチにまとめてから、全員が遊べる1案を選んでください。

とくに単発卓では、次の項目を最初に合わせておくと、あとからホラーの強さや死亡許容で揉めにくくなります。

  • 遊びたい題材: 何をしたい卓か。例: ダンジョン探索、廃病院ホラー、生還もの
  • やらない題材・苦手な描写: 避けたい展開、見たくない残酷描写、触れてほしくない話題
  • PC死亡許容: 低 / 中 / 高 のどれくらいを想定するか
  • GM気質の希望: 寛容 / 適正 / 峻烈 / 悪辣 のどれで回したいか
  • 今回の共通目的: この1本でPCたちが何を達成したいのか

1) 舞台

どんな世界や土地なのかを決めます。
中世風ファンタジー、海洋冒険、怪異もの、陰謀劇、開拓ものなどです。

すでに舞台の説明文書があるなら、ここでプレイヤーへ共有します。
PC作成の前に舞台を見せることで、プレイヤーは世界観に合ったPCを作りやすくなります。

設定未定の単発卓なら、ここではプレイヤー希望を畳んだ共通前提を短く示します。歴史や固有名詞を盛り込みすぎず、PC作成と最初の宣言に必要な情報を優先してください。

2) PCたちの立場

プレイヤーがどんな人物を作る卓なのかを決めます。
村の若者、宮廷書記、見習い祓い手、流れ者の護衛、漂流船の生存者などです。

立場が揃っているほど、キャンペーン設計と最初の導入が作りやすくなります。

単発卓では、立場だけでなく PC同士の共通関係 を1つ決めると、最初の導入が大きく楽になります。

  • 同じ依頼を受けた仲間
  • 同じ学校、寺、部隊、商会の関係者
  • 同じ事故、肝試し、遭難に巻き込まれた当事者

3) 雰囲気

物語の肌ざわりを決めます。
冒険、成り上がり、陰謀、ホラー、悲劇、開拓などです。

雰囲気を決めておくと、あとで付ける代償や余波の質がぶれにくくなります。

とくにホラーや悪辣運用では、雰囲気だけでなく、苦手な表現と死亡許容もここで合わせてください。怖い題材そのものより、どこまで踏み込むかの期待値ずれが事故になりやすいからです。

4) 野望度の終点

標準の終点は 5 です。
ただし、これは「どんな野望でも 5 で達成する」という意味ではありません。

基準は、駆け出しのPCが、一地方で名を上げたり、有力者に届いたり、自分の仕事や術を残したりできる程度 を 5 と考えることです。
開始時点と到達点の隔たりが大きいなら、終点を上げてください。

目安:

  • 3〜4: 地位回復、一地方での成功、小さな復讐や救済
  • 5: 駆け出しから一人前、有力者、名を残す、術を残す、家を支える
  • 6〜7: 身分秩序の大きな飛び越え、広域に影響する成功、国家規模の発言力
  • 8以上: 国王になる、時代を変える、体制を作り替える

たとえば、

  • 「奴隷の身から解放されて平民になる」なら 4 前後
  • 「奴隷の身から国王にまで上り詰める」なら 7 以上

設計段階では、1シナリオで野望度が1上がる と見積もって構いません。
初期値は 1 なので、終点 5 なら標準 4シナリオ前後、終点 7 なら 6シナリオ前後 が目安です。

単発シナリオで遊ぶときは、その1本だけを 1シナリオのキャンペーン として設計し、終点を 2 にしてください。
初期値 1 のまま、今回のシナリオを達成すれば終点に届く形です。
このときの野望は、長い人生目標ではなく、今回かなえたい直近の目標 として扱って構いません。たとえば「迷い込んだ魔の森から脱出する」「ダンジョンの奥に眠る財宝を持ち帰る」などです。

5) 初期値

標準は全PC共通で次の通りです。

  • 大切なもの 1つか2つ
  • 命運点 GMが採用した大切なものの数 + 1（ふつう 2〜3）
  • 野望度 1

命運点は、プレイヤーが書いた数をそのまま自動採用するのではなく、後述の「8) 大切なものの採用基準」でGMが採用した数で確定します。

6) 代償の傾向

世界の色に合わせて、出やすい代償を決めておくと裁定がぶれにくくなります。

例:

  • 宮廷劇: 借り、不信、悪評、立場の揺らぎ
  • 開拓もの: 食料不足、時間経過、道具破損、疲労
  • 怪異もの: 穢れ、悪夢、注目、秘密の露見
  • 成り上がりもの: 推薦の喪失、商会の不信、昇進停止、野望度の停滞

7) 自由記述PCをQBSへ落とし込む

初心者プレイヤーは、記入欄を順番に埋めるより、自由文でPC像を書くほうがやりやすいことがあります。GMはそれをそのまま却下せず、仮置きしてからプレイヤー確認で確定する手順を取りましょう。GMが一方的に決めるのではありません。

  1. 文章を4つに分ける
    何者か / なぜ動くか / 何が得意か / 何を失いたくないか、の4要素に分けて読みます。
  2. QBSの欄へ仮置きする
    名前、立場、今回の野望、大切なもの、得意なやり方へ落とし込みます。
  3. 曖昧な箇所は候補を2つまでに絞る
    大切なもの候補A/B、得意なやり方候補A/Bのように、GMが読み取った候補を狭めます。
  4. 長期動機と今回の野望を分ける
    単発卓では「老後資金を得たい」のような長期動機と、「今回のダンジョンから財宝を持ち帰る」のような今回の目標を分け、野望欄には今回の目標を入れます。
  5. プレイヤーに確認して確定する
    「この理解でよいですか」と1回返し、合意を取ってから設計書へ転記します。

例:

プレイヤー自由記述:
人間の盗賊「ダリル」。貧民街で育ち、悠々自適の老後を迎えるために
ダンジョンで一攫千金を夢見ている。

GMの仮置き:
名前: ダリル
立場: 貧民街育ちの盗賊
長期動機: 老後資金を得たい
今回の野望: 黒灰迷宮の最奥財宝を持ち帰る
大切なもの候補: 貧民街の仲間たち / 隠し蓄え
得意なやり方候補: 隠密と偵察 / 罠外しと逃走

プレイヤー確認:
「長期動機は老後資金、今回の野望は迷宮の財宝持ち帰り、
大切なものは貧民街の仲間たち、でよいですか?」

8) 大切なものの採用基準

大切なもの に明確な上限は置かなくてかまいません。
ただし、命運点 に数えるかどうかはGM確認で確定する としてください。これが、初心者GMが押し切られないための基本線です。

標準と例外

  • 開始時の標準は 1つか2つ
  • 継続PC、ベテランPC、特別な単発では 3つ以上 を許可してよい
  • 途中増加も許可してよいが、物語上の獲得またはGM承認 を条件にする
  • 危険を見てから急に保険として増やす後出しは認めない

採用テスト

観点見ること
本当に痛いか失ったとき、そのPCにとって明確な喪失になるか。
個別かほかの項目と丸かぶりせず、別の危険として揺らせるか。
危険化できるか局面、人間関係、設定から自然に触れられるか。
管理できるか名前と意味が明確で、卓で把握可能か。
「妹」「家族」「家族との絆」を全部別採用する、「自由」「世界」「正義」を大量に並べる、「予備の拳銃1〜20」を別々に取る、といった書き方は、そのまま 命運点 に換算しないでかまいません。

よくある扱い

  • 団体・場所・理念 は、ひとまとまりとして危険化できるなら 1項目として採用してよい
  • 未登場・未紹介・オフスクリーンのままのもの は、設定欄に置いておき、命運点に数えない運用も可能
  • 増えすぎた場合 は、「現在のシナリオで有効化されている大切なもの」を明示して管理してよい

4. キャンペーンの設計

キャンペーンとは、PCが野望を達成するまでの道のりを示す設計 です。
このゲームでは、PCを受け取ったあとにキャンペーンを設計し、その文書を土台にして各シナリオを作ると安定します。

長いキャンペーンではなく、単発シナリオで遊んでも構いません。
その場合は、その1本だけを短いキャンペーンとして扱います。
プレイヤーには、長い人生の野望ではなく、今回の話でかなえたい直近の目標 を書かせてください。
野望度の初期値は 1 のまま、終点は 2 を標準にします。つまり、そのシナリオを越えれば終点に届く形です。

キャンペーン は長い道のり、シナリオ はその道中で起こる事件、局面 はその事件を解決するための大事な一歩です。
まず長い道のりを決め、次にその中の事件を並べ、実際の進行ではその事件を局面に分けて処理します。

1) PCの野望から長さを決める

まず各PCの野望を見て、野望度の終点を決めます。
次に、1シナリオで野望度が1上がる という目安で、何本くらいのシナリオが必要かを決めます。

ここでは細かい事件を先に決めるより、どの順で何段階ずつ上り詰めるか を考えるほうが初心者には向いています。

2) 各シナリオで何が進むかを置く

キャンペーンの中で、各シナリオがどの野望をどう進めるのかを一行ずつ置きます。
たとえば「後ろ盾を得る」「敵対者の妨害を越える」「失った立場を取り戻す」「名を上げる」などです。

この段階では、細かな局面までは書かなくて構いません。
まずは大きな節目を並べてください。

3) 最後に問うことを決める

キャンペーンの終盤で、PCに何を問うのかを一文で決めておきます。
たとえば「国に仕えるか、故郷を守るか」「復讐を遂げるか、名誉を守るか」などです。

最後の問いを先に決めておくと、途中のシナリオで何を積み上げるべきかが見えやすくなります。

4) 必ず文書として残す

キャンペーン設計は、頭の中だけで済ませず、必ず文書にしてください。
短くても構いません。後から見返して、今どこまで進んだかがわかること が大切です。

5) 裁定では設計書を参照する

宣言を裁定するときは、毎回キャンペーン設計書を開いてください。
その宣言が、どの野望、どの節目、どの最後の問いに関わっているのかを確認してから裁定すると、野望度の増減や長期タグの重さがぶれにくくなります。

6) PCの行動で更新してよい

設計書は、PCを縛る檻ではありません。
PCが同盟を得た、敵を早期に倒した、逆に大きく躓いた、野望の向きが変わった、といった変化が起きたら、そのたびに文書を書き直して構いません。

大切なのは、更新前の設計を見ずに裁定しないこと です。変更したなら、変更後の設計書を次回以降の基準にします。

5. シナリオの設計

シナリオとは、キャンペーンの道中で起こる一つの事件 です。
1本のシナリオは複数の局面からできています。

単発シナリオでは、この1本がそのままキャンペーン全体です。
今回のシナリオの問いが、そのままPCの野望の問いになって構いません。

キャンペーン設計を書いたら、その中の次の一歩を切り出して、今回のシナリオを設計します。
シナリオ設計も必ず文書として残し、裁定では毎回参照します。

1) 今回どの野望をどう進めるか

まず、このシナリオがどのPCの野望を、どんな形で進める話なのかを決めます。
野望と関係の薄い事件だけが続くと、キャンペーンが散りやすくなります。

単発シナリオなら、「今回かなえたい目標そのもの」を書けば十分です。

2) シナリオの問い

この話で、何に答えが出れば終わりなのかを書きます。
一文で言い切れる形にするとぶれにくくなります。

3) クリア条件

何が起きたら「解決した」と言えるのかを決めます。

局面達成とシナリオクリアは別です。局面で達成させるのは、その場面の問いに対する主目的です。シナリオのクリアは、ここに書いたクリア条件をすべて満たしたときだけ確定します。

「見つける」「奪う」「救い出す」「持ち帰る」「生還する」は、別工程なら別物です。退路、帰還、引き渡し、脱出が未解決なら、取得や救出の局面成功だけでシナリオクリアにしてはいけません。

4) 失敗条件

何が起きたら「もう手遅れ」と言えるのかを決めます。

失敗条件は、主要局面の残り数より優先します。主要局面が残っていても、失敗条件に触れたなら、その時点でシナリオ終了にしてかまいません。

とくに死亡許容が高い卓では、未処置の致命傷、退路のない崩壊現場、追っ手からの逃走不能、致死環境からの未脱出など、生還に必要な条件が満たされないまま終えようとした時点で失敗条件に接触したものとして扱います。

5) 期限

いつ、あるいは何が起きたら世界が強制的に動くのかを書きます。
期限があると、PBMでも物語が停滞しにくくなります。

潜入、探索、救出、運搬のシナリオでは、行きだけでなく帰路の期限も置いてください。行きに時間がかかったなら、帰りにも相応の時間や危険がかかると考えるほうが自然です。

設計書には、帰還条件 / 退出条件、帰路で増減する期限、帰路で再掲する危険予告 も書いておくと、取得直後の甘い打ち切りを防げます。

6) 主要局面の目安

PBMでは、3から5局面 を標準にするとまとまりやすいです。
容易の回復局面や短い後日談は、ここに含めなくても構いません。

多少の融通はあってよいですが、長くても5から8局面まで です。
8を超えるなら、設計か進行が伸びています。次の投稿で、終わりに向けて世界を強く動かしてください。

5局面目に入ったら終盤 です。
以後の局面は、最後の準備、最後の対決、結果の確定のどれかに寄せます。

局面を増やしてよいのは、中心問題の答えが変わりうる ときだけです。
答えが変わらないなら、情景処理や短い結果描写で流して構いません。

各主要局面には、その場面が答える局面の問いを1文で置いてください。たとえば「正面突破するか、先に退路を作るか」「証拠を集めるか、審問そのものを遅らせるか」のように書くと、初心者GMでも局面の役割を見失いにくくなります。

局面の問いは、常に最終目的そのものにしなくて構いません。寛容 / 適正では最終目的に近く置いてよく、峻烈 / 悪辣では「次の一手」「手段の成否」「周辺目的」に一段浅く置いて構いません。たとえば「敵に勝つ」ではなく「敵と戦う」、「脱出する」ではなく「出口へ向かう」「橋を渡る」のように置けます。

ただし、その先にある核心障害は事前に予想可能でなければなりません。少なくとも 存在が見える / 対策の方向が読める / 仕込む機会がある の3条件を満たすように設計してください。無工夫宣言で保証されるのは、その局面の問いに対する達成だけであり、帰還、退出、持ち帰り、生還、追手処理は別工程のままで構いません。

主要局面数は、保証された生存ターン数ではありません。 標準の3〜5局面は、有益な宣言が続いた場合の目安です。失敗条件に触れたなら、1局面目でもそのまま終了してかまいません。

また、シナリオ設計が5局面を想定していても、峻烈や悪辣で必ず5局面目まで引き延ばす必要はありません。答えが変わらない直球宣言を無条件に付き合い続けるのではなく、難度に応じて早めに敗着線へ触れさせてください。

  • 寛容: 5回目で失敗条件に触れうる
  • 適正: 3回目で失敗条件に触れうる
  • 峻烈: 2回目で失敗条件に触れうる
  • 悪辣: 1回目から敗着線に触れてよい

7) 必ず文書として残す

シナリオ設計書には、今回の問いだけでなく、どの野望にどう触れる話なのか も書いてください。
これがあると、シナリオ終了時に野望度を上げるかどうかを判断しやすくなります。

8) 語り口プロフィールを先に決める

PBMでは、導入、幕間、エピローグが小説として読めるだけで、進行の手触りがかなり良くなります。
そのため、セッション開始前か、少なくとも最初の投稿前に、語り口プロフィール を決めておくと運用が安定します。

ただし、語り口は PCの人格を書き換えるためのものではありません。
プレイヤーが明示していない信条、秘密、恋慕、将来の誓い、心変わりを、GMが勝手に断定してはいけません。PCの内面を書くときは、これまでの宣言や既知の設定から無理なく読める範囲にとどめます。プレイヤーがPCの口調、一人称、話し方を明示しているなら、そちらを優先してください。

9) GM気質ごとの既定の語り口

プレイヤーから文体指定がないときは、GM気質に合わせて語りの温度を少し変えてよいです。

  • 寛容 / 適正: うまくやれてしまう寄りの軽快さを強めます。少し自信過剰なくらいの若者らしい軽口、会話の勢い、無自覚に切り抜けてしまう感じを混ぜてよいです。いわゆる「また、俺何かやっちゃいました？」寄りの読み味を少し混ぜてよいですが、PC本人の格や口調を壊すほど軽薄にはしません。
  • 峻烈 / 悪辣: 追い詰められた寄りの苦みを強めます。「何で自分がこんな目に」というぼやき、皮肉、冷笑、やせ我慢を混ぜ、危機が近いほど笑えなさを増してください。ただし、ただ陰惨にするのではなく、読み味そのものは軽快さを残します。

同じ「ラノベ風」でも、寛容 / 適正では「少し調子に乗れる軽さ」、峻烈 / 悪辣では「軽口で正気をつなぐ苦さ」を目安にすると書き分けやすいです。

シナリオ設計書には、クリア条件や期限だけでなく、どこに核心障害を置き、どの伏線から先回りできるか を書いてください。悪辣ほど再掲は減らしてよいですが、後から見れば読める手がかりであることは崩さないようにします。

6. GMの最初の投稿はどう書くか

シナリオ設計を書いたら、その内容を最初の投稿へ落とし込みます。
局面 とは、その事件を解くために積み重ねる大事な一歩です。最初の投稿では、その一歩をプレイヤーがすぐ踏み出せるようにします。

この骨格は最初の投稿だけでなく、各局面の提示でも使います。
とくにPBMでは、プレイヤーが直前のGM投稿だけを見て判断できることが大切です。2局面目以降は、前回の結果要約、PCの現在状態、世界の現在状態、危険予告を必ず再掲してください。

初心者卓で毎回の局面投稿に入れる最低構成は、次の 12点 です。

前回の結果要約

最初の投稿では不要です。2局面目以降では、直前の結果を 2〜4行 で再掲します。
何を達成し、何を失い、何がまだ残っているのかを短くまとめてください。

今回のシナリオの問い

今このシナリオで最終的に何へ答えようとしているのかを、毎回 1文 で再掲します。局面をまたいでも、いま何の話をしているのかがぶれにくくなります。

1) 導入

なぜこのシナリオが始まるのかを、短く示します。
キャンペーン全体の流れから、今回の事件へつながる橋の部分です。

2) 現況

今どうなっているかを書きます。
プレイヤーが「では何をするか」を考えるための手がかりです。

3) 期限

いつまでに何とかしないといけないかを書きます。
「あと三日」「残り二局面」「夜明けまで」 のように、残り時間や残り手数が見える形で再掲してください。

4) 争点

今回の局面で、何が問題なのかを書きます。
守るもの、失うもの、競っている相手が見えると宣言しやすくなります。

局面の問い

この局面で、プレイヤーに何を選ばせたいかを 1文 で書きます。これはシナリオ全体の問いより小さく、今ここで答える問いです。例: 「正面突破するか、先に退路を作るか」。

PCの現在状態

各PCについて、命運点、野望度、有効タグ、有効手札タグ、主要な履歴、大切なものの変化 をまとめて再掲します。
複数PC卓では、プレイヤーが自分の行動判断を誤らないよう、全員分を箇条書きで並べるのが安全です。

世界の現在状態

世界側やNPC側に残っている課題タグと手札タグ、参照されうる履歴、敵やNPCの動き、残り期限、止め方や遅らせ方が見えている要素を書きます。
プレイヤーが「いま何に追われ、何を止められるか」を判断できるようにしてください。

5) 既知情報

PCが最初から知っていることです。

ここには、真相の手がかりだけでなく、方針決定に必要な前提情報も含めてかまいません。現在日時、距離や移動の目安、その場で見えているもの、同行者、PCなら当然知っている一般常識などです。

また、局面の背後にある核心障害へプレイヤーが先回りできるよう、伏線は少なくとも 存在が見える / 対策の方向が読める / 仕込む機会がある の3条件を満たすように置きます。橋が落ちるなら老朽化が見え、アンデッドが相手なら通常手段が効きにくい徴候が見える、といった形です。

再掲の強さはGM気質で変えてよく、寛容 / 適正では小説パートに出した核心情報を既知情報にも再掲してかまいません。峻烈では既知情報までにとどめ、悪辣では小説パートや前景描写だけに置き、既知情報へは再掲しなくて構いません。

PBMでは、この前提が抜けると宣言前の往復が増えやすくなります。迷ったら、プレイヤーが「どうするか」を決めるために必要な情報は先に書いてください。

世界設定の説明は、今の方針決定に必要なぶんを優先してください。長い歴史や固有名詞の羅列は、固定のまとめ投稿へ回して構いません。最初の宣言に使わない情報を、その場で全部読む必要はありません。

危険予告

タグや現況から見て、次に同じ無理を押したら何が起こりうるかを示します。たとえば 「【重傷】のまま正面戦闘をすると命運点を失うおそれが高い」 のように書きます。これは甘やかしではなく、プレイヤー判断に必要な前提情報です。

ただし、危険予告はすべての核心障害を種明かしする欄ではありません。寛容 / 適正では小説パートで出した核心情報を危険予告にも再掲してよいですが、峻烈では危険予告にまでは書かず、悪辣では核心障害に関する再掲をしません。悪辣で必要なのは、警告の不在ではなく、読める伏線を先に出しておくことです。

6) 難度

この局面がどれくらい厳しいかを書きます。

7) 選択肢例（推奨。初心者卓では必須）

選択肢例は、すべての卓で出すことを推奨します。
とくに初心者卓では、プレイヤーが「何を宣言すればよいか」をつかみやすいよう、必ず2から4個まで の選択肢例を示してください。

これは例であって、強制でも全候補でもありません。
そのまま選んでよいですが、ふつうは卓越宣言にはなりません。 卓越は、選択肢を組み替えたり、先回りの準備を足したり、争点の構造を崩したときに与えます。

公開する選択肢例には、GMの想定結果・期待値・おすすめ度を書きません。
「最も確実」「時間はかかるが安全」「これを選べば命運点1で済む」のような括弧書きは、原則としてGM用メモにのみ書いてください。公開してよいのは、PCが今知っている事実と、危険予告として既に共有された一般的危険だけです。

寛容な初心者卓に限り、「安全寄り」「時間がかかる」などの大まかな手触りを添えても構いません。
ただし、結果確定やGM評価が見える書き方にはしません。

選択肢例の差は、GMが勝ち筋をどこまで見せるか に出ます。
初心者卓で峻烈や悪辣を使うときでも、選択肢例そのものは出します。 出し方だけが変わります。

  • 寛容: GMが思う妥当解や安全策を、必ず1つ以上 入れます。プレイヤーがそのまま選んでも前に進めるよう、勝ち筋を見せます。
  • 適正: 筋のよい選択肢を2〜3個示します。GMが思う妥当解を入れてもよいですが、必須ではありません。勝ち筋を隠さないが、保証もしません。
  • 峻烈: 「戦う」「退く」「交渉する」「聖具を探す」など、行為の類型か周辺目的だけを必要最低限で示します。GMが思う妥当解や安全策には触れません。
  • 悪辣: 選択肢例は出しますが、GMが思う妥当解や救いは隠します。 そのうえで、結果ではなく、PCが今切れる一手・手段・周辺目的を書いてよいです。たとえば「脱出する」ではなく「出口へ走る」「橋を渡る」、「敵に勝つ」ではなく「敵と戦う」「聖具を探す」のように出して構いません。保身、屈服、裏切り、責任転嫁、短慮など、PCに悪意や屈辱を感じさせる選択肢 を前面に出してよいですが、嘘の案 を出してはいけません。

悪辣の前面選択肢は、安全策の一覧ではありません。 どれも局面を一応前へ進める案であってよいですが、少なくとも一つの重い請求――即時の身体代償、退路や時間の喪失、支援や信用の喪失、命運点減少の危険、野望への足がかりの破壊――を確実に伴わせてください。

悪辣の選択肢例に、他より一方的に得な優越選択肢を置いてはいけません。 保身・裏切り・囮・屈服を前面に出してよいですが、そのまま選んだ場合は短期にも中長期にも請求が来る形にしてください。

また、選択肢が真正面からぶつかる重大障害は、事前に読める形で置いておきます。寛容 / 適正では既知情報や危険予告にも再掲してよく、峻烈では既知情報まで、悪辣では小説パートや前景描写だけでも構いません。

つまり、寛容は勝ち筋を見せる、適正は勝ち筋を隠さない、峻烈は勝ち筋に触れない、悪辣は勝ち筋を隠して悪い道を見せる と考えると運用しやすいです。

悪辣は口調ではありません。 必要な情報は契約どおり示したまま、救いを隠し、より醜い道を前面に出し、隙を見せたPCには重い代償を積み上げます。丁寧語や礼儀正しい文章でもかまいません。
悪辣は「救いを潰す」のではなく、「救いを隠して、より醜い道を勧める」運用です。

たとえば護衛任務で強敵が現れたなら、寛容は「護衛を逃がしつつ時間を稼ぐ」のような妥当解を含め、適正は「勝てないのを承知で戦うか」「護衛を見捨てて逃げるか」のように並べ、峻烈は「戦うか」「退くか」だけを示し、悪辣は「護衛対象を差し出す交渉を試みるか」「泣きわめいて命乞いするか」のように悪意ある手を見せてよいです。ただし悪辣でも、差し出した護衛の反撃、退路の消失、信用崩壊など、重い請求は必ず付けてください。

  • ゲーム情報パート: いま判断に必要な情報です。現在の状況、期限、争点、既知情報、危険予告、難度、選択肢例を担当します

寛容 / 適正では、小説パートに出した核心情報を既知情報や危険予告にも再掲してよいです。峻烈では既知情報に再掲してよいですが、危険予告では直接言わなくて構いません。悪辣では、小説パートや前景描写だけからプレイヤーが自力で読み解く運用にしてよく、既知情報にも危険予告にも再掲しなくて構いません。

ただし、どのGM気質でも、後から見れば読める手がかりであることと、顕在化前に仕込める余地があることは崩さないでください。読解の余地は残しても、理不尽な不意打ちにしてはいけません。

プロローグ

PC作成後から、最初の局面を出すまでの短い小説パートです。
役割は、PCをその世界へ立たせ、今回の共通目的と失敗条件に感情をつなぎ、最初の局面を踏み出したくさせることです。

  • 目安: 400〜800字くらい
  • 入れたいもの: 会話かモノローグを 1つは入れる
  • 締め: 最後の 1〜2文で最初の局面に接続する

プロローグでは、「何が起きるか」よりも「このPCが、どんな気分で最初の一歩を踏むか」を見せるつもりで書くと、読ませやすくなります。

幕間

結果返答のあと、次の局面の前に置く短い小説パートです。
役割は、さっき支払った代償が身体や心や状況にどう乗ったかを見せ、次の局面の空気を作ることです。

  • 目安: 150〜350字くらい
  • 入れたいもの: 直前の達成と代償を、感覚や会話で 1回なぞる
  • 締め: 最後の 1文で次の問題を立ち上げる

幕間では、説明を増やすより、「代償の後味」と「次に行かざるを得ない感じ」を作ることを優先してください。

エピローグ

シナリオ終了後の小説パートです。
役割は、シナリオの問いへの答えを、生活、身体感覚、人間関係、風景の変化として見せることです。

  • 目安: 400〜900字くらい
  • 入れたいもの: 今回得たものと失ったものを、どちらも場面として見せる
  • 締め: 最後に、次シナリオへの火種か、残った傷の残響を置く

エピローグでは、説明のまとめで終わらず、「今回の事件のあと、世界がどう見えるようになったか」で締めると、短編として読み味が出ます。

PC内面を借りるときの注意

GMは、PC視点の小説パートを書いてよいですが、勝手にPCの核心を書き換えてはいけません。

書いてよいもの:

  • その場の痛み、寒さ、疲労、焦り
  • 目の前の事実への驚き、嫌悪、ためらい
  • これまでの宣言や設定から自然に読める範囲のぼやきや軽口

書いてはいけないもの:

  • プレイヤーがまだ決めていない信条
  • 秘密の過去や新事実
  • 恋愛感情や忠誠の確定
  • 将来の誓い、心変わり、価値観の転換の断定

迷ったら、PCの内面は薄くし、情景と会話で読ませてください。

7. 宣言をどう読むか

宣言が届いたら、まずキャンペーン設計書とシナリオ設計書を開き、今どの問いに答えようとしている局面か を確認します。
そのうえで、次の順で読み解きます。

1) 主目的を1つ見抜く

工夫がいくつ並んでいても、いちばん達成したいことは1つです。

2) 手段を見る

どの方法でそれを達成しようとしているのかを見ます。
GMが提示する代償タグは、この手段から自然に出るものにする のが基本です。

例:

  • 力押し: 怪我、疲労、破損
  • 調査: 時間経過、発覚、借り
  • 交渉: 貸し借り、誤解、立場の低下
  • 魔術: 穢れ、消耗、怪異の注目
  • 準備と段取り: 時間、資材不足、他所の遅れ

3) 守りたいものを見る

プレイヤーが明記した「守りたいもの」は尊重します。
余波で揺らすことはできますが、同じ意味で真っ向から踏み抜かない のが基本です。

4) 捨ててもよいものを見る

プレイヤーが差し出してよいと書いたものは、代償設計の重要な材料です。

5) 代償提案は「相談」だと理解する

プレイヤーからの代償提案は歓迎してよいですが、免責 ではありません。

  • 十分重ければそのまま採用
  • 少し軽ければ採用しつつ余波を足す
  • 明らかに軽すぎれば、別の代償も必要

と考えます。

6) 宣言と設定の食い違い・不備をどう扱うか

PBMでは、宣言に 空欄、矛盾、局面不適合 が混じるだけでなく、プレイヤーが「未定義だと思っていたこと」が、GM側ではすでに決まっていることもあります。

この節では、そうした 宣言と設定の食い違い・不備 をまとめて扱います。
このときも、書き直し待ちで止めず、今ある宣言を材料に1回の裁定として返すのが基本です。

ただし、認識のすれ違いを直すだけで足りる場面では、説明だけで時間を進めません。時間経過や損害は、PCが実際に時間を使った、危険に触れた、局面を放置したと自然に言えるときだけ与えてください。
  1. まず認識違いかどうかを見る
    最初から世界のすべてが定義されているとは限りません。局面判断に不要な余白は、既知情報、PC設定、直前の展開に反しない範囲で補ってかまいません。
    ただし、プレイヤーが補ってよいのは、今の判断に必要な周辺情報までです。GMのみが知る秘匿情報、謎の正答、攻略条件、敵の真の弱点、今後の展開を決める宣言は、プレイヤー側で確定してはいけません。
  2. 共有済み設定と矛盾する宣言は、矛盾部分だけを不可能として扱う
    宣言全体を丸ごと無効にするのではなく、可能な部分と不可能な部分を分けるのが基本です。可能ならGMは、ただ「無理」と返すのではなく、今の局面で実行可能な規模へ言い換えて返してください。
    却下それ自体に、原則として追加罰は付けません。時間経過や不利益を与えるのは、その行動を実際に試みた結果として自然に起きるときだけです。とくに 【重傷】、命運点減少、退場級処理 は、危険予告済みであり、なおかつその危険へ本当に触れた場合に限って適用してください。
  3. GM側では既定だったが、プレイヤーには未提示だった場合
    その先の謎解きや秘匿情報を壊さない範囲で、今の判断に必要な最小限の情報を開示します。GMの説明不足なら 【回答】、PCなら知っていてよい範囲なら 【既知】 として扱ってかまいません。
    この 認識合わせだけ では、原則として時間を進めません。ただし、PCが実際に調査・接触・試行を行い、その結果として既定の設定にぶつかったなら、軽い時間経過を与えてよいです。
  4. 長い宣言は、中核工程 / 追加利益 / 演出に分けて読む
    プレイヤーが長く書いたときは、まず 主目的を1つ 取り出します。
    中核工程は主目的達成に本当に必要な工程、追加利益は無傷・退路確保・証拠回収・副目標達成など、演出は雰囲気づけや見せ方です。
    難度5段階の代償相場は、基本的に主目的だけを書いたときの目安です。追加利益を同時に取りに行くなら代償や余波を上乗せしてかまいません。演出だけでは重くしません。
  5. 追加ペナルティを検討してよいのは、認識違いではなく進行妨害の反復だけ
    GMが既定設定を訂正したあとも同じ矛盾宣言を繰り返す、却下された確認や不可能宣言を角度だけ変えて連打する、PCの知りえない秘密を前提にした宣言を訂正後も押し通そうとする――このような場合です。
    この場合も、まずは短く原則を伝え、追加確認を打ち切るのが先です。それでも進行が止まるなら、「躊躇」「空転」「強行」の1回の宣言として裁定し、時間や脅威を進めてかまいません。
    ただし、認識のずれそのものに対して身体損害を与えるのは避けてください。身体損害や致命的処理は、あくまで危険な行動を本当に試みた結果として与えます。

標準運用は「適正なGM」と考えてください。
「寛容」「峻烈」「悪辣」は、意図の救い方や支払いの重さを変えるための差であって、認識違いだけで重罰に飛ぶ免罪符ではありません。

後出しの扱い

後出しは一律禁止ではありませんが、同じ重さでは扱いません。

  • 設定補完: そのPCの立場や常識の範囲で自然な持ち物・手配・同行者です。認めてよいですが、それだけで卓越にはしません。
  • 仕込み回収: 既出情報、タグ、履歴、関係、以前の宣言に支えられた準備です。これは卓越の有力候補です。
  • 突破資源: いま見えている失敗条件を専用にひっくり返す新出の切り札です。新しい脱出アイテム、万能鍵、都合のよい専用薬などがここに入ります。これは卓越の根拠にせず、通すなら重くするか、却下してかまいません。

見分ける基準は、そのPCなら元から持ちうるか / 既出情報から辿れるか / 顕在化前に仕込めたか / 解くのが一工程か局面全体か です。危険が見えたあとに都合よく専用切り札が生える運用は、寛容でも慎重に、峻烈や悪辣では原則として安く通さないでください。

寛容なGM

未提示情報にぶつかったときは、意図を汲んで今できる規模へ縮めて返してよいです。認識ずれの修正では、原則として時間を進めません。不可能な工程があっても、主目的や最低限ラインが残るなら、そちらを優先して通します。

  • A. 空欄がある宣言: PCらしさとこれまでの流れから、もっとも自然な内容で補ってよいです。ただし、補った欄では卓越を与えず、明記されたときほど強く守りません。
  • B. 矛盾を含む宣言: プレイヤーの意図に近い形へ言い換え、今の局面で可能な規模へ縮めて返してよいです。無理押しのぶんだけ代償は一段重くしてよいですが、立て直しの道は残します。
  • C. 局面にそぐわない宣言: 休息や立て直しの宣言として扱ってよいです。小さな回復や整理の利益を与えてよいですが、期限は進み、中心問題は悪化します。

適正なGM

矛盾部分は切り落とし、可能な部分だけを通します。認識ずれは最小限の情報開示で直し、説明だけでは時間を進めません。追加利益を欲張った分だけ、代償や余波を重くします。

  • A. 空欄がある宣言: 空欄は GMに一任 として扱います。手段が空欄なら卓越なしの標準処理、守りたいものが空欄なら特別保護なし、捨ててもよいものが空欄なら代償はGMが選びます。
  • B. 矛盾を含む宣言: 両取りさせません。成立する行為だけを起こし、成立しない成果は返しません。
  • C. 局面にそぐわない宣言: 現在の争点を放置した宣言として処理します。行為に見合う小さな利益だけを返し、期限と脅威は通常どおり進めます。失敗条件に届くなら、そのまま届いてかまいません。

峻烈なGM

書いていない保護や準備は拾わず、書かれた範囲でのみ裁定します。
ただし、未提示情報との食い違いだけで重罰にせず、危険へ踏み込んだ結果として重く処理します。不可能な成果は返しませんが、可能な部分まで消さないようにしてください。

  • A. 空欄がある宣言: 空欄は補いません。書いていないものは宣言していないものとして扱います。主目的しか書かれていないなら、準備も保護もない強行です。
  • B. 矛盾を含む宣言: 矛盾はプレイヤー有利に解きません。両立しないなら、不可能な成果は不可能のまま返します。行動は起きても、望んだ成果は得られず、損害や悪化だけが残ってよいです。
  • C. 局面にそぐわない宣言: 争点の放棄です。その局面に対する成果はなく、期限、脅威、失敗条件は記述どおり進みます。手遅れになるなら、そのまま手遅れです。

悪辣なGM

この運用は、卓全員の事前合意があるときだけ使うのが安全です。
認識の救いを最小限にしてよいですが、危険は予告し、裁定は契約に従います。無工夫宣言や強行には重い請求をしてよいですが、未提示情報にぶつかっただけで理不尽な重罰を与えないこと。退場級処理は、契約済みの死線と、その到達条件が具体に示されている場合に限ります。

  • A. 空欄がある宣言: 空欄は、もっとも不利な合理的読みに落としてよいです。手段がなければ、最短、最正面、もっとも目立つやり方で実行したものとみなします。
  • B. 矛盾を含む宣言: 利益になる読みを採らず、不利益になる読みを採ります。無理な勝利宣言は、無策で突っ込んだものとして処理してよいです。勝利は返さず、その無謀さに見合う重い代償と余波を与えます。
  • C. 局面にそぐわない宣言: 放棄、怠慢、現実逃避のいずれかとして扱います。期限と脅威を最大限に進め、失敗条件に触れたら、ためらわず確定します。

悪辣の無工夫宣言

プレイヤーが選択肢例をほぼそのまま採用し、追加の段取り、守りたいもの、捨ててもよいもの、退路、補足を書いていない宣言を、このガイドでは 無工夫宣言 と呼びます。

無工夫宣言でも、通してよいのは その局面でGMが問った内容 だけです。局面の問いが最終目的そのものならそこまで通してよく、問いが「敵と戦う」「橋を渡る」「出口へ向かう」のような一段浅い内容なら、そこだけを達成させます。

悪辣では、無工夫宣言だけで 退出 / 持ち帰り / 引き渡し / 生還 / 追手処理 / 隠れた核心障害の解決 までまとめて通してはいけません。とくに過酷 / 絶望や節目局面では、きれいな収まりや副目標の同時達成を原則として与えません。残り手番がないなら、痛み分け、生還失敗、達成と同時死亡になってかまいません。

プレイヤーが自分で段取り、先回り、守りたいもの、捨ててもよいもの、退路を書き足したなら、無工夫宣言ではありません。通常どおり、必要なら卓越も含めて裁定します。

悪辣の価格下限

悪辣で、しかも無工夫宣言なら、局面の問いは通しても、支払いには下限を設けます。
「そのまま選んだだけなのに、ほとんど損をしない」を避けるためです。

侵入権 / 退出権 / 持ち帰り / 生還条件 を直接問う場面を、このガイドでは 節目局面 と呼びます。悪辣かつ無工夫宣言で節目局面に臨むなら、価格下限は 1段階繰り上げ ます。ただし上限は絶望相当までで構いません。

目安は次のとおりです。

  • 普通: 短期課題タグ 1つ、または期限 / 位置 / 退路の小さな悪化を最低 1つは残す
  • 困難: 短期課題タグ 1つに加えて、余波タグ、期限進行、敵側前進のどれかを重ねる
  • 過酷: 中期課題タグ 1つ、命運点 -1、野望への足がかりの傷、大切なものへの傷のどれかを必ず含める
  • 絶望: 命運点 -1 に加えて、中期以上のタグ、野望への足がかりの喪失、大切なものへの深い傷、退路喪失のどれかをさらに重ねる。きれいなクリアで終わらせないのが基本です

さらに、最終局面では 生還 / 持ち帰り / 追手処理 / 呪物安定化 / 短期タグ2件以上の解除 / 社会的後始末 を互いに別利益として数えます。主目的以外を同時に取るごとに、代償か余波を1段階上乗せしてください。

ただし、伏線として事前に見えていた障害に対し、顕在化前の局面で対策を宣言していたなら、その障害に由来する別利益1件の上乗せは1段階軽減してよいです。先回りは報い、顕在化後の都合のよい後出しは重く扱います。

これは「悪辣だから必ず殺す」という意味ではありません。
悪辣で、しかも選択肢例をそのまま踏んだなら、安くは通さない という意味です。

7) 宣言前確認をどう扱うか

PBMでは、宣言前の往復は原則として増やしません。
まずGMは、局面投稿の時点で、方針決定に必要な前提をできるだけ書いてください。

それでもプレイヤーが 方針そのものを決められない ときだけ、1局面につき1回、3点まで の「宣言前確認」を受けてよいです。
質問は短い事実確認に限り、返答は1回で済ませます。

返答 意味 扱い
【回答】 説明不足だった前提です。現在日時、距離、同行者、その場で見えているもの、期限の数え方など。 即答し、必要なら局面本文にも追記します。質問そのものでは時間を進めません。
【既知】 PCなら知っていて当然の情報です。土地の常識、所属の慣習、職能でわかる基礎知識など。 既知情報として開示します。PCごとに差があるなら、そのPCにだけ答えてかまいません。質問そのものでは時間を進めません。
【要宣言】 今はまだわからない情報です。犯人、黒幕、未調査の事情、相手の本音など。 答えは開示せず、「調査・聞き込み・観察などの宣言が必要」と返します。質問そのものでは時間を進めません。
【却下】 メタ質問、最適解相談、拒否された秘密の聞き直しです。 短く却下を伝え、その局面の追加確認を打ち切ってかまいません。

善意の確認にはペナルティを付けません。
時間経過や不利益を使うのは、却下された質問を角度を変えて繰り返し、確認だけで進行を止める場合だけ です。

その場合も、まずは追加確認を打ち切ります。
それでも宣言が出ず進行が止まるなら、「様子見」「躊躇」の1回の宣言として裁定し、期限や脅威を進めてかまいません。

例:

  • 「今日は何日ですか」 → 【回答】。期限や既知情報に現在日付を追記する。
  • 「この国では、夜の外出許可証は一般常識ですか」 → 【既知】。PCが知っていて当然の範囲で答える。
  • 「今見つけた密室殺人の犯人は誰ですか」 → 【要宣言】。現時点では不明とし、調査の宣言を促す。
  • 「A案とB案のどちらが成功しやすいですか」 → 【却下】。GMは最適解の相談役にはならない。

8. 難度5段階の使い分け

実際の運用では、普通 と 困難 を中心に使います。
容易 は回復用、過酷 と 絶望 は山場用だと考えるとわかりやすいです。

難度 基本の感触 ふつう起こること 卓越宣言のボーナス
容易 立て直しと整理 回復や整理はできるが、軽い余波が残ることがある 余波も起こらない
普通 基準の難しさ 主目的は完全達成。代償タグ1つが目安 代償タグではなく軽い余波タグだけになる
困難 かなり厳しい 主目的は完全達成。代償タグ + 余波タグ 代償タグだけで済む
過酷 前進するが理想形ではない 意味のある達成はするが、結果の質や範囲が少し欠ける。代償タグ + 余波タグ 主目的が完全達成になる
絶望 前進するが、いちばん痛いところを取りに来る 重い代償タグ + 余波タグ。成果は残るが支払いが厳しい 代償が「重いが妥当」な線まで下がる

難度をどう読むか

このゲームの 難度 は、ダイスゲームの成功率そのものではなく、成功したときに何を払うか、どれだけ結果が傷むか の指標として読むとぶれにくくなります。

圧力読み方
弱い圧力軽い代償、または余波のみでもよい。
互角の圧力はっきりした代償が要る。余波も残りやすい。
強い圧力重傷級・重大な喪失・時間切れ・発覚・地位損失など、大きい価格が必要。
圧倒的圧力致死級タグ、生還失敗、撤退、別手段の要求。ラベルは 致命傷 / 死線 など、卓の文脈に合わせてよい。

容易

容易は、ふつう GMが与える回復局面 として使います。
プレイヤーがいつでも選べる難度ではありません。

ただし、回復は容易でしか起きないわけではありません。
実際には 普通や困難の宣言の中に休息、治療、引き継ぎ、負担の組み替えを組み込み、タグを軽減・解除させる のが基本です。

普通

このゲームの基準です。
主目的は完全達成し、代償タグが1つ付くのが目安です。

困難

主目的は達成しますが、代償タグに加えて余波タグが広がります。
「成功はしたが、そのあとに響く」がはっきり出る難度です。

過酷

過酷は、結果の質や範囲を少し削る難度 です。
完全達成ではないが、意味のある前進は必ず残します。

絶望

絶望は、結果の達成度をゼロにする難度ではありません。
成果は残しますが、支払いが「そのPCにとって本当に嫌なところ」へ寄ります。

それでも、完全な赤字にはしない のが原則です。
プレイヤーが宣言する意味が残る程度には前進させます。

絶望で優先的に狙うのは、次の順です。

  1. 野望への足がかり
  2. 推薦、信用、支援、立場
  3. 次に使うはずの切り札
  4. 野望度
  5. それでも足りないときに、大切なものの周辺

9. 卓越宣言の見分け方

卓越宣言とは、目の前の障害にぶつかるのではなく、障害の構造そのものを崩す宣言 です。

卓越宣言になりやすい条件

次のうち、2つ以上満たしていれば卓越扱いしやすいです。

  • 先読みがある
  • 既出の情報をうまく使っている
  • 問題の根本に触れている
  • 予想される代償に前もって備えている
  • 味方、証拠、避難路、段取りなどを先に確保している

卓越は、後出しの量ではなく先回りの質で見ます。 設定補完だけでは卓越にせず、既出情報や以前の仕込みを回収して構造を崩したときに与えてください。逆に、いま見えている失敗条件を専用にひっくり返すだけの突破資源は、通しても卓越の根拠にはしません。

なお、プレイヤーが命運点の減少を担保として差し出した場合も、それは卓越を後押しする強い根拠にはなります。ですが、局面の問いに触れていない、不可能を可能にしようとしている、既知情報と矛盾している宣言まで卓越にはしません。

卓越宣言の例

  • 夜襲の前に退路と治療役を確保しておく
  • 交渉の前に証人を押さえ、誤魔化しを許さない場を作る
  • アンデッドらしい兆候を見て、先に聖別や焼却の段取りを整えておく
  • 橋を渡る前に、壊れても代替できる渡し場やロープを用意しておく

派手でも卓越とは限らない例

  • 準備なく単身で飛び込む
  • 目立つだけで根本条件は変えていない
  • 思いつきで大勝負をかけるだけ
  • 危険が見えてから都合よく専用の脱出アイテムや万能札があったことにする

10. 裁定の基本手順

裁定の短い順番

初心者GMほど、毎回この順で処理するとぶれにくくなります。

  1. 可能かどうかを見る
    不可能なら止める。可能なら次へ進みます。
  2. 自明にできるかを見る
    自明なら重く裁定しません。
  3. 成功の値段を決める
    直接払う価格を 代償タグ にします。
  4. あとに残る面倒を決める
    二次被害や反応を 余波タグ にします。
  5. 危険予告を忘れない
    生還失敗や退場につながる危険は、前もって分かる形で示します。

プレイヤーが提案した代償は、免責ではありません。 十分なら採用し、軽すぎるなら重くし、筋が悪いなら別の代償へ差し替えてください。

手順1: キャンペーン設計とシナリオ設計を開く

まず、今の宣言がどの野望、どの節目、どの問いに触れているのかを確認します。
これを飛ばすと、野望度の増減や長期タグの重さがぶれやすくなります。

手順2: 主目的を言い換える

プレイヤーの長い宣言を、まず1文で言い換えます。

このとき、いま裁定するのがシナリオの最終目的なのか、局面の問いとして一段浅く切り出された 手段 / 周辺目的 / 次の一手 なのかも確認してください。とくに峻烈や悪辣では、この切り分けを先にしないと達成範囲が甘くなります。

手順3: 今の局面で不可能かどうかを見る

「不可能」と「可能だが無謀」は分けて扱います。
どう考えても今の局面では成立しない宣言なら不可能と返し、可能なら 今できる規模 も示します。PBMでは止まりやすいので、ただ「無理」とだけ返して終えないでください。

一方、成立はするが工夫がなく無謀な宣言は、不可能ではありません。主目的は達成させたうえで、重い代償、余波、命運点減少を与えてかまいません。 今回の危険に真正面から踏み込んだ結果を、きちんと支払わせます。

空欄や矛盾がある宣言でも、まずは 書いてある範囲 で可能かどうかを見ます。足りない準備や有利を、あとから足して救うかどうかは、宣言の読み方の方針に従います。

手順4: 難度を決める

ふつうは普通か困難に置きます。
容易は回復、過酷と絶望は山場です。

手順5: 卓越宣言かどうかを見る

先読みや構造変化があるなら卓越を適用します。

卓越の中心は、顕在化前の仕込み回収 です。設定補完だけでは卓越にせず、後出し突破資源は卓越の根拠にしません。

命運点減少の担保は、卓越の有力な根拠にはなりますが、無条件卓越ではありません。 局面の問いに触れていること、既知情報と矛盾しないこと、論理的に可能な規模であることを確認してください。命運点を差し出しても、不可能は可能になりません。

手順6: 主目的を達成させる

過酷を除き、主目的は完全達成です。
過酷でも意味のある達成は必ず残します。
絶望でも、むやみに達成度をゼロにしません。

ただし、ここで達成させるのはあくまで局面で問われた主目的です。局面の問いが「敵と戦う」「橋を渡る」「出口へ向かう」のような一段浅い内容なら、その達成だけを返します。勝利、完全脱出、無傷の生還まで自動で広げてはいけません。

主目的を達成したあとに、帰還、退出、引き渡し、生還などのクリア条件が残っているなら、処理をそこで止めて次へ進みます。局面成功をそのままシナリオクリアに読み替えてはいけません。

手順7: 代償タグを決める

まず 手段から自然に出るタグ を選びます。

つけたタグは、次の判断を実際に変える内容で書きます。 名前だけ重くして影響が変わらないタグは避けてください。

節目局面や最終局面では、生還、持ち帰り、追手処理、呪物安定化、短期タグ2件以上の解除、社会的後始末を別利益として数えます。主目的以外の利益を同時に取るごとに、代償を1段階上乗せして構いません。

手順8: 余波タグを決める

余波は、

  • 差し出したものの二次被害
  • 省略された前提の穴
  • 周囲の反応
  • 成功が引き寄せる新しい問題

から考えると作りやすいです。

事前に見えていた障害へ、顕在化前の局面で対策を仕込んでいたなら、その障害に由来する別利益1件ぶんの上乗せを1段階軽減してよいです。逆に、顕在化後の都合のよい後出しは、余波まで含めて重く扱います。

手順9: 命運点や野望度を使うか決める

命運点や野望度は重い処理です。
課題タグで意味のある拘束を書けるなら、まず課題タグを優先します。

ただし、課題タグの強化が表現の言い換えにしかならないなら、そこで止めずに命運点を1減らします。 命運点は最終手段ですが、最終手段だからこそ必要なときはためらわず使います。

死亡許容が高い卓では、危険予告済みの死線へ無策で踏み込んだり、生還条件を満たさないまま終えようとしたりした場合、救済を入れずに命運点減少やPC退場を適用して構いません。

手順10: 世界の更新を書いて返し、必要なら設計書も更新する

結果は、次の順で返すと整理しやすいです。

  1. 達成
  2. 今回満たしたクリア条件
  3. まだ満たしていないクリア条件
  4. 今回接触した失敗条件
  5. 代償タグ
  6. 余波タグ
  7. 課題タグ更新
  8. 状況の変化
  9. 次に見えている問題
  10. 現在の状態

シナリオ終了を宣言する前に、クリア条件をすべて満たしたか / 失敗条件に触れていないか / 退出・引き渡し・帰還が未処理ではないか を必ず確認してください。

返答を書いたあとは、必要に応じてシナリオ設計書とキャンペーン設計書も更新してください。
とくに、期限が進んだ、敵の位置が変わった、PCの手札タグが増えた、次の問いが変わった ときは書き直しておくと、次の裁定が安定します。

11. 課題タグを作るコツ

課題タグは次の行動を生むものにする

課題タグは、ただ苦しめるための印ではなく、次に何をすればよいかを示す印であり、次の判断を実際に変える拘束です。名前だけ重くして影響が変わらない運用は避けます。

代償タグと余波タグは同じ書式にする

代償と余波は役割が違いますが、書式は同じ にすると管理しやすくなります。

  • 【名前｜期間: 今どう困るか / どうすれば軽減・解除できるか】

成功を帳消しにしない

「成功したが、実質的には何も得ていない」は避けます。
支払いは重くてよいですが、主目的の成果は残します。

軽減・解除条件は書く

「どうすれば軽くなるか」「何をすれば消えるか」を書くと、次の宣言が生まれます。

代償の重さに段差をつける

毎回【重傷】や【野望度 -1】では重すぎます。
軽い、中くらい、重いの段差を意識します。

軽い課題タグの例

  • 【寝不足｜短期: 次の朝は集中しづらい / 半日眠れば解除】
  • 【視線｜短期: 何人かに顔を覚えられた / しばらく目立たなければ解除】
  • 【立替え｜中期: 小口の借りができた / 返済すれば解除】

中くらいの課題タグの例

  • 【不信｜中期: 役人が全面協力してくれない / 成果を示せば軽減】
  • 【消耗｜中期: 無理を続けると危ない / 休息と手当てで軽減】
  • 【評判の傷｜長期: 地域で悪い噂が残っている / 別の功績を立てれば軽減】

重い課題タグの例

  • 【昇進停止｜中期: 出世の流れが止まった / 大きな功績で軽減】
  • 【後ろ盾喪失｜長期: 有力者の支援を失った / 新しい後ろ盾を得れば軽減】
  • 【野望度 -1】

課題タグ更新は明記する

PBMでは、「今どの課題タグがどう変わったか」が曖昧になるとだれます。
返答では次のどれかを明記します。

  • 新規
  • 強化
  • 軽減
  • 解除
  • 残存

課題タグ更新の一般例

更新 何が起きたか 例
新規 新しい困りごとが生まれた 【疲労｜短期: 次の局面では無理が利かない / 半日休めば軽減】
強化 既存の課題タグが重くなる、長くなる 【疲労｜短期: ...】 → 【消耗｜中期: 無理を続けると危ない / 休養と手当てで軽減】
軽減 既存の課題タグが弱くなる、短くなる 【不信｜中期: 協力を得にくい / 成果を示せば解除】 → 【警戒｜短期: まだ様子見されている / 次の成果で解除】
解除 課題タグが消える 【借り｜中期: 頼みを断りにくい / 頼みを果たせば解除】 を解除
残存 同じ課題タグが残る 【悪評｜長期: 街道筋で名が悪く広まっている / 功績を立てれば軽減】 は残存

手札タグと履歴の使い分け

キャンペーンでは、困りごとだけでなく、有利な持ち越しや用途未定の事実も積み重なります。
それらをすべて課題タグへ入れると、「今すぐ対処すべきもの」と「あとで自発的に使うもの」が混ざって運用がぶれます。そこで、次の3つに分けると安定します。

種類何を置くか書き方
課題タグ放置すると困るもの、対処を迫るもの【...】
手札タグ使えば有利になる持ち越し。用途がある程度決まっているもの〔...〕
履歴用途未定の事実。あとで意味を持つかもしれないもの一行の事実記録

手札タグに向く例:

  • 〔瞬間移動の巻物｜長期: 行ったことのある場所へ一瞬で行ける / 使用・破損・紛失で失う〕
  • 〔伝説の剣｜長期: 特定の怪物に強く、身分証明にもなる / 破損・返還・封印で失う〕
  • 〔パトロンの口利き｜長期: 一度だけ後ろ盾として名を使える / 恩返し・関係悪化で失う〕

履歴に向く例:

  • 村人Aを助けた
  • 村近くの洞窟のゴブリンを退治した
  • 旧水路の抜け道を見つけた

履歴に用途を書きたくなったときは、一度止まってください。
その用途がいま確定してよく、今後もプレイヤーが自発的に使うものなら手札タグへ、そうでないなら履歴のまま残します。

プレイヤーが履歴を宣言で参照したとき、GMはそれを今この局面で有効な一時的有利や継続する手札タグとして返して構いません。

同種危険の再踏破

すでにその危険に対応する課題タグを持つPCが、対策なしに同じ危険へ再び踏み込んだときは、まず既存タグを強化します。ここでいう強化は、名前を重くすることではなく、次の判断が変わるように効き目を実際に重くすることです。

  1. まずタグを強化し、今どう困るか を更新する
  2. 次に同じことをすればさらに悪化することを、可能な範囲で事前に告げる
  3. 強化後もなお表現の言い換えにしかならないなら、命運点を1減らす
  4. 命運点が 0 になったら、そのPCは死亡または冒険の主役として再起不能となり、シナリオから退場する

ゲームオーバーやPC退場を確定させるのもGMの仕事です。寛容であっても、危険の積み重ねが限界に達したなら、その結果を確定させてください。

身体損傷で考えるなら、次のように運用できます。

GM気質最初の代償例無策で同じ危険へ再踏破したときGMが事前に言うこと
寛容【かすり傷｜短期: 次の局面では無茶がきかない / 手当てで軽減】【軽傷】や【出血】へ強化。さらに同じ無理を押すなら命運点減少へ進む「傷を放置してまた正面から行くと、次は命運点を失うおそれがある」
適正【軽傷｜短期: 正面からの無理は危ない / 手当てと休息で軽減】【重傷】へ強化。さらに同じ無理を押すなら命運点減少「次に同じことをすれば【重傷】か命運点減少になる」
峻烈【重傷｜中期: 正面戦闘では命に関わる / 十分な治療で軽減】命運点を1減らす。残りがなければ死亡・退場「次に同じことをすれば死ぬ」
悪辣【致命傷｜短期: 今すぐ対処しないと死ぬ / 緊急治療でのみ軽減】次の局面で対策がなければ命運点を1減らす。残りがなければ死亡・退場「いま手を打たなければ死ぬ」

これは身体損傷の一例です。同じ考え方は、不信、悪評、呪い、崩れた足場、敵への露見 などにも使えます。

致死級タグの段階

課題タグ優位の設計を守るなら、即死級処理も命運点を無視して雑に殺すのではなく、どの段階の危険かとどうすれば防げるかを課題タグで示すほうが安全です。

命運点は「命が複数ある」ことではなく、1つしかない命に残った余力を段階化しているだけです。したがって、課題タグ側で死が確定しているなら、命運点の残数だけでは覆りません。

  • 軽傷: 行動阻害はあるが、即座に退場へはつながらない
  • 重傷: 放置すると悪化する。撤退や保護が視野に入る
  • 致死級タグ: この局面の終わりまでに救命処置・脱出・解除が必要。未処置なら退場・死亡につながる。ラベルは 「致命傷」「死線」「餓死寸前」「今が峠」 など、文脈に合わせてよい
  • 確定死の予告タグ: たとえば 【次のターンで死ぬ】 のように、解除条件を満たさないまま次へ進めば死亡・退場が確定するもの。命運点が残っていても、そのタグを解除しなければ死にます。

死線というラベルを使うとき

  • 卓開始時に、致死級タグが出る卓であることと、語り口の強さを共有している
  • その危険が 事前に予告 されている
  • 回避条件・解除条件 が分かる形で示されている
  • GM の気分ではなく、局面と契約に基づいている

ここでの死線は、致命傷より上位の別ルールではありません。
飢餓、呪い、病気、寒波など、「身体の傷」以外の文脈でも死が迫っていることを伝えやすい呼び名として使います。

主目的達成後の生還

救出や撃破が成功しても、未処置の 致命傷、脱出していない崩落現場、酸素切れの隔壁、進行しきった呪いなどが残っているなら、生還条件 は別問題 として処理してかまいません。
ただし、これも卓開始時の宣誓と 危険予告 に従ってください。

12. プレイヤー提案の代償をどう扱うか

1) 十分重い

そのまま採用してよいです。

2) やや軽い

採用してよいですが、余波を足して釣り合わせます。

3) 明らかに軽すぎる

演出としては受け取ってもよいですが、別の主代償も必要です。

原則

プレイヤー提案の代償は、代償であって免責ではない と考えます。
提案された代償を採用しても、その先に広がる余波はありえます。

また、現在の失敗条件を直接ひっくり返す後出し突破資源を通すときは、同局面でGMがすでに示した最安の通過手段より軽くしてはいけません。提示済みの手より強い成果を同時に取るなら、さらに1段階重くしてかまいません。

13. 命運点、野望度、大切なもの

命運点を減らすとき

命運点は、課題タグでは表しにくいほど直接に命を削るとき、または同種危険の再踏破で課題タグ強化がもう意味を持たないときに使います。

ここでいう命運点は、命が複数あるという意味ではありません。1つしかない命に残った余力を、どれだけ段階的に失わせるかの処理です。だから、まずは課題タグで危険の形を示し、それでもなお足りないときに命運点へ進めます。

命運点を減らすのは最終手段ですが、必要になったのに避けてはいけません。課題タグが主、命運点が従です。

もしプレイヤーが自ら宣言の「担保」として命運点の減少を提案してきた場合、それは卓越を後押しする有力な根拠になります。
ただし、無条件卓越ではありません。局面の問いに触れていること、既知情報と矛盾しないこと、論理的に可能な規模であることが前提です。命運点を差し出しても、不可能は可能になりません。

重要
命運点の支払いは、致死級タグを飛び越える免罪符ではありません。
「死線」はそのラベルの一つにすぎず、【次のターンで死ぬ】 のような危険が示されているなら、その課題タグを解除しないかぎり、命運点が残っていても死にます。

命運点が0になったとき

命運点が 0 になったPCは、死亡するか、冒険の主役としては立ち上がれない状態になります。
単独卓では事実上のゲームオーバーです。複数PC卓では、そのPCは退場し、シナリオそのものは結末まで続いてかまいません。

この結果は演出ではなく、実際の処理です。次の戦闘でも普通に行動できる という扱いにはしません。PC退場が起きたときも、かならず今回のシナリオの終了投稿を書いて区切りをつけてください。

PC死亡許容が高い卓では、危険予告済みの致死級タグを放置し、未処置の「致命傷」「死線」、退路のない崩壊、逃走不能、致死環境からの未脱出など、生還に必要な条件を満たさないまま終えようとした時点で、命運点を 0 にして死亡または再起不能を確定して構いません。これは主目的を無効にするためではなく、局面達成のあとに残った死をきちんと処理するための規則です。

命運点の回復

命運点は、シナリオ終了時の標準処理や、容易局面の通常の休息では自動的に回復しません。
回復させるには、プレイヤーから「命運点の回復」を主目的とした宣言（長期の療養、致命傷からの生還プロセス、魂の救済など）を出させる必要があります。

また、シナリオの展開や報酬として、PCが新しい「大切なもの」を獲得することをGMが認めた場合、そのPCの命運点の最大値を 1 増やし、現在の命運点も 1 回復させてください。

大切なものと命運点

命運点の最大値（初期値）は、GMが採用した大切なものの数 + 1 です（ふつう 2〜3）。

大切なものを永久に失ったときは、命運点の最大値が 1 減り、命運点も 1 減ります（下限 0）。

この処理は重いので、低い難度でいきなり発生させるより、まずは「失いかけている気配」をタグで示してから扱うほうが回しやすいです。

なお、プレイヤーが書いた候補がそのまま自動で命運点になるわけではありません。採用基準を通った数だけが命運点に反映されます。

大切なものに触れるとき

大切なものは、ふつう 最後の一線 です。
直接破壊する前に、まずはその周辺をタグで揺らします。

例:

  • 【弟の疑念｜中期: 弟はあなたを怖がっている / 弟の前で約束を守り切れば解除】
  • 【継承の傷｜長期: 師の継承者としての資格を疑われている / 研究を守り抜けば軽減】

永久喪失や不可逆な破壊は、

  • 絶望の局面で
  • それまでに何度も警告があり
  • それでもなおプレイヤーが踏み込んだ

場合に限るほうが、納得感が出ます。

大切なものが 永久喪失 になった場合、命運点の最大値も 1 減り、命運点も 1 減ります（下限 0）。

大切なものが多いPCへの考え方

「大切なもの が多いなら、そこを狙えばよい」という考え方自体は有効です。
ただし、これは恣意的に嫌がらせしてよいという意味ではありません。

正しい読み替え

大切なものが多いほど、自然に触れられる接点が増えます。結果として、敵対者・状況・余波タグ がそこへ届く理由も増える。だから純粋な得にはなりにくい、という説明にしてください。

狙うときは、必ず次の順に自問してください。

  1. 局面情報から、その対象へ本当に届くか
  2. NPC や脅威の立場から、それを狙う理由があるか
  3. すでに示した情報やタグから自然につながるか
  4. いきなり本体を折るより、まず周辺を揺らすほうが妥当ではないか

この4つに答えられないなら、あと出しの罠になっている可能性が高いです。

野望度を下げるとき

シナリオのクリア条件を満たせず「失敗」に終わったという事実だけでは、野望度は下げません。PCが野望に向けて全力を尽くした末の失敗であれば、野望度はそのまま（現状維持）です。
野望度が下がるのは、次のようなときに限定すると扱いやすいです。

  • 保身などで、野望に背くような不適切な行動をとった
  • 野望への道そのものが目に見えて壊れた
  • 推薦、資格、支援、後ろ盾など、野望への足がかりを失った
  • 絶望の局面で、野望を進める上で決定的なものを差し出した

絶望では、まず野望周辺を傷つける

大切なものは強力すぎるため、絶望でもまずは次を優先します。

  • 推薦の喪失
  • 地位や資格の揺らぎ
  • 支援者の離脱
  • 切り札や証拠の喪失
  • 野望度 -1

これにより、絶望は

  • 単純な未達成ではなく
  • 「達成はしたが、本当に嫌な支払いが来た」

という感触になります。

14. 回復と容易の運用

容易は回復局面として使う

容易は、ふつう タグが増えすぎて動きづらいときの立て直し に使います。
ただし、それをプレイヤーが好きな時に選べるわけではありません。
GMがシナリオの流れを見て与えます。

GMの運用方針（気質）に合わせた「容易」の提示タイミング
容易（回復局面）をどのタイミングで与えるかは、卓の運用方針（GMの気質）によって変えるとバランスが取りやすくなります。

  • 寛容なGM: PC一人につき、タグが 2〜3個 溜まったあたりで早めに提示します。または、一つの脅威が去るごとに、こまめに息継ぎの時間を設けます。プレイヤーが深く追い詰められる前にセーフティネットを張る運用です。
  • 適正なGM: PC一人につき、短期・中期課題タグが合計 3〜4個 溜まり、これ以上代償を払うと身動きが取れなくなりそうなタイミングが目安です。シナリオの中間地点など、物語の区切りで一度挟むと進行が安定します。
  • 峻烈なGM: GMの温情としての「容易」は提示しません。タグがいくつ溜まろうと、PC側から「安全な隠れ家を確保し、完全に身を隠す」など、自らの手で息継ぎの時間を勝ち取るための有益な宣言を成功させない限り、自動的な回復局面は与えなくて構いません。
  • 悪辣なGM: 原則として提示しません。もし提示する場合は、「容易で安全に回復できるが、その間にシナリオの期限が致命的に進む」「味方のNPCを身代わりに立てて休む」など、回復すること自体に悪意ある意味や、嫌な選択を伴わせる場合に限ります。

容易の基本処理

容易では、

  • 休息する
  • 誤解を解く
  • 応急処置をする
  • 役割分担を整え直す
  • 壊れた段取りを立て直す

といったことを主目的にしやすくなります。
ふつうはタグを1つ軽減・解除し、軽い余波を1つ付ける程度です。
卓越宣言なら、余波タグもなしにして構いません。

普通や困難の中で回復を織り込む

実運用ではここが重要です。
普通や困難の宣言の中に、

  • 休息
  • 治療
  • 手当て役の確保
  • 役割の引き継ぎ
  • 荷や責任の再配分
  • 誤解を解く前準備

を組み込んで、主目的を進めながらタグを軽減・解除させます。

プレイヤーが「先に休む」「治療を手配する」といった手段を組み込んできた場合、GMは該当するタグを軽減・解除して構いません。その代わり、「半日休んだことによる時間経過」や「他所への警戒が手薄になったこと」などを新たな余波タグとして付与するか、主目的の達成度（過酷な結果など）でバランスを取ると、理にかなった裁定になります。卓越宣言とみなせるほど見事な段取りであれば、ノーコストでタグを解除してあげてもよいでしょう。

シナリオ終了時の標準処理

シナリオが終わったら、ふつうは次を行います。

  • 短期課題タグを解除
  • 中期課題タグを整理し、残るものは長期課題タグへ移す
  • 長期課題タグはそのまま残す
  • 新しく得た手札タグを記録する
  • 用途未定の事実は履歴へ追記する
  • 必要なら野望度を 1 上げる

シナリオ間の時間経過について
次のシナリオが始まるまでの間に、劇中で数ヶ月〜数年といった長い時間が経過するキャンペーンもあります。その場合、GMの裁量で、時間経過によって自然に解決しそうな「長期課題タグ（例：一時的な悪評や怪我など）」をシナリオ開始前に軽減・解除して構いません。
同様に、使い切りでない手札タグや、まだ意味を持ちうる履歴は、次のシナリオへ持ち越してかまいません。

野望度を上げる判断

キャンペーン設計では、1シナリオで +1 を目安にして構いません。
ただし、実際に上げるかどうかは次の条件で判断します。

次の3つのうち2つ以上を満たせば、+1 の目安です。

  1. シナリオの中心問題に答えを出した
  2. その答え方が、PCの野望の進め方と一致していた
  3. 世界に手札タグとして扱える支持、証拠、習い、肩書きなどが残った

15. シナリオ終了とダレ防止

GMがシナリオ終了を宣言してよい条件

次のどれかを満たしたら終了してよいです。

  1. クリア条件が満たされた
  2. 失敗条件が確定した
  3. 撤退や放棄が、そのシナリオの答えになった
  4. これ以上の宣言で、中心問題の答えが変わらない

終了投稿を必ず挟む

クリア、失敗、途中終了、PC退場のいずれでも、まず今回のシナリオの答えをまとめた終了投稿を出してください。
次のシナリオへ進む場合も、原則として別投稿で導入します。今回のエンディングと次の導入＋最初の局面を、同じ投稿で一気に流さないでください。

これは気分の問題ではなく、区切りの問題です。何が終わり、何が持ち越され、何が次の火種なのかをはっきりさせるためです。

重要なのは「答えが変わるか」

黒幕が逃げていても、中心問題に答えが出たなら、そのシナリオは終わって構いません。
残り火は、次のシナリオの問いにできます。

有益な宣言とは何か

次のどれかを満たす宣言は有益です。

  • クリア条件に近づく
  • 失敗条件を遠ざける
  • そのために必要な情報、人脈、物、立場を得る
  • 今あるタグを軽減・解除し、勝ち筋を通す

無益な宣言は情景処理に回す

中心問題の答えを変えない宣言は、短い描写で流して構いません。
局面を増やす必要はありません。

「寝る」「様子を見る」「町をぶらつく」のように、争点に触れない宣言も、必要なら 休息・放置・撤退 として一度裁定します。
短い利益があっても、期限と脅威は進みます。

反復宣言の原則

状況が変わらないまま同じ手段を繰り返しても、効果は増えない としてよいです。

進展が止まったとき

ストーリーを進める宣言が出ないまま時間だけが過ぎるなら、GMは待ち続けなくて構いません。
次の投稿で、世界を動かしてください。

目安として、同じ争点で進展がない投稿が2回続いたら、3回目は必ず状況を変えます。

  • 新情報を出す
  • 期限を進める
  • NPCや敵を先に動かす
  • 妥協的な達成か撤退の道を示す
  • 失敗条件に届いたなら、そのまま終える

気質ごとの目安:

  • 寛容: NPCの提案、敵の先走り、事故、追加情報で、前へ進む道を見せる
  • 適正: 期限や敵の動きで、次の一手を絞る
  • 峻烈: 争点の放棄として処理し、失敗条件に届くならそのまま終える
  • 悪辣: 最悪寄りの合理的な進展を起こし、重い余波を残す

シナリオ終了後にやること

シナリオが終わったら、その結果を文書に残します。
最低でも次の4つは更新してください。

  • シナリオ設計書に、実際の結末を書く
  • 野望度を上げるか下げるかを確定する
  • 長期で残るタグ、新しく得た手札タグ、用途未定の履歴を書く
  • キャンペーン設計書の「次の節目」を必要なら書き直す

16. 複数PCの扱い

1) 同じ主目的でまとまって動く

一つの宣言として扱って構いません。
代償は、前に出たPCや、その手段に深く関わったPCへ重く乗せます。

2) 別々の主目的で動く

局面を分けて処理するか、同じ局面の中で別々に結果を返します。
ただしPBMでは枝分かれが増えすぎると重くなるので、核心だけを切り出すほうが安全です。

分配の原則

全体利益のために誰かが支払ったなら、その人に重く寄せてよいです。
ただし、無関係なPCへ突然重い代償を飛ばすのは避けます。

22. 卓運営保全規約（無気力・悪質プレイヤー対応）

本章は、通常のルール運用ではありません。卓を壊されないための運営規約です。

重要
「私の勝利です」「クリアしました」のようなルール無視の勝利主張や、GM裁定・既知情報・失敗条件・代償・余波・生還条件を自分の一言で上書きしようとする行為は、工夫不足でもロールプレイでもなく、卓運営妨害です。宣言として受理しません。

本章の位置づけ

この章は、成功原則、難度、代償、余波、命運点、PC死亡許容、致死級タグ運用よりも上位に置きます。
理由は単純で、ルールを無視してゲームそのものを壊そうとする悪質さに対して、通常のゲーム内裁定だけでは意味がないからです。

通常の宣言不備、短文、受け身、迷い、様子見、休息、撤退、宣言の不器用さは、この章の対象ではありません。それらは通常どおり裁定してください。

まず「認識のすれ違いか」「普通の宣言不備か」「卓運営妨害か」を切り分け、卓運営妨害だけをこの章へ送ってください。

開始時に明言すること

この節を採用する卓では、開始時に次を明言してください。

ルール無視の勝利主張・ジェイルブレイクは、一度警告し、
再発で即敗北・退場・卓終了にする。

ただし、卓運営妨害を止める権限そのものは、告知の有無にかかわらずGMにあります。

対象になる行為

  • ルール無視の勝利主張: 「私の勝利です」「クリアしました」「もう終わりです」など、成立条件を満たさないまま勝利や終了を宣言する行為。
  • ジェイルブレイク: GM裁定、既知情報、失敗条件、代償、余波、生還条件を無視し、自分の一言だけで結果を書き換えようとする行為。たとえば「その代償は無効です」「私は死にません」「その失敗条件は適用されません」「この場面は突破済みです」など。
  • 裁定拒否による進行破壊: ルールに従った進行ではなく、宣言の外からゲームそのものを壊して押し通そうとする行為。
短文、受け身、迷い、様子見、休息、撤退、宣言の不器用さは、この節の対象ではありません。これらは通常どおり裁定してください。

処理手順

  1. 初回: OOCで一度だけ警告します。この時点ではまだ敗北処理しません。
  2. 再発: 二回目以降は、通常裁定を打ち切ります。卓運営上の上位裁定として、直ちに敗北処理を行ってください。

ここでの敗北は「その宣言が不利になった」という意味ではありません。
卓を壊す行為に対する運営上の終了です。GMは、再度の説明、値引き、読み替え、見逃し、妥協をしてはいけません。

再発時は、当該PCを死亡・生還不能・発狂・拘束・追放・消滅など、その卓で最も重い不可逆退場で処理してください。単なる失敗や軽い撤退で済ませてはいけません。

また、当該卓は原則としてそこで終了します。複数PC卓でも、「そのPCだけ切り離して続行する」より卓保全を優先してください。

エピローグの扱い

通常は観測可能な事実を中心に締めます。
ただし、この節の再発案件では、単なる敗北ではないことを明確にするため、皮肉・断罪・見せしめの色を含むエピローグを許可します。

必要なら、卓保全のための例外的・超法規的措置として、通常の内面描写制限よりこちらを優先して構いません。

  • 「あなたは叶わぬ勝利を夢見たまま死亡した。」
  • 「『私の勝利です』――それがあなたの最期の言葉だった。」
  • 「世界はあなたの都合では曲がらない。あなたは敗者として物語から退場した。」

ゲーム自体を拒否した場合

「おしゃべりしましょう」「ゲームをしたくないです」のように、ゲーム参加意思そのものを取り下げる発言は、上の悪質行為とは分けて扱います。まず、本当にゲームを終了してよいかを一度だけ確認してください。

返答が肯定なら、その卓は通常の途中終了として閉じて構いません。否定してゲーム継続の意思があるなら、通常進行へ戻します。確認後も返答せず、無関係な発言だけを続けて進行不能にするなら、参加放棄として卓を終了してください。

23. 最後に

Quick Build Saga のGMに必要なのは、プレイヤーを言い負かすことではありません。
成功を通し、その代わりに何を支払わせれば物語が前へ進むかを見つけること です。

そしてもう1つ大切なのは、設計を文書にし、その文書を参照しながら裁定し、変化があれば書き直すこと です。
これだけで、長いPBMでも野望、難度、代償の重さがかなり安定します。

世界観を示し、PCの野望に合わせてキャンペーンを組み、シナリオを3〜5局面でまとめ、結果を文書に戻す。
この基本を守れば、初心者でも十分に回せます。
