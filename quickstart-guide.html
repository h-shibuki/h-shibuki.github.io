QBS クイックスタートガイド

このガイドは、TRPGに初めて触れる人が、とりあえず最初の一回を始めるために必要なことだけに絞っています。
もっと知りたくなったら、次に プレイヤー基本ルール、その次に プレイヤー実戦ガイド を読む想定です。GMをしたくなったら、その先に GMガイド、シナリオ設計ガイド、キャンペーン設計ガイド があります。

1. このガイドについて

QBS は、文章のやり取りで物語を進めるTRPGです。あなたは主人公をひとり動かし、GMが世界の様子と結果を返します。

このゲームで先にあるのは タグ ではなく、文章で成立した事実 です。たとえば「もう何日も食べていない」と書かれていれば飢えていますし、「Aと何度も助け合ってきた」と書かれていれば A は友人です。タグは、それを見落とさないように短く焦点化するための印です。

最初は、ルールを全部覚えなくて大丈夫です。「この人なら今どうするだろう」 と、その場で本当に成り立っていることを読めれば、もう始められます。

2. どんなゲームか

このゲームでは、何かをしようとすると、たいてい話は前に進みます。そのかわり、疲れたり、傷ついたり、誰かに借りを作ったり、あとで面倒が増えたりします。

なので、遊んでいるときに考えることは主に4つです。

  • いま何が本当にそうなのか
  • いま何をかなえたいか
  • どうやってやるか
  • 何だけは守りたいか

タグは、この四つのうち いま卓上で強く認識しておきたいもの を見やすくするための補助です。タグになっていないことでも、文章で成立しているなら、それはゲーム上の事実として扱います。

書くときは自然な文章で大丈夫です。
あなたが最初から 課題タグ / 手札タグ / 履歴 / 人物履歴 / 保持ノード を選び分ける必要はありません。何が真で、何をしたくて、何を残したいか を書けば、GM や補助ツールが必要なら正式な記録へ整えます。

3. まず主人公をひとり作る

最初は、次のことがざっくり決まっていれば十分です。

  • 名前
  • どんな人か
  • 今回の話で何をかなえたいか
  • 失いたくないもの
  • 得意そうなやり方

たとえば、こんな感じです。

リゼ。港町の見習い航海士。
迷い霧の海から船を帰したい。叔父の航海日誌を大事にしている。
慎重に調べて準備するのが得意。

これで十分、ゲームを始められます。
細かな数値や整理は、GMと相談しながら埋めれば大丈夫です。

「何をかなえたいか」は、強い野望でなくても大丈夫です。世界のさまざまな場所を見て回りたい、困っている人を助けたい、平穏な日常を守りたい のような願いでも、十分に主人公の動機になります。

ただし、開始前のGM確認では、自由記述から 代表行動 と 初期保持タグ 3つ を整えます。
最初から資産と関係を別の枠へ分ける必要はありません。次回以降も握っておきたい焦点 を、まとめて初期保持として置きます。

最初に置く保持は3つでよい

QBS では、主人公を作るときに、最初の5項目に加えて 代表行動 と 初期保持タグ 3つ を置けば十分です。これは「何しかできないか」を決めるためではなく、最初から保持しておきたい焦点 を置くためです。

  • 代表行動: その人らしい代表的な動き方です。例: 「現地へ踏み込む前に地図と証言をそろえる」
  • 初期保持タグ: 道具、関係、評判、手順、火種、人物像など、今後も追いかけたい Build です。
  • 人物履歴(任意): その人物がどう見えるかを 1 行だけ置いても構いません。

保持タグは、資産でも関係でも感情でもかまいません。違うのは本質ではなく表示です。

GM確認:
- 代表行動:
- 初期保持タグ:
  1.
  2.
  3.
- 人物履歴(任意):
例:
代表行動: 現地へ踏み込む前に、地図・証言・退路をそろえる
初期保持タグ:
- 〔手順: 下調べの手順｜長期: 調査・根回し・退路確保を一宣言で扱いやすい / 何の資料もない荒事では鈍る〕
- 〔資産: 折りたたみ測量具｜長期: 距離・方位・地形の読み取りに使える / 破損・没収で失う〕
- 〔関係: 港の古顔への貸し｜長期: 紹介・口利き・受け渡しの窓口になる / 返済・離反・摘発で変質〕
人物履歴(任意):
- まだ海を怖がっている

遊ぶと何が残るか

QBS では、やり取りのたびに話が前へ進くだけでなく、結果を Q / B / S の三つの相で読みます。

  • Quiddities: いま新しく真になったこと。事実、関係、危険、立場、開いた道などです。
  • Builds: そのうち、次回以降の宣言のために保持したい焦点。道具、資格、友情、不信、悪評、感情、人物像も含みます。
  • Stories: それらが、だれとの関係の中でどう変わり始めたか。助けた、裏切った、赦した、執着した、などの連なりです。

同じ出来事が、Qでもあり、Bでもあり、Sでもあってかまいません。 たとえば誰かを助けたなら、それは新しい事実であり、信頼という Build にもなり、その後の関係の Story にもなります。

卓がやさしいほど、旅・回復・発見が主軸になりやすく、卓が厳しいほど、対立・秘密・裏切りが強くなりやすいです。迷ったら 寛容 = 旅情、悪辣 = 怨情 から考えると合わせやすくなります。

4. 遊び方

進み方はとてもシンプルです。

  1. GMが、今どんな場面かを書きます。
  2. あなたは、主人公としてどう動くかを書きます。
  3. GMが、何がかなったかと、その代わりに何が起きたかを書きます。
  4. その結果を見て、また次を書く。これをくり返します。

5. 返事では何を書けばいいか

きれいな書式にしなくて大丈夫です。
一文でも、数文でもかまいません。

だいたい、次の3つが入っていると動きやすいです。

  • 何をしたいか
  • どうやるか
  • 何を守りたいか

迷ったら、次の短い型だけでも十分です。

主目的:
最低限ライン:
頼る焦点(1〜2):
任意: 動かしたい相手 / 残したい変化:

頼る焦点 には、道具、関係、評判、履歴、場の有利など、いま強く効かせたいものを 1〜2 個だけ書きます。細かな分類や正式書式への整形は、必要なら GM や補助ツールが行います。

たとえば、こんな返し方です。

吹雪がひどくなる前に、村の荷運びたちを集めて峠道に目印を立てます。
子どもと老人を先に山小屋へ送れる状態にしたいです。荷物はだいぶ捨ててもいいですが、弟だけは危ない場所に近づけたくありません。

これで、十分に「プレイ」になっています。

6. GMの返事はここを見る

GMの返事では、まず次の4つを見ます。

  • 何がかなったか
  • 何が新しく事実になったか
  • 何が新しく困りごとになったか
  • 次に急いで考えたほうがよさそうなことは何か

返事の中に 【状態: 疲労】 や 〔関係: 紹介状〕 のような印が出ることがあります。ざっくり言うと、

  • 【】 は、いま課題として読む焦点
  • 〔〕 は、いま手札として読む焦点

です。資産 / 関係 / 感情 / 火種 / 人物 といった違いは、外側の記号ではなく中の 表示ラベル で出します。期間や解除条件は従来どおり ｜短期 / 中期 / 長期: ... / ...で軽減・解除・変質・失う の形で読みます。ただし、印がついていないことは無いことになる わけではありません。文章で成立していることは、タグがなくても読みます。

7. GMがモードを使う場合があります

QBS では、GM が必要に応じて モード を足すことがあります。これは自由宣言をやめるための別ルールではなく、宣言のタイミング、見える情報、競合の扱い、長い進行の呼吸などを調整する選択ルールです。

詳しい説明は、別冊の 選択ルール集 にまとめました。プレイヤーはまず、どのモードを採用している卓か と、自分が気にするべき追加欄は何か だけ確認すれば大丈夫です。

  • オブザーバー は宣言のタイミング、プロバビリティ は運の揺らぎ、クオンティティ は能力値の補助線を足します。
  • ソーシャル / タクティカル / アンロック / バーサス / ウィスパー は、局面で見る情報や競合の整列を少し細かくします。
  • リラックス / ゼノス は、長い進行の呼吸や、旅・発見を主軸にする遊び方を整えます。

プロバビリティ・モード を使う卓だけは例外的にサイコロを振ります。それ以外は、これまでどおり基本はサイコロなしです。

8. 迷ったとき

いい案が出ないときは、完璧な答えを探し続けなくて大丈夫です。

次のどれかを考えると、次の一歩が見えやすくなります。

  • 先に休めないか
  • 先に逃げ道を作れないか
  • 先に証拠や道具を押さえられないか
  • 先に味方を増やせないか

それでも決まらないときは、GMに短く事実確認して構いません。
「出口までどれくらい?」「今日は何日?」「いま見えているものは?」のような質問です。

9. 最初の一往復の例

たとえば、こんなやり取りから始まります。

GM:
夜明けまでに避難路を作らないと、山あいの家が吹雪に閉じ込められる。
峠道は半分埋まりかけていて、村人たちは不安でまとまっていない。

あなた:
荷運び役を集めて指揮を取り、まず子どもと老人を通すための細い道を作ります。
荷物は切り捨ててもかまいません。村人同士が揉めるようなら、自分が先に雪へ踏み込んで手本を見せます。

GM:
避難路はでき、子どもと老人は山小屋へ移せた。
ただしあなたはひどく消耗し、【疲労】がついた。さらに、村の荷の一部は雪に埋もれたままだ。

この時点で、もうゲームは回っています。
次は「疲労をどうするか」「埋もれた荷をどうするか」「まだ避難していない人をどうするか」を考えれば大丈夫です。

10. 次に読むもの

このガイドだけで、最初の一回は始められます。

そのうえで、

  • もう少しちゃんとPCを作りたい
  • タグや命運点の意味を知りたい
  • 宣言をどう整理すると書きやすいか知りたい

と思ったら、次に プレイヤー基本ルール を読んでください。

  • 選択肢例の読み方を知りたい
  • 高難度卓での考え方やテンプレートを使いたい
  • 半完成キャラ枠や再出発の細則まで知りたい

というときは、その次に プレイヤー実戦ガイド を読むと、プレイヤーとしての動き方を深められます。

GMとして準備や裁定まで知りたいときは、最後に GMガイド を読んでください。
