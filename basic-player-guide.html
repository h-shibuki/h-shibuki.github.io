QBS プレイヤー基本ルール

このガイドは、クイックスタートガイドの次に読み、プレイヤーとして遊ぶのに必要な基本ルールをまとめています。
もっと上手に立ち回りたい人向けの実戦的な話は プレイヤー実戦ガイド、GM向けの運用と裁定は GMガイド、自分で単発シナリオを作るなら シナリオ設計ガイド、長いキャンペーンを設計するなら キャンペーン設計ガイド に分けています。

1. はじめに

QBS は、基本はサイコロを振らず、プレイヤーの宣言で物語を前へ進めるPBM向けTRPGです。
できるかどうかを細かく争うより、何を守り、何を失い、それでも何を取りに行くかを選ぶゲームだと考えるとつかみやすくなります。

このガイドは、プレイヤーが遊ぶときに必要なことだけをまとめたものです。
GMがどんな順番でシナリオを作るか、どこまで内部で較正しているか、といった話はここでは扱いません。

QBSの三要素

QBS では、同じ出来事を Quiddities / Builds / Stories の三つの相で読みます。

  • Quiddities（いま真であるもの）: 描写、会話、宣言の結果として、いま成立している事実、関係、危険、立場、可能性です。タグの有無ではなく、言語で成立しているかどうか で決まります。
  • Builds（保持された焦点）: Quiddities のうち、将来の宣言のために、あらかじめ焦点化し、保持し、資産化したものです。道具、資格、航路、友情、不信、悪評、感情、人物像、ドラマ上の火種も Build になりえます。
  • Stories（変質の連続）: Quiddities と Builds が、他者との関係や時間の経過の中でどう変わり続けるかです。信頼が負い目へ変わること、努力が評価へ変わること、裏切りが赦しや断絶へつながることも Story です。

大事なのは、この三つが別々の棚ではなく、ひとつの宣言を三方向から読むための補助線だということです。同じものが、同時に Quiddity であり、Build であり、Story の一部でもありえます。

QBS では、タグが真実を作るのではありません。先に文章ややり取りで成立した事実があり、タグはそのうち いま卓上で強く認識しておきたいもの を短く焦点化する記法です。

1分でわかる成長点

成長点は、原則としてシナリオ終了時の成長処理で使います。
プレイヤーはまず 残す / 始める / 伸ばす / つなぐ / 整える の 5 動詞で希望を言えば十分です。正式な操作名と細かいコストは GM や補助ツール側の処理 なので、プレイヤーは覚えなくて構いません。

  • 獲得: シナリオ終了時や、シナリオ中に世界課題タグを達成したときに増えます。
  • 使用: 使うのはシナリオとシナリオの間、つまり シナリオ終了時の整理 です。
  • 申告: GMの進行に合わせて、プレイヤーはまず「今回はこれを 残したい / 始めたい / 伸ばしたい / つなぎたい / 整えたい」と相談して決めます。

1分でわかる Q / B / S と保持

Q はいま成立していること、B はそのうち将来のために保持した焦点、S はそれらが使われた結果どう変質したかです。
資産、関係、感情、火種、人物像の違いは、本質の違いではなく、読み味と記録のための表示差です。

  • Q: いま真になった事実、危険、立場、関係、可能性
  • B: 次回以降の宣言のために保持した焦点。道具、関係、評判、感情、人物像も含みます
  • S: 使った結果のねじれ、深まり、壊れ、言い換え、再配置

保持はタグでもノードでも書けます。 期間や解除 / 消失条件は、従来どおり ｜短期 / 中期 / 長期: ... / ...で軽減・解除・変質・失う の形で読みます。外側の記号は 【課題】か〔手札〕か を示し、資産 / 関係 / 感情 / 火種 / 人物などの違いは内側の表示ラベルで出します。

2. まず知っておけばよいこと

あなたがやること

プレイヤーとしてやることは、かなり単純です。

  • GMの投稿を読む
  • 自分のPCがこの場面で何をしたいか決める
  • どうやるか、何を守りたいかを書いて返す
  • GMの結果を受けて、また次を考える

一番大事なのは、正解を当てることよりも、そのPCなら今どう動くかをはっきりさせることです。

1往復の流れ

順番何が起きるか
1GMが、今の状況、期限、問題、見えている情報を出します。
2あなたは、PCとしてどう動くかを書いて返します。
3GMは、何がかなったか、何を支払ったか、何が残ったかを返します。
4その結果を受けて、次の局面へ進みます。

GMの返事で見るところ

GMの返事では、とくに次の4つを見ると次が考えやすくなります。

  • 何がかなったか
  • 何が代償や余波として残ったか
  • 次に困ることは何か
  • 今のPCの状態（命運点、タグ、履歴、手札など）

3. このゲームでよく使う言葉

この章は、プレイヤーが遊ぶうえで必要になる語を、正式名称にそろえてまとめた用語集です。まずは キャンペーン / シナリオ / 局面 / 宣言 / 裁定 / 課題タグ / 手札タグ / 履歴 / 野望 / 野望度 を押さえれば十分です。

物語の単位

キャンペーン は PC が野望に届くまでの長い道のり、シナリオ はその道中で起きるひとつの事件、局面 はその事件を進めるための大事な一場面です。

登場人物

プレイヤー はあなた、PC はあなたが動かす主人公、NPC は GM が担当する人物です。GM は世界の様子、敵や味方の動き、時間経過、結果の返答を担当します。

宣言と結果

宣言 は、その局面で PC がどう動くかを書く文章です。その中でいちばんかなえたいことが 主目的、そのために使う方法が 手段 です。GM が宣言を読んで、達成と支払いを決めて返すことを 裁定 と呼びます。

残っていくもの

課題タグ は、すでに成立している困りごとや危険のうち、いま強く認識しておきたいものです。手札タグ は、すでに成立している資源・関係・条件のうち、プレイヤーが自発的に使えるよう焦点化したものです。履歴 は、すでに起きて真になったことのうち、まだどの形式で保持するかを決めていない記録です。旅履歴 は履歴のうち旅や探索の持ち帰りにラベルを付けたもの、人物履歴 は人物像に関する Build を記録する形式です。Build のうち、装備・拠点・制度線・手順のように台帳で管理するものは 保持ノード で扱い、その成長材料を 素材 と呼びます。

覚えておくと便利な言葉

命運点 は今ここで運命をねじ曲げるための点、大切なもの は失いたくない支え、致死級タグ は放置すると死亡・退場・生還失敗へつながる重い課題タグ、生還条件 は主目的達成後も生きて帰るために満たす条件です。野望 は PC が人生をかけて目指す到達点、野望度 はそこへどれだけ近づいたかを示す段階です。

本質語 / 記録語 / 表示語

本質語 と 記録語 と 表示語 を分けて読むと、QBS はかなり整理しやすくなります。

層語役目
本質語Quidditiesいま言語上すでに成立しているもの
本質語Buildsそのうち将来に備えて焦点化・保持したもの
本質語Storiesそれらが他者との関係の中で変質し続ける連なり
記録語履歴成立済みだが、まだ保持形式を決めきっていないもの
記録語保持タグ一行で保持するもの。課題なら 【…】、手札なら 〔…〕
記録語保持ノード多段の情報で保持するもの。装備・拠点・制度・重い関係など
記録語人物履歴人物表示で保持 / 変質を追う記録様式
表示語資産 / 関係 / 感情 / 火種 / 人物 / 制度 / 場 / 路 / 手順 / 状態本質を変えず、読み味だけを変える表示ラベル

つまり、資産と人間関係は QBS の本質上は同じ Build です。違うのは本質ではなく、表示と読み味です。現金、土地、パトロンの庇護、恩義、執着、言えなかった本音は、いずれも保持された焦点としては Build であり、使われた結果の変質が Story です。

課題 / 手札 と 表示ラベルは分けて読む

QBS では、外側の記号 と 内側の表示ラベル を分けて読みます。

  • 【…】 … いま課題として読む焦点
  • 〔…〕 … いま手札として読む焦点
  • 表示ラベル … 資産 / 関係 / 感情 / 火種 / 人物 / 制度 / 場 / 路 / 手順 / 状態

一般表示ではラベルを省略して構いません。期間や解除 / 消失条件は、従来どおりタグ本文の中で扱います。

〔資産: 折りたたみ測量具｜長期: 距離・方位・地形の読み取りに使える / 破損・没収で失う〕
〔関係: 港湾商会への貸し｜長期: 紹介・口利き・受け渡しの窓口になる / 返済・離反・摘発で変質〕
【感情: 執着｜中期: 対象が絡むと撤退や切り捨てが高くつく / 告白・断念・和解で変質】
〔火種: 言えなかった本音｜長期: あの相手が絡むと話を動かしやすい / 告白・露見・断絶で解消〕
【状態: 疲労｜短期: 無理押しが高くつく / 休息や手当で軽減】

4. キャラクターの作り方

プレイヤーとして遊ぶだけなら、次の7項目がそろえば十分です。
全部きっちり欄埋めしなくても、短い自由記述からGMと一緒に整えて構いません。

項目何を書くか
名前PCの名前です。
立場どんな人物かを一言で表します。例: 港町の見習い航海士、山寺育ちの祓い手。
野望そのPCが最終的にかなえたい目標です。単発では「今回かなえたい目標」で構いません。
大切なもの失いたくないものです。人、立場、持ち物、信念、居場所などから選びます。
得意なやり方そのPCがよく使う方法です。例: 調査と準備、交渉と根回し、祈りと供養。
命運点初期値は、GMが採用した大切なものの数 + 1です。ふつうは 2〜3 です。
野望度初期値は 1 です。ふつうは 5 が終点の目安、単発では 2 が目安です。

自由記述で出してよい

たとえば、次のような出し方でも十分です。

港町で育った見習い航海士です。
叔父の航海日誌を受け継いでいて、
いつかどの港でも通用する海図を作りたいと思っています。
慎重に調べて準備するのが得意です。

GMはそこから、立場、今回の野望、大切なもの、得意なやり方を整え、さらに卓開始用のゲーム的データとして 代表行動 / 初期保持タグ 3つ / 人物履歴(任意) を仮置きします。
違うと思ったら、その場で言い直して大丈夫です。最初から資産と関係を別の枠へ分ける必要はありません。

開始時に決めること

QBS では、名前 / 立場 / 野望 / 大切なもの / 得意なやり方 / 命運点 / 野望度 に加えて、次の要素も最初に決めます。

項目最初の目安役目
代表行動1つそのPCらしい代表的な動き方です。以後の保持と成長の核になります。
初期保持タグ3つ道具、関係、感情、手順、評判、人物像など、最初から保持しておきたい焦点です。
人物履歴任意で1行人物像をどう読んでほしいかを一言だけ置けます。空欄でも構いません。
成長点0保持ノードを育てるための点です。シナリオ終了時や小課題の達成で増えます。

自由記述で始める場合も、GM確認の時点で 代表行動 / 初期保持タグ 3つ を必ず整えます。
大切なのは、最初の宣言から 保持 と 変質 の種が置かれていることです。

GM確認テンプレート:
- 代表行動:
- 初期保持タグ:
  1.
  2.
  3.
- 人物履歴(任意):

書き方の例

得意なやり方: 調査と準備
代表行動: 現地へ踏み込む前に、地図・証言・退路をそろえる
初期保持タグ:
- 〔手順: 下調べの手順｜長期: 調査・根回し・退路確保を一宣言で扱いやすい / 何の資料もない荒事では鈍る〕
- 〔資産: 折りたたみ測量具｜長期: 距離・方位・地形の読み取りに使える / 破損・没収で失う〕
- 〔関係: 叔父の航海日誌｜長期: 海図・港・古記録にまつわる宣言で背中を押す / 焼失・盗難・改竄で失う〕
人物履歴(任意):
- まだ海を怖がっている

人物履歴は空欄でも始めてよい

人物履歴 は、そのPCがどう見え、どんな傷や矛盾や秘密があとで効きそうかをまとめる補助欄です。必須ではありません。 気楽に始めたいなら 空欄 のままでも、GMにおまかせ とだけ書いて始めても構いません。

【人物履歴】(空欄可 / GMにおまかせ可)
主類型:
裏面:
傷 / 欠落:
秘密:
矛盾:
輪郭を増幅する相手:

全部を埋める代わりに、開始時は 人物履歴 を 1 行だけ置いても構いません。これは職業や能力ではなく、今回どの人物線をいちばん触りたいか を示す仮置きです。

【人物履歴】(開始時は1行だけで可 / 後で更新可)
- いまは〜型に見える:
- まだ言えない欠けがある:
- この相手といると輪郭が出る:

三つ全部を書く必要はありません。どれか一つだけ で十分です。これは数値ボーナスではなく、人物像に関する Build をどこに置くか の足場として使います。

この欄は、開始時に完成して終わる設定ではありません。セッション中の宣言と GM の裁定で、あとから埋まり、変わり、増えていきます。

  • 危険を引き受けて仲間を逃がした → 傷 / 欠落 や 主類型 がはっきりする
  • 人をうまく扇動して自分だけは危地から外れた → 裏面 や 矛盾 や 秘密 に残りうる
  • だれかにだけ弱さを見せた → 輪郭を増幅する相手 や 関係 が更新される

人物履歴は、直接の数値ボーナスだけを与える欄ではありません。信じられ方、疑われ方、期待され方として将来の宣言価格を変える Build だと考えてください。最初は白紙でも、シナリオが終わったあとに「この人はこういう人だった」と見えてきたら書き足して構いません。GM や補助ツールが候補を 1〜2 件出し、プレイヤーが採用 / 修正 / 見送りを選ぶ 形でも大丈夫です。

5. ゲーム開始までの流れ

最初の流れは、だいたい次のようになります。

  1. GMが舞台、雰囲気、今回の共通前提を示す
  2. プレイヤーがPCを作る
  3. GMが最初のシナリオと最初の局面を出す
  4. そこから実際のやり取りが始まる

単発で遊ぶときは、最初に「どんな題材で遊びたいか」「避けたい題材はあるか」「どれくらい危ない卓がいいか」を共有しておくと始めやすくなります。

GMから卓の厳しさについて説明があることもあります。
細かな運用名を全部覚えなくても、どれくらい案が見える卓か、どれくらい痛い代償や死亡がありうる卓か がわかれば、プレイヤーとしては十分です。

モードは別冊「選択ルール集」で確認する

QBS のモードは、自由宣言そのものを置き換えるルールではありません。宣言のタイミング、見える情報、運の揺らぎ、長い進行の呼吸、競合の整列などを調整するための選択ルールです。

モードはすべて任意で、併用できます。 先に決めるのは 卓等級 と 裁定傾向 / 指針提示度 / 劇的展開度 / 死亡許容度 で、その上で必要な取扱領域だけをモードとして足します。とくに ソーシャル / タクティカル / アンロック は、1つの局面の中に複数の 局内工程 を立てて読むモードです。

詳しい説明は 選択ルール集 にまとめました。プレイヤーは、卓の開始時に「どのモードを使うか」「自分が気にするべき変化は何か」だけ確認すれば十分です。

モードプレイヤーが最初に確認すること詳しく読む場所
オブザーバー局面ごとの宣言ではなく、観測方針を出す卓か。選択ルール集
プロバビリティ今回の卓で 2d6 の揺らぎを使うか。選択ルール集
クオンティティ能力値を補助線として使うか。選択ルール集
ソーシャル / タクティカル / アンロック局面文にどんな 局内工程 や追加欄が出るか。選択ルール集 / 選択ルール集 / 選択ルール集
バーサス / ウィスパー競合や秘匿情報を扱う卓か。選択ルール集 / 選択ルール集
リラックス / ゼノス長く遊ぶ呼吸か、旅と発見を主軸にするか。選択ルール集 / 選択ルール集
ヨーク / ゼロット連帯や逸脱をどこまで卓の主軸にするか。選択ルール集 / 選択ルール集

既定値は各ガイドにありますが、最優先なのはシナリオの性質とプレイヤーの希望です。上級卓でも今回は切る、初級卓でもこの回だけ入れる、といった運用で構いません。

どのモードでも、QBS の中心は変わりません。何をしたいかを文章で書くことが先で、モードはその前後を調整するだけです。

劇的展開度も最初に決める

QBS では、卓の基準を 卓等級 + 裁定傾向 / 指針提示度 / 劇的展開度 / 死亡許容度 の五項目で共有します。劇的展開度 は、どんな感情や余韻で世界や NPC が返りやすいかを表す軸です。最初に迷ったときは、次の対応から決めると合わせやすくなります。

卓等級 / 裁定傾向合わせやすい劇的展開度その卓で起こりやすいこと
初級 / 寛容旅情旅、発見、休息、助力、伴走、穏やかな達成が主軸になりやすいです。
中級 / 適正人情義理、信頼、ためらい、日常のぬくもり、小さなすれ違いと和解が主軸になりやすいです。
上級 / 峻烈激情告白、対立、犠牲、反転、覚悟の衝突が強く出やすいです。
熟練 / 悪辣怨情敵視、報復、裏切り、破局、濁った情念の衝突が起こりやすいです。

これは出発時の目安です。「熟練だけど旅情」「初級だけど怨情」 のように別軸で決めても構いません。大切なのは、どんな物語とどんな返り方を主軸にする卓なのか を最初に合わせることです。

6. 局面の読み方

GMの局面投稿には、ふつう次のような情報があります。

  • 現在の状況: 今どこで何が起きているか
  • 期限: いつまでに動く必要があるか
  • 争点: 今回いちばん問題になっていること
  • 既知情報: PCが知っていてよいこと
  • 難度: どれくらい厳しい場面か
  • 現在状態: 命運点、課題タグ、手札タグ、履歴など
  • 選択肢例: こういう方向で動ける、という出発点

選択肢例は、そのまま選んでもいいし、少し変えてもいいし、まったく別の案を出しても構いません。
「GMが出した3つしかやってはいけない」という意味ではありません。

モードが有効な局面ではここを見る

GM がモードを使う局面だと明言したときは、いつもの局面文に加えて、次を見ます。

モードまず見ること
オブザーバーいま有効な観測方針と、その局面で GM がどの行動原理を参照しそうか。
プロバビリティ今回が「運の揺らぎを入れる場面」かどうか。難度とタグの噛み合わせがいつもより重要です。
クオンティティ文章とタグだけで読み切れない余白があるか、あるなら主能力は何になりそうか。
ソーシャル表層サイン、共有圧、次防御予告、残っている共有状態です。
タクティカル誰がどこにいて、どの共有圧を止めるべきかです。
ウィスパーどこまでが共通情報で、どこからが自分だけの事情かです。
アンロック可視構成要素、既知の物理条件、構造圧、次の構造反応、残っている共有安全状態です。
バーサス対象、争う層、共有状態、外部圧です。
リラックス今回の候補が、回復・仕込み・後始末・主野望前進のどれかです。

どのモードでも、追加されるのは 考えるための足場 です。何をしたいかを文章で宣言する基本は変わりません。

GM がシーン種別を公開しているとき

GM は局面の見出しや冒頭で、導入 / 親密 / 対立 / 行動 / 余韻 のどれとして出しているかを公開してかまいません。これは選択肢の強制ではなく、いまどの方向へ書きやすいか を知らせる補助線です。

【公開シーン種別】導入 / 親密 / 対立 / 行動 / 余韻
【火種表示使用】可 / 不可（親密 / 対立 / 余韻では可になりやすい）
  • 導入: 状況確認や方針決めを書きやすい
  • 親密: 相談、告白、回復、和解、弱さの開示を書きやすい
  • 対立: 言い分の衝突、立場の揺らぎ、秘密の火種を書きやすい
  • 行動: 主目的と手段を明快に書きやすい
  • 余韻: 言えなかったこと、後味、次へ残す火種を書きやすい

GM が公開していない卓でも遊び方は変わりません。見えているときは、白紙負荷を減らす目印 として使ってください。

バーサス・モードが有効な局面ではここを見る

バーサス・モードが有効な局面では、ふだんの局面投稿に加えて、誰と何を争っているか が明示されます。まずは次の欄を見てください。

欄まず見ること
対象今回、誰 / 何の最終状態を争っているのか。
争う層身体、位置、物品、社会のどの層が主戦場なのか。
共有状態すでに確定している追い詰め、露見、退路半壊、疑惑などは何か。
同時提出方式公開同時か、GM宛て秘匿同時か。投稿順が有利にならないか。
外部圧教師、警察、締切、増援、世論など、放置すると第三の悪化になるものは何か。

大事なのは「相手に勝つ方法」だけではありません。今回どの層だけを強く争うか と 勝っても何が残るか を読むことです。

7. 宣言の考え方

宣言は、自由記述で大丈夫です。長くても短くてもかまいません。
だいたい次の3つが入っていると、GMが読みやすくなります。

  • 何をしたいか
  • どうやるか
  • 何を守りたいか、何なら差し出せるか

プレイヤーは、タグ種別や記録形式を先に選ばなくて構いません。
何が真で、何をしたくて、何を残したいか を書けば十分です。GM や補助ツールは、それを必要に応じて 課題タグ / 手札タグ / 履歴 / 人物履歴 / 保持ノード へ正規化します。分類が未記入だから差し戻す ことはしません。

オブザーバー・モードでは観測方針を書く

オブザーバー・モードが有効な PC は、局面ごとの宣言の代わりに、シナリオ開始時に次のような 観測方針 を提出します。

【観測方針】
最優先:
取りにいくもの:
平時の第一手:
危機時の第一手:
捨ててもよいもの:
禁止事項:
退路基準:

プレイヤー側で大切なのは、何を優先し、どの線を越えたら引くのか を言葉にしておくことです。局面の途中で細かく書き換えるのでなく、シナリオの節目で更新するモードだと考えてください。詳しい運用は 選択ルール集 にまとめています。

まずは自由に書いてよい

たとえば、こんな書き方で十分です。

吹雪が強まる前に、荷運び仲間を集めて峠道に道しるべを立てます。
子どもと老人を先に山小屋へ動かしたいです。荷物はかなり捨ててもかまいませんが、弟だけは危ない場所に近づけません。

この一文の中に、やりたいこと、やり方、守りたいもの、払ってよいもの が入っています。

書きやすい人向けの型

自由記述で困るときだけ、まずは次の 4 欄 だけ埋めれば十分です。
これはプレイヤー向けの短縮入力で、GM や補助ツールが必要なら詳細書式へ広げます。

主目的:
最低限ライン:
頼る焦点(1〜2):
任意: 動かしたい相手 / 残したい変化:

頼る焦点 には、道具、関係、評判、履歴、場の利、味方、地図など、今回強く効かせたいものを 1〜2 個だけ書きます。詳しい手段や代償の候補は、必要になったときに GM が聞き返したり、補助ツールが追加入力欄を出したりして構いません。

細かく書きたい人は、このあとに自由に補足を足してください。

迷ったらの既定

QBS では、まず いま真になっていること を読み、そのうえで 今回の主目的 を通す前提で考えます。GM が請求してよいのは、見えていた / 読めた / 仕込めた障害 が生む価格です。迷ったときは、いきなり不成立にせず、まず 別利益の切り捨て → 時間や位置 → 露見や消耗 → 関係や評判 → 長期線の傷み の順で支払いを置きます。タグがないことだけを理由に、成立済みの事実が消えることはありません。大きな成功や傷は、演出で流さず 履歴 / Builds / Stories のどこかへ残ります。

宣言のコツ

  • 主目的は1つに絞ると通りやすくなります。
  • 全部を一度に取ろうとすると、支払いは重くなりやすいです。
  • 「無傷で」「証拠も回収して」「追手も消して」「全員生還して」のように利益を重ねるほど、当然ではなくなります。
  • 困ったら、このPCが今いちばん守りたいものから逆算すると動きやすいです。

また、まだ決まっていない余白なら、世界観に矛盾しない範囲で「この土地にはこういう慣習があるのでは」「保守用の導線が残っていそうだ」のように提案して構いません。詳しい使い方は プレイヤー実戦ガイド の NPCや組織も動かしてよい を見るとつかみやすいです。

複数PC卓での書き方

複数PC卓では、自分のPCが何をするかを書けば十分です。
連携したいなら、「誰と合流したいか」「何を担当するつもりか」を一言添えると伝わりやすくなります。

ウィスパー・モード が有効な卓では、あなたのPCだけが知っている事情や、あなたのPCだけが強く望む着地点があるかもしれません。そのときも、まずは 自分のPCが何をしたいか を書けば十分です。秘密の中身を毎回全部明かす必要はありません。

ただし、まだ共有していない事実を他PCが当然知っている前提で連携を書くことはできません。逆に、相談していないこと自体が失敗や妨害に読み替えられることもありません。同じ対象の排他的な終着点 を取り合うときだけ、GM が競合として扱います。

複数PC卓では同行メモを置いてよい

フルのヨーク・モードを使わない卓でも、複数PC卓では 同行メモ を 1 行だけ置くと連帯を書きやすくなります。

【同行メモ】(各PC1行 / 任意)
今回いっしょに守るもの:
帰るときに確かめたいこと:

これは新通貨ではありません。だれと、どの関係線を今回前に出すか を見える化するだけです。救援、肩代わり、回復、後始末、継承の宣言が書きやすくなります。

バーサス・モード局面では一段だけ細かく書く

バーサス局面でも、基本は自由記述のままで構いません。ただし、何を争うか を一言添えると、GMがかなり読みやすくなります。強く争う層は、1回の宣言につき1つまで が基本です。

主目的:
対象 / 層:
手段:
守りたいもの:
捨ててもよいもの:

他PCの内面を直接決めることはできません。「恐怖した」「改心した」「恋に落ちた」 のような内面は、GMや相手プレイヤーの応答余地が残ります。

迷ったときは事実確認してよい

方針が決まらないときは、短い事実確認をして構いません。たとえば、

  • 今日は何日か
  • 出口までどれくらいか
  • その場で何が見えているか
  • PCなら知っていて当然の常識か

といったことです。

ただし、どれが正解か、どちらが成功しやすいか、真相そのもの はGMに聞くものではありません。
その先を考えるのがプレイの中心です。

ビルドは「行動の一覧」ではなく、宣言の値段を変えます

QBS の Builds は、「この職業だからこの行動しかできない」 というルールではありません。
保持タグや保持ノードは、自由宣言に対して次の4種類の働きをします。

  • 許可: ふつうは別工程や重い段取りになりやすいことを、1宣言で扱いやすくする
  • 値引き: 必要な担保、代償、別利益の切り捨てを1段軽くする
  • 保険: 余波を軽くしたり、顕在化を遅らせたりする
  • 変換: 成果や代償を、素材、保持タグ、火種表示へ変える

つまり、ビルドとは宣言の射程と価格を変えるものです。

例: 〔狙撃姿勢｜長期: 遠距離の精密処理を一宣言で扱いやすい / 乱戦や狭所では鈍る〕
→ 「狙撃しかできない」のではなく、
   「遠距離の精密処理を 自由宣言 で安全に大きく扱える」

通し例: 〔裏社会の顔役とのコネ〕

たとえば盗賊の PC が 〔裏社会の顔役とのコネ｜長期: 闇市・紹介・口利きを扱いやすい / 公権力が前面に出る場では鈍る〕 を持っているとします。

  • 許可: 顔役を通して闇市へ入り、情報集めと買い物を 1宣言 でまとめやすくする
  • 値引き: 潜入が露見しても、【指名手配】 ではなく 【見回り強化】 くらいで済ませやすくする
  • 保険: 騒ぎを起こしても、衛兵の到着を翌朝まで遅らせる、といった逃げ道を作りやすくする
  • 変換: 盗んだ書類を横流しして、〔闇市場の通行証〕 や 〔関係: 顔役への大きな貸し〕 に変えやすくする

Build があるから新しい行動が解放されるのではなく、もともと自由に宣言できる行動の「通しやすさ」が変わる と読むと、QBS のビルドは理解しやすくなります。

「誰の心をどう動かしたいか」も書いてよい

Stories が入ったQBS では、主目的を「物を手に入れる」「敵を止める」だけにしなくて構いません。
誰の心を、どう動かしたいか をはっきり書くと、関係の進展や秘密の開示が強く動きます。

たとえば、

  • 「妹を説得する」より 「妹に、私をまだ信用してよいと感じさせたい」
  • 「隊長を止める」より 「隊長に、自分ひとりで背負わせないと受け取らせたい」
  • 「黒幕を追い詰める」より 「黒幕に、こちらが真相へ届いたと悟らせて動揺させたい」

のように書くと、結果が「関係」として残りやすくなります。

8. 難度と結果の受け取り方

このゲームでは、宣言は基本的に前へ進みます。
ただし、何もかも理想通りになるとは限りません。とくに高い難度では、成功しても代償や余波が重くなります。

難度感触起こりやすいこと
容易立て直しや整え直し回復や整理がしやすい
普通基準の難しさ主目的は達成し、代償がつく
困難かなり厳しい主目的は達成し、代償と余波が残る
過酷理想通りではない山場意味のある前進はするが、結果の質や範囲が少し欠ける
絶望大きく前へ進むが痛い成果は残るが、かなり嫌な支払いが来る

主目的以外は自動ではついてこない

このゲームでは、その局面で成功した主目的がかないます。
でも、そこから先の利益まで自動で全部ついてくるとは限りません。

たとえば、

  • 証拠をつかむこと
  • 安全に持ち帰ること
  • 追手を振り切ること
  • 生きて帰ること

は、場面によっては別々に問われます。
「取れた」ことと「無傷で全部片づいた」ことは同じではない、と覚えておくと読みやすくなります。

不可能なことはある

一度の宣言で、手がかりゼロのまま黒幕を断定したり、遠く離れた土地の問題を全部片づけたりはできません。
GMはできるだけ「今できる規模」を返してくれますが、宣言の規模そのものは場面に合っている必要があります。

プロバビリティ・モードでは、先に質を読み、そのあとで振ります

プロバビリティ・モードでは、出目だけでなく宣言の書き方が結果に響きます。主目的、手段、備えがはっきりした宣言ほど結果が安定しやすく、薄い宣言ほど出目だけでは伸びません。

つまり、このモードで意識するべきなのは「高い目を出すこと」より、何をどうやるかを文章で通すことです。詳しい数式や裁定順は 選択ルール集 を参照してください。

クオンティティ・モードでは、数値は文章より下に置きます

クオンティティ・モードでは、能力値があっても、タグ > 履歴・関係・準備 > 能力値 の順で読みます。数値だけで不可能を可能にはできませんし、数値が低くても十分な準備と筋の通った宣言があれば通ります。

9. タグ・保持・命運点・大切なもの

タグは3種類で考えるとわかりやすい

タグは、世界に新しい真実を生むものではありません。すでに成立していることのうち、いま強く認識しておきたいもの を短く焦点化する印です。

種類意味
課題タグ今すぐ、あるいは優先的に対処したい困りごとです。放置すると悪化します。
手札タグあとで自分から使える資源です。協力者、証拠、肩書き、道具、関係、習慣などがここに入ります。
履歴今は用途を固定しない事実の記録です。あとで宣言の根拠にでき、必要なら Build に育ちます。

課題タグは次の判断を変える

課題タグは飾りではありません。
短期ならすぐ響き、中期ならその事件の間じゅう効きやすく、長期ならシナリオをまたいで残ることがあります。

同じ危険に、同じ対策不足のまままた踏み込めば、タグは本当に悪化します。
ついたタグは、次の宣言に織り込んで考えてください。

命運点は「残機」ではない

命運点は、PCの命の余力です。0になると、そのPCは死亡するか、冒険の主役として立ち上がれない状態になります。

  • 初期最大値は、GMが採用した大切なものの数 + 1
  • ふつうは、まず課題タグで危険が示されます
  • 命運点は、それでも足りないほど直接に命を削るときに使われます

だから、命運点は「何回でも死を無効にできる回数」ではありません。
たとえば 【次の局面に進む前に治療が必要 / 未処置なら退場】 のような重いタグがついているなら、それを放置したまま次へ進めば危険です。

大切なものは弱点でもある

大切なものは、そのPCの支えです。
同時に、物語の中で揺らされる場所でもあります。

だからといって、低い難度でいきなり永久に壊れるのが普通というわけではありません。まずは、疑い、距離、傷つき、失いかけ、といった形で危険が見えます。

なお、大切なものを永久に失ったときは、命運点の最大値が 1 減り、命運点も 1 減ります。

回復の考え方

回復は、専用の休憩場面だけで起きるとは限りません。
ふつうの局面の中で、

  • 休んでから動く
  • 治療役を確保してから踏み込む
  • 自分で抱えず誰かに引き継ぐ
  • 荷を捨てて負担を減らす

といった書き方をすると、前へ進みながらタグを軽くできます。

保持タグ・保持ノード・素材

手札タグのうち、シナリオをまたいで固定されたものを、QBS では 保持タグ と呼びます。
書式は通常の手札タグと同じですが、「今後のあなたらしさ」として扱われます。

種類主な役目例
手順表示宣言の許可・値引き・保険・変換を支える〔手順: 下調べの手順〕〔手順: 応急手当の型〕〔手順: 顔役の作法〕
資産表示道具、武具、現金、土地、資格、工房の鍵などを保持する〔資産: 連発式の小弩〕〔資産: 護符の束〕〔資産: 測量具一式〕
場 / 関係表示基地、工房、船、相棒、紹介網、庇護などを保持する〔場: 港の工房〕〔場: 修道会の宿〕〔関係: 荷運び仲間〕
素材あとで工作や改造や保持ノード化の材料にする〔資産: 竜骨片〕〔資産: 旧式封印器〕〔制度: 監査印章〕

素材は、それだけでは保持の完成形ではありません。成長点・工作・幕間処理 と組み合わせて、資産表示・制度表示・場表示・関係表示などの保持へ育ちます。

保持ノード方式の考え方

QBS では、保持全体 と 保持ノード を分けて読みます。
保持ノード は、Build のうち装備・拠点・制度・共同体・重い関係・手順・ルートのように、台帳で継続管理する形式です。

大事なのは「何を持っているか」より、その保持がどの宣言をどう安くするか です。保持ノードは、まず次の三つが読めれば使えます。

  • 何に効くか: 調査、搬出、受取、面会、避難、封鎖など、どの宣言へ触れるか
  • どこで鈍るか: 広域では弱い、公権力案件では鈍る、通信断では鈍る、など
  • どう傷むか: 破損、疲労、監査、露見、離反、更新失敗、など
表示例どう効くか
資産表示〔資産: 携帯結界具〕封鎖・浄化・足止めを扱いやすくする
関係表示〔関係: 港の走り子ミナ〕配達・聞き込み・受け渡しを扱いやすくする
制度表示〔制度: 港湾検分補佐資格〕面会・検品・搬入・受取の価格を軽くする
路表示〔路: 夜間搬出網〕人手・運搬・隠匿・後始末の足場になる
手順表示〔手順: 退路管理の型〕撤収・再集合・避難誘導を扱いやすくする

単なる評判や肩書は、そのままでは Quiddities 寄りです。
繰り返し使えて、面会・通行・受取・紹介の窓口になる ときに、保持ノードとして扱います。

1回の宣言で、主に数える保持ノードはふつう 2つまで です。たくさん書いてもよいですが、値引きや保険の根拠として強く数えるのは 2つまでだと覚えると遊びやすくなります。

保持タグを関係・感情・火種として書く

関係表示 / 感情表示 / 火種表示 は、別ルールではありません。どれも同じ 保持タグ を、関係・感情・火種として読みやすくするための表示名です。

  • 関係表示: 信頼、貸し借り、庇護、不信、約束、所属など。例: 〔関係: リアへの信頼｜長期: 協力・庇い合い・相談の宣言を強くできる / 裏切り・離別・沈黙で変質〕
  • 感情表示: 執着、恐れ、憧れ、怒り、後ろめたさなど。例: 【感情: 執着｜中期: 対象が絡むと撤退や切り捨てが高くつく / 告白・断念・和解で変質】
  • 火種表示: 今すぐ使わなくても、あとで関係や場面をひっくり返しそうなもの。例: 〔火種: 言えなかった本音｜長期: あの相手が絡むと話を動かしやすい / 告白・露見・断絶で解消〕
  • 人物表示: 人物像、評判、見られ方、癖、役回り。長めに追いたいときは 人物履歴 を使います。

同じものが、課題にも手札にもなりえます。 たとえば `〔関係: リアへの信頼〕` を切ることも、`【関係: リアへの信頼】` が重荷として働くこともあります。外側の記号は役割、内側のラベルは読み味です。

火種専用の記号は使わなくても構いません。正式記法では、火種も課題 / 手札のどちらかに置く と考えると運用が安定します。

10. 成長点・野望度・シナリオ終了

野望度は、戦闘力ではなく、野望への近さです。シナリオが終わったあと、GM は「今回ほんとうに野望へ近づいたか」を見て、野望度を上げるか判断します。

  • +1: 野望への足場がきちんと前へ進んだ
  • 0: 事件は越えたが、野望そのものはあまり進んでいない
  • -1: 野望への足場を壊した、支援や資格を大きく失った

単発シナリオでは、野望度 2 を終点にすることが多いです。つまり、今回の話を越えれば、そのまま完結まで行ける形です。

ただし、単発の終了時に付く +1 / 0 / -1 は、まずその 1 本の達成度採点 です。単発の最終採点としての 0 は、直ちに引退や再出発を意味しません。継続して同じ PC を使う場合に、実際の現在野望度が 0 になったときだけ、引退か再出発の処理へ進みます。

成長点はいつ使うか

成長点を使うタイミングは、原則としてシナリオとシナリオの間です。
GMがシナリオ終了を宣言し、成長点・素材・一時手札を確定したあと、プレイヤーが「何を固定するか」「どこを増設するか」を申告します。

  • 獲得のタイミング: シナリオ終了時や、シナリオ内で世界課題タグを達成したとき
  • 使用のタイミング: シナリオ終了時の整理
  • 再訓練: 既存ノードの寄せ直しは幕間のダウンタイムで行います

シナリオ終了時の成長点

QBS では、野望度とは別に 成長点 を得ます。野望度が「人生の到達度」なのに対して、成長点は コツコツ積み上げた腕前 です。

目安は次の通りです。

  • シナリオを達成した: 成長点 +2
  • 完全達成ではないが、意味のある前進や生還があった: 成長点 +1
  • 追加で +1（最大 2 回まで）:
    • 世界課題タグやサブ目標を片づけた
    • 素材や拠点につながる発見・工作・お使いを達成した
    • 告白 / 裏切り / 和解 / 犠牲 / 秘密開示などの大きな転換点が起きた

つまり、ふつうは 1〜4 点 の範囲で増えます。

使い道目安
小さな保持タグを取る2点
一時手札を保持タグへ固定する2点
強い保持タグ、保持ノードの強化、重い関係や制度の保持化3点
再訓練、軽い工作、表示ラベルの整理1点

成長はメニューから選ぶだけでなく、プレイ中に作った一時手札を固定する ことでも起こります。これにより、「プレイした宣言そのものがビルドになる」流れを保てます。

手札タグは、まずタグ本文の期間に従って残るかどうかを見ます。 短期なら今回限りになりやすく、中期・長期なら次回へ持ち越しうる、という読みです。固定 は「全部の手札に EXP を払って残す」処理ではなく、一時手札・臨時協力・素材を Build に変える 処理だと覚えてください。

選択ルールを使う卓では追加条件が付くことがありますが、詳しくは別冊の選択ルール集に従います。

シナリオ終了後にやること

プレイヤーがまず答えるのは、次の 3 問 + 任意 1 問 で十分です。

今回、真になったこと:
次回も持ちたい焦点:
自分 / 関係で変わったこと:
任意: 成長希望:

成長希望 は、残す / 始める / 伸ばす / つなぐ / 整える の 5 動詞で書けます。GM や補助ツールは、この入力をもとに内部で タグ整理 → 達成度判定 → 報酬確認 → 成長処理 → 台帳更新 の順へ正規化します。

基本は GM が進行し、プレイヤーが短い希望を返す 流れです。「今回はこの協力者を残したいです」「この隠れ家を伸ばしたいです」と相談しながら決めてください。

旅や発見を一言だけ残したいなら、上の 3 問のどこかに混ぜて書いて構いません。 独立した旅履歴が必要なのは、旅情や探索の余韻をあとで拾いたいときだけです。

任意: 旅履歴を一言残す

旅や発見が中心の回では、成長点や素材とは別に 旅履歴 を短く残して構いません。これはその回で見たもの、出会った人、その土地の風習、持ち帰った余韻の記録です。

【旅履歴】(任意 / 1本ごとに1つで可)
景 / 人 / 風習 / 持ち帰り のどれか1つを一言:

旅履歴はその場では数値ボーナスではありません。ただし、あとで同系統が積み重なり、この旅で自分がどう変わったか が一文で見えたなら、GM は Stories や 素材 に結びつけてかまいません。

白紙から毎回ひねり出す必要はありません。 GM や補助ツールがログから候補を 1〜2 件出し、プレイヤーが 採用 / 修正 / 見送り を選ぶ形で構いません。

成長点の使い方（保持ノード方式）

QBS では、成長点の使い道を「何を買うか」だけでなく、保持のうち、台帳で管理する部分をどう育てるか で見ます。

プレイヤーの言い方何をしたいかGM / 補助ツール側の正式処理
残す今回得た一時手札や臨時協力を、次回以降も握りたい固定 / 端緒化
始める新しい保持ノードを立てたい開設
伸ばす既存ノードの用途や広さを増やしたい増設 / 昇格
つなぐ二つの保持を一緒に効くようにしたい接続
整える文言や寄せ先をまとめ直したい再訓練 / 圧縮

プレイヤーはこの 5 動詞で希望を言えば十分です。正式な操作名や細かなコストは GM や補助ツール側の処理 として扱えます。

変質として残すもの

シナリオの終わりには、何を保持するか だけでなく、何が変質したか も見ます。

  • 関係が深まる / ねじれる / 壊れる
  • 感情が軽くなる / 別の感情へ移る / 重荷になる
  • 火種が残る / 露見する / 解消する
  • 人物像の見え方が変わる

ここで決めるのは「Build か Story か」の二者択一ではありません。何を保持し、何が変質したか を書き、そのあとで必要なら資産 / 関係 / 感情 / 火種 / 人物の表示名を付けます。

リラックス・モードでは何が成長になるか

このモードでは、毎回のシナリオで 野望度が動かなくてもかまいません。 野望度は人生の到達度なので、節目や価値選択が起きた回だけ動く、くらいで十分です。

そのかわり、ふだんの満足は 成長点、素材や保持タグ、関係の進展、秘密の開示、世界課題タグの解決 で受け取ります。小さな依頼、補給、工作、和解、聞き込み、後始末も、ちゃんと成長になります。

回復回や穏やかな回でも、何か一つは次に効くものを残す と遊びやすくなります。新しい協力者、通い慣れた道、工房の足場、信頼の回復など、次の宣言が少し楽になるものが残れば十分です。

ゼノス・モードではどんな野望でも始められるか

始められます。野望は「成り上がり」や「復讐」だけではありません。世界のさまざまな場所を見て回る、平穏な日常を守る、困っている人々を助ける、各地の祭りや作法を知る のようなものでも構いません。

ゼノス・モードでは、まだ何者になりたいか決めきれていないが、まず世界を歩いて確かめたい という薄い野望から始めてもよいです。旅の途中で野望が言葉になっていくこと自体を前進として扱えます。

11. 卓越宣言

卓越宣言は、奇抜な発想や難しい理屈だけを指しません。
見えている情報を使って、一手先に備えるだけでも立派な卓越の入口です。

たとえば、

  • 追っ手が来る前に退路を確保しておく
  • 説得の前に証人や証拠を押さえておく
  • 危険な探索の前に休息や治療を入れておく
  • 怪異らしい兆候を見て、先に対策材を用意しておく

といった宣言は卓越になりやすいです。

逆に、準備なしで危ない場所へ突っ込む、根拠なく一発逆転を狙う、といったものは、派手でも卓越とは限りません。

12. うまく遊ぶためのコツ

  • 主目的は1つに絞る
  • 困ったら、何を守りたいかから考える
  • タグ、手札、履歴を見て次の宣言を組み立てる
  • 選択肢例はそのまま使ってもいいが、少し自分のPCらしさを足すと通り方が見えやすくなる
  • 完璧な答えを探し続けるより、今のPCが選びそうな前進を選ぶ

13. よくある疑問

Q. GMの選択肢がしっくりきません

A. そのまま選ばなくて大丈夫です。選択肢例は出発点です。
「自分のPCならこうしたい」を書いて構いません。

Q. 代償が怖いので、動かないほうが得では?

A. 動かないことにも、時間切れや状況悪化があります。
このゲームは、何も失わずに止まるより、何を払って前へ進むかを選ぶ方向に向いています。

Q. 何を書けばいいかわからなくなります

A. まずは、何をしたいか / どうやるか / 何を守りたいか の3つだけで十分です。
短くても大丈夫です。

Q. 型どおりに書かないとだめですか

A. だめではありません。自由記述で構いません。
型は、整理しやすい人向けの補助です。

Q. 大切なものを書いたら、そこばかり狙われませんか

A. 物語の中で揺らされることはあります。
でもそれは、そのPCにとって本当に意味がある場所だからです。そこがあるほど、何を守る話なのかがはっきりします。

Q. いい案が思いつきません

A. そんなときは、先に休めないか、先に証拠を押さえられないか、先に逃げ道を作れないか、先に味方を増やせないか を考えると動きやすくなります。

Q. Build とは何ですか? ジョブやスキルと何が違いますか

A. Build は、あなたの自由宣言に 許可・値引き・保険・変換 を与える 保持された焦点 です。
現金、土地、道具、資格、パトロン、恩義、執着、人物像は、どれも本質的には Build になりえます。違うのは本質ではなく、資産 / 関係 / 感情 / 火種 / 人物 といった表示名です。ジョブや固定コマンドの一覧ではなく、「この行動しかできない」ではなく「この行動を安く安全に通しやすい」 を作るものだと考えてください。

Q. ビルドが増えると、選択肢を選ぶゲームになりませんか

A. なりません。QBS のビルドは「できることの一覧」ではなく、自由宣言の許可・値引き・保険・変換 として働きます。
同じ宣言でも、どのPCならどんな値段で通せるかが変わる、と考えてください。

Q. 成長点はいつ使えますか

A. 原則として シナリオ終了時の成長処理 で使います。
GMが成長点を配り、今回得た一時手札や素材を確認したあと、プレイヤーはまず「残したい / 始めたい / 伸ばしたい / つなぎたい / 整えたい」と申告します。必要なら GM や補助ツールが、それを 固定 / 増設 / 開設 / 昇格 / 接続 / 圧縮 / 再訓練 へ正規化します。再訓練 はふつう幕間で行います。

Q. 成長点は、野望度と何が違うのですか

A. 野望度は人生の到達度、成長点は地道な研鑽です。
野望度が大きく上がらない回でも、調査、工作、お使い、和解、秘密開示で成長点は増えてかまいません。

Q. 絆を使うと、毎回つらい話になりませんか

A. つらくなることもありますが、それだけではありません。旅情や人情の卓なら、絆は 安心して潜れる居場所 や 回復の理由 にもなります。大事なのは、使ったあとに 何も変わらない を続けないことです。

Q. リラックス・モードだと、終わらないだけになりませんか

A. なりません。各シナリオはいつもどおり終わります。 ただし、キャンペーン全体の「最後の大決戦」を必須にしないだけです。区切りごとに次回候補を選べるので、止まりたくないときは続け、十分だと思ったらその節目で締めても構いません。

14. 最後に

このガイドで覚えてほしいことは、実はそんなに多くありません。
何をしたいか、どうやるか、何を守りたいか。まずはそれを書ければ、もう遊び始められます。

TRPGが初めてで、もっと軽い入口がほしいなら クイックスタートガイド を先に読んでください。
プレイヤーとしてもう一歩上手くなりたいなら プレイヤー実戦ガイド、GMとして卓を回したいなら GMガイド、自分で単発シナリオを設計したいなら シナリオ設計ガイド、長いキャンペーンを設計したいなら キャンペーン設計ガイド へ進んでください。
