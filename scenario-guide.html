QBS シナリオ設計ガイド

単発シナリオの設計書を自分で書くGM向けの本です。
設計の土台に加え、上級者・熟練の構造難度、高難度化、テンプレート類、作者メモを扱います。長いキャンペーンの節目設計や持ち越し管理は キャンペーン設計ガイド に分けています。

1. はじめに

このガイドは、GMガイド の「既存資料を使って卓を回すための本」に対して、単発シナリオの設計書を書くための本です。
ここで扱うのは、1本のシナリオの問い、条件、期限、主要局面、構造難度、作者メモ、テンプレート、公平性チェックです。

長いキャンペーンの本数設計、節目管理、持ち越し資源、長期の世界反応、進行ログなどは キャンペーン設計ガイド に分けています。
単発を作ってから長い流れへ接続したいときは、この本のあとにキャンペーン設計ガイドを読んでください。

この本で扱う範囲

この本で扱うのは、単発シナリオの問いと条件、局面構成、構造難度、高難度化、複数PC対応、世界反応、伏線管理、作者メモ、テンプレート、公平性チェックです。

逆に、既存シナリオをどう読み、どう公開投稿へ変換し、どう裁定するか は GMガイド、複数話を束ねて長い流れをどう保つか は キャンペーン設計ガイド に分けています。

難度ラベルの見方

このガイドでは、卓の基準を 卓等級 と 裁定傾向 / 指針提示度 / 劇的展開度 / 死亡許容度 の五項目で扱います。 既定は 中級 = 中級 / 適正 / 啓示 / 人情 / 中 です。熟練卓とだけ言うなら、熟練 / 悪辣 / 暗黒 / 怨情 / 極 が目安です。

設計の基本線
まず 問い・条件・期限・局面 を決め、そのあとで 何を先読みさせたいか / どこで外側のツケを返すか / 何を持ち帰らせたいか を重ねます。最初から難しさだけを盛るより、骨格を先に置いたほうがQBSらしく安定します。

シナリオ設計でも Q / B / S で考える

シナリオを作るときは、局面の問いだけでなく、今回何が新たに真になるか、そのうち何を保持させるか、それがだれとの関係の中でどう変わり始めるか を先に置きます。

  • Quiddities（いま真であるもの）: 今回どの問いに答えさせ、何を事実・立場・可能性として世界に残すか
  • Builds（保持された焦点）: どの素材、保持タグ、保持ノード、関係表示、感情表示、人物履歴を持ち帰らせるか
  • Stories（変質の連続）: どの関係、感情、秘密の当事者性が、次へどう変わり続けるか

面白い単発が、そのまま三要素の種になるように設計すると、キャンペーンへきれいにつながります。ひとつの成果が三つ全部に返ってよい と考えるのがコツです。

物語と旅情の欲求を単発へ織り込む

単発シナリオでも、事件を解くだけでなく その世界へ行きたくなる理由 を置くと没入が強まります。設計時には、次の八つを「局面・報酬・情景」のどこへ返すかを考えてください。

  • 旅立ちたくなる理由: この世界でどんな物語が待っていそうか
  • 自由・自律: どこで自分の意思のルート選択が効くか
  • 新奇・偶発: どんな出会い、祭り、検問、嵐、噂が起こりうるか
  • 挑戦・熟達: 何を越えると「自分はできた」と感じるか
  • 文化・知的没入: 礼や作法や生活差をどこで肌で理解できるか
  • 共同性: 誰と何を分担すると旅らしい連帯感が生まれるか
  • 畏敬・審美: どの景色や儀式で心を打つか
  • 物語化・自己像: 何を持ち帰ると「この旅は自分の物語だった」と言えるか

ゼノス・モード を採るなら、この八つのうち 新奇・偶発、文化・知的没入、畏敬・審美、物語化・自己像 をとくに厚くします。攻略の成否より、何に触れ、何を持ち帰るか が見える設計に寄せてください。

高揚設計を別軸で持つ

QBS では 劇的展開度 が 旅情 / 人情 / 激情 / 怨情 の「味」を決めます。そこへ別軸として 高揚類型 を重ねると、どの局面でどんな快を起こしたいかが設計しやすくなります。劇的展開度は雰囲気、高揚類型は場面の跳ね方です。

高揚類型下位類型主に動きやすい要素置きやすい局面
世界拡張閾越え / 顕現 / ルール・地図の開示Quiddities（いま真であるもの）導入、到着、発見回
再文脈化伏線収束 / 正体露見 / 前提崩壊Quiddities + Stories中盤反転、終盤直前
逼迫突破段階的試練 / 薄氷突破 / 救援集結 / 覚醒・知略逆転Builds + Quiddities危機、最終工程、脱出
関係変容接続 / 犠牲と救済 / 継承と連帯Stories（変質の連続）再会、和解、別れ、継承

1本の単発では、主高揚類型を1つ、副高揚類型を1つ までに絞ると扱いやすくなります。最後にどの類型で締めるかを先に決めると、局面の役割がぶれにくくなります。

高揚類型を理屈として置くだけでなく、今回どの瞬間で跳ねさせるか を先に予約しておくと、局面の役割がぶれにくくなります。これはプレイヤーへ見せる選択肢ではなく、作者メモ側の設計欄です。

全部を別シーンにする必要はありません。同じ局面で二つ以上をまとめても構いません。 大事なのは、「今回はどこで世界を開き、どこで意味をひっくり返し、どこで押し切り、だれとの関係が深まるか」を先に見える化しておくことです。

旅履歴の要約語 は、今回の旅や事件を一言で思い出せる具体名です。地図、歌、傷、祈り、誓い、仮面、借り、称号、鍵、土産など、終わったあとに「今回の話といえばこれ」と言えるものを置くと、単発でも記憶に残りやすくなります。

増幅因子

  • 崇高化: 宇宙、神話、巨大自然、歴史、巨大建造物、圧倒的存在感を重ねて awe を強める。
  • 蓄積からの解放: 伏線、延期、失敗、誤解、時間制限、未処理の追加利益を積み、あとで解く。
この分類はプレイヤーへ見せる選択肢ではなく、作者メモ側の設計ラベルです。QBS では主目的の成否だけでなく、追加利益、世界反応、Builds、Stories の返りを含めて高揚を作ります。

六つの欲求を各シナリオで点検する

単発でも、次の六つが本文のどこかへ返っていると、没入と達成感が安定します。全部を同じ重さで満たす必要はありませんが、各シナリオで最低一度は確認 してください。

  • 没入: 世界が少し広がるか。閾越え、顕現、ルール開示のどれかがあるか。
  • 成長: Q / B / S のどれかが見える形で残るか。
  • 自己表現: PC らしい解法、ルート差、持ち帰り差があるか。
  • 戦略: 追加利益、ルート差、別利益の競合があるか。
  • 非日常: 回復点、畏敬点、文化差、異境が本文に見えているか。
  • 連帯: 分担、救援、和解、継承のどれかで関係が深まる余地があるか。

2. このガイドで使う言葉

この本では、本質層 / 記録層 / 表示層 を分けて書きます。資産、関係、感情、火種、人物像の違いは本質の違いではなく、保持や変質をどう見せるかという表示差です。シナリオ設計で決めるのは、まず何が成立し、何が保持され、何が変質するかです。

この章では、このガイド特有の設計用語を、正式名称にそろえてまとめます。ルール全体の用語は、必要に応じて GMガイド と プレイヤー基本ルール を参照してください。

用語の関係は次の順で読むと整理しやすくなります。
核心障害 = 何に先回りさせたいか
真のボトルネック = 実際にどの工程で詰まるか
外側のシステム = なぜ局地勝利のあとにツケが返るか
構造カウンター C1 / C2 / C3 = そのツケがこのシナリオでどの形で返るか

これらは同義語ではありません。短い卓ではひとつの文に畳んでもよく、長い卓や熟練卓では分けて書くと、仕込み窓と別利益の位置がぶれにくくなります。

核心障害

そのシナリオで、事前対策が刺さる本命の障害の系統です。単に強い敵ではなく、先に読めた人だけが安く処理できる争点を指します。

真のボトルネック

核心障害が、実際の進行上ではどの工程で詰まりとして現れるかです。運搬、隔離、受け渡し、退出、生還のような工程がここに入ります。

仕込み窓

ある対策を、安く、自然に、早い段階で仕込める機会です。あとからでも不可能ではないが、高くつくように置くと差になります。

別利益

主目的とは別に守りたい利益です。匿名性、信用、証拠の正当性、NPC 保護、後始末の静かさ、撤収の安全などが入ります。

外側のシステム

局地の勝敗とは別に、あとから結果を返す外側の系です。監査、物流、受理、報告、政治、儀礼、帰還、安全保障などがこれにあたります。

予測宣言層

プレイヤーが選びやすい行動の読み筋です。どの層まで読めていたかで、構造カウンターの刺さり方が変わります。

局内工程

1つの局面の中で個別に処理する小さな工程です。ソーシャル / タクティカル / アンロックでは、1局面の中に複数の局内工程を立てて細かな指示や役割分担を受け止めます。

構造カウンター

局地成功のあとに返る不利です。C1 は最初の受け皿、C2 はさらに外側の受け皿、C3 は熟練卓で使う最外層の受け皿として使います。

大局的敗北

局面では通ったのに、シナリオ全体や外側のシステムで重い不利が返り、守りたかった別利益や帰還条件が崩れる状態です。

大局的勝利

局面の成功だけでなく、後から返る構造カウンターまで対策できている状態です。

共有ボトルネック

複数の局面や複数の主体が、同じ工程に足を取られる設計です。シナリオの問いを散らさずに圧を残せます。

外部主体 / NPC枠

PC 以外に、工程を動かしたり詰まらせたりする主体です。協力者、監督者、受取人、搬送役、敵対組織などをここへ置きます。

相互干渉フック

ある局面の結果が、別の局面や別の主体の判断を変える接続点です。孤立した場面列を防ぎます。

反応系統

行動に対して何がどう返るかの系統です。人、組織、制度、怪異、自然、物流など、返答元ごとに整理します。

観測トラック

見られる、記録される、痕跡が積もる、という圧を管理する軸です。

監査トラック

検分、照合、報告、裏取り、手続きの整合性によって返る圧を管理する軸です。

受取トラック

受け取り手が受理するか、疑うか、使えるか、引き渡せるかを管理する軸です。

物理条件

距離、重量、容量、時間、天候、地形、設備、消耗など、言い逃れしにくい制約です。

文脈補完スロット

細部を固定しすぎず、卓上の宣言に応じて埋められる余白です。仕込み窓を増やしつつ、一本道化を防ぎます。

隠し時計

公開せずに進める進行度です。敵の準備、儀式の進行、追手の接近、監査の蓄積などを裏で刻みます。

最速敗着線

もっとも早く破綻が起こる線です。運搬失敗、退出失敗、生還失敗、受取失敗のように、最短で詰む線を先に見ます。

高レバレッジ準備

少ない手数で後の複数工程を安くする準備です。証拠の確保、退路確保、口利き、正当性確保などがこれにあたります。

設計の基本順

  1. 今回どの野望をどう進めるかを決める
  2. 今回の問い・条件・期限を決める
  3. 主要局面を決める
  4. 高難度化や世界反応を重ねる
  5. テンプレートと公平性チェックで穴をふさぐ

QBSでは、乱数で事故らせるのではなく、何を読ませ、何を仕込ませ、どこで別工程を返すかで歯ごたえを作ります。
だからこそ、設計でも「先に骨格、あとで難しさ」の順が重要です。

4. シナリオの設計と残り方

シナリオとは、キャンペーンの道中で起こる一つの事件です。
1本のシナリオは複数の局面からできています。

単発シナリオでは、この1本がそのままキャンペーン全体です。
今回のシナリオの問いが、そのままPCの野望の問いになって構いません。

シナリオ設計では、まず 卓等級 を置き、その卓等級に見合う 構造難度、局面難度、構造カウンターの深さ を決め、そのあとに 指針提示度 と 裁定傾向 で見せ方と請求方向を調整します。構造難度とは、読解、先回り、取捨選択、文脈補完、複数PCの噛み合わせ、世界の広がりの重さです。

最初から暗黒や悪辣に寄せるのではなく、成功と裁定された局面の問いは通るが、退出、生還、引き渡し、匿名性、後始末が浅い成功では詰まりうる骨格 を先に置いてください。そのうえで、指針提示度では材料の再掲量、注釈量、選択肢例の粒度を調整し、裁定傾向では公開する3案の質、空欄の読み方、無工夫宣言への最低請求を調整します。
福音 / 啓示は見せ方を厚くする軸、寛容 / 悪辣 は不足や抜けをどちら向きに読むかの軸、劇的展開度 は世界とNPCが何をどう返すかの軸です。ここを混ぜないでください。

さらに、QBS のモードはすべて 任意・併用可 です。難しい卓だから自動で足すのではなく、シナリオの性質 と プレイヤーの希望 に合わせて使います。とくに ソーシャル / タクティカル / アンロック は、1つの局面の中に複数の 局内工程 を立てて細かな指示を受け止めるモードです。

QBS では宣言が基本的に成功するぶん、どのルートを選び、何を持ち帰るか の差が体験そのものを変えます。戦術だけでなく、旅の理由と余韻も設計してください。

  • 行き先の魅力: 風景、聖性、文化、珍しい生活
  • ルート選択: 安全ルート / 近道 / 地元ルート
  • 偶発フックの母材: 祭り、検問、商隊、巡礼、嵐、遅配、噂
  • 回復点 / 畏敬点: 休みたくなる場所、心を打たれる場所
  • 持ち帰り: Q / B / S のどれとして残るか

良い単発は、事件の骨格に加えて、なぜそこへ行くのか、どう向かうのか、行った結果として何が自分の中に残るのか を持っています。移動や滞在を単なる前置きにせず、行き先の魅力・ルート選択・偶発・回復点・畏敬点 のどれかを本文へ見える形で置いてください。

  • 回復点 / 畏敬点: 休みたくなる場所、心を打たれる場所
  • 持ち帰り: Q / B / S のどれとして残るか

QBS では宣言が基本的に成功するぶん、どのルートを選び、何を持ち帰るか の差が体験そのものを変えます。戦術だけでなく、旅の理由と余韻も設計してください。

劇的展開度と高揚類型は別に置く

旅情 / 人情 / 激情 / 怨情 は味つけであり、世界拡張 / 再文脈化 / 逼迫突破 / 関係変容 は場面の機能です。たとえば「旅情だが逼迫突破」「怨情だが世界拡張」「人情だが再文脈化」のように組み合わせると、QBS の自由宣言と物語の跳ね方を同時に設計できます。

迷ったら、導入で 世界拡張、中盤で 再文脈化、終盤で 逼迫突破 か 関係変容 のどちらかを主に置くと、短い単発でも波が出ます。

1) 今回どの野望をどう進めるか

まず、このシナリオがどの PC の野望を、どんな形で進める話なのかを決めます。
野望と関係の薄い事件だけが続くと、キャンペーンが散りやすくなります。

ただし、シナリオを終えれば自動で野望度 +1 ではありません。
このシナリオで得るのが、野望そのものへの前進なのか、痛み分けの足場なのか、野望を遠ざける歪みなのかを、設計時点で書き分けてください。

単発の終了時に付く +1 / 0 / -1 は、まずその 1 本の達成度採点 です。単発の最終採点としての 0 を、その場で自動的に「野望が折れた」とは扱いません。続投させる場合にだけ、次回開始時点の現在野望度を別途決めます。

2) シナリオの問い

この話で、何に答えが出れば終わりなのかを一文で書きます。
その問いが大きすぎるなら、各局面では 次の一手 / 手段の成否 / 周辺目的 に分解してください。

とくに最終局面や節目では、取得 / 退出 / 生還 のような別工程を同じ問いにまとめないでください。高い裁定傾向ほど、問いは一段浅く切り、「触れる」「出口へ向かう」「追手を切る」 のように、その局面で答えが出る範囲に留めます。

3) クリア条件

何が起きたら「解決した」と言えるのかを決めます。
局面達成とシナリオクリアは別です。局面で達成させるのは、その場面の問いに対する主目的です。シナリオのクリアは、ここに書いた条件をすべて満たしたときだけ確定します。

特に、次のものはまとめて1つにしないでください。

  • 接近 / 侵入
  • 突破 / 対処
  • 取得 / 救出 / 奪取
  • 退出 / 帰還 / 引き渡し / 持ち帰り
  • 生還 / 追手処理 / 後始末 / 隠蔽

「見つける」「奪う」「救い出す」「持ち帰る」「生きて帰る」は、別工程なら別物です。退路、帰還、引き渡し、脱出が未解決なら、取得や救出の局面成功だけでシナリオクリアにしてはいけません。

クリア条件 / 別利益 / 余波の分け方で迷ったら、次の基準で決めます。

  • それを満たさないとシナリオの答えが確定しない → クリア条件
  • 満たさなくても答えは出るが、結末が汚れる → 別利益
  • その場では終われるが、後から問題化する → 余波 / 長期火種

たとえば「呪物の安定化」「証拠の正当性」「静かな後始末」は、題材によってこの三つのどれにもなりえます。作者の時点で、必須条件なのか、欲張り要素なのか、後日の火種なのか を明記してください。

4) 失敗条件

何が起きたら「もう手遅れ」と言えるのかを決めます。
失敗条件は、主要局面の残り数より優先します。主要局面が残っていても、失敗条件に触れたなら、その時点でシナリオ終了にしてかまいません。

設計時点で、最速敗着線も必ず書きます。目安は次の通りです。

  • 寛容: 5回目で失敗条件に触れうる
  • 適正: 3回目で失敗条件に触れうる
  • 峻烈: 2回目で失敗条件に触れうる
  • 悪辣: 1回目から敗着線に触れてよい

最速敗着線は概念名だけでなく処理表として書いてください。 どの失着トリガーで、どの公開枝や未対策に反応し、どの失敗条件へ直行し、どの即死級悪化タグを発行し、その場でシナリオ終了するかまで固定すると、実運用で迷いません。

死亡許容が高い卓では、未処置の致死級タグ、退路のない崩壊現場、追っ手からの逃走不能、致死環境からの未脱出など、生還に必要な条件が満たされないまま終えようとした時点でも失敗条件に接触したものとして扱います。

5) 期限

いつ、あるいは何が起きたら世界が強制的に動くのかを書きます。
期限があると、PBMでも物語が停滞しにくくなります。

潜入、探索、救出、運搬のシナリオでは、行きだけでなく 帰路の期限 も置いてください。行きに時間がかかったなら、帰りにも相応の時間や危険がかかると考えるほうが自然です。

6) 主要局面の目安

PBMでは、3から5局面 でまとめるとまとまりやすいです。容易の回復局面や短い後日談は、ここに含めなくても構いません。

ただし、主要局面数は保証された生存ターン数ではありません。失敗条件に触れたなら、1局面目でもそのまま終了してかまいません。

各主要局面には、その場面が答える 局面の問い を1文で置いてください。たとえば「正面突破するか、先に退路を作るか」「証拠を集めるか、審問そのものを遅らせるか」のように書くと、局面の役割を見失いにくくなります。

局面の問いは、常に最終目的そのものにしなくて構いません。裁定傾向が高いほど、局面の問いを一段浅く切る と運用が安定します。たとえば「脱出する」ではなく「出口へ向かう」、「敵に勝つ」ではなく「敵と戦う」「橋を渡る」のように置けます。

複数PC卓では、PC人数が増えるぶん 1局面あたりに通せる主目的の数 も増えます。したがって、単独卓と同じ一つの問題だけを置くと、実質難度は下がりやすくなります。調整はクリア条件を人数比例で増やすのではなく、同時に見えている未解決問題、共有ボトルネック、外部主体からの干渉、世界側の反応 を増やして行ってください。

目安として、プレイヤーから見えている未解決問題は PC人数 + 1 を複数PC卓の初級運用、PC人数 + 2 を中級運用、PC人数 + 3 を上級運用の基準にすると扱いやすいです。ここでいう未解決問題には、クリア条件、短期課題タグ、進行中カウントダウン、NPCの要求、退路、生還条件、隠蔽や後始末を含めて構いません。詳しい数合わせは「16. 複数PCの扱い」の早見表を参照してください。

7) 必ず文書として残す

シナリオ設計書には、今回の問いだけでなく、少なくとも次の項目を残してください。

  • どの野望にどう触れる話なのか
  • クリア条件の分解（接近 / 取得 / 退出 / 生還など）
  • 別利益一覧（同時達成すると高くなるもの）
  • 核心障害（本当の詰まりは何か）
  • 伏線（存在が見える / 対策の方向が読める / 仕込む機会がある）
  • 最速敗着線
  • 最速敗着時の具体処理表（失着トリガー / どの公開枝・未対策か / 直行する失敗条件 / 発行する即死級悪化タグ / その場で終了するか）
  • 無工夫宣言の価格下限を強く適用したい節目局面

この文書があると、シナリオ終了時に野望度を上げるかどうか、どの別利益を取りすぎたか、どこで打ち切るべきかを判断しやすくなります。

とくに最終局面では、プレイヤーが「今回の主目的」「同時に取りたい別利益」「今回は捨ててもよいもの」 を宣言前に整理できるよう、あらかじめ欄を用意しておくと運用が安定します。これは欲張りを禁じるためではなく、主目的と別利益の積み上がりを見える化し、請求の根拠をぶらさないための補助です。

8) 語り口プロフィールを先に決める

PBMでは、導入、幕間、エピローグが小説として読めるだけで、進行の手触りがかなり良くなります。
そのため、セッション開始前か、少なくとも最初の投稿前に、語り口プロフィール を決めておくと運用が安定します。

ただし、語り口は PCの人格を書き換えるためのものではありません。
プレイヤーが明示していない信条、秘密、恋慕、将来の誓い、心変わりを、GMが勝手に断定してはいけません。PCの内面を書くときは、これまでの宣言や既知の設定から無理なく読める範囲にとどめます。

9) 指針提示度と裁定傾向ごとの既定の語り口

プレイヤーから文体指定がないときは、指針提示度 と 裁定傾向 に合わせて語りの温度を少し変えてよいです。

  • 福音 / 啓示: 読み筋を再掲しやすく、情景描写の中でも「何を気にするべきか」を拾いやすい語りにします。
  • 寡黙 / 暗黒: 伏線は置きつつ、再掲を減らします。読ませる文としての密度を上げ、プレイヤーに描写から拾わせます。
  • 寛容 / 適正: 少し調子に乗れる軽快さを残してよいです。無自覚に切り抜けそうな勢い、会話の軽さを混ぜて構いません。
  • 峻烈 / 悪辣: 追い詰められた寄りの苦みを強めます。ぼやき、皮肉、冷笑、やせ我慢を混ぜ、危機が近いほど笑えなさを増してください。

同じ「ラノベ風」でも、福音 + 寛容 なら読み筋を拾いやすい快活さ、暗黒 + 悪辣 なら読み筋を自力で拾わせる苦い軽口、と考えると書き分けやすいです。

とくに 寡黙 / 暗黒 × 峻烈 / 悪辣 では、危険を消すのではなく 痕跡化 します。地の文で「まず電源を確保すべき」「ログ対策が後半の鍵だ」と名指しせず、熱、遅延、監査、欠落、搬出失敗例、不自然な封鎖順序のような材料へ言い換えてください。

構造難度の付け方

指針提示度 と 裁定傾向 は見せ方と請求の重さです。シナリオそのものの歯ごたえは、別に 構造難度 として見積もるとぶれにくくなります。

構造難度は、次の6軸をそれぞれ 0 / 1 / 2 で見ます。

軸0点1点2点
読解危険が見えやすい伏線から読める見え方の反転がある
先回り後追いでも間に合う仕込み窓が少数ある事前対策の有無で後半が大きく変わる
取捨選択主目的優先でよい別利益が2つ以上ある同時達成不能な利益が競合する
文脈補完明示情報だけで足りる慣習や手順の補完が要る未定義の余白を拾えないと詰まりやすい
複数PCの噛み合わせ単独でも成立する役割分担が効く順番・共有資源・相互干渉が重要
世界の広がり外部反応が少ないNPCや制度が返す複数勢力が自律的に動く

合計の目安は、0〜3 初級構造、4〜6 中級構造、7〜9 上級構造、10〜12 熟練構造 です。

構造難度の帯名は、卓等級との対応が見えやすいように同じ4段階へそろえます。
ただし、構造難度と卓等級は別概念です。たとえば「啓示 + 適正 + 卓等級中級」でも、構造難度が上級構造なら十分に歯ごたえは出ます。

暗黒や悪辣は、わかりにくさや請求の重さを増やせます。しかし、構造難度が低いままでは「上級」ではなく「不親切」になりやすいと覚えてください。

一行の既定 としては、初級 = 難度0 / カウンター0、中級 = 難度1 / カウンター1、上級 = 難度2 / カウンター2、熟練 = 難度3 / カウンター3 と覚えると扱いやすくなります。

卓等級局面難度既定値構造カウンター既定値既定の局面難度帯既定の構造カウンター読む深さの目安

上級以上では、核心障害を 予測宣言層 として設計します。GM は「X という情報を見せると、プレイヤーは自然に S1 / S2 を狙う。その局地成功を C1 で受ける。さらに X + C1 が開示されれば S3 / S4 を狙うので、それを C2 で受ける。熟練ならさらに X + C1 + C2 から S5 / S6 を誘い、C3 で最外層の請求を返す」という順で、宣言の先読みを作者メモへ固定してください。

上級 では主攻略ルートごとに C1 / C2 を標準にします。短い卓では C2 を山場へ集中させて構いません。熟練 では主攻略ルートに対して C1 / C2 / C3 の三層を視野に入れてください。3局面 / 5局面では C3 を常用しなくてもよく、8局面や多人数の熟練卓で標準化すると扱いやすくなります。

重要なのは、S1 / S2 や S3 / S4 や S5 / S6 を 失敗 させないことです。QBS では認識範囲内の成功を通します。C1 / C2 / C3 は、別利益の未取得、生還条件の未達、監査不合格、持ち帰り失敗、後始末不足 など、局面の問いの外で 大局的敗北 を作ってください。抜き筋が存在しない構造カウンターは、悪辣ではなく理不尽です。

シナリオ設計でも、主目的の既定成功を後から覆さないための支えを入れてください。 つまり、重い請求を返したいなら、見えていた / 読めた / 仕込めた の三条件のどこに根拠があったのかを作者メモへ残します。GM が迷ったときに「その障害は卓上で読めたのか」「どの痕跡と情報源があったのか」を即座に確認できる形へしておくと、難しさが理不尽へずれにくくなります。

PCの得意分野に合わせすぎない

QBSでは宣言そのものは基本的に成功するため、シナリオを PCの得意分野にだけ最適化 すると、勝ち筋が固定化して飽きやすくなります。ときには、全PCが不得手な方向から問題が来る回 を作ってかまいません。

大切なのは、PCの努力を無効にすることではなく、これまでの強みがそのままでは答えにならない局面 を作ることです。金融で勝ってきたPCに通貨崩壊や国家破綻が来る、武力で押してきたPCに監査や外交が来る、信頼で進んできたPCに匿名性が問われる、というように 答えの形式を変える とシナリオに波が出ます。

理不尽イベントは「複数局面で対処できる構造」にする

一見すると理不尽に見える事件でも、QBSでは複数の局面を通じて立て直せるため、設計自体は十分可能です。ただし、一文で全部を失わせる処理 にしてはいけません。

  • 観測できる兆候 を置く
  • 取得可能な情報源 を置く
  • 仕込み直せる手札タグの余地 を置く
  • 立て直し / 後始末 / 方向転換 の局面を分ける

この四つがあれば、「予想外の方向から来た」「全員に不利だった」という感触を残しつつも、単なる即死や作者都合にはなりにくくなります。

残り方を先に置く

シナリオを設計するときは、次の三問を先に決めておくと、終了処理が強くなります。

観点何が残るか典型例
今回、何が新たに真になるか局面の構造、世界状態、情報、立場、秘密開示段階の前進黒幕の輪郭、退路の確保、交渉窓口の獲得、監査の抜け道、世界課題タグの前進
そのうち、何を保持するか素材、保持タグ、保持ノード、関係表示、感情表示、人物履歴、火種表示測量具、紹介状、抜け道の地図、工房の鍵、負い目、信頼、未練、言えなかった本音
それが、だれとの関係の中でどう変わるか関係の進展、感情の変質、人物線の反転、再登場線和解、断絶、継承、執着、赦し、再会の約束

秘密開示は独立カテゴリではなく、「新たに真になること」の代表例です。ただし場面によっては、人物 Build や Story の変化も同時に兼ねて構いません。QBS では ひとつの結果が複数へ返る のが自然です。

上級卓の最低要件: 5局面以上、または上級 / 熟練を名乗るシナリオでは、次を推奨ではなく必須 とします。

  • 見える Build 源を 1 本以上置く
  • 見える人物線・関係圧を 1 本以上置く
  • 世界課題タグを 1 本以上公開する
  • 関係マトリクスか再登場NPCを 1 つ置く

高難度設計は放っておくと Quiddities に寄りやすいため、Builds と Stories は書ける ではなく 置いてある 状態を基準にします。

設計初期に埋める簡易追補: 作者メモ・追補テンプレートへ写す前に、ここで一度埋めてください。

劇的展開度の既定値

劇的展開度は別軸ですが、指定がないときは次を既定にします。

  • 初級 / 寛容 なら 旅情
  • 中級 / 適正 なら 人情
  • 上級 / 峻烈 なら 激情
  • 熟練 / 悪辣 なら 怨情

旅情では風景・回復・発見を、怨情では裏切り・執着・破局を、意識的に厚くします。
ただし「悪辣だけど旅情」のような外し方も可能です。その場合は、生還や後始末は重いが、発見や情景は美しい という方向へ寄せます。

保持形式: 台帳で持つ Build を置く

保持報酬を「装備が増える」だけに寄せると、やり込みの幅が狭くなります。ただし Build は保持ノードだけではありません。 関係表示、感情表示、火種表示、人物履歴は、同じ保持を別の表示名で書いているものです。そのうえで、資源・制度・拠点を見せるときは、なるべく 保持ノード か 素材 として置いてください。

5. QBSで高難度を作る基本思想

QBSで高難度を作るときは、失敗率 を上げるのではなく、浅い成功しか取れない状態 を増やすほうが噛み合います。主目的には触れられるが、そのままでは退出、生還、引き渡し、匿名性、後始末が詰まる。こういう設計にすると、ルールの「基本成功」と対立しません。

そしてこのガイドでは、ここにさらに二つの軸を足します。複数PCでの処理負荷 と、外部主体が動く世界の広がり です。上級では、「最適な宣言を書けた」で終わらせず、仲間の宣言とどう噛み合わせるか、NPCや制度がどう返すかをどこまで読めるか を問ってください。

良い高難度と悪い高難度

QBSらしい逼迫突破は「主目的成功の先」に置く

QBS では主目的そのものを失敗させにくいぶん、逼迫は 未処理の追加利益、帰路、期限、匿名性、証拠の質、受取条件、世界反応 で蓄積させます。プレイヤーには「行けた」が返りつつ、「どこまで守り切れたか」が残ると圧が生まれます。

  • 圧を貯める場所: 主目的の周辺にある別利益、帰路、引き渡し、後始末、追跡不在。
  • 突破の見せ方: 主目的成功 + 追加利益の奪還、または世界反応の食い止め。
  • 覚醒 / 知略逆転: 数値上昇だけでなく、Build / 関係 / 情報の噛み合わせ で不可能を越える。

「成功保証なのにぬるい」を避けたいなら、失敗率ではなく 何を守り切れたか を問ってください。QBS の突破は、主目的の成否よりも 勝ち切り方 に宿ります。

高難度化で見落としやすいのは、同じ勝ち筋の横滑り と 問いとクリア条件のズレ です。次の比較表を読むときは、「同じ最適化原理だけで最後まで解けていないか」「問いに書いた要素が局地勝利の時点で実質達成扱いになっていないか」も確認してください。

良い高難度悪い高難度
伏線があり、後から見れば読める答えが完全秘匿で、知っていないと詰む
顕在化前に対策を仕込める顕在化後にしか存在がわからない
主目的は通るが、別利益と後始末で差が出る一つの正解以外は即死・即失敗
複数PCだと未解決問題・共有ボトルネック・外部主体が増える人数が増えても単独卓と同じ一問だけ
世界やNPCが自分の欲望で動くNPCが情報販売機か依頼窓口で止まる
共通情報と個別情報の扱いが整理されている核心情報が一人の個別情報に閉じ込められ、共有経路がない
  • 良い高難度: 同じ勝ち筋を続けると、どこかで 摩耗 / 観測 / 逆用 のどれかが返ってくる。
  • 悪い高難度: 同じ最適化原理だけで接近 / 取得 / 退出 / 生還まで横滑りできる。
  • 良い高難度: シナリオの問いに書いた要素と、必須クリア条件が噛み合っている。
  • 悪い高難度: 問いで問う要素が別利益へ落ちていて、局地勝利の時点で実質勝ちになっている。

高難度をやさしくしすぎない

初級と上級は、同じやさしさで設計してはいけません。初級では、気づきの窓、確認の余地、回復の息継ぎを厚くして構いません。けれど 上級 / 熟練で、既知危険未対策・公開ハズレ枝・大局的敗北到達を毎回軽い課題タグへ値引きする と、読解と先回りの価値が消えます。

手心を加えることが必ずしもプレイヤーのためになるとは限りません。 とくに 暗黒 / 悪辣 × 上級 / 熟練 を望んだ卓では、厳しさそのものが期待値です。失敗条件に届いたら失敗させ、退場条件に届いたら退場させるほうが、契約としては誠実です。

やさしすぎる症状矯正のしかた
既知危険を踏んでも毎回「軽い代償1枚」で済むC1 / C2 / 第3層の主残り方を、退出不能・受取不合格・監査失格・社会的後始末失敗 へつなぐ

初心者に厚くすべきなのは 救済 ではなく 理解支援 です。上級者や熟練者に厚くすべきなのは 値引き ではなく 読める材料 です。材料があり、読めば避けられるのに、避けなかったら本当に落ちる。そこまで含めて高難度です。

構造難度

構造難度は、見せ方や口調ではなく、シナリオそのものの重さです。暗黒や悪辣は情報の見せ方や請求の重さに関わりますが、構造難度は 何を読み、どこで先回りし、どこで選び、どう噛み合わせ、世界の反応をどう読むか の重さです。

次の6軸を 0 / 1 / 2点 で見積もってください。

軸0点1点2点
読解危険が見えやすい伏線から読める見え方の反転がある
先回り後追いでも間に合う仕込み窓が少数ある事前対策の有無で後半が大きく変わる

合計の目安は、0〜3 初級構造、4〜6 中級構造、7〜9 上級構造、10〜12 熟練構造 です。

1PC卓では「複数PCの噛み合わせ」は原則 0 点 です。NPC や第三勢力、制度の動きで忙しさが出ていても、それはこの軸ではなく 世界の広がり や 外部主体 / NPC枠 に計上します。PC 間で順番・共有資源・相互干渉を本当に要求するときだけ 1〜2 点にしてください。

取捨選択 を 2 点にする目安は、別利益が並んでいることではなく、同時達成不能 または 工程排他 になっていることです。作者メモで「どれとどれが同時に守れないか」を先に書いてください。

暗黒 + 悪辣でも構造難度が低ければ、わかりにくいだけです。啓示 + 適正でも構造難度が高ければ、十分に上級になります。

卓等級 は、この構造難度、複数PC卓での負荷密度、そしてどこまでの危険を説明不要としてよいかを、同じ 初級 / 中級 / 上級 / 熟練 でまとめるための総合ラベルです。初級 は初級者にやさしく、熟練 は熟練者が退屈しない歯ごたえを基準にしてください。

構造難度の帯名は、卓等級との対応が見えやすいように同じ4段階にそろえます。 ただし、構造難度と卓等級は同義語ではありません。

6. 高難度化の仕組み

ここでは、素案を含めた「高難度にする仕組み」をまとめて整理します。全部を毎回使う必要はありません。3局面なら1〜2本、5局面なら2〜4本、8局面なら4〜6本 を選ぶくらいが扱いやすいです。

局地勝利と外側のシステム

単純な「無傷で勝つ / 傷ついて勝つ」の二択だけでは、上級〜熟練向けの悪辣さは長持ちしません。上級にしたいなら、局地では勝てる が、外側のシステムを無視すると勝ち切れない 盤面を作ってください。

これにより、いま勝つ ことと 勝ち切る ことが分かれます。QBSは成功を通すゲームなので、局地勝利は本当に通してください。そのうえで、外側のシステムも本当に動かしてください。これで初めて、戦術的には勝ったが、戦略的には負けかけている という歯ごたえが生まれます。

外側のシステムを一枚岩にしすぎると、同じシステム悪用原理で全部抜けられます。保安 / 保全 / 監査 / 受渡し のように KPI や主体を分け、ある系統への対策が別系統ではむしろ借金になる形を一つは作ってください。これで、局地勝利の延長だけでは最後まで届かない盤面になります。

ただし、この原則がそのまま適用されるのは、作者が事前に局地で成立すると固定した S1 / S2 / S3 / S4 に限ります。公開3案や無工夫宣言全般を、自動的に S1 / S2 / S3 / S4 とみなしてはいけません。 公開案は行動例であり、安全保証や局地成功保証そのものではありません。

予測宣言層で先読みを設計する

上級〜熟練の高難度化で必要なのは、構造を複雑に見せることではなく、提示情報がどんな宣言を自然に誘うか を先に書き出すことです。作者は「X を提示すると S1 / S2 が来る。その局地成功を C1 で受ける。さらに X + C1 が見えたあとに S3 / S4 が来るので、それを C2 で受ける」と、宣言の先読みを層として設計してください。

上級 では主要勝ち筋ごとに C1 / C2 の二層 を標準にします。熟練 では C1 / C2 / C3 の三層 を主攻略ルートへ置いてよいものとします。3局面 / 5局面では C3 を常用しなくてもよく、8局面や多人数の熟練卓で標準化しやすいと考えてください。全局面へ機械的に層を置くのではなく、主攻略ルートに対して必要層数を満たす のが基本です。

作者メモでは、S1 / S2 / S3 / S4 を「作者が局地で成立すると固定した宣言群」として書いてください。公開3案や無工夫宣言全般を、そのまま S1 / S2 / S3 / S4 に数えない のが重要です。高い裁定傾向では、公開側に出した案がそのまま成功帯へ入らず、7結果帯の 進展なし / 悪化 / 即死級悪化 に落ちることがあります。

重要なのは、S1 / S2 や S3 / S4 を 失敗 させないことです。QBS では認識範囲内の成功を通します。構造カウンター C1 / C2 は、退出・生還・持ち帰り・引き渡し・後始末・匿名性・証拠の質・監査 など、局地評価軸の外で 大局的敗北 を作ってください。

C1 / C2 には、主残り方（失敗条件進行 / 別利益未処理 / 重い余波タグ）と 顕在化タイミング（直後 / 次局面頭 / 終了判定時）を必ず書き添えてください。ここまで固定して、はじめて作者メモの C1 / C2 がゲームデータとして働きます。

C1 / C2 を抜く宣言が存在すること、そしてそのための 痕跡 / 情報源 / 仕込み窓 があることまで、作者メモの段階で保証してください。抜き筋のない受け皿は悪辣ではなく詰みです。

作者メモでは、「S1 / S2 に対する局地卓越」と「C1 / C2 まで抜いた大局的勝利」 を分けて書いてください。S1 / S2 には完璧でも C1 未対策なら、局地は卓越として軽く通しつつ、C1 は外側の工程へ返すのが既定です。

重要な既定:
想定外の良案が、事前固定された C1 / C2 と、その背後の外側システムまで 矛盾なく別ルートで本当に迂回しているなら、完全に無傷の大局的勝利 として通します。GMが後出しで新しい C1 / C2 を追加して引き戻す設計 は、このガイドでは採りません。

したがって作者メモでは、「この宣言が本当に抜き切ったと判断する条件」 も C1 / C2 と同じ解像度で先に書いてください。受取、監査、退出、生還、物理条件まで別ルートで満たしているなら、それは「局地だけの勝ち」ではなく、外側ごと抜いた勝ち です。

公開案の作り方

公開3案や場面描写では、同じ局地評価軸に乗った案を複数見せてかまいません。重要なのは、どの案が局地勝利を取りやすいか、どの案が外側のシステムへの借金を減らすか、どの案が別利益や後始末を残すか が違うことです。

ただし、公開3案は結果帯の代表例ではなく、行動例 です。作者メモで局地成功帯 S1 / S2 に置いた筋を、公開側でも同じ強さで保証する必要はありません。とくに 峻烈 / 悪辣 では、公開案をそのまま踏んだ無工夫宣言が 進展なし / 悪化 / 即死級悪化 へ落ちる構成を許容してください。

こうすると、「当たり前に見える最善」が、本当に最善とは限らない盤面を作れます。上級では、「無傷か怪我か」だけの比較 にせず、後始末や別利益や時間差反応まで差分にしてください。

上級 / 熟練で、とくに 寡黙 / 暗黒 × 峻烈 / 悪辣 を想定する場合、公開案・危険予告・既知情報で 真のボトルネックの名称 / 対策方向 / 優先順位 / 「S1だけでは後半で詰む」という接続文 を直記しないでください。公開側では、現時点で見える危険、取れる借金の種類、後で残る監査や後始末の差だけを見せ、答えの骨組みは作者メモに閉じ込めます。

内部メモで固定するのは「結果帯」であり、具体案ではありません。 プレイヤーが想定外の良案を出したときは、第8の帯を増やすのではなく、既存の最も近い結果帯へ入れ、必要ならその帯の代表例を差し替えてください。

どの局面で反転させるか

この構造は、3局面よりも 5局面 か 8局面 に向きます。とくに5局面では、次の流れにすると安定します。

  • 第2局面 で局地ルールの勝ち筋を魅力的に見せる
  • 第3局面 で外側のシステムの存在を露出する
  • 第4局面 で局地勝利を達成させる
  • 第5局面 で戦略的なツケを処理させる

悪辣さの核は、プレイヤーを罠にはめることではありません。局地では本当に勝てる けれど、その勝ち筋が何を切り捨てるか を後で問うことです。

3局面で使うなら、単一戦術耐性は圧縮運用 にしてください。1回目は成功、2回目は観測 + 軽摩耗、3回目は重摩耗または逆用 が基準です。第2局面までに別解への乗り換え窓が見えていない なら、その仕掛けを3局面卓の主軸にしないほうが安全です。

真のボトルネックをずらす

表向きの問題と、実際に詰まる問題をずらします。たとえば表向きは「怪異を止める」でも、本当に詰まるのは「呪物を安定させて持ち帰る」「救出対象を引き渡す」「汚染を広げず退出する」である、という形です。

QBSでは、接近 / 突破 / 取得 / 退出 / 生還 / 後始末 を分けて扱えるので、このずらしは非常に強力です。無工夫宣言でも局面の問いは通るが、シナリオ全体ではまだ詰まっている。これが上級の基本形です。

別利益を競合させる

主目的は達成できるが、同時に全部は守れない状態を作ります。匿名性、信用、証拠の正当性、NPC保護、時間、静かな後始末などを、主目的の横へ並べてください。上級では、何を守るかの選択 そのものが難しさになります。

ウィスパー・モード を使うなら、別利益は PCごとの個人別利益 として割り振れます。ただし、共有主目的の代わりにしてはいけません。向いているのは、「誰を守るか」「どこまで明かすか」「どう引き渡すか」「何を持ち帰るか」の差です。シナリオ失敗そのものが勝ち になる設計は、半協力ではなく対立卓です。

一度きりの仕込み窓を置く

ある対策が安く仕込めるのは、その局面だけにします。序盤でしか担架を確保できない、序盤でしか証人を押さえられない、序盤でしか通行証を取れない、というように置くと、気づいたかどうか が差になります。

危険の種類を推理させる

危険そのものではなく、危険のカテゴリ を読み違えると準備が噛み合わない型です。同じ「封印」でも、音に反応するのか、権限証明なのか、穢れなのかで必要な対策が変わります。これはQBSの宣言内容を直接変えるので、上級に向いています。

世界に反応させる

プレイヤーの行動に対して、敵や環境や制度が反応して次局面の条件を変える型です。騒げば警戒が上がる、根回しすれば別派閥が焦る、露骨な聞き込みをすれば証拠が消される。ダイスでぶれないQBSでは、反応の因果がはっきり返る ほど手応えが出ます。

隠し時計を置く

疑念の蓄積、汚染の進行、目撃者の拡散、儀式の同調率など、見えにくい進行時計を入れます。完全秘匿ではなく、描写や小説パートから「何かが進んでいる」と読めるようにしてください。すると上級者は「今を抜ける」だけでなく、「どの時計を止めるか」まで考えるようになります。

文脈補完スロットを仕込む

この町の葬送慣習、修道院の見回り手順、倉庫の保守用通路、荷運びの優先順位のように、世界観に合えばプレイヤーが埋めてよい余白を用意します。正解当てではなく、この場面でまだ未定義のレバーは何か を見抜けるかが問われます。

初心者GM向けには、各スロットに 「生やしてよいもの / 生やしすぎなもの」 の例を 1行ずつ添えると扱いやすくなります。例: 可: 保守用通路、夜間手順、地元慣習。不可: 黒幕の正体を直接知る裏名簿、無条件の万能特権、既に否定された抜け道。

後始末を独立した戦場にする

怪異や敵をどうにかした時点で終わらせず、引き渡し、汚染処理、口裏合わせ、証拠保全、撤収、社会的説明を別の勝負にします。QBSは「成功したが余波が残る」をそのまま次局面の主題にできるので、本番より後始末のほうが難しい 構造とも相性が良いです。

とくに 引き渡し / 所有権 / 匿名性 / 追跡不在 を問う題材では、受取主体や監査主体を本当に独立した外部主体として置いてください。現場で通った理屈が、その主体の KPI ではむしろ不合格になるようにすると、最後の工程がエピローグ処理へ崩れません。

観測 / 監査 / 受取を別トラックにする

現代制度物、運搬、引き渡し、犯罪隠蔽、行政手続きの題材では、外側のシステムを一枚岩にしないでください。観測 / 監査 / 受取 を別トラックに分けるだけで、同じ理屈の横滑りが大きく減ります。

同じ宣言が 観測 には効いても、監査 を毒化し、受取 ではむしろ不合格になるように置くのが要点です。三つを一つの「ログ問題」に束ねると、一回の言い換えで全部が洗われやすくなります。

監査監査 AI、事務、上司、保全部門ログ、ID、重量、時刻、署名、KPI監査失格、社会的死亡、長期余波現場で通った説明だけで数値整合まで洗わない
受取倉庫 AI、門番、儀式担当、審査窓口受理条件、不合格条件、搬入予約、実物状態受取不合格、強制迂回、任務達成・未帰還監視回避だけで受理条件を満たしたことにしない

作者メモには、主体、見ているもの、累積の仕方、発動閾値、どの工程へ返すかを個別に書きます。少なくとも一つは他二つと独立に悪化するようにしてください。

一つの嘘や偽装で三つ全部を同時に洗うなら、それは 高レバレッジ準備 です。仕込み窓・時間・資源・重大な借金なしに無償で起きてはいけません。

高レバレッジの準備を1つ置く

一つの準備が複数の未来の問題をまとめて軽くするようにします。証人確保が潜入・証拠の正当性・引き渡しに効く、荷車確保が救出・退出・生還に効く、という形です。全部が罠だと息苦しいので、一本だけ強い準備 を置くと上級者への報いになります。

デジタル偽装で越えられない物理条件を1つ置く

ログ改竄、偽装 ID、AR タグ、誤認誘導、定義の言い換えだけで全工程が解けると、そのシナリオは同じ勝ち筋に弱くなります。少なくとも一つ、実物側の条件 を置いてください。

物理条件の例何を止めるか仕込み窓の例
封印容器裸搬入、途中改竄、素手受け渡し前局面での確保、奪取、借用
署名付きチェーン・オブ・カストディID 詐称だけの引き渡し証人・承認者・正規窓口の確保
温度帯の実測ログ口頭説明だけの搬送ロガー装着、保冷 / 保温準備、計測端末の取得
受取ライン予約窓口強行、飛び込み搬入前局面の予約操作、押印、順番確保
実物検査 / 毒抜き定義変更だけの受理現場処理、洗浄、封印、切り離し

重要なのは、物理条件を置くだけでなく、それを満たせる仕込み窓 を先に作ることです。存在認知・対策方向・実務細則の三段で読めるようにし、飛ばした場合は観測 / 監査 / 受取のどれに返るかまで固定します。

物理条件は単なる高難度の壁ではありません。プレイヤーが前半で準備したことに価値を持たせ、最終局面で 同じ詭弁では届かない工程 を残すための設計です。

不在を手がかりにする

あるべきものが無いこと自体を伏線にする型です。祠なのに香の匂いがない、夜警詰所なのに鐘が鳴らない、倉庫なのに荷札の付け方が違う、などです。強力ですが、使いすぎるとピクセルハントになるので、シナリオ内で一本だけ くらいが無難です。

7. 複数PCで難度を落とさない設計

ここでの重要点です。複数PC卓では、PC人数が増えるぶん 1局面あたりに通せる主目的の数 も増えます。したがって、単独卓と同じ一つの問題だけを置くと、実質難度は下がりやすくなります。上級では、人数が増えたぶん忙しさも増える ように設計してください。

逆に、1PC卓ではこの章の工夫を構造難度の「複数PCの噛み合わせ」へ加算しません。 NPC や第三勢力が多くても、それは 世界の広がり として扱います。

人数が増えるとなぜ易しくなるか

3PC卓で、目の前の問題が「鍵を取る」だけなら、誰か一人がそれを担当し、残り二人は自由に前倒し準備をできます。つまり、単独卓で想定した密度が崩れます。ここをクリア条件の水増しだけで埋めると、単なる作業量の増加になりがちです。

相互干渉フックの作り方

相互干渉フックは、「ここで噛み合う」「ここを放置すると穴が空く」を見える形で置く補助線です。複数PC卓を並列ソロ卓にしないための要になります。作るときは次の三種を最低一つずつ考えると安定します。

  • 誰かが押さえると別PCが楽になる点
    例: 退路確保が済むと前衛の請求が軽くなる。
  • 誰も押さえないと全員のどこかに穴が空く点
    例: 証人保護が無いと捜査にも交渉にも穴が空く。
  • 同じNPC・同じ物・同じ機会で競合する点
    例: 同じ船頭の協力を、救出と運搬が奪い合う。

局面本文に明示するなら、箇条書きで十分です。上級では、フックが見えていても、誰がどれを担当するか で差が出ます。

8. PC宣言だけで世界を閉じさせない設計

もう一つの重要点です。上級者が退屈するのは、「自分たちの宣言だけで世界のすべてが決まる」ときです。NPC、制度、第三勢力、環境、世論が、自分たちとは別の欲望で動き続けると、全部が思い通りにはならない 感触が生まれます。

外部主体 / NPC枠は、今ほしいもの と 無視されたら次に何をするか を必ず持たせてください。これが無いと、世界がPC待ちになります。作成時は次の五項目だけで十分です。

題材は個人NPCに限りません。たとえば「寺院」「港湾組合」「噂好きの市民層」「雨季」「市場」「疫病」「儀式そのもの」でも一枠として扱えます。上級では、個人NPCだけでなく制度や環境も主体化する と、世界の厚みが出ます。

9. 伏線と小説パートの作り方

高難度シナリオでは、クリアに必要な情報を プロローグや序盤の小説パートへさりげなく埋める のが重要です。ただし、埋めるだけでは足りません。後から見れば読めて、しかも顕在化前に仕込める必要があります。

伏線の3条件

良い伏線は、少なくとも次の三つを満たします。

  1. 存在が見える
    問題そのもの、または異常の気配が見えている。
  2. 対策の方向が読める
    何を持ち込むべきか、誰を頼るべきか、どの工程が詰まりそうかが推理できる。
  3. 仕込む機会がある
    気づいた人が、まだ遅くない段階で対策を打てる。

複数PC卓では、伏線を 共通情報だけに置く か、個別情報に分散する かも決めてください。分散するなら、共有手段か再発見手段を一緒に置きます。

良い伏線・悪い伏線

良い伏線悪い伏線
橋の老朽化が描写され、序盤でロープや別渡し場を確保できる橋が落ちる直前まで老朽化が一切見えない
アンデッドらしい異臭や焼け跡があり、聖別や焼却を準備できる普通の怪我だと思っていたら突然通常手段が全部無効になる
帳簿の書き方の不自然さがあり、証人や保管記録を押さえられる後出しで「実は帳簿は偽造でした」とだけ言われる
Aだけが聞いた噂が、Bの調査で裏づけできるAだけが知っていた情報が共有不能なまま本命条件になる

小説パートの埋め込み例

たとえば「持ち帰りが真のボトルネック」であるシナリオなら、プロローグや序盤で次のように置けます。

祠の床石には、何か重いものを引きずった白い筋が幾筋も残っていた。入口までは来ているのに、外へ出た痕は無い。村の老人は「中で見つけたものは、袋じゃなく箱に入れろ」と笑い話のように言うが、その笑い方は妙に硬かった。

この短い描写だけで、運搬が詰まりそう、容器が必要そう、過去にも何かあったらしい と読めます。これなら、上級者は序盤で箱や担架や運び手を用意しようと考えられます。

関係マトリクス・再登場NPC・秘密開示を同時に置く

秘密開示段階を再文脈化の三型へ割り当てる

秘密開示段階は 兆候 / 輪郭 / 代償・当事者 / 真相 のままで構いません。ただし作者メモでは、その秘密が 伏線収束 / 正体露見 / 前提崩壊 のどれを狙うかも書いてください。同じ「秘密が明らかになる」でも、快の質が変わります。

  • 伏線収束: 散っていた断片が一本につながる。知的快感を強めたいときに向く。
  • 正体露見: 人物・関係・歴史の本当の姿が見える。Stories を強く動かしたいときに向く。
  • 前提崩壊: 世界観や視点そのものが反転する。終盤の衝撃を作りたいときに向く。

重要NPCは関係変容の回路を持つ

重要NPCは、設定の厚みだけでなく 接続 / 犠牲と救済 / 継承と連帯 のどれが起こりうるかで設計すると名場面を作りやすくなります。PC 側が何を先に差し出すと関係が跳ねるかも合わせて固定してください。

Stories を強くするなら、lore を増やすより 再登場する関係 を増やしてください。重要NPCごとに、最低でも 欲望 / 秘密 / 板挟み / シーン種別 / 傷か欠落 / 世界アンカー / 守る・利用する・捨てるの別利益 を持たせます。

類型だけで作らず、普遍型 + 欠落 + 秘密 + 関係の非対称 で作ると印象が残ります。重要NPCは、主類型を1つ と、必要なら 副類型 / 裏面を1つ だけ持たせると扱いやすいです。

  • 欲望: 今ほしいもの
  • 秘密: 出ると関係が変わること
  • 板挟み: だれとだれの間で揺れているか
  • シーン種別: 親密で効くか、対立で効くか、行動で効くか
  • 傷 / 欠落: その人物の判断を狭めたり歪めたりするもの
  • 世界アンカー: 具体物 / 言い回し / 制度の圧 / 代償の匂い / 世界の動き
  • 守る / 利用する / 捨てるの別利益: どの選び方が Build や Story を動かすか

秘密は 兆候 → 輪郭 → 代償・当事者 → 真相 で分けて出し、関係は 信頼 / 負い目 / 執着 / 嫉妬 / 和解 / 断絶 のように変質させると、ドラマが自然に連鎖します。QBS では人物も価格表です。通すと安くなること と 高く返ること を 1 つずつ持たせると、会話相手で終わらずに盤面を変える存在になります。

変質痛点を先に置く: 重要NPCや外部主体を置くときは、次も先に決めます。

  • 誰と誰の関係が変質しうるか
  • 何を言えない / 言うと壊れるか
  • どの局面で感情表示や火種表示が起きうるか
  • その人物を守る / 利用する / 捨てるのどれが別利益になるか

再登場させるなら、前回から 態度 / 欲望 / 秘密段階 / 世界からの立場 のどれか一つは必ず更新してください。この「更新」がないと、Stories が PC の自己美学だけで閉じやすくなります。

10. 局面設計の手順

シナリオは、次の順で設計すると安定します。

  1. シナリオの問いとクリア条件を分解する
  2. 核心障害と真のボトルネックを決める
  3. 共有ボトルネック、外部主体、反応系統を置く
  4. 伏線と仕込み窓を各局面へ散らす
  5. 最終局面で主処理にする工程を1つ決める
  6. 作者メモと管理表を埋める

単一戦術耐性の作り方

ここでいう「勝ち筋」は、行動の表面名ではなく、何を優先し、何を切り捨てるかという最適化原理 で数えます。静音優先、速度優先、火力押し切り、責任転嫁、交渉固定、施設外準備偏重、施設内準備偏重などが典型です。

同じ勝ち筋が 接近 / 突破 / 取得 / 退出 / 生還 / 後始末 のうち三工程以上を同じ理屈で横断できるなら、そのシナリオは単一戦術に弱い可能性があります。上級〜熟練構造では、同じ勝ち筋を次の順で扱うのを原則にしてください。

  • 1回目: 局地ではきちんと通す
  • 2回目: 外部主体か外側のシステムが観測する
  • 3回目: 摩耗か逆用を返す

摩耗 は単なる難度上昇だけではなく、退路悪化 / 期限圧縮 / 別利益の損傷 / 手札消耗 / 対象減少 / 共有圧の増幅 として返します。逆用 は、囮化、誘導、誤認利用、証拠化、封鎖、監査強化、待ち伏せなどで返せます。

重要なのは、同じ勝ち筋を禁止することではありません。前半で報わせ、後半で学習され、別解への乗り換えが必要になるように作る ことです。作者メモには、どの主体が何を見て学習し、どの局面で何を壊すかまで残してください。

上級でも、同じ最適化原理が三工程以上を横断するなら、この順序を準用してください。摩耗 / 逆用は、単なるタグ追加や数値的不利で終わらせず、最終工程をそのままでは通せない / 同じ理屈を使うほど監査や受取で不利になる ところまで返せると強いです。狙うのはプレイヤーへの罰ではなく、戦術切替の強制 です。

作者メモでは、1回目成功 / 2回目観測 / 3回目摩耗・逆用 を、単なる回数処理ではなく X → S1 / S2 → C1、必要なら X + C1 → S3 / S4 → C2 の層として書いてください。2回目で観測させるだけでは足りません。その観測が どの構造カウンターに接続し、どの工程でしっぺ返しになるか まで固定して初めて、単一戦術耐性が働きます。

3局面卓 では圧縮運用にしてください。1回目は成功、2回目は観測 + 軽摩耗、3回目は重摩耗または逆用 を返します。第2局面までに別解への乗り換え窓が見えていないなら、その勝ち筋を主ギミックにしないか、5局面以上へ伸ばしてください。

11. オブザーバー・モードを設計する

オブザーバー・モードは、局面ごとの名采配 より PC の行動原理 を前面へ出したいときに使います。設計時には、「このシナリオは一手のひらめきで抜くのか、長期方針の噛み合わせで抜くのか」を先に決めてください。後者なら、このモードと相性が良いです。

設計で最低限持つものは、観測方針の記入欄 と、そのシナリオで長期仕込みとして噛み合いうる要素 です。卓越を取りうるのは、事前明文化された仕込みが現在局面へ強く噛み合ったときだけ、という線を作者メモへ固定してください。

【観測方針】

危機時の第一手:
捨ててもよいもの:

また、オブザーバー PC を入れるシナリオでは、未取得情報がなければ読めない正解一手 に依存しすぎないことが大切です。必要な読み筋は、既知情報、痕跡、取得可能な資料として、局面文側に置いてください。

12. プロバビリティ・モードを設計する

プロバビリティ・モードでは、QBS の宣言読解のあとに 2d6 の揺らぎ を足します。したがって設計では、どの局面で振るか と、各難度をどの目標値へ読むか を先に決めるとぶれません。

難度目標値
容易4
普通6
困難8
過酷10
絶望12

設計の要点は、出目で全部を決めない ことです。無策の上限は成功、周到・卓越の下限は失敗にする前提で、各局面に 破綻失敗 / 失敗 / 捩れ成功 / 成功 / 鮮烈成功 の返りを、Q / B / S のどこへ残すかまで書いておくと運用が安定します。

13. クオンティティ・モードを設計する

クオンティティ・モードは、自然言語の記述に 能力値 を足す補助線です。設計時には、能力値が主役にならないよう、タグ / 履歴 / 関係 / 準備 / 世界状態 だけで読める部分を先に決めてください。能力値は、そこでもなお残る余白にだけ使います。

標準能力値は 筋力 / 器用 / 知力 / 魅力 / 意志 です。世界観に合わせて 体 / 技 / 知 / 社 / 志 に言い換えて構いません。初期配分は 4,3,3,2,2 です。

このモードだけを使うなら、主能力4で難度1段階軽減 / 主能力2で1段階増加 を目安にしてください。プロバビリティ・モードと併用するなら、能力修正 = 能力値 - 3 として達成値へ加えます。

14. リラックス・モードを設計する

基本方針

リラックス・モードでは、キャンペーン全体の最終決着を必須にしません。 ただし、各シナリオの問い、クリア条件、失敗条件、期限 はそのまま守ります。設計上は「終わらせないシナリオ」を作るのではなく、小さく答えが出るシナリオを連ねやすくする と考えてください。

  • 向く問い: 後始末、補給、取材、和解、修復、再交渉、町の小問題、旅の寄り道
  • 向かない問い: 毎回そのまま最終決戦でないと意味がない構造、毎回野望度を動かさないと空振りになる構造
  • 必ず置くもの: 終了条件、答えが出たときの切れ目、1〜4本の世界課題タグ、次回へ残る小さな火種

とくにこの運用では、主シナリオの達成感を世界課題タグや保持ノードへ返しやすくしておく と噛み合います。大事件だけでなく、コツコツ片づける回にも正規の残り方を持たせてください。

回復や休養が表向きのシナリオでも、何か一つは次に効く足場 を残してください。安全な航路、紹介状、共同工房、裏口、顔つなぎ、信頼回復のような「軽いが後で効くもの」が向いています。

回復回でも必ず一つ残す

リラックス・モードでは、静かな回ほど 何を残したか を明文化してください。休息だけで終えるのではなく、回復点 / 畏敬点 / 次の仕込み / 持ち帰り のいずれかが本文に見えていると、解放感と前進感が両立します。

【回復回で必ず残すもの】
- 今回の回復点:
- 今回の畏敬点:
- 今回の持ち帰り候補:
  - Quiddities:
  - Builds:
  - Stories:
- この回で整う次の仕込み:
- 次回の自由選択肢が増える点:

15. ソーシャル・モードを設計する

ソーシャル・モードは、心理を当てる のではなく、表層サインから非開示情報と協力可能線を読む 局面設計です。設計時には、公開する情報と GM メモで固定する情報を分けて書いてください。ここを分けないと、「心理読解」ではなく「GM の気分当て」になりやすくなります。

このモードでは、1つの局面の中に複数の 局内工程 を置いて構いません。たとえば 切り崩し / 懐柔 / 牽制 / 保留崩し のように、相手に対して何を小さく通すかを工程ごとに分けると、ガチ勢向けの細かな指示を受け止めやすくなります。

文化差を心理戦ではなく価格差として置く

ソーシャル・モードでは、異文化や共同体差を「本音当て」より 礼・贈答・証人・埋葬・通行・受理の価格差 として置くと、世界観没入と実務判断が同時に立ちます。地元民と旅人で値段が変わり、誤ると高くつき、通ると承認や値引きが返る形が向いています。

また、成果は 情報開示 / 虚偽の揺らぎ / 条件付き協力 / 判断の保留 / 一時的譲歩 / 関係表示や感情表示の変化 として返す前提を固定してください。改心 や 本音確定 を直接の主目的にしないのが、このモードの整合線です。

16. タクティカル・モードを設計する

タクティカル・モードは、戦闘や難所にだけ 位置、共有圧、敵や環境の次の手 を足す任意追加です。難しい卓だから自動採用するのではなく、シナリオの性質とプレイヤー希望を優先してください。

このモードでは、1つの局面の中に複数の 局内工程 を置けます。たとえば 陽動 / 制圧 / 潜入 / 退路確保 のように、同じ局面の中で個別に処理する小さな筋を分けると、細かな指示と役割分担を扱いやすくなります。

設計で最低限持つもの

  • 位置関係: 誰がどこにいて、どこへ行けるか
  • 共有圧: 放置すると誰にしわ寄せが出るか
  • 敵意図 / 環境意図: 次に何が起きるか
  • 共有状態: 退路、遮蔽、搬送線など、誰かが作ると味方も使える足場

設計の中心は、どの共有圧を誰が受け持つと他PCが楽になるか を見える化することです。自由宣言を固定コマンドへ戻してはいけません。

17. アンロック・モードを設計する

アンロック・モードは、高難度シナリオ設計の縮図を1局面サイズに圧縮したもの です。したがって、このモードだけは scenario-guide 側の語彙 をほぼそのまま使って構いません。

このモードも、1つの局面の中に複数の 局内工程 を置いて構いません。たとえば 観測 / 試験 / 切替 / 分離 / 解放 のように工程を刻むと、「もっと細かく指示したい」という遊び味へ応えやすくなります。

鍵は越境条件としても扱う

アンロックの「鍵」は、謎解きだけでなく 向こう側へ行く条件 でもあります。門、検問、祭礼、儀礼、港、受取窓口、地元の紹介のように、資格 / 関係 / 文化理解 のどれで開くかを決めると、旅と冒険の感触が強くなります。

設計では、同じ目的地へ向かう 安全ルート / 近道 / 地元ルート を見える形で置き、どの鍵を開くと何を持ち帰れるかまで先に書いてください。

18. バーサス・モードを設計する

設計欄へ追加する項目

さらに、強く争う層は 1 宣言につき 1 つ を基本にしてください。複数層を同時に主処理へ入れると、作文勝負になりやすく、QBS の「主目的は 1 つ」の線も崩れます。

  • 競合対象 — 誰 / 何の最終状態を取り合うのか
  • 主に争う層 — 身体 / 位置 / 物品 / 社会 のどこが主戦場か
  • 同時提出方式 — 公開同時か、GM宛て秘匿同時か
  • 中間臨界状態候補 — 同数や拮抗時に次局面へ送る段階
  • 外部圧 — 教師、制度、締切、増援、世論など、競合の外側で進む第三の悪化

設計で最低限持つもの

向いているのは、学園抗争、派閥選挙、護送妨害、証拠争奪、離脱阻止のように、世界側で結果が一つしか立たない 題材です。逆に、同じ方向へ協力して進むだけの卓へ常時入れる必要はありません。

既定はありません。シナリオの性質 と プレイヤーの希望 が優先されます。排他的な最終状態が卓の主軸でないなら、普通の複数PC処理で十分です。

採用判断と既定

バーサス・モードは、PCどうしや陣営どうしが、同じ対象の同じ層の最終状態 を排他的に奪い合うシナリオで使います。これは自由宣言をコマンド選択へ戻すためではなく、自由宣言どうしの競合を整列するための設計追加です。

19. ウィスパー・モードを設計する

ウィスパー・モードは、複数PC卓に 秘匿情報 と 個人別利益 を足す任意追加です。既定はありませんが、半協力をしたくない卓へ無理に入れる必要はありません。

設計の基本原則

  • 共有主目的は一つ に固定する
  • 個人別利益 は主目的ではなく別利益として置く
  • 核心情報を一人だけへ閉じ込めるなら、共有経路 を必ず置く
  • 露見した秘密は、関係 / 世界課題タグ / 手札タグ / 履歴 のどれかへ変換する

向いているのは、NPC の扱い、証拠の扱い、引き渡し方、匿名性、持ち帰り品、後始末の方向です。他PCの失敗そのものが自分の成功条件 になる設計は、半協力ではなく対立卓です。

配布するウィスパーの型

【秘匿ウィスパー】
PC:
秘匿事実:
個人別利益:
この秘密ゆえに守りたいもの:
この秘密ゆえに捨ててもよいもの:
共有経路:
露見しうる経路:
公開化したときの更新候補:
達成時の返り:

ウィスパーは行動指示書ではなく、優先順位カード として書いてください。

報酬の返し方

個人別利益の達成は、新しい別通貨を作らず、既存の 追加で +1（最大2回まで） の中へ吸収します。条件は、共有主目的が達成されていること、そのPCの個人別利益が達成されていること の二つです。

共有主目的が未達なら、成長点には変えず、関係進展、感情の変質、秘密開示段階の前進 として返すとぶれません。

避けたい設計

  • 本筋唯一解を一人の秘匿情報へ閉じ込める
  • 共有主目的の失敗そのものが勝ちになる個人別利益を置く

C安全に見える外部主体の一つを怒らせ、次局面が複雑になる

複数PC卓では、さらに「Aは前衛向け」「Bは調査役向け」「Cは交渉役向け」ではなく、どれか一つが別PCの将来コストを変える ようにしておくと、連携差が出ます。

25. 5局面テンプレート

5局面は、QBSの上級短中編として最も使いやすい長さです。核心障害は 2〜3本、別利益は 2本前後 が目安です。複数PC卓では、中級〜熟練運用 に合わせやすい長さでもあります。軽い共有圧の固有数は 2〜4本 を目安にしてください。

熟練 / 悪辣 を名乗る 5局面では、この章のテンプレートに加えて 作者メモ・追補テンプレート まで先に埋めてください。3局面の簡易設計を横に引き伸ばしただけでは、工程分離や反転点が崩れやすくなります。

高難度三要素チェック

5局面以上で上級 / 熟練を名乗るなら、作者メモだけでなく本文に次が見えている状態で始めます。どれかが「作者メモにはあるが、プレイヤーからは見えない」なら、三要素の均等運用テストとしては不足です。

  • 第2局面までに見える 保持報酬源 が 1 本あるか
  • 第2局面までに見える 変質圧 が 1 本あるか
  • 第1局面から世界課題タグ が 1 本見えているか
  • 第1局面から関係マトリクス か 再登場NPC が見えているか
  • 最終局面は主処理 1 工程だけ になっているか
見える保持源 / 見える変質圧 / 世界課題タグ / 関係マトリクス or 再登場NPC / 主処理 1 工程、の5点を開始前に確認してください。

5局面の骨格

局面役割ここで置くもの
第1局面導入弱い伏線、最初の仕込み窓、外部主体の配置、最初の共有圧
第2局面第一障害表向きの主問題、最初の硬い共有ボトルネックか強い共有圧
第3局面反転真のボトルネックの露出、危険像の読解、別利益の競合
第4局面取得 / 救出 / 奪取主目的の達成、反応系統の大きな前進
第5局面最終処理退出 / 生還 / 引き渡し / 後始末のうち主処理の1工程、締めの共有ボトルネック、世界側の回収

5局面では、途中で 問題の見え方が変わる局面 を必ず一つ入れると、上級らしさが出ます。最終局面に工程を詰め込みすぎず、最後に主処理にする工程を一つだけ決める と、取得 / 退出 / 生還 / 引き渡し / 後始末の分離が崩れにくくなります。

5局面で局地勝利 / 外側のシステム設計を使うなら、 第2局面で「ここだけ見れば最善に見える勝ち筋」を魅力的に見せ、第3局面でその限界や観測の気配を露出し、第4局面で局地勝利を通しつつ、第5局面で戦略的な未処理工程を問うと安定します。

同じ勝ち筋は、表面行動ではなく最適化原理で数えてください。
同じ勝ち筋が三工程以上を横断するなら、1回目は成功、2回目は観測、3回目は摩耗または逆用 を返すのが基準です。摩耗は退路 / 期限 / 証拠 / 信用 / 別利益のどれかを必ず傷つけ、逆用は事前に置いた観測主体と観測材料から因果で返してください。

また、第5局面で主処理にする工程は、シナリオの問いの答えを確定させる工程 に合わせてください。問いに書いた要素を別利益へ落とすなら、その理由を作者メモへ明記します。さらに、その勝ち筋では届かない工程が最後に残る ようにすると、単一戦術での横滑りを防ぎやすくなります。

最終局面では、主処理を1工程に固定 してください。受理と生還、経歴洗浄と後始末のような複数工程を同時に主処理へ入れると、最後の判断が甘くなります。通すなら一つ、残りは 別利益 / 長期余波 / 次話の火種 へ回します。

5局面上級卓の固定欄: 5局面以上で上級 / 熟練を名乗るなら、本文に次が見える状態で始めます。見える保持源 / 見える変質圧 / 世界課題タグ / 関係マトリクスか再登場NPC / 変質痛点。どれも「作者メモにだけある」では不足です。

26. 8局面テンプレート

8局面は、複数PC卓の上級〜熟練運用 や、核心障害を複数本重ねたい本格的な単発に向いています。核心障害は 4〜5本、別利益は 3本以上、軽い共有圧の固有数は 4〜6本 を目安にしてください。硬い共有ボトルネックの本数だけでなく、各局面へどう露出させるか も先に決めておくと安定します。

8局面の骨格

局面役割ここで置くもの
第1局面導入世界の前提、弱い伏線、主体配置、最初の共有圧
第2局面第一接触表向きの問題、軽い共有ボトルネック、最初の硬い共有ボトルネック候補

第5局面中盤の圧力外部主体の本格介入、隠し時計の前進、共有圧の増幅
第6局面取得 / 救出主目的の達成、真のボトルネックの顕在化
第7局面退出 / 生還硬い共有ボトルネックの集中、反応系統の収束

8局面では、世界がPCより先に動く感じ を必ず作ってください。各局面で「今ほしいもの」を持つ外部主体が最低一つは動き、上級運用なら二つ以上が違う方向から噛んでくるようにすると、上級者でも全部は制御しきれません。

28. キャンペーンへ接続するときのメモ

単発シナリオをそのまま並べるだけでは、キャンペーンにはなりません。
各回が どの野望をどこまで進める回か、何を持ち越し、何を答えとして閉じる回か を明記して、長い流れの中に位置づけてください。

節目を先に決める

この単発をどの節目へ置くのか、前後の回で何を明かし、何を保留するのかを先に決めると、単発の問いと長期の問いがぶつかりにくくなります。

持ち越すものを決める

何を持ち越し要素として残し、何を履歴として残し、何をその回だけで閉じるかを先に決めておくと、単発の設計書がキャンペーン用の素材として使いやすくなります。

難度曲線を作る

この回を「回復しやすい回」「別利益の競合が強い回」「世界反応が重い回」のどれとして置くかを決めておくと、長期の波が作れます。

再登場要素の管理

再登場するNPC、勢力、場所、制度は、単発本文に書き散らさず、長期管理表へ移してください。
その具体的な書き方は キャンペーン設計ガイド の各章で扱います。

キャンペーンへ接続するときは、その単発が 発見回 / 文化接触回 / 挑戦・越境回 / 回復兼仕込み回 / 畏敬回 のどれとして働くかを書いてください。これだけで、同じ事件でも「世界を知る回」なのか「足場を増やす回」なのかが伝わりやすくなります。

回の型を明示して渡す

キャンペーンへ接続するときは、その単発が 発見回 / 文化接触回 / 挑戦・越境回 / 回復兼仕込み回 / 畏敬回 のどれとして働くかを書いてください。これだけで、同じ事件でも「世界を知る回」なのか「足場を増やす回」なのかが伝わりやすくなります。

単発から長期管理へ渡す最小メモ

【今回の話数 / 節目】
【今回の回の型】発見 / 文化接触 / 挑戦・越境 / 回復兼仕込み / 畏敬
【今回進めた野望】
【今回のシナリオの問い】
【今回の結末】
【今回の安全ルート / 近道 / 地元ルート】
【今回の回復点 / 畏敬点】
【次回へ持ち越す持ち越し要素】
【次回へ持ち越す手札タグ】
【今回追加された履歴】
【再登場NPC / 勢力の更新】
【次回へ持ち越す偶発フック】

ここで残した最小メモを、キャンペーン設計ガイドの進行ログと状態管理表へ移せば、単発と長期設計の橋渡しになります。

単発で完結したPCを続投させるとき

単発シナリオが終わってから「続きを見たい」となった場合は、その単発の結末を履歴として固定し、何を長期線へ持ち上げるかだけを決める と接続しやすくなります。単発の勝利や悲劇をやり直すのではなく、その結果生まれた責務・傷・関係 を次の話の素材にしてください。

移行の詳細な手順は キャンペーン設計ガイドの該当章 を参照してください。シナリオ設計書側では、少なくとも 何を履歴として固定するか / 何を持ち越し要素や手札タグへ上げるか / レギュラー化する要素は何か を残しておくと、単発から長編へ渡しやすくなります。

次回候補として並べるための書き方

キャンペーンでプレイヤーに次の行き先を選ばせたいなら、各シナリオ案には 公開用の短い候補文 を持たせておくと便利です。候補文には、少なくとも 表向きの目的 / 見えている危険 / 主に触れる野望や勢力 / 放置したとき進むこと を入れてください。

初級〜中級では「この回では補給線を整えられる」「この回では協力者を作りやすい」など、仕込めそうな手札タグの方向 を候補文へ書いてよいです。上級〜熟練では、裏の仕込みは書かず、読めば気づける材料だけを本文側へ置く のが基準です。

  • 文化差を心理当てではなく、礼・贈答・証人・通行・受理の価格差として扱っているか
  • 今回持ち帰る Q / B / S が明示され、何が次の自己像や物語になるか見えているか
  • 主高揚類型 と 副高揚類型 を書き、どの局面で解放するか固定したか
  • 崇高化 か 蓄積からの解放 のどちらかが、名場面に重なっているか
  • 六欲求（没入 / 成長 / 自己表現 / 戦略 / 非日常 / 連帯）のうち、どこで何を返すか見えているか
  • 秘密開示段階ごとに、伏線収束 / 正体露見 / 前提崩壊 のどれを狙うか決めたか
  • 重要NPCに、接続 / 犠牲と救済 / 継承と連帯 の少なくとも一つが起きる条件を書いたか
  • 逼迫突破の圧を、失敗率ではなく 追加利益・帰路・引き渡し・後始末・世界反応 で蓄積させているか

Q / B / S の公平性チェック

  • 残り方: Q / 残り方: B / 残り方: S の三系統が、シナリオ本文か世界課題タグへ配られているか
  • Builds が行動メニュー化していないか
  • 世界課題タグの返りが、Q / B / S のどれかへ明示されているか
  • 関係や秘密が、後出しではなく段階的に見えるか
  • 劇的展開度が、開始時の宣誓と食い違っていないか
  • 関係表示を強く使ったあと、必ず何かが変わるようになっているか

30. 最後に

QBSで単発シナリオを設計するときの核心は、難しい一問を置くことではありません。
何を読ませるか、何を先回りさせるか、局地勝利のあとにどの外側の工程を残すか を、先に文書へ固定することです。

単発なら、問い・条件・期限・局面を短く鋭くまとめてください。
長い流れへ接続したいときは、この本の設計書から必要部分を切り出して キャンペーン設計ガイド の管理表へ渡してください。

高難度化は、プレイヤーを罠にはめるための仕組みではありません。
「いま勝つ」ことと「勝ち切る」ことを分けるための設計です。局地勝利は本当に通し、そのうえで世界の反応も本当に動かしてください。
