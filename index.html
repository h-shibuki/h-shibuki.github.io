長期運用向けのルール

新しい HTML を追加するときは、最初に front matter を書いておくのがおすすめです。

  • featured: true なら上段の主要ガイドに出ます
  • nav_order で並び順を制御できます
  • badge と page_type で見た目を揃えられます
  • listed: false で一覧から外せます

新しいページを増やすときの流れ

普段は index.html を触らなくても大丈夫です。新しい HTML 側に必要な情報を書けば、トップページが追従します。

1

front matter を書く

title、description、nav_order、badge を先頭に追加します。

2

HTML の head もそろえる

<title> と <meta name="description"> を front matter と揃えると、検索向けにもきれいです。

3

push すると自動反映

GitHub Pages の Jekyll ビルドで一覧が再生成され、トップページのカードに反映されます。

補足: 一覧に出したくないページは listed: false、上段の主要ガイドに出したいページは featured: true を使ってください。
