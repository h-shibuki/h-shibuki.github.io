QBS 選択ルール集

この別冊は、QBS の各モード（選択ルール）を一か所にまとめたものです。
プレイヤーは 「プレイヤーが読むところ」 を、GM は 「GMの運用」 を、キャンペーン設計者は 「キャンペーンでの置き方」 を見れば、必要な情報に最短で届くようにしています。

1. この本の使い方

モードは、自由宣言をやめるための別ゲームではありません。いつ宣言するか / 何が見えるか / 何で補うか / どこで揺らぐか / 長い進行をどう呼吸させるか を調整するための選択ルールです。資産 / 関係 / 感情 / 火種 / 人物といった違いは、本質差ではなく表示差として扱います。

  • プレイヤー は、採用されているモードの「プレイヤーが読むところ」だけ読めば十分です。
  • GM は、公開情報の出し方、内部処理、採用基準を「GMの運用」で確認します。
  • キャンペーン設計 では、常時採用か、特定回だけか、どの長期線へ返りを積むかを「キャンペーンでの置き方」で決めます。

より細かい書式例や運用の手触りまで見たいときは、各モードの 実戦補足 を参照してください。

2. 採用時の共通確認

卓の開始時に、まず次の三点をそろえます。

  • 採用有無: 今回どのモードを使うか。
  • 何が変わるか: 宣言タイミング、追加公開情報、運の揺らぎ、能力値の補助、競合、秘匿、長期進行の呼吸のどこが変わるか。
  • 読む場所: プレイヤーは必要部分だけ、GM は内部処理まで読むこと。

モードはすべて任意で、併用できます。 難しい卓だから自動的に増える仕組みではありません。先に決めるのは 卓等級 と 裁定傾向 / 指針提示度 / 劇的展開度 / 死亡許容度 で、その上で必要な取扱領域だけをモードとして足します。

整理すると、モード は「何を扱うか」、卓等級と四軸 は「それをどう返すか」です。たとえば同じオブザーバーでも、寛容 なら PC の核や成長線を壊しにくい既定宣言、悪辣 なら同じ甘えや抜けが破滅へ届きうる既定宣言になります。旅情 なら助力や伴走が返りやすく、怨情 なら敵視や報復が返りやすくなります。

また、ソーシャル / タクティカル / アンロック は、1つの局面の中に複数の 局内工程 を立てて読むモードです。局面が大きな問い、局内工程がその中で個別に処理する小さな筋だと考えてください。

物語パッケージで選ぶ

はじめてモードを選ぶときは、個別ルールを一つずつ見るより、「どんな物語を遊びたいか」の束から選ぶと入口が強くなります。最初にパッケージで方向を決め、そのあと必要なら個別モードを足してください。

「何が遊べるか」を先に束で見せると、選択ルールが設計者向けの記号ではなく、プレイヤーへ渡す約束として読みやすくなります。

モードが増えても正本は変わらない

どのモードを併用しても、公開正本は最新の公開現在状態、ウィスパー時の個別秘匿正本は各 PC に最後に渡した個別現在状態です。内部メモやツール出力は裁定補助であり、投稿に出るまで正本ではありません。

差分欄と完全再掲が食い違うなら完全再掲、途中の候補と結果返答が食い違うなら結果返答を優先します。

モード併用でも訂正と再提出の既定は同じ

GM / AI / 補助ツールの訂正で前提が変わったときは、次の確定結果が出る前なら再提出可、訂正そのものでは時間を進めない のが既定です。

結果確定後の誤りは、モードにかかわらず、まず主成果を保ったまま未来側で補正します。巻き戻しは全員が望む例外だけにしてください。

モードは既定裁定を上書きしない

モードを足しても、QBS の既定は変わりません。まず 卓上で真になっていること を読み、主目的は成功で仮置きし、見えていた / 読めた / 仕込めた障害 だけを請求対象にします。請求は、まず否定ではなく 価格 で返します。大きな変化は、演出だけで流さず 履歴 / Builds / Stories のどこかへ残します。

ソーシャル / タクティカル / アンロック で切る 局内工程 は、必要ならそれぞれ 応対 / 作戦 / 手順 と呼び替えて構いません。ただし、正式にはどれも 局面の下位にある局内工程 として扱います。

リラックス / ゼノス / ヨーク / ゼロット は、オンにすると長い呼吸、旅の余韻、連帯、逸脱を前へ出しやすくするモードです。オフでも、旅・協力・詩的な演出そのものが禁止されるわけではありません。

3. ナラティブ・モード

結末の成否だけでなく、見え方の変化、関係更新、余韻と次回への引き を主報酬にする選択ルールです。成功失敗を軽くするのではなく、どのように終わり、何が残るとその回が満足になるか を先にそろえます。

プレイヤーが読むところ

このモードでは「勝ったか負けたか」だけでなく、「何が言い切られたか」「誰との距離が変わったか」「何がまだ気になるまま残ったか」を大事にします。宣言では、主目的に加えて、何を問いにしたいか、どの関係を揺らしたいか、何を言い切らずに残したいか を一言添えると強くなります。

GMの運用

GM は各シナリオに、未解決の問い、見え方が変わる点、関係更新点、感情回収点、次回への引き を最低 1 つずつ置きます。局面の成否より先に、どれが確定するとその回を閉じられるか と どれを次回まで残すか を決めてください。

このモードの終了条件は、主に次の四つです。見え方が変わる、関係が更新される、価値選択が確定する、自己像が言い換わる。どれか一つが確定したなら、その回は十分に終われます。

迷ったらの既定（卓等級と四軸の反映）

項目このモードで主に変わること
卓等級初級では変化点を分かりやすく一つへ絞り、熟練では複数の未解決問いや関係線を抱えたまま終われます。
裁定傾向寛容は余韻や人物履歴として残しやすく、悪辣は露見・断絶・取り返しのつかなさとして返りやすくなります。
指針提示度福音は問いと引きを明示し、暗黒は結果のあとで意味だけが浮かび上がる構成を取りやすくなります。
劇的展開度旅情は余韻や再訪、人情は和解や義理、激情は告白や衝突、怨情は執着や断絶を前へ出します。
死亡許容度低では退場より関係更新を、高や極では退場や断絶そのものを確定点として扱えます。

実戦補足

ナラティブ・モードは「物語っぽい雰囲気にする」ための曖昧な合図ではありません。終わり方と残り方を設計するモードです。次のメモがあるだけで、運用はかなり安定します。

【ナラティブ設計メモ】
未解決の問い:
見え方が変わる点:
関係更新点:
感情回収点:
次回への引き:
この回で確定すると終われるもの:
この回のあとに残すもの: 未解決の問い / 人物履歴 / 約束 / 断絶 / 次回予告文

失敗は停止ではなく、再文脈化、秘密露見、関係のずれ、選択肢の狭まり として返してください。主目的未達でも、何の意味が変わったか、誰との距離がどう変わったかは必ず残します。

相性がよいのは ヨーク、アンロック、ウィスパー、ゼノス です。関係変容、真相反転、半秘匿の群像、旅情年代記のどれにもつながります。

キャンペーンでの置き方

長編では、各アークに最低 1 本、見え方が変わる回 と 関係が更新される回 を置いてください。公開用のシーズン・ログライン、未解決の問い、次回予告文を先に持つと、このモードは強く機能します。

4. オブザーバー・モード

宣言のタイミングを前倒しし、局面ごとの細かな入力よりも、PCの行動原理で進める選択ルールです。

プレイヤーが読むところ

プレイヤーは局面ごとに長文を書く代わりに、シナリオ開始時に「このPCが何を優先して動くか」を観測方針として出します。意識するべきなのは、最優先・危機時の第一手・捨ててもよいもの・禁止事項をはっきり書くことです。

GMの運用

GMは未取得情報で最適化せず、観測方針・既知情報・履歴に沿って行動を決めます。

オブザーバー・モードでは、プレイヤーは局面ごとの宣言を出さず、シナリオ開始時に 観測方針 を提出します。GM は、その方針・既存タグ・履歴・関係・世界状態から、局面ごとの 既定宣言 を作ります。

【観測方針】
最優先:
取りにいくもの:
平時の第一手:
危機時の第一手:
捨ててもよいもの:
禁止事項:
退路基準:

GM がしてはいけないのは、未取得情報で最適化すること です。既定宣言の質は、観測方針だけなら原則 妥当、既出の準備や手札タグが噛み合えば 周到、事前明文化された長期仕込みが強く噛み合ったときだけ 卓越 としてください。

結果返答では、今回 PC が取った行動 / その根拠になった観測方針・タグ・履歴 / 生じた結果と変化 を明記すると、プレイヤーが次の更新判断をしやすくなります。

迷ったら、観測方針から作る既定宣言にも通常の既定をそのまま適用します。 まず観測方針で狙う主目的を一文にし、見えていた / 読めた / 仕込めた障害だけを価格へ変えます。未取得情報で安全最適化したり、読めなかった上位障害で主目的をあとからひっくり返したりはしません。

迷ったらの既定（卓等級と四軸の反映）

項目オブザーバー・モードで主に変わること
卓等級どこまで先回りしないと安定しないか。熟練では、局面内での成功だけでなく、外側の工程や帰路まで読めて初めて安定します。
裁定傾向観測方針の空白をどう読むか。寛容は PC の核や成長線を壊しにくい既定宣言、悪辣は同じ甘えや抜けが破滅へ届きうる既定宣言になります。
指針提示度観測方針から何を再掲するか。福音は「今回この原理が参照されそう」と厚め、暗黒は結果だけに寄せます。
劇的展開度既定宣言の成功や失敗を、助力・義理・衝突・報復のどちら向きで返すか。旅情なら伴走、怨情なら敵視や追い打ちが強くなります。
死亡許容度悪い既定線の到達上限。低なら重傷や重大喪失止まり、極なら不慮の死や退場まで到達しえます。

実戦補足

オブザーバー・モードでは、プレイヤーは 局面ごとの宣言を行いません。 代わりに、PC作成後またはシナリオ開始時に、その PC がどういう人物として動くかをまとめた 観測方針 を提出します。以後、GM はその観測方針・既存タグ・履歴・関係・世界状態に従って、その局面の 既定宣言 を作って裁定します。

これは自由宣言の放棄ではありません。「その場で宣言する」代わりに「どういう人物として動くかを先に宣言しておく」 モードです。

観測方針

観測者 PC は、通常の PC 記入欄に加えて、次を持ちます。

【観測方針】
最優先:
取りにいくもの:
平時の第一手:
危機時の第一手:
捨ててもよいもの:
禁止事項:
退路基準:
  • 最優先: 最後まで守ろうとするもの。1〜2個。
  • 取りにいくもの: 積極的に得ようとするもの。1〜2個。
  • 平時の第一手: 情報不足のとき、まず何をするか。
  • 危機時の第一手: 危険が見えたとき、まず何をするか。
  • 捨ててもよいもの: 代償にしてよいもの。1〜2個。
  • 禁止事項: どれだけ得でもしないこと。1個。
  • 退路基準: どの線を越えたら撤退・離脱・見切りを選ぶか。

基本処理

  1. GM は局面文を通常どおり提示します。
  2. 観測者 PC のプレイヤーは、局面ごとの宣言を返しません。
  3. GM は、観測方針・既存タグ・履歴・関係・現在の世界状態から、その局面でもっとも自然な 既定宣言 を1つ作ります。
  4. 既定宣言は、内部的には通常の宣言と同じく、主目的 / 最低限ライン / 手段 / 守りたいもの / 捨ててもよいもの を持つものとして扱います。
  5. GM はその既定宣言を裁定し、結果を返します。

GM の結果で見るところ

観測者 PC への返答では、GM は結果の中に次の 3 点を明記します。

1. 今回 PC が取った行動
2. その根拠になった観測方針・タグ・履歴
3. 生じた結果、タグ、履歴、関係の変化

プレイヤーが見るべきなのは、自分の PC がどう動いたか と、次に方針を書き換えるべき変化が起きたか の二つです。

GM がしてはいけないこと

GM は、その PC がまだ得ていない情報を使って 最適化 してはいけません。行動選択に使ってよいのは、その PC が知っていること / 持っているもの / 観測方針に書かれていること だけです。

複数の行動が同程度に自然なら、禁止事項に触れない → 退路基準を守る → 最優先を守る → 平時 / 危機時の第一手に従う → 不可逆損失がより小さい の順で選びます。

既定宣言の強さ

  • 観測方針だけ から生じた既定宣言は、原則として 妥当 までです。
  • 既出の準備、退路、支援、証拠確保、手札タグがそのまま噛み合うなら、周到 まで上げて構いません。
  • 卓越 は、事前に明文化された長期仕込みや準備が現在局面に強く噛み合ったときだけです。

観測更新

観測方針は、原則として シナリオ開始時と終了時 にだけ書き換えてよいものとします。長期キャンペーンでは、各シナリオの合間に更新して構いません。観測者でも、命運点、履歴、手札タグ、関係・通信タグ、失ったものは通常どおり変化します。

【オブザーバー quick ref】
観測方針を書き、局面ごとの宣言は GM が既定宣言化する
GM は未取得情報で最適化しない
既定宣言の質は 原則: 妥当 / 噛み合いで: 周到 / 長期仕込みで: 卓越
観測更新は シナリオ開始時・終了時（長期では幕間も可）

キャンペーンでの置き方

長編では、観測方針の更新窓と、観測者だけが不利にならない情報配置を先に決めておくと安定します。

オブザーバー・モードは、各局面のひらめき より PC の一貫した行動原理 を見たい長編と相性が良いです。使うなら、どの PC が観測者か を最初に決め、観測方針の更新窓を 各シナリオの開始時と終了時 に置いてください。

キャンペーン側で大事なのは、観測者 PC だけが不利になるような「未取得情報がなければ選べない正解一手」に寄せすぎないことです。必要な読み筋は、既知情報、痕跡、取得可能な資料として出しておくと、長編でも破綻しにくくなります。

5. プロバビリティ・モード

宣言読解に 2d6 の揺らぎを足し、運の要素を少しだけ入れる選択ルールです。

プレイヤーが読むところ

プレイヤー側で大切なのは、出目任せにせず、主目的・手段・備えを言葉で示すことです。深い宣言ほど結果が安定し、薄い宣言ほど出目だけでは伸びません。

GMの運用

GMは先に宣言の質を読み、その後で 2d6 を振ります。出目だけで全部を決めないことがこのモードの核です。

このモードでは、自由宣言の読み取りに加えて、運の揺らぎ を判定に入れます。QBS の基本はあくまで宣言読解です。したがって、出目が良くても薄い宣言は伸びにくく、出目が悪くても深い宣言は壊れにくくします。

6. クオンティティ・モード

文章とタグを主にしたまま、能力値を補助線として足す選択ルールです。

プレイヤーが読むところ

数値は便利ですが主役ではありません。文章で何をしたいかを書き、そのうえで能力値が補助する、と考えると読みやすくなります。

GMの運用

GMはタグ・履歴・関係・準備で読めることを先に確定し、能力値は足りない余白を補うために使います。

クオンティティ・モードでは、PC に 筋力 / 器用 / 知力 / 魅力 / 意志 の能力値を持たせます。初期配分は 4,3,3,2,2 です。ただし、優先順位は常に タグ > 履歴・関係・準備・世界状態 > 能力値 としてください。

このモードだけを使うなら、主能力4で難度1段階軽減、主能力2で1段階増加を目安にします。プロバビリティと併用するなら、能力修正 = 能力値 - 3 として達成値へ加えます。

迷ったらの既定（卓等級と四軸の反映）

項目このモードで主に変わること
卓等級初級では能力値は補助線として軽く使い、熟練では能力差と Build / 履歴 / 関係の重ね合わせまで読ませる。
裁定傾向寛容は数値不足を関係や準備で救いやすく、悪辣は数値不足をそのまま請求へ接続する。ただし成立済みの事実を数値で否定しない。
指針提示度福音は参照能力と有効な補助線を再掲し、暗黒は能力名だけに留める。
劇的展開度旅情は能力差が役割分担や助け合いへ、人情は信頼や依存へ、激情は張り合いへ、怨情は侮りや報復へ返る。
死亡許容度低では能力差が即死へ直結しにくく、極では能力差の読み違いが致命傷まで届きうる。

実戦補足

このモードでは、自然言語で書かれた PC 記述に加えて、一般的な RPG 風の 能力値 を持たせます。ただし、能力値は主役ではありません。タグ、履歴、関係、準備、世界状態で直接読めることを、能力値で上書きしてはいけません。

標準能力値

標準能力値は 筋力 / 器用 / 知力 / 魅力 / 意志 です。世界観に合わせて、体 / 技 / 知 / 社 / 志 へ言い換えて構いません。

  • 筋力: 力、耐久、踏ん張り、重作業、肉体的強行
  • 器用: 速さ、精密さ、潜入、操作、身のこなし
  • 知力: 分析、記憶、知識、工作、読解
  • 魅力: 説得、交渉、演技、信用、場の支配
  • 意志: 勇気、集中、信念、平静、誘惑や恐怖への抵抗

初期値

PC を通常どおり作成したあと、5 つの能力値に 4, 3, 3, 2, 2 を 1つずつ割り振ります。4 は明確な得意分野、2 は明確な不得意分野です。4 と 2 には、それぞれ根拠になる一文を添えます。

器用4: 屋根伝いの移動と潜入に慣れている
意志2: 仲間を切る判断が遅れる

基本処理

  1. まず タグ / 履歴 / 関係 / 準備 / 世界状態 を読みます。
  2. それだけで結果が十分に読めるなら、能力値は使いません。
  3. まだ余白があるときだけ、主能力を1つ決めて参照します。

プレイヤーは宣言末尾に 主能力: 器用 のように添えて構いません。ただし、それだけで有利になることはなく、最終的に何を参照するかは GM が決めます。

クオンティティだけを使うとき

  • 主能力が 4 なら、難度を 1段階軽く してよい
  • 主能力が 2 なら、難度を 1段階重く してよい
  • 主能力が 3 なら、変化なし

この補正は、タグや準備で直接読める部分には使いません。 あくまで GM がまだ迷う余白だけに使います。

プロバビリティと併用するとき

プロバビリティと併用するなら、能力値は達成値修正に読み替えます。

能力修正 = 能力値 - 3
4 → +1 / 3 → 0 / 2 → -1

優先順位

文章で書かれたタグ
> 履歴・関係・準備・世界状態
> 能力値

能力値だけで不可能を可能にはしません。逆に、能力値が低くても、十分な準備と筋の通った宣言があるなら通します。

能力値の変化

能力値は原則として 固定 です。QBS における成長の主役は、あくまでタグ、履歴、関係、拠点、仲間です。ただし、再訓練 / 長期療養や後遺症 / 身体改造 / 身分や立場の大きな変化 / 信念や覚悟の決定的変化 があったときは、シナリオ終了時に 1 点だけ入れ替えて構いません。

【クオンティティ quick ref】
能力値: 筋力 / 器用 / 知力 / 魅力 / 意志
初期値: 4,3,3,2,2 を1つずつ配る
単独運用: 4なら難度1段階軽減 / 2なら1段階増加 / 3なら変化なし
プロバビリティ併用: 4なら+1 / 3なら0 / 2なら-1
優先順位: タグ > 履歴・関係・準備 > 能力値

キャンペーンでの置き方

長編では数値成長の配分が Build の成長線を食わないよう、保持タグや素材の保持形式と並べて管理します。

クオンティティ・モードは、キャラクターシートの手触り を強めたい長編で使います。能力値の基本形は 4,3,3,2,2 を維持し、成長の主役はあくまでタグ、履歴、関係、拠点、仲間に残してください。

長編で運用するなら、能力値は原則固定 とし、再訓練、後遺症、身体改造、身分変化、信念変化のように PC の基礎そのものが変わったときだけ、シナリオ終了時に 1 点入れ替える窓を用意すると安定します。

7. リラックス・モード

各シナリオの区切りは保ったまま、キャンペーン全体の終点だけを急がない選択ルールです。

プレイヤーが読むところ

旅、後始末、取材、補給、関係修復のような回を無理なく重ねたいときに向きます。だらだら引き延ばすためではなく、小さな完結を重ねるためのモードです。

GMの運用

GMは「各シナリオの問いは毎回きちんと終える」ことを守りつつ、次回候補の出し方と見直し間隔を整えます。

迷ったらの既定（卓等級と四軸の反映）

項目このモードで主に変わること
卓等級初級では「各話に答えは出るが全体終点は急がない」がそのまま遊び味になる。熟練では、全体終点を急がなくても各話の工程管理は重くできる。
裁定傾向寛容は小さな達成や回復を拾いやすい。悪辣は大成を急がせないだけで、各話の請求は容赦しない。
指針提示度福音は次回候補や休み方を厚く出し、暗黒は寄り道の意味を読ませる。
劇的展開度旅情は休息・補給・同行の豊かさへ、人情は関係修復へ、激情はくすぶる対立へ、怨情は静かな遺恨や報復の仕込みへ返る。
死亡許容度低なら休息回で死線を跨がせにくく、極なら「ゆっくり進むが、油断した回で死ぬ」は普通にありうる。

何を緩め、何を守るか

このモードで緩めるのは、キャンペーン全体を最終局面へ一直線に押す圧 だけです。逆に、GM は次のものを緩めません。

  • 今回の問い
  • クリア条件 / 失敗条件
  • 期限
  • 答えが出たら終了する という原則

QBS は、自由宣言の結果が世界に残ることで締まります。ここを失わないため、一つの問題をだらだら続ける 運用にはしません。毎回の小さな終わりを保ったまま、キャンペーン全体の終点だけを急がないでください。

次回候補の出し方

シナリオ終了ごとに、GM は次回候補を 2〜4本 出します。役目の違う候補を混ぜると、プレイヤーの自発性が出やすくなります。

候補の役目何を扱う回か短い例
主野望寄り大きな目標や敵対勢力へ一歩近づく「王都の書庫で禁書の出所を探る」
回復兼仕込み休息、補給、拠点、協力者、退路づくり「港町で補給線と船医を確保する」
後始末 / 修復余波、誤解、借り、破損、人間関係を整える「疑われた衛兵隊に弁明し信用を戻す」
横道の発見世界課題、別利益、再登場線、秘密開示「古い観測所で消えた地図師の記録を追う」

候補文は 選択肢例 です。プレイヤーが「この勢力と和解したい」「今のうちに工房を整えたい」と別案を出したなら、同じ重さの候補として採用してかまいません。

野望度・成長点・命運点の扱い

リラックス・モードでは、普段の回では野望度を原則動かさない でください。野望度を動かすのは、立場の更新、価値選択、大きな勝利や敗北など、人生の節目だと感じる回だけで十分です。

日常の達成感は、成長点、世界課題タグの回収、素材、保持ノード、関係や秘密の進展 で返します。小さな依頼、補給、工作、交渉、和解、後始末にも、正規の保持形式を用意してください。

命運点の回復を、このモードだけで自動的に厚くしてはいけません。 穏やかな長編にしたいときも、命運点は既存の回復原則に従って扱い、無料の全快装置にしないでください。緊張を薄めるなら、命運点そのものではなく、危険度の低い候補や仕込み回 を増やして調整します。

回復回とタグ整理

回復回は、ただ休ませる回ではありません。休息・再配置・関係修復・次の仕込み のいずれかを必ず含めます。休んだ結果として、次の宣言が少し通しやすくなるものを一つ残してください。

  • 隠れ家、連絡網、航路、紹介状の確保
  • 工房、宿、診療所、祈祷所の足場づくり
  • 誤解の解消、謝罪、和解、同行の約束
  • 後で使える証拠、部材、情報の整理

また、長く続く卓では タグを増やすこと より 統合して閉じること が重要です。終了ごとに、少なくとも次の三択で整理してください。

  • 残す: まだ次の判断を変える
  • 閉じる: 役目を終えたので消す
  • 履歴へ送る: 事実としては重要だが、いまは手札ではない

似た向きの持ち越し要素が増えたら、一枚に圧縮してかまいません。リラックス・モード運用では、この整理を怠ると管理がすぐ破綻します。

通常キャンペーンへ戻す / 締める

このモードは、キャンペーンの終点を消すものではありません。急がない だけです。大きな敵や価値選択が見えてきたら、「ここからは通常どおり終点へ向かう」と宣言して戻して構いません。

逆に、プレイヤーが「ここで満足した」と感じたなら、その節目で静かなエピローグへ入って終了してもかまいません。重要なのは、いつ終えるかをプレイヤーと共有すること であり、最終決戦を義務にしないことです。

実戦補足

何をするモードか

標準のキャンペーンでは、シナリオの積み重ねが最終的に野望の終点や最終局面へ向かいます。リラックス・モードでは、その直線を弱めます。今回の事件を解き、今回の問いに答え、次はどこへ行くかをそのつど選ぶ 運用に寄せます。

  • 残すもの: シナリオの問い / クリア条件 / 失敗条件 / 期限 / 終了処理
  • 弱めるもの: 毎回野望度を動かす圧、最終決着へ一直線に進む圧
  • 増やすもの: 次回候補の選択、休養や仕込みの回、世界課題タグの細かな回収

「終わらない一つの問題」をだらだら引き延ばすのではありません。終わる単位を小さく保ったまま、次の行き先をプレイヤーが選び続ける のが核心です。

次回候補へどう希望を出すか

シナリオ終了時には、GM が次回候補を 2〜4本 出すのが基本です。あなたはその中から選んでもよいですし、同じ重さの別案 を提案してもかまいません。

このときは、次の三つを書くと希望が通りやすくなります。

  • 今いちばん触りたいこと: 人、場所、問題、勢力、後始末
  • 次に仕込みたいもの: 協力者、退路、補給線、工房、証拠、関係修復
  • 今回は避けたい重さ: 連戦、怨情の激化、致死級危険、政治抗争など

候補を「正解当てクイズ」と思う必要はありません。このPCなら今どこへ向かいたいか を短く出せば十分です。

次に行きたい方向:
- 触りたいもの:
- 仕込みたいもの:
- 今回は避けたい重さ:
- もし候補外なら、やってみたい別案:

野望度が動かない回の読み方

このモードでは、野望度が動かない回 = 無意味な回 ではありません。野望度は人生の節目に使い、日常の充実や小さな達成は別の場所で受け取ります。

その回で主に進むものよくある内容主な返り
Quiddities（いま真であるもの）聞き込み、調査、後始末、土地勘づくり、世界課題の回収新しい事実、世界状態の改善、秘密段階の前進
Builds（保持された焦点）補給、工作、工房整備、拠点開設、協力者の固定素材、保持ノード、保持された関係線
Stories（変質の連続）和解、謝罪、再会、里帰り、同行、看病、祭礼関係の変質、感情の変化、火種表示、再登場線

表示ラベルの推奨先: ソーシャルは関係 / 感情 / 制度、タクティカルは資産 / 手順 / 路、アンロックは制度 / 場 / 路 / 火種、ヨークは関係 / 人物、ゼロットは火種 / 感情 / 人物、ゼノスは場 / 路 / 文化を推奨します。これは本質差ではなく、読み味のための表示差です。

成長点は地道な研鑽、野望度は人生の到達度 と切り分けて読むと、このモードでも停滞しません。

回復回でも何かを残す

休む回、遊ぶ回、取材する回、里帰りする回でも、次の宣言が少し楽になるもの を一つ残すと、リラックス・モード運用が締まります。

  • 安全な宿や隠れ家を押さえる
  • 道案内や仲介役を得る
  • 補給線や連絡網を整える
  • 謝罪や対話で絆の壊れ方を変える
  • あとで使える小さな証拠や素材を持ち帰る

逆に、何も変えずに全快だけする 回を続けると、QBS らしい「宣言が残る」感触が薄くなります。穏やかな回でも、何が残ったかは一行でよいので意識してください。

キャンペーンでの置き方

長編では本数ではなく見直し間隔を決め、野望度・成長点・命運点の回り方を通常キャンペーンと切り分けておくと扱いやすくなります。

基本原則

リラックス・モードでは、キャンペーン全体の最終決着を必須にしません。 ただし、各シナリオの問い、クリア条件、失敗条件、期限、終了処理 は通常どおり持ちます。

つまり、シナリオは終わる / キャンペーンは急がなくてよい、が原則です。QBS の自由宣言は「今回の答え」を変えることで強く働くため、各話の切れ目は残します。

既定値と採用基準

既定は 初級ではあり、中級・上級・熟練ではなし とします。ただし、これは機械的な強制ではありません。旅情、人情、日常、後始末、補給、調査、共同体の運営を重く見るキャンペーンでは、どの段階でも採用して構いません。

逆に、短本数で鋭く終点へ向かう設計、毎回大きな敗着線が動く設計、最終局面の圧を主軸にする設計では、切るほうが向きます。

進行の基本ループ

  1. 各シナリオで今回の問いに答えを出す
  2. 終了処理でタグ整理と成長処理を行う
  3. GM が次回候補を 2〜4 本出す
  4. プレイヤーが候補を選ぶか、同等の別案を出す

候補の役目は分けると安定します。主野望を押す回 / 回復兼仕込み回 / 後始末や修復の回 / 横道の発見回 を混ぜると、自由宣言の方向が生まれます。

野望度・成長点・命運点

このモードでは、普段の回では野望度を原則変動させません。 野望度が動くのは、立場の更新、大きな勝利や敗北、価値選択の固定など、人生の節目だと感じる回だけで十分です。

通常の達成感は、成長点、世界課題タグの解決、素材、保持ノード、関係・感情・秘密の進展 で返します。これにより、「終わりを急がない」ことと「何も進まない」ことを切り分けられます。

命運点の回復を、このモードだけで厚くしてはいけません。 穏やかさを作りたいときは、命運点を無料で戻すのではなく、危険の薄い候補や回復兼仕込み回を増やして調整します。

回復回と持ち越し要素の整理

回復回は、全快だけして元に戻る回 にしません。少なくとも一つは、次の宣言が軽くなるものを残します。隠れ家、航路、紹介状、工房、協力者、謝罪、信用回復、記録整理のような軽い足場が向きます。

また、長く続くほど 持ち越し要素の増えすぎ が最大の敵になります。終了ごとに、残す / 閉じる / 履歴へ送る の三択で整理し、似た向きのものは一枚へ圧縮してください。

通常キャンペーンへ戻す / ここで締める

リラックス・モードは、最終局面を禁止するものではありません。大きな敵や終わりの問いが見えてきたら、通常のキャンペーン進行へ戻して終点へ向かってよい です。

逆に、プレイヤーが「この辺りで十分満足した」と感じたなら、その節目で静かなエピローグへ入って終えてもかまいません。終点を必須にしない ことが、このモードの核です。

キャンペーン級の対立を扱うなら、誰が誰と敵対するか だけでなく、何を取り合うときにバーサスへ入るか を先に決めておくと、長編でも対立が濃くなりすぎません。

8. ソーシャル・モード

本心当てではなく、見えているサインから非開示情報や協力可能線を読む選択ルールです。

プレイヤーが読むところ

相手の心を言い当てる必要はありません。表層サイン、共有圧、次に守ろうとしていることから、「どこなら話が通りそうか」を読むのが中心です。

GMの運用

GMは内面そのものを確定せず、読むための材料を出します。成果は情報開示、虚偽の揺らぎ、条件付き協力として返します。

ソーシャル・モードで大事なのは、内面そのものを確定させない ことです。局面では、心理の代わりに 表層サイン / 共有圧 / 次防御予告 / 残っている共有状態 を出してください。

このモードでは、1つの局面の中に複数の 局内工程 を立てて構いません。たとえば 切り崩し / 懐柔 / 牽制 / 保留崩し のように、相手に対して何を小さく通すかを工程ごとに分けると、細かい指示をしやすくなります。

【表層サイン】言い淀み / 話題回避 / 過剰反応 / 矛盾 / 強調しすぎる点 / 誰の前では言い方が変わるか

【次防御予告】論点ずらし / 虚偽の補強 / 逆質問 / 退席 / 第三者を盾にする / 条件闘争へ持ち込む
【残っている共有状態】二人きり / 安全保証 / 面子の退路 / 証拠突合せ / 第三者退席 / 限定協力の土台

GM メモ側では、真の心理ボトルネック / 非開示情報 / 開示や協力の最低条件 / 虚偽継続時の次防御 を事前固定します。返りは、情報開示 / 虚偽の揺らぎ / 条件付き協力 / 判断保留 / 一時的譲歩 / 関係表示や感情表示の変化 として返してください。

内面そのものを確定させない のが、このモードの一番大事な制限です。「改心した」「恐怖した」「罪悪感に屈した」のような断定を主目的にしないでください。QBS らしい強さは、内面断定ではなく 共有状態を変えること にあります。

キャンペーンでの置き方

長編ではレギュラーNPCの防御線や協力条件をアーク単位で持っておくと、交渉の積み重ねがきれいに残ります。

ソーシャル・モードは、尋問、密談、交渉、審問、評議、派閥調整のように、表層サインと共有圧が主軸 の回へ置きます。毎回入れるより、人間関係の節目、情報開示の節目、立場が動く節目 に置くほうが呼吸が良くなります。

長編では、ソーシャルの成果を 情報開示 / 条件付き協力 / 関係表示や感情表示の変化 として持ち越すと、Stories 線が太くなります。本心の断定より、次の局面で何が言えるようになったか を残してください。

9. タクティカル・モード

位置取り、共有圧、敵や環境の次の手を見える化して、自由宣言のまま連携しやすくする選択ルールです。

プレイヤーが読むところ

コマンド選択のゲームに戻るわけではありません。誰がどこにいて、何を止めると楽になるかを読んで、自由宣言で役割分担しやすくするための補助線です。

GMの運用

GMは位置関係と圧力を公開して連携の芯を示しますが、固定コマンドへ落とし込みすぎないようにします。

タクティカル・モードは、戦闘や難所にだけ 位置取り、共有圧、敵や環境の次の手 を足す任意追加です。自由宣言をやめるためではなく、共有ボトルネックを読ませる ために使います。

このモードでは、1つの局面の中に複数の 局内工程 を置けます。たとえば 陽動 / 制圧 / 潜入 / 退路確保 のように、同じ局面の中で個別に処理する小さな筋を分けると、ガチ勢向けの細かな指示を受け止めやすくなります。

迷ったらの既定（卓等級と四軸の反映）

項目このモードで主に変わること
卓等級初級では位置と退路が見えていれば回る。中級では役割分担、上級では作戦の連鎖、熟練では局地成功後の外側工程まで含めた三手読みを要求する。
裁定傾向寛容は作戦の曖昧さを安全寄りに補い、悪辣は見えていた退路不足や役割穴をそのまま圧へ接続する。
指針提示度福音は位置・共有圧・次手予告を厚く再掲し、暗黒は現状態だけ見せて次を読ませる。
劇的展開度旅情は援護や庇い合いへ、人情は役割信頼へ、激情は突破と犠牲へ、怨情は執拗な追撃や見せしめへ返る。
死亡許容度低では壊滅より撤退や重傷に寄せ、極では退路読み違いが即死へ届きうる。

いつ使うか

毎戦闘に付ける必要はありません。位置関係、退路、役割分担 が主軸になる局面だけに付けると、PBM でも読みやすくなります。

局面で見せること

  • 誰がどこにいるか
  • いま放置すると重くなる共有圧
  • 敵や環境が次に何をしようとしているか
  • 誰かが作ると味方も使える共有状態

見せるのは 読む材料 であって、行動メニューではありません。

やってはいけないこと

タクティカル・モードを入れたからといって、自由宣言を「移動 / 攻撃 / 防御」のような固定コマンドへ戻してはいけません。プレイヤーは引き続き、何をどうしたいか を文章で宣言します。

実戦補足

タクティカル・モードは、戦闘や難所にだけ 位置取り、共有ボトルネック、ラウンド圧、敵や環境の次の手 を見える化する追加です。自由宣言をコマンド選択へ戻すためのルールではありません。

何が増えるか

  • 位置関係: 誰がどこにいて、どこへ行けるか
  • 共有圧: 放置すると全員のどこかへしわ寄せが出る問題
  • 敵意図 / 環境意図: 次に何が起こりそうか
  • 共有状態: 退路、遮蔽、時間稼ぎ、搬送線など、誰かが作ると味方も使える足場

増えるのは 読む材料 であって、選べる行動が固定されるわけではありません。

まず何を読むか

最初に読むべきなのは、どの圧を誰が受け持つか です。個人最適だけでなく、誰が共有ボトルネックを押さえると他PCが楽になるか を見ると、自由宣言の質が一段上がります。

次に、敵や環境が次に何をしようとしているか を見ます。勝つことそのものより、いま止めるべきものを止める ほうが強い局面が多いです。

宣言に一行添える

タクティカル・モードが有効な局面では、いつもの宣言に次の一行を足すだけでも十分です。

戦術補足:
- どの共有圧を受け持つか
- 誰の退路や生還線を支えるか
- 何を次ラウンドへ残したいか

これは書式の強制ではありません。連携の芯 を明文化するための補助です。

キャンペーンでの置き方

長編では、護送・脱出・防衛・追跡など、戦闘以外の難所にもどこまで使うかを先に決めておくとぶれません。

タクティカル・モードは、戦闘や難所にだけ 位置、共有圧、敵意図 を足す任意追加です。既定はありません。シナリオの性質 と プレイヤーの希望 を優先し、必要な回だけ採用してください。

10. アンロック・モード

鍵、封印、機械、手順など、構造そのものを読む局面を扱う選択ルールです。

プレイヤーが読むところ

「正解を当てる」よりも、「何を見てそう判断したか」を書くことが重要です。今の状態、つながっている先、固定された条件を確かめると読みやすくなります。

GMの運用

GMは可視構成要素・物理条件・構造圧・共有安全状態を公開し、設計メモ側では真のボトルネックと、結果の残り方 / 請求工程を固定します。

アンロック・モードは、鍵、封印、爆弾、隔離工程、再起動手順のように、構造の読解 が主軸の局面で使います。公開するのは 可視構成要素 / 既知の物理条件 / 構造圧 / 次の構造反応 / 残っている共有安全状態 です。

このモードも、1つの局面の中に複数の 局内工程 を置いて構いません。たとえば 観測 / 試験 / 切替 / 分離 / 解放 のように工程を刻むと、「もっと細かく指示したい」という遊び味へ応えやすくなります。

GM メモでは、核心障害 / 真のボトルネック / 第1構造カウンター C1 / 必要なら第2構造カウンター C2 / 最速敗着線 / 痕跡 / 情報源 / 仕込み窓 / 抜き筋 を固定してください。読み違えたが前進はした、を C1/C2 で返し、触ってはいけない失着を最速敗着線で返す二層処理が基本です。

このモードだけは、宣言前確認を少し厚くして構いません。同一工程・同一対象 に限り、1回の確認で 現状態 / 連動先 / 既知の物理固定条件 のような原子的事実を束ねて返してよいです。ただし、どの順が正解か / 何を先にすべきか / これで安全に生還できるか は返しません。

項目このモードで主に変わること

裁定傾向寛容は観測・試験で安全状態を拾いやすく、悪辣は見えていた連動や物理条件の見落としを厳しく返す。
指針提示度福音は可視構成要素・共有安全状態・次の構造反応を厚く示し、暗黒は事実だけを置く。
劇的展開度旅情は未知構造との出会いを探検の手応えへ、人情は協力解法へ、激情は禁忌突破へ、怨情は罠・封鎖・報復起動へ返る。
死亡許容度低では失着は閉鎖・損壊・重傷止まり、極では致死起動や閉じ込め死が通常帰結に入る。

アンロック・モードは、高難度シナリオ設計の縮図を、1局面サイズへ圧縮したもの です。鍵開けに限らず、儀式停止、隔離解除、爆発物処理、手順解体、医療工程、設備再起動のように、構造そのもの を読む場面で使います。

11. バーサス・モード

排他的な最終状態を、自由宣言のまま競合処理する選択ルールです。

プレイヤーが読むところ

相手に勝つための専用コマンドではありません。同じ対象を「捕らえる / 逃がす」のように取り合うとき、何を争っているのかを一段だけ細かく書くモードです。

GMの運用

GMは競合する層、同時提出の方式、致死や恒久喪失の扱いを先に合わせ、両立部分があるならまずそこを残します。

バーサス・モードは、PCどうしや陣営どうしが、同じ対象の同じ層の最終状態 を排他的に奪い合う局面で使います。QBS の自由宣言をやめるためではなく、自由宣言どうしの競合を整列し、共有世界の結果を一つへ畳むための任意ルールです。

迷ったらの既定（卓等級と四軸の反映）

項目このモードで主に変わること
卓等級初級では争う層を一つに絞り、中級では別利益の衝突、上級では多層競合、熟練では複数工程にまたがる排他的最終状態を扱う。
裁定傾向寛容は競合しても両立する周辺利益を広めに拾う。悪辣は排他的な最終状態をきっちり切り分ける。
指針提示度福音は何が排他的で何が両立可能かを示し、暗黒は結果の衝突だけを見せて宣言から整列させる。
劇的展開度旅情は競い合いのあとに共存や譲歩が返りやすく、人情は遠慮や貸し借りへ、激情は決着と遺恨へ、怨情は潰し合いと報復へ寄りやすい。
死亡許容度低では競合の帰結は敗北・喪失止まり、極では対立が死へ届きうる。

いつ使うか

次の三条件がそろうときに使うと、過不足が出にくくなります。

  • 排他的な最終状態 がある — 片方が拘束すれば、片方の離脱成功は同じ瞬間には立たない
  • 対象が共有世界で一つ しかない — 同じ証拠、同じ窓口、同じ退路、同じ役職、同じ当事者
  • 競合それ自体が卓の主軸 である — 学園抗争、派閥選挙、証拠争奪、護送妨害、離脱阻止など

逆に、同じ方向へ動いている複数PC卓では、まず普通の複数PC処理を優先してください。バーサスは、排他的な最終状態だけ を扱う追加です。

局面で見せること

バーサス局面では、通常の局面欄に加えて、次を明示するとぶれにくくなります。

【バーサス状態 / 共通部分】(任意)
- 競合ラウンド:
- 今回主に競合しうる対象:
- 今回争われやすい層:
- 同時提出方式:
- 先に確保済みの共有状態:
- 競合候補になりやすい一回性機会:
- 今回効きやすいタグ / 資産 / 位置取り:
- 能動的外部主体:
- 放置時に自然進行する第三の悪化:
- 同数時に置きやすい中間状態候補:

ここで重要なのは、投稿順を優先順位にしない ことです。PBM では、先に書いた人が先手になると卓の納得感が壊れやすいので、原則は 同時提出 を既定にしてください。

プレイヤーに出してもらう宣言

プレイヤーには、ふだんの宣言を大きく崩さず、次の形だけを足してもらうと扱いやすくなります。

【バーサス宣言】
主目的:
対象:
争う層: 身体 / 位置 / 物品 / 社会 から1つ
手段:
守りたいもの:
捨ててもよいもの:
別利益(任意):
今回あえて取らないもの(任意):

強く争う層は 1 宣言につき 1 つ を既定にしてください。ほかの層へ触れる内容は、別利益、演出、余波、次局面の争点へ落とします。

やってはいけないこと

  • 他PCの内面を直接確定させる — 恐怖、改心、恋愛感情、忠誠の書き換えを、一宣言で通さない
  • いきなり恒久喪失へ飛ばす — 致死、大切なものの喪失、関係の破局は卓宣誓の範囲で段階的に扱う
  • 世界を止める — 競合処理のたびに、外部主体、制度、世論、締切のどれかを最低 1 つは進める
  • 勝者を無傷にする — 勝者にも露出、消耗、監視、悪評、後始末を残す

競合そのものの処理手順と中間臨界状態表は、27章の複数PC処理で使います。ここでは、どういう局面でバーサスを宣言し、何を見せるか を先に固めてください。

実戦補足

何をするモードか

バーサス・モードは、同じ対象の同じ層の最終状態を、PCどうしや陣営どうしで奪い合うときに使います。たとえば、同じNPCを「捕まえたい / 逃がしたい」、同じ証拠を「確保したい / 隠したい」、同じ退路を「押さえたい / 通したい」といった場面です。

大事なのは、ここでもなお 自由宣言のゲームである ことです。コマンドを選ぶのではなく、主目的 と 手段 を文章で書き、その結果をGMが一つの共有世界へ畳みます。詳しい競合解決手順や中間臨界状態表は、GM側の運用として GMガイド にまとめられています。プレイヤーとしては、今回どの層を争うか、何を守り、何を捨てるか を明確に書ければ十分です。

主に争う層は四つに分ける

層何を争うか置きやすい中間状態
身体拘束、負傷、行動不能、とどめ体勢不利 / 拘束 / 行動不能 / 瀕死
位置接敵、遮断、離脱、包囲、退路確保捕捉 / 退路半壊 / 遮断 / 包囲
物品奪取、保持、破損、証拠確保所在露見 / 争奪中 / 破損 / 機能停止
社会疑惑、失脚、支持、評判、審問停止注目 / 疑惑 / 審問停止 / 失脚寸前

バーサスでは、強く争う層を1宣言につき1つだけに絞ると安定します。ほかの層に触れる内容は、演出、別利益、余波、次局面の争点へ回します。

宣言では何を書くか

主目的は1つ、強く争う層も1つが基本です。たとえば「相手を拘束しつつ、証拠を奪い、世論も落とす」は一文では書けても、標準価格では全部は通りません。

【バーサス宣言】
主目的:
対象:
争う層: 身体 / 位置 / 物品 / 社会 から1つ
手段:
守りたいもの:
捨ててもよいもの:

バーサス局面では、ふだんの宣言に 対象 と 争う層 を明示すると読みやすくなります。とくにPBMでは、同時提出 を前提にしたほうが投稿順の有利不利が減ります。

GM返答は三段で読む

GMの返答では、まず 共通成果 を見ます。追い詰めた、退路候補を一つ潰した、証拠の所在を露見させた、疑惑を公にした、といった部分です。ここは、競り負けても残ることがあります。

次に 排他的な最終状態 を見ます。拘束されたのか、離脱を許したのか、証拠を持ち去ったのか、失脚させたのか、という一つしか立たない結果です。

最後に、決着が持ち越されたなら 中間臨界状態 を見ます。ここが次局面の争点です。QBSのバーサスは、即断の勝敗よりも、まだどちらにも転びうる臨界点 を作ったほうが面白くなりやすいです。

やってはいけない書き方

  • 他PCの内面を直接確定する — 「恐怖した」「改心した」「恋に落ちた」は、宣言だけでは決まりません。
  • 合意のない恒久喪失へ一気に飛ぶ — 致死や大切なものの喪失は、卓宣誓で許可された範囲で扱います。
  • 一宣言で複数層を強く取り切る — 身体 / 位置 / 物品 / 社会 の全部を一気に主処理へ入れないでください。
  • 勝ったら無傷だと思う — 勝者にも、注目、悪評、消耗、監視、後始末は残ります。

キャンペーンでの置き方

長編では、どの回に置くかと、対立が終わったあとの関係の残し方を先に決めておくと後腐れが少なくなります。

キャンペーン設計書への追記欄

重要なのは、バーサスを置いても 世界反応を止めない ことです。教師、制度、警察、世論、怪異、締切のどれかを毎ラウンド最低 1 つは進め、PC同士だけで世界を閉じさせないでください。

  • 中盤の反転回 — 味方だったはずの派閥が別利益を優先する
  • 学園 / 都市の対立回 — 役職、証拠、世論、放送、後援を取り合う
  • 終盤前の離脱 / 護送回 — 同じ当事者や同じ退路の最終状態を奪い合う

どの回に置くか

既定はありません。ただし、シナリオの性質 と プレイヤーの希望 を優先してください。長編で常時バーサスにすると、関係管理と情報管理が遊びを食いやすいので、使う回を選ぶ ほうが安定します。

既定と卓宣誓

キャンペーンでバーサス・モードを使うときは、毎回入れる よりも、派閥対立が主軸の回、証拠や窓口を取り合う回、離脱や護送の成否を奪い合う回 に絞るほうが呼吸が良くなります。QBS のバーサスは、対立を別ゲームへ切り離すためではなく、自由宣言どうしの競合を整列するためのモードです。

12. ウィスパー・モード

秘匿情報と個人別利益を持つ、半協力卓向けの選択ルールです。

プレイヤーが読むところ

秘密を毎回全部明かす必要はありません。守りたいもの、捨ててもよいもの、どこまで共有済みかを意識して書けば十分です。

GMの運用

GMは共有主目的と個人別利益を両立しやすい形で配置し、秘密が露見したときの関係・タグへの反映先を決めておきます。

ウィスパー・モードは、複数PC卓に 秘匿情報 と 個人別利益 を足す任意追加です。完全対立ではなく、共有主目的は同じだが、望む着地点が少しずつ違う 卓を作るために使います。

迷ったらの既定（卓等級と四軸の反映）

劇的展開度旅情は秘密が信頼や告白へ、人情は貸しやためらいへ、激情は対立や決断へ、怨情は疑心と裏切りへ返る。
死亡許容度低では秘匿失敗は関係悪化や機会損失に寄り、極では共有不足がそのまま致死線を開きうる。

個別情報は、合流 / 通信確立 / 公開描写 / 共有宣言 のどれかがあるまで、他PCの宣言根拠にはできません。逆に、まだ共有していないこと自体を理由に、GM が勝手に妨害や失敗へ読み替えることもありません。

競合はどこで起きるか

半協力卓で本当に競合になるのは、同じ対象の排他的な終着点 を取り合ったときです。たとえば「同じNPCを捕縛したい / 逃がしたい」「同じ証拠を提出したい / 隠したい」のような場合です。

相談していないこと、動機が少しずれていることだけでは競合になりません。QBS の基本どおり、GM はまず 両立する部分 を優先して裁定します。

成長点の返り

ウィスパー・モードでは、共有主目的を達成したうえで 個人別利益まで通したPCに、追加の +1 候補 がつくことがあります。ただしこれは新しい別通貨ではなく、既存の 追加で +1（最大2回まで） の中に入ります。

共有主目的が失敗した場合、個人別利益だけを通しても成長点ボーナスにはなりません。その代わり、関係の進展、感情の変質、秘密開示段階の前進 として返ることがあります。

キャンペーンでの置き方

長編では秘密の露見タイミングと、共有主目的を失わないための基準を長期的に管理します。

ウィスパー・モードは、複数PCキャンペーンへ 秘匿情報 と 個人別利益 を足す任意追加です。既定はありません。半協力をしたくない卓へ無理に入れる必要はありません。

どの回に置くか

毎回入れるよりも、中盤の反転回、関係が揺れる回、最終局面の着地点が割れる回 に絞るほうが扱いやすくなります。長編で常時秘匿にすると、情報管理が遊びを食いやすいからです。

長編で崩さないための基準

  • 共有主目的は一つ に固定する
  • 個人別利益 は別利益として置く
  • 主進行でないPCにも、保持か変質の進展 を必ず返す
  • 露見した秘密は、持ち越し要素 / 関係 / 履歴 / 世界課題タグ へ変換する

秘密が回をまたいで残るなら、誰が知っていて、どの回で共通情報へ上がるか をキャンペーン側で先に持っておくと、長編でも散りにくくなります。

成長点の返り

個人別利益の達成は、既存の 追加で +1（最大2回まで） の中へ吸収します。条件は、共有主目的が達成されていること と そのPCの個人別利益が達成されていること です。共有主目的が未達なら、関係進展や秘密開示段階の前進で返すとぶれません。

大きな野望がまだ言葉になっていなくても始められます。何を攻略したかより、何に出会い、何を持ち帰ったかが重くなります。

オフでも旅や異文化は通常の局面として扱えます。オンにすると、GM は 文化差 / 偶発 / 持ち帰り を毎回の収穫として返しやすくなり、失敗も止まるより枝分かれしやすくなります。

GMの運用

GMは失敗をただの停滞で終わらせず、別の案内人、寄り道、新しい驚きへ変換しやすくします。

迷ったらの既定（卓等級と四軸の反映）

項目このモードで主に変わること
卓等級初級では驚きと文化差を楽しみやすく、中級では旅の価格表を読み、上級では異文化対応と持ち帰りの整理、熟練では旅情の外側にある危険工程まで読む。

ゼノス・モードでは、未知の土地へ行く理由、文化差や通行差、偶発フックの母材、回復点か畏敬点、持ち帰り候補 を毎回のシナリオに最低ひとつずつ置くと安定します。安全ルート / 近道 / 地元ルートの差が見えていると、旅の宣言が自然に立ちます。

このモードでは、交流失敗、作法違い、祭りへの遅刻、珍品の買い損ねを、そのままシナリオ失敗へ落とさないでください。誤解、借り、別の案内人、思わぬ招待、新しい寄り道 に変換し、旅の枝として返すほうが合います。重く扱う失敗条件は、生還不能 / 帰路断絶 / 通行不能 / 関係破綻 / 補給崩壊 のような 旅の継続不能 です。

終了投稿では、今回の収穫 を一行で明文化してください。新しい事実、関係、手札、余韻のどれでも構いません。野望度が動かない回でも、何が残ったか が見えていれば、プレイヤーは「この旅は自分の物語だった」と受け取りやすくなります。

実戦補足

何をするモードか

ゼノス・モードでは、大きな野望の達成や差し迫った任務の成功よりも、未知の土地、異文化、奇習、祭り、風景、驚きに触れる旅 を前面へ出します。QBS の基本は変わらず、宣言は局面単位で行い、世界は反応し、代償や余波も起きます。ただし、重く見るのは 何を攻略したか より 何に出会い、誰と交流し、何を持ち帰ったか です。

リラックスとの違い

リラックス・モード は、野望は残したまま歩幅をゆるめるモードです。ゼノス・モード は、野望そのものが薄くても、異境への引力で旅を続けられるモードです。回復や後始末が中心ならリラックス、驚きや越境や交流が中心ならゼノス、と考えると読み分けやすくなります。

このモードでは、交流失敗、作法違い、祭りへの遅刻、珍品の買い損ねをそのままシナリオ失敗へ直結させません。誤解、借り、別の案内人、思わぬ招待、新しい寄り道 へ枝分かれさせるほうが、QBS らしい旅の感触が残ります。

同じ場面の見え方

同じ「祭りの街へ少し遅れて着く」場面でも、カメラの寄り方が変わります。

運用返り方の例
通常時祭り本番に間に合わず、交渉や探索がやや不利になる。主目的への遅れとして扱いやすい。
ゼノス・モード祭り本番には遅れるが、後夜祭の案内人、裏路地の屋台、撤収する旅芸人とつながり、別の縁や持ち帰りが立ち上がりやすい。

旅の収穫の読み方

このモードでの収穫は、新事実 / 新しい縁 / 新しい手札 / 新しい余韻 のどれかです。野望度が毎回動かなくても、回復点 / 畏敬点 / 素材 / 関係の変化 / 持ち帰り品 / 世界の見え方の変化 が残るなら、その回は十分に前進しています。

野望も「成り上がり」だけではありません。世界のさまざまな場所を見て回る、平穏な日常を守る、困っている人々を助ける のようなものでも構いません。ゼノス・モードでは、まだ何者になりたいか決めきれていないが、まず世界を歩いて確かめたい という薄い野望から始めてもよいです。

キャンペーンでの置き方

長編では旅路そのものが収穫になる回を候補として並べ、成長と持ち帰りの残し方を先に置いておくと機能します。

既定値と採用基準

既定は なし とします。ただし、これは出発点にすぎません。旅・発見・異文化交流・偶発 を主軸にしたい長編なら、どの段階でも採用して構いません。リラックス・モード が「野望は残したまま歩幅をゆるめる」配置なら、ゼノス・モード は「野望そのものが薄くても、異境への引力で回を選べる」配置です。

候補として並べたい回の型

ゼノス・モードでは、候補の役目を次のように分けると選ぶ意味が立ちます。

候補の型プレイヤーが得やすいもの放置したとき進みやすいもの
発見回新事実、地図、噂、記録、景観見逃した情報、世界の見え方の固定
交流回新しい縁、協力者、紹介、理解誤解、借り、関係のすれ違い

このモードでは、野望度が毎回動かなくても構いません。薄い野望が言葉になる、旅の動機が深まる、持ち帰りが次の旅を呼ぶ といった変化を前進として扱えます。終了ログには、今回の収穫 を一行で残してください。

14. ヨーク・モード

同行・肩代わり・共同作業・帰着確認を見える化し、複数PC卓の連帯を自由宣言のまま太くする選択ルールです。

プレイヤーが読むところ

一人で全部を背負う必要はありません。シナリオ開始時に、今回だれと何を一緒に背負うか を短く決めます。これがあると、救援、分担、待機、援護、相談、回復、継承の宣言が書きやすくなります。

【同行誓約】
結ぶ相手:
今回いっしょに守るもの:
その相手の何を肩代わりするか:
その相手にだけ見せる弱さ:
帰るときに確かめたいこと:

これは自動で罰が発生する契約ではありません。今回この相手とどういう関係で旅をするか を先に見える化するための欄です。前に出した連帯線を自分で破れば、関係・信頼・共同作業の価格が返ることはあります。

フルの同行誓約を重く感じる卓では、次の 2 行だけでも十分です。

【同行メモ】(各PC1行)
今回いっしょに守るもの:
帰るときに確かめたいこと:

これだけでも、救援・肩代わり・回復・後始末の線が見えやすくなります。

ヨークがオフでも、協力・庇い合い・関係更新は通常どおり起こります。ヨークはそれを禁止 / 解禁するルールではなく、今回だれと何を一緒に背負うか を見える化する補助線です。

ヨーク・モードを使うシナリオでは、二人以上で越えるとおいしい共有圧、回復点での非戦闘の共同作業、肩代わりや弱さの開示で関係が変わる局面 の三つを最低1回ずつ置くと安定します。

同行誓約に沿った行動が本文で成立したら、GM は Q / B / S のどれかに、共有の返り を 1 つ置いてください。これは新通貨ではなく、既存の保持形式へ吸収します。

迷ったらの既定（卓等級と四軸の反映）

項目このモードで主に変わること

良いヨークは、「いっしょに戦う」だけではありません。片方が前に出て、片方が後始末を背負う、片方が社交を受け持ち、片方が退路を管理する、片方だけが知る弱さを、もう片方だけが見ている といった運用まで含みます。

ヨークは主に複数 PC の連帯線を太くするモードであり、NPC の有無や存在感を決めるモードではありません。NPC との協力や感情線も、ヨークの有無にかかわらず通常どおり扱えます。

残り方置きやすいもの

重要なのは、だれと組むかをGMが命令しないことです。だれと何を背負うかを、プレイヤーが自分で決めやすくする のがこのオプションです。

既定は なし です。ただし、複数PCで旅する長編、護送、巡礼、学園、共同体防衛、隊商、遠征では非常に入れやすい追加です。毎回全員が全員と結ぶ必要はありません。その回で前に出す線を 1〜2 本だけ選ぶ ほうが呼吸が良くなります。

相性がよいのは タクティカル、ゼノス、ウィスパー です。救援・分担・秘密を抱えた連帯が見えやすくなります。

15. ゼロット・モード

現実的整合性の代わりに、遊戯的・詩的・批評的・夢幻的・実存的・メタ的な整合性を前景化し、良いナンセンス を卓で扱いやすくする選択ルールです。

プレイヤーが読むところ

このオプションでは、変な宣言、大げさな比喩、夢の logic、滑稽な飛躍、妙に詩的な処理、第四の壁ぎりぎりの言い回しを出して構いません。ただし、「何を壊すか」と「何だけは壊さないか」を一言添えてください。

【ゼロット補足】
逸脱:
合図:
残す一貫性:
狙う報酬:

変だから通るのではありません。この卓では、この変さを面白さとして扱う と共有されているから通ります。

通るかどうかは、変さの強さではなく、逸脱 / 合図 / 残す一貫性 / 狙う報酬 が書けているかで決まります。

局所発動

シナリオ全体をゼロット回にしなくても、採用卓では各 PC が 1回 / シナリオ を目安に局所逸脱を提案してかまいません。

【局所逸脱】
逸脱:
合図:
残す一貫性:
狙う報酬:
支払い: 火種表示1 または 目立つ余波タグ1

これは危険の無償取消しではありません。苦境や失敗を、笑い / 詩性 / 批評 / 夢性 / メタ性へ変換する権利 として使います。

通る例と通りにくい例

  • 通る例: 法廷回で、被告ではなく法廷そのものに証言を求める。批評風刺や夢幻の合図があり、判決の重みと代償は消さない。
  • 通りにくい例: 爆発しそうな炉を「今日は喜劇回だから」で無害だったことにする。共有済みの危険を、合図や支払いなしで無効化している。

GMの運用

ゼロット・モードを使うときは、シナリオ開始時に 今回のゼロット枠 を 1〜2 個だけ告知してください。

【今回のゼロット枠】
- 解放喜劇
- 批評風刺
- 詩的・言語遊戯
- 夢幻・異世界自然化
- 実存的不条理
- メタ・異化

GM は、変な宣言を「現実的ではないから」で捨てるのではなく、逸脱があるか / 合図があるか / 局所的一貫性があるか / 報酬があるか の四条件で読みます。成立したら、必ず Q / B / S のどれかに返してください。

迷ったらの既定（卓等級と四軸の反映）

項目このモードで主に変わること
卓等級初級では変な宣言を通しやすい入口として働き、中級では変さを Build や Story へ残し、上級では逸脱の代償管理、熟練では逸脱を勝ち筋にも破滅筋にも変える設計になる。
裁定傾向寛容は変さを新しい窓口や物語の推進力として拾いやすい。悪辣は逸脱が偏見・孤立・自壊へ返りやすい。
指針提示度福音はどの程度まで変でも世界が受け止めるかを示し、暗黒は結果だけ返して後から意味を浮かび上がらせる。
劇的展開度旅情は祝祭や珍道中へ、人情は愛嬌や共犯へ、激情は狂熱と対決へ、怨情は悪夢・風刺・破滅喜劇へ返る。
死亡許容度低では逸脱の代償は社会的ズレや喪失へ留め、極では逸脱そのものが死や退場の引き金になりうる。

実戦補足

ゼロットは無償通過の免罪符ではありません。価格は消えません。大事なのは、意味の回路を切り替える ことです。

残り方置きやすいもの
Quiddities群衆の注目、制度の空転、禁忌の抜け道、夢の扉、場の反転
Builds奇妙な小道具、言い回し、合言葉、仮面、奇縁、芝居の足場
Stories共有の笑い、気まずさ、幻滅、親密化、畏怖、不穏な通じ合い

やってはいけないのは、既に共有された危険や代償を、ただのギャグで無効化すること、他PCの内面を変なノリだけで直接確定すること、一人だけが面白くて他の人の没入を壊すこと です。

良いゼロットは、「意味がない」のではなく、現実的整合性を少し壊す代わりに、遊戯的・詩的・批評的・夢幻的・実存的・メタ的な整合性を与える ものです。

キャンペーンでの置き方

既定は なし です。祭り回、夢回、道化回、法廷風刺回、奇習交流回、異界越境回、あるいは重い回のあとの解放回に向きます。毎回常時採用にするより、「このアークだけ」「この土地だけ」「この回だけ」で使う ほうが扱いやすいです。

相性がよいのは ゼノス、ソーシャル、アンロック、ウィスパー です。異境の鮮度、社交的反骨、夢 logic、仮面劇の秘密が立ちやすくなります。

16. 併用時の順番

複数モードを併用するなら、基本順は次のとおりです。

ウィスパー配布 / 観測方針の確定 / 同行誓約と今回のゼロット枠の確認
→ 局面ごとの可視情報（ソーシャル / タクティカル / アンロック / バーサス）
→ 宣言
→ タグ / 履歴 / 関係 / 準備で読む
→ クオンティティで補う
→ プロバビリティで揺らぐ
→ Q / B / S へ返す

リラックス・モード と ゼノス・モード は、この流れの外側で、シナリオの切れ目やキャンペーン全体の呼吸に効きます。ヨーク・モード はシナリオ開始時の関係線、ゼロット・モード は今回許可する逸脱の枠を確認してから入るとぶれにくくなります。

同締切の宣言は同時に読む

バーサスだけでなく通常の複数PC卓でも、同じ締切で集まった宣言は投稿順ではなく同時提出として扱います。処理順は、共有事実の確定 → 個別達成 → 個別代償 → 共有余波 が既定です。

秘密事故は既知化し、責任側が補う

ウィスパーを含む併用卓で秘密が事故で漏れたときは、完全に忘れたことにはしません。既知になったものとして扱い、その事故を起こした側が補償する のを既定にします。

GM / AI 側の事故なら、代替秘密、別の有利情報、個別利益の補填など、なるべく同種の補償を返してください。

併用プリセット集

最初は 2〜3 個まで を推奨します。宣言タイミング系 0〜1、局面可視化系 0〜1、関係 / 秘匿 / 旅 / トーン系 1〜2 くらいで始めると扱いやすいです。

18. このモードを非対応にする理由の書き方

一般用で 非対応 と書くこと自体は問題ではありません。問題なのは、理由が曖昧で、何を足せば対応できるのかが読めないこと です。配布用では、「未想定です」だけで終わらせず、何が欠けているため非対応なのか を書いてください。

悪い書き方 / 良い書き方

非対応にしやすい主な理由
