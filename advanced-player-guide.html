QBS プレイヤー実戦ガイド

このガイドは、クイックスタートガイドと プレイヤー基本ルール を読んだあと、プレイヤーとしてもっと上手に立ち回りたい人向けに、実戦寄りの話をまとめています。
宣言の骨格、高難度卓での読み方、テンプレート類、半完成キャラ枠、再出発の細則を扱い、GM専用の内部設計や裁定手順までは扱いません。

1. はじめに

QBS は、基本はサイコロを振らず、プレイヤーの宣言を基本成功させ、その代わりに代償を支払って前へ進むTRPGです。
メール、掲示板、チャット、SNSのスレッドなど、文章を往復させる PBM形式 を前提にしています。

このゲームで大事なのは、できるかどうか を細かく争うことではありません。
何を失い、何を守り、それでもどこまで前へ進むか を選ぶことです。

このゲームでは、キャンペーン → シナリオ → 局面 の順で話が小さくなります。
キャンペーン はPCが野望を達成するまでの道のり全体、シナリオ はその道中で起こる一つの事件、局面 はその事件を解決するために積み重ねる大事な一歩です。

プレイヤー基本ルールが「遊ぶための基本」をまとめた文書なら、このガイドは「うまく遊ぶための実戦補助」です。

最低限ライン / 担保 / 選択肢例の読み方 / NPCや組織を動かす宣言 / 半完成キャラ枠 / 語り口の希望 / 野望度0からの再出発 など、プレイヤー側で知っていると判断しやすい内容をここに集めています。

たとえば、

  • 友人を救うことには成功したが、追っ手に顔を覚えられた
  • 吹雪の前に避難路を作ることには成功したが、体を壊した
  • 王都への足がかりは得たが、有力者に借りを作った

といった形で、宣言は実ります。
ただし、その実りにはたいてい何かしらの値段がつきます。

このガイドは、初めて読む人がこれだけで遊び方を理解できるように、用語の説明から順番に書いてあります。
まず言葉を覚え、次にPCを作り、ゲームがどう始まるかを知り、そのあと宣言の書き方を読んでください。

QBSの三要素と実戦での読み方

このガイドでは、QBS の宣言を Quiddities / Builds / Stories の三つの読み口で扱います。

  • Quiddities: その宣言が何を前進させ、何を世界に残すかを読みます。
  • Builds: その宣言にどんな許可・値引き・保険・変換がかかるかを読みます。
  • Stories: 誰の心がどう動き、どんな痛みや約束が残るかを読みます。

実戦では、この三つを分けて選ぶのではなく、ひとつの宣言の中で同時に立てます。
このガイドは、その書き方と読み方を実戦向けに整理したものです。

2. このゲームで使う言葉

この章は本文で使う用語をまとめた辞典です。先頭から読むときは、GM / PC / シナリオ / 局面 / 宣言 / 裁定 / タグ / 命運点 / 野望 / 野望度 / 卓越宣言 を先に押さえれば十分です。細かな項目は、本文中のリンクから必要になったときに参照してください。

TRPG

会話や文章のやり取りで物語を進めるゲームです。
あなたは主人公を演じ、GMは世界の様子と結果を担当します。

PBM

「Play by Mail」の略です。
QBSでは、対面で即座に話すよりも、文章を投稿して順番に進める形式を想定します。

セッション

実際に遊ぶひとまとまりの時間です。
PBMでは一度に集まらず、投稿の往復で進むこともあります。
このガイドで「セッション開始前」と書くときは、最初の投稿を始める前 という意味です。

GM

ゲームマスターです。
局面を提示し、あなたの宣言の結果を返します。
敵、町の人、家族、怪異、天気、時間経過なども担当します。

運用強度

このゲームで、その卓をどれだけ初級者にやさしくするか、どれだけ上級者・熟練者が退屈しない歯ごたえにするかをまとめて示す合言葉です。初級 / 中級 / 上級 / 熟練 の4段階で扱います。

  • 初級: 初級者にやさしい基準です。読むべき危険が見えやすく、事前対策の入口も拾いやすい卓です。
  • 中級: 慣れた人の基準です。一般的な先読み、段取り、共有ボトルネックへの対処を求める卓です。
  • 上級: 上級者が退屈しない歯ごたえを基準にする卓です。危険の再授業は減り、先回りや噛み合わせを自分で拾う比重が上がります。
  • 熟練: 熟練者向けの基準です。危険の存在、対策方向、実務細則のいずれも自力で拾う比重が高く、公開情報だけでは安全策が足りないことがあります。

運用強度 は別々の意味を持つラベルではありません。卓全体の歯ごたえと、どこまでを説明不要としてよいか の基準を、同じ 初級 / 中級 / 上級 / 熟練 でそろえて扱います。

  • 初級相当の危険: その題材に触れる人なら、基礎として読める危険
  • 中級相当の危険: 少し考えれば読める一般教養・社会常識の危険
  • 上級相当の危険: 舞台固有の制度、専門実務、専門知識、怪異固有の禁忌など、説明なしでは読みにくい危険
  • 熟練相当の危険: 複数工程の連結、例外手順、専門細則の噛み合わせなど、整理に経験を要する危険

五軸は独立です。 運用強度、裁定厳格度、指針提示度、劇的展開度、死亡許容度は別々に指定できます。
ただし卓の印象をまとめて言うなら、初級者向け = 初級 / 寛容 / 福音 / 旅情 / 低、中級者向け = 中級 / 適正 / 啓示 / 人情 / 中、上級者向け = 上級 / 峻烈 / 寡黙 / 激情 / 高、熟練者向け = 熟練 / 悪辣 / 暗黒 / 怨情 / 極 を目安にして構いません。既定は 中級者向け です。

峻烈 では 初級相当まで、悪辣 では 中級相当まで が、危険予告や既知情報で親切に再掲されないことがあります。悪辣 + 熟練運用では、上級相当まで を再授業しないこともあります。
熟練相当の危険 は後出しにしてよい、という意味ではありません。そこは先に痕跡か情報源が置かれます。

指針提示度と裁定厳格度は別です。
「福音 + 峻烈」のように、読み筋は手厚く見えても、初級相当の危険 は自分で拾う前提の卓があります。

裁定厳格度

このゲームで、無工夫宣言をどれだけ高く請求するか、どこまで早く敗着線へ触れさせるか、空欄や未定義の余白を善意に読まれやすいか・通常どおり読まれやすいか・厳しく読まれやすいか・不利な合理読みに寄りやすいか を示す軸です。
このガイドでは、寛容 / 適正 / 峻烈 / 悪辣 の4段階で説明します。

  • 寛容: 初心者救済が強く、余白も善意に拾われやすく、選択肢をそのまま選んでも比較的前へ進みやすい卓です。
  • 適正: 基準の厳しさです。余白は通常どおり読まれ、無工夫宣言は卓越の根拠にならず、成功保証もありません。読みが浅ければ進展なしや失敗もありえます。
  • 峻烈: 余白を安易に有利補完されにくく、無工夫宣言への値引きが少なく、2回目のまずい踏み方から敗着線へ触れうる卓です。節目や既知危険未対策では、進展なしや悪化 も普通に起こります。
  • 悪辣: 第1局面から敗着線に触れうる卓です。余白は不利な合理読みに寄りやすく、救いは隠され、無工夫宣言は安く通りません。公開された3案に妥当解や安全策が混ざらず、勝ち筋は第四案として掘ることもあります。 踏み方が悪ければ、達成なしのままゲームオーバー に届きます。

指針提示度

このゲームで、どこまで材料が再掲され、どこまで具体的な選択肢例が文章で添えられるか を示す軸です。
指針提示度は 情報の量と解像度 の話であり、提示された案の安全さそのものは決めません。
このガイドでは、福音 / 啓示 / 寡黙 / 暗黒 の4段階で説明します。

  • 福音: 案に注釈、必要担保、危険の読みどころまで書かれやすい卓です。
  • 啓示: 具体的な案は見えますが、結果予想や注意書きは薄めで、読み切りはプレイヤーに任されやすい卓です。
  • 寡黙: 案は短い動詞句か周辺目的までに圧縮され、原則として1手だけが書かれる卓です。「その隙に」「時間を稼いでから」「確実に」「無傷で」 のような、補助手順や成果予告は書かれにくい卓です。
  • 暗黒: 選択肢例が出ないことがあります。ただし、局面を読むための材料まで消えるわけではありません。

ただし、何が再掲されるかの上限は裁定厳格度が決めます。
福音や啓示でも、峻烈や悪辣で 説明不要 とされた危険を、危険予告や既知情報で親切に再授業するとは限りません。

劇的展開度

このゲームで、どんな感情や余韻を物語の主軸に置くか を示す軸です。
このガイドでは、旅情 / 人情 / 激情 / 怨情 の4段階で説明します。

  • 旅情: 景色、移動、発見、驚き、畏敬、余韻が主軸になりやすい卓です。
  • 人情: 義理、信頼、暮らし、交流、和解が主軸になりやすい卓です。
  • 激情: 告白、対立、犠牲、反転、感情の噴出が主軸になりやすい卓です。
  • 怨情: 執着、嫉妬、報復、裏切り、破局、濁った情念が主軸になりやすい卓です。

死亡許容度

その卓で、あなたの PC がどれくらい死亡・退場しうるかの目安です。
死亡許容度は 低 / 中 / 高 / 極 の4段階で共有されます。

  • 低: 死亡や再起不能はまれです。救命導線や撤退余地が厚めです。
  • 中: 基本線です。未処置の致死級タグや未達成の生還条件を放置すると、退場や死亡につながります。
  • 高: 危険の重さが卓の中心に入ります。致死級タグや敗着線が早く出やすくなります。
  • 極: 熟練者向けの基準です。退路喪失や競合の隙を放置すると、局面成功後でも死亡や再起不能に至りえます。

プレイヤー

あなたのことです。
自分のPCが何をするかを宣言します。

PC

プレイヤーキャラクターです。
あなたが演じる主人公です。

NPC

GMが担当する人物です。
村人、兵士、商人、家族、王、敵、協力者などがここに入ります。

シナリオ

野望を達成するまでの道中で起こる、ひとまとまりの事件や問題です。
たとえば「呪われた灯台を鎮める」「王都で偽書状の真相を暴く」「雪山で遭難者を救う」がシナリオです。

局面

シナリオを解決するために積み重ねる、ひとまとまりの大事な場面です。
局面にはたいてい 期限・争点・脅威 があります。

例:

  • 三日後の審問までに証拠をそろえる
  • 夜明けまでに吹雪の前の避難路を確保する
  • 巡察隊が来る前に隠れ家を移す

宣言

その局面で、PCがどう動くかを書く文章です。
このゲームでは細かい一手より、局面全体に対する方針を書くことが多いです。

裁定

GMが宣言を読んで、何が達成され、どんな代償や余波が起きたかを決めて返すことです。

主目的

その宣言で、いちばん達成したいことです。
1宣言につき1つ にしてください。

高難度卓では、局面成功はまず主目的1つの達成を意味します。 生還、持ち帰り、後始末、匿名性の確保まで自動で付いてくるわけではありません。そこまで取りたいなら、宣言の中で分けて書いてください。

良い例:

  • 審問を先延ばしにし、時間を稼ぐ
  • 吹雪の前に避難路を作る
  • 灯台の怪異を静める

悪い例:

  • 審問を止め、黒幕を捕まえ、王の信頼も得て、出世の足がかりも作る

手段

主目的を達成するために使う方法です。
交渉、調査、準備、祈り、魔術、変装、力押し、根回しなどがここに入ります。

タグ

このゲームで「タグ」とは、【】や〔〕のように書式が決まっている短い印の総称です。
PCや世界に残り、あとから参照しやすくするために使います。

タグには大きく2種類あります。課題タグ は、今すぐ、あるいは優先的に対処すべき強制的な状況です。手札タグ は、いつ・どう使うかをプレイヤーが選べる能動的なリソースです。用途をまだ固定したくない事実は、タグにせず 履歴 として残します。

書式の目安は次の通りです。

  • 課題タグ: 【名前｜期間: 今どう困るか / どうすれば軽減・解除できるか】
  • 手札タグ: 〔名前｜期間: 何に使えるか / どう失うか〕

期間の目安は、どちらも 短期 / 中期 / 長期 の3種類です。

タグはPCだけでなく、NPC、組織、町、噂、橋、調査の進み具合など、世界側に置いても構いません。

課題タグ

今すぐ、あるいは優先的に対処すべき「強制的」な状況です。
放置すると困り、宣言や処置で軽減・解除することが期待されます。

このゲームでは、成功の価格である代償タグも、その結果として広がる余波タグも、基本的には課題タグとして表現します。

例:

  • 【疲労｜短期: 次の局面では無理が利かない / 半日休めば軽減】
  • 【不信｜中期: 役人に疑われており協力を得にくい / 目に見える成果を示せば解除】
  • 【悪評｜長期: 街道筋で名が悪く広まっている / 次の土地で功績を立てれば解除】

手札タグ

PCや世界に残る、今後の宣言で使える「能動的」なリソースです。
支持、証拠、肩書き、協力者、安全地帯、貴重な所持品、使い切りの切り札などがここに入ります。

手札タグは、課題タグと違って今すぐ対処を迫るものではありません。 プレイヤーが「ここで使う」と宣言してはじめて力を持ちます。GMは、課題タグのように自動で軽減・解除させたり、勝手に消費したりしません。

書式の目安は次の通りです。

  • 〔名前｜期間: 何に使えるか / どう失うか〕

例:

  • 〔瞬間移動の巻物｜長期: 行ったことのある場所へ一瞬で行ける / 使用・破損・紛失で失う〕
  • 〔書記長の紹介状｜中期: 書庫や役所で顔が利く / 期限切れ・不正発覚で失う〕
  • 〔救国の名誉｜長期: 第一印象が良く、面会や事情聴取が通りやすい / 大失態・失脚で弱まる〕

履歴

主要履歴

履歴のうち、次の宣言の根拠になりうるものを最新の状態欄へ再掲したものです。

主要に入っていない履歴が消えるわけではありませんが、PBMでは最新のGM投稿だけで次の宣言を判断できるのが理想なので、次の判断に使いそうな履歴が優先して再掲されます。

用途をまだ決めない事実の記録です。
助けた人、退治した脅威、見つけた場所、交わした約束など、あとで意味を持つかもしれないが、今は効果を固定したくないものをここへ残します。

履歴はタグではありません。固定効果も、決まった書式も持ちません。プレイヤーが宣言で参照したときに、GMは既知情報、状況の変化、一時的な有利、継続する手札タグなどとして裁定して構いません。

履歴には、原則として用途を書きません。一行の事実記録で十分です。

  • 村人Aを助けた
  • 村近くの洞窟のゴブリンを退治した
  • 旧水路の抜け道を見つけた

代償タグ

成功のために直接支払った価格である課題タグです。
成功の値段そのものです。

余波タグ

代償や行動の結果として、あとから広がる面倒である課題タグです。
世界の反応や二次被害にあたります。

致死級タグ

致死級タグ は、放置すると死亡・退場・生還失敗につながる、効果の重い 課題タグ の総称です。
代償タグ か 余波タグ かは問いません。

重要なのはラベル名ではなく、タグ本文の期限句です。 QBS では、この種の重い危険は 致死級タグ のラベルへ一本化して扱います。身体損傷でも呪いでも環境死でも、処理は本文で決めます。

  • 【致死級タグ｜この局面の終わりまでに緊急治療が必要 / 未処置なら退場】
  • 【致死級タグ｜次の局面に進む前に脱出が必要 / 未処置なら死亡】

同じ 致死級タグ でも、締切が「この局面の終わりまで」なのか「次の局面に進む前」なのかで扱いは変わります。読むべきなのは名称ではなく、期限句と解除条件です。

生還条件

主目的 を達成したあとも、あなたの PC が生きて帰るために満たす必要のある条件です。
脱出、帰還、引き渡し、救命処置などがここに入ります。

命運点

PCの命は1つです。
命運点は、その1つの命に残った余力を、どれだけ段階的に扱えるかを示す値です。

  • 最大値（初期値）は GMが採用した「大切なもの」の数 + 1（開始時はふつう 2〜3）
  • 0 になると、死亡するか、冒険の主役としては立ち上がれない状態になります
  • ふつうは 課題タグが先 に使われます。命運点は、課題タグでは表しにくいほど直接に命を削るときの最終手段です
イメージ
命運点 1 なら、その命をほぼ丸ごと賭けて無理をしている状態です。命運点 2 や 3 なら、同じ1つの命を 1/2 や 1/3 ずつ削るように、危険を段階的に受けられます。

ただし、命運点は残機ではありません。致死級タグや生還条件の不達成を、それだけで無効化する回数ではありません。

たとえば 【次のターンで死ぬ】 や、致死級タグが示されているなら、その課題タグを解除しないまま次へ進めば、命運点が残っていても死にます。

また、大切なものを永久に失ったときは、命運点の最大値が 1 減り、命運点も 1 減ります（下限 0）。

大切なもの

PCにとって、できれば失いたくないものです。
人、立場、持ち物、信念、居場所などから選びます。

  • 開始時は 1つか2つ（初心者は 1つ で十分です）
  • 3つ以上 を候補にしてよいですが、命運点に数えるかどうかはGM確認で確定します
  • GMが採用した数が、命運点の最大値（採用数 + 1）に関わります

大切なものを書くときは、名前だけでなく、その意味と、どう危険になりうるかを一言添えると遊びやすくなります。

  • 師の護符袋（師の継承者である証 / 奪われれば継承の正当性が揺らぐ）
  • 村の畑（家族の生活そのもの / 焼かれれば冬を越せない）
  • 妹の信頼（自分がまだ人でいられる最後の支え / 裏切れば二度と戻らない）

野望

PCが最終的にかなえたい大きな目標です。
成り上がり、復讐、発見、救済、栄光、名誉、研究の完成など、長期的な目的を指します。

単発シナリオで遊ぶときは、長い人生目標ではなく、今回の話でかなえたい直近の目標 を野望として扱って構いません。
たとえば「迷い込んだ魔の森から脱出する」「ダンジョンの奥に眠る財宝を持ち帰る」などです。

野望は大きすぎても構いません。
ただし、終点の目安 5 は基準値です。GMは開始時に、野望の大きさに応じて終点を 6 以上へ調整してよいです。
単発シナリオでは、GMが終点を 2 にすることがあります。

野望度

PCが自分の野望にどれだけ近づいたかを表す値です。
これは戦闘力のレベルではなく、人生の目標への到達度 です。

  • 初期値は 1
  • 終点の目安は 5
  • GMは開始時に、野望の大きさに応じて終点を 6 以上へ調整してよい
  • 0 になると、その野望はいったん折れます

卓越宣言

目の前の障害に正面からぶつかるのではなく、障害の構造そのものを崩す宣言 です。
先回り、根回し、段取り、避難路の確保、証人の手配、敵の条件つぶしなどがここに入ります。

難しい理屈や奇抜な発想だけが卓越宣言ではありません。
すでに見えている情報を使って、一手先に備える だけでも十分です。

キャンペーン

PCが野望を達成するまでの道のり全体です。
複数のシナリオを続けて遊ぶ長い物語で、野望度や長期タグだけでなく、手札タグや履歴も積み重なります。

単発シナリオでは、その1本だけを 1シナリオのキャンペーン として扱います。
このときは、シナリオクリア（野望度が 2 になる）でキャンペーン終了です。

単発では、+1 に届かなくてもその1本で終了してよい と覚えてください。野望度の +1 / 0 / -1 は、その1本の達成度採点として扱われます。続編フックが残るかどうかは、GMと卓の合意次第です。

つまり、キャンペーンの中にシナリオがあり、シナリオの中に局面があります。
あなたは、局面ごとに宣言を積み重ね、1本のシナリオを終え、それを重ねてキャンペーンの終わり、つまり野望の達成へ近づいていきます。

Quiddities / Builds / Stories で頻出する語

語実戦での見方
Quiddities宣言が局面 / 世界 / 秘密 / 立場をどう動かしたかを見る読み口です。勝ち筋の「残り方」をここで読みます。
得意な行動「このPCならまずこう動く」という一文です。迷ったときの起点になります。
能力タグ宣言の値段を変える恒久タグです。許可 / 値引き / 保険 / 変換 のどれをしてくれるかで読みます。
世界課題タグシナリオ本筋とは別に、片づけると Quiddities / Builds / Stories のいずれかが進むサブ目標です。
絆タグを切るその結びつきを大きく使って、強い成功を取りに行くことです。使ったあとは関係が変質します。
秘密開示真相が一気に全部出るのではなく、兆候 → 輪郭 → 代償 / 当事者 → 真相 の段階で見えていくことです。
劇的展開度旅情 / 人情 / 激情 / 怨情 のどれを主軸にする卓か、という見方です。
長期資産ノード装備・仲間・共同体・制度窓口・資格・手順など、Builds として長く残る資産の最小単位です。
台帳で一枚ずつ持ち、主要作用 / 適用 / 鈍り / 反応 で読みます。
Build 素片部材 / 手順 / 縁故 / 正当性 / 場 / 証左 の6種です。
装備だけでなく、仲間・共同体・資格・拠点の成長材料になります。
接続 / 圧縮接続は二つのノードのシナジー、圧縮は二つのノードを一つへまとめる処理です。
便利になりますが、鈍り と 反応 は残すのが原則です。

3. プレイヤーは何をするゲームか

GMは、局面を文章で提示します。
そこには、ふつう次のものがあります。

  • 導入や現況: 今どこで何が起きているか
  • 期限: いつまでに動かなければならないか
  • 争点: 今回の局面で何が問題なのか
  • 既知情報: PCが知っていてよいこと
  • 難度: どれくらい厳しい局面か
  • 選択肢例: 宣言の出発点になる案。福音 / 啓示の初心者卓では必ず示され、寡黙では短く、暗黒では出ないことがあります
  • PCの現在状態: 命運点、野望度、大切なもの、短期 / 中期 / 長期の有効課題タグ、有効手札タグ、主要な履歴、関係・通信タグ、失ったものや変化
  • 世界の現在状態: 世界側の課題タグや手札タグ、参照できる履歴、敵やNPCの動き、残り期限、進行中カウントダウン
  • 危険予告: 次に同じ無理を押すと何が起こりうるか

PBMでは、最新のGM投稿だけで次の宣言を判断できることが大切です。局面投稿、結果返答、シナリオ終了のたびに、状態やタグや履歴が完全再掲されるのが基本です。差分欄があっても、それだけで読まず、現在状態欄まで必ず見てください。

あなたはそれを読んで、PCとして

  • 何を達成したいか
  • どんな手段を使うか
  • 何を守りたいか
  • どこまでなら失ってよいか

を宣言します。

そしてGMは、その宣言を成功させたうえで、代償タグや余波タグ、状況の変化を返します。
あなたはその結果を受けて、次の局面でまた宣言します。

4. キャラクターの作り方と成長ライン

初心者は、次の7つを決めれば始められます。

項目ごとに埋めても、短い自由記述で出してもかまいません。GMは必要なら整理を手伝い、最後はあなたに確認してから確定します。

補助ツール: PCを下書きからGMへ渡す本文まで整えたいときは、プレイヤー用の補助ツールとして PC作成ツール が使えます。7項目の入力、単発 / 長期の目安、半完成キャラ枠、自由記述の下書きまで、1画面で整理できます。

1) 名前

PCの名前です。

2) 立場

どんな人物かをひとことで表します。

例:

  • 港町の見習い航海士
  • 宮廷書庫の下級書記
  • 山寺育ちの祓い手
  • 雪国の荷運び人

3) 野望

PCが最終的にかなえたい大きな目標です。

例:

  • 王の顧問となり、国の進路を変える
  • どの港でも通用する海図作りで名を残す
  • 怪異を鎮める術を体系化し、各地に残す
  • 家族を貧しさから救えるだけの財を築く

単発シナリオで遊ぶときは、長い人生目標ではなく、今回の話でかなえたい直近の目標 を書いて構いません。

例:

  • 迷い込んだ魔の森から脱出する
  • ダンジョンの奥に眠る財宝を持ち帰る

野望は大きすぎても構いません。
ただし、終点の目安 5 は基準値です。GMは開始時に、野望の大きさに応じて終点を調整します。
単発シナリオでは、GMが終点を 2 にすることがあります。

書くときは、今の立場から何を大事にして、どんな方向へ進みたいのか がわかるようにしてください。ゼノス・モードでは、まだ何者になりたいかは決めきれていないが、まず世界を歩いて確かめたい という薄い野望から始めても構いません。

4) 大切なもの

1つか2つ選びます。初心者は 1つ で十分です。
書くときは、名前 + その意味 にしてください。

  • 亡き師の研究ノート（師の遺志そのもの）
  • 弟との約束（卑怯者にならないための支え）

開始時は 1つか2つ です。継続PCや特別な単発では、GM確認のうえで3つ以上にしてもかまいません。明確な上限はありません。

ただし、命運点に数えられる大切なものは、自動的に全部採用されるとは限りません。次の条件を満たしていると通りやすくなります。

  • 本当に痛い: 失ったとき、そのPCにとって明確な損失や後悔になる
  • 個別である: ほかの大切なものと丸かぶりせず、別の危険として扱える
  • 危険にさらせる: 局面や登場人物との関係から、実際に揺らしたり脅かしたりできる
  • 卓で管理できる: 名前と意味が明確で、全員が把握できる

重複・抽象・量産は、そのままでは数えられないことがあります。たとえば「妹」「家族」「家族との絆」を全部別に取る、広すぎる理念を大量に並べる、替えの利く所持品を数で並べる、といった書き方は、統合や設定扱いになることがあります。

物語の中で新しい大切なものを得たら、GMの承認であとから増えることもあります。危険が見えてから急に保険として増やすことはできません。

5) 得意なやり方

そのPCがよく使う方法や発想です。

例:

  • 調査と準備
  • 交渉と根回し
  • 力仕事と現場指揮
  • 祈りと供養
  • 嘘と変装

6) 命運点

命運点の初期値は、GMが採用した大切なものの数 + 1 で始めます（ふつう 2〜3）。

記録は 現在/最大 として書くと管理しやすいです（例: 命運点 2/2）。

途中で新しい大切なものが正式に増えたなら、最大値や現在値が増えることがあります。

7) 野望度

1 で始めます。
終点の目安は 5 ですが、GMは開始時に野望の大きさに応じて終点を調整できます。
単発シナリオでは、終点 2 にすることがあります。

記入例

  • 名前: リゼ
  • 立場: 港町の見習い航海士
  • 野望: どの港でも通用する海図作りで名を残す
  • 大切なもの: 叔父の航海日誌（自分が海へ出る理由そのもの）
  • 得意なやり方: 調査と準備
  • 命運点: 2/2
  • 野望度: 1

自由記述で提出してよい

PCは、7項目を順番に埋めなくてもかまいません。たとえば「人間の盗賊ダリル。貧民街で育ち、老後のために一攫千金を夢見ている」のような短い文章でも十分です。

そのときは、何者か、なぜ今回動くか、何が得意か、何を失いたくないかがわかると整理しやすくなります。単発シナリオでは、長い人生目標ではなく、今回の話で達成したい目標を書いてください。

自由記述の出し方の例:
名前:
どんな人物か:
なぜ今回動くのか:
何が得意か:
何を失いたくないか:

自由記述で始める場合も、卓開始前には 得意な行動 / 初期恒久タグ / 絆タグ の3行が埋まっている状態へ整えます。ここが空いたまま始めると、Builds と Stories の初速が落ちます。

GM確認で確定する

GMは、あなたの自由記述を読んで、立場、今回の野望、大切なもの、得意なやり方を仮置きします。さらに、得意な行動 / 初期恒久タグ / 絆タグ も必ず仮置きします。その内容は、あなたが確認してから確定します。違うと思ったら、遠慮なく言い直してください。

単発シナリオでは、「老後資金を得たい」のような長期動機と、「今回のダンジョンから財宝を持ち帰る」のような今回の野望が分けて扱われることがあります。野望欄には、今回の目標が入ります。

GMの確認例:
立場: 貧民街育ちの盗賊
今回の野望: 黒灰迷宮の最奥財宝を持ち帰る
大切なもの候補: 貧民街の仲間たち
得意なやり方候補: 隠密と偵察
得意な行動: まず逃げ道を作ってから本題へ手を出す
初期恒久タグ: 〔逃走経路の勘｜長期: 退路確保を一宣言で扱いやすい / 地形不明の密室では鈍る〕
絆タグ: 〔貧民街の仲間たち｜長期: 路地・噂・貸し借りに関わる宣言で背中を押す / 裏切りや摘発で変質〕

「この理解でよいですか?」と返ってきたら、
合っているかを確認してください。

GM確認では、得意な行動 / 初期恒久タグ / 絆タグ の3行が埋まるまで開始しません。自由記述PCでも、この3行が最初の宣言の足場になります。

まず キャンペーン という長い道のりの土台が決まり、その中の最初の シナリオ が決まり、さらにその最初の 局面 が提示されます。

単発シナリオで遊ぶときは、その1本だけを短いキャンペーンとして扱います。
その場合、最初のシナリオがそのまま最後のシナリオになります。シナリオクリア（野望度が 2 になる）でキャンペーン終了です。

モードも最初に決める

QBS では、オブザーバー・モード、プロバビリティ・モード、クオンティティ・モード、リラックス・モード、ゼノス・モード、ソーシャル・モード、タクティカル・モード、アンロック・モード、バーサス・モード、ウィスパー・モード を必要に応じて追加できます。各モードは、宣言の前後や見え方を調整するためのもので、自由宣言そのものを置き換えるものではありません。

既定値は各節に従いますが、これは機械的な強制ではありません。シナリオの性質 と プレイヤーの希望 が優先されます。上級卓でも今回は切る、初級卓でもこの回だけ入れる、という運用で構いません。

どのモードでも、QBS の中心は変わりません。与えられた選択肢から選ぶ のではなく、あなたが何をしたいかを自由に書く のが先です。

リラックス・モードも最初に決める

これは、キャンペーン全体の終着を必須にせず、1本ごとのシナリオの問いにはきちんと答えを出しながら 長く遊ぶためのモードです。旅情、人情、後始末、取材、補給、工房整備、関係修復のような回を、野望度の大きな上下と切り離して回しやすくします。

既定は 初級ではあり、中級・上級・熟練ではなし です。ただし、これはあくまで出発点です。シナリオが回復や準備に向く、プレイヤーが長く遊びたい、単発の続きを自然に続けたい、といった理由があるなら、どの段階でも使って構いません。

このモードで緩めるのは キャンペーン全体の終点 だけです。各シナリオの問い、クリア条件、失敗条件、期限 はそのまま残ります。QBS の強みは「自由宣言で答えを変える」ことなので、1本ごとの答えは必ず出します。

バーサス・モードの有無も最初に合わせる

モードは「宣言の代用品」ではなく、宣言の前後を調整する

QBS のモードは、自由宣言を捨てるための別ゲームではありません。いつ宣言するか / 何が見えるか / 何で補うか / どこで揺らぐか / 長い進行をどう呼吸させるか を調整するための層です。

層モード変わること
宣言タイミングオブザーバー局面ごとの宣言を、シナリオ前の観測方針へ前倒しします。
解像度クオンティティ文章とタグで読み切れない余白を、能力値で補います。
揺らぎプロバビリティ宣言の質を読んだあとに、運の揺らぎを足します。
局面の見え方ソーシャル / タクティカル / アンロック / バーサス何を読ませる局面かを明確にします。
情報の配布ウィスパー個人別の秘匿情報と着地点を持たせます。
長編の呼吸リラックス1本ごとの答えは守ったまま、全体終点だけを急がない進行にします。

したがって、モードを増やしても、QBS の中心は変わりません。与えられた案の中から選ぶのではなく、自分のPCは何をしたいかを書くのが先です。

複数モードを重ねるときの順番

複数モードを併用するなら、基本順は次のとおりです。

ウィスパー配布 / 観測方針の確定
→ 局面ごとの可視情報（ソーシャル / タクティカル / アンロック / バーサス）
→ 宣言 or 既定宣言化
→ タグ / 履歴 / 関係 / 準備で読む
→ クオンティティで補う
→ プロバビリティで揺らぐ
→ Quiddities / Builds / Stories へ返す

リラックス・モード と ゼノス・モード だけは、この流れの外側で、シナリオの切れ目 に効きます。

PCどうしや陣営どうしが、同じ対象の同じ層の最終状態を排他的に奪い合う卓では、バーサス・モードを使います。これは自由宣言を捨てるためのものではなく、自由宣言どうしの競合を整列するための任意モードです。

既定は 初級 / 中級ではなし、上級 / 熟練ではあり です。ただし、シナリオの性質 と プレイヤーの希望 が優先されます。学園抗争、派閥選挙、証拠争奪、脱出路の奪い合いのように、排他的な最終状態 が卓の主軸になるときだけ入れると扱いやすくなります。

最初に合わせたいのは、同時提出方式、致死や恒久喪失の有無、他PCの内面を直接確定できるか の三つです。ここが曖昧だと、ルールの問題より先に卓の約束がぶれます。

単発卓では先に題材を合わせる

設定がまだ決まっていない単発卓では、この流れに入る前に、プレイヤーが「どんな題材で遊びたいか」を短く出し合います。GMはそれを1つの共通前提にまとめ、遊びたくない題材や苦手な描写も同時に確認します。

あなたが伝えるとよいのは、遊びたい題材、避けたい題材、苦手な描写、死亡許容度の希望、そして どれだけ初級者にやさしく / 熟練者が退屈しない歯ごたえを求めるか、どれだけ読み筋を見せてほしいか、どれだけ厳しく請求してほしいか です。

この段階で、指針提示度、裁定厳格度、運用強度、劇的展開度、死亡許容度、致死級タグ の有無、生還条件 を別に扱うかどうかも確認してかまいません。
五軸は独立ですが、既定は 中級者向け = 中級 / 適正 / 啓示 / 人情 / 中 です。厳しい卓かどうかは、途中で察するのではなく、先に聞いて大丈夫です。

とくに峻烈や悪辣を希望するときは、どの運用強度を目指すか と、どこまでを説明不要にするか も最初に確認してください。既定では、峻烈は初級相当まで、悪辣は中級相当まで を説明不要として進みます。悪辣 + 熟練運用なら、上級相当まで再授業しないこともあります。

1) GMが舞台と雰囲気を示す

GMは、世界観、PCたちの立場、雰囲気を先に示します。
事前に設定集がある卓なら、ここでその要点が共有されます。

単発卓では、舞台だけでなく、今回の共通目的、PC同士の関係、PCが知っていてよい常識 もここで共有されます。PC作成に必要な前提は、先に出してもらってかまいません。

2) プレイヤーがPCを作る

あなたは、その世界に合うPCを作ります。
とくに 野望 は大切です。あとでGMがキャンペーンを作る土台になるからです。

7項目で埋めても、自由記述で出してもかまいません。GMが仮置きした内容は、あなたが確認してから確定します。

3) GMがキャンペーンを決める

GMは、提出されたPCの野望を見て、そのPCが野望に届くまでの道のり を決めます。
これがキャンペーンです。あなたの野望が大きいほど、長いキャンペーンになることがあります。

4) GMが最初のシナリオと最初の局面を出す

GMは、キャンペーンの道中で最初に起こる事件、つまり最初のシナリオを用意します。
さらに、その事件を解くための最初の一歩として、導入と最初の局面を投稿します。

つまり、PCを作ること自体が、キャンペーンの設計に関わっています。
このゲームでは、PCの野望を書くことが、そのまま長い物語の種になります。

PBMでは、局面投稿とは別に「今回の共通前提まとめ」が固定投稿で置かれることがあります。迷ったら、まずそこへ戻って、舞台、共通目的、既知情報を確認してください。

物語の語り口も希望してよい

このゲームでは、GMが出すプロローグ、幕間、エピローグの語り口も希望してかまいません。指定がなければ、読みやすいラノベ風の会話劇が既定値になります。

ただし、語り口は雰囲気の話です。GMがあなたのPCの信条や秘密を勝手に決める、という意味ではありません。PCの内面は、あなたが書いた設定や、これまでの宣言から無理なく読める範囲だけが使われます。

自分のPCの口調、一人称、よく言う台詞、避けたいノリがあるなら、最初に一言書いておくと、かなり合わせやすくなります。

指定がないときの既定値

文体指定をしないときは、次のような語り口が既定値になります。

  • 文体: 日本の若者が読みやすいラノベ風。会話と短いモノローグが多め
  • 視点: PC寄りの三人称
  • 会話量: 多め
  • 福音 / 啓示: 読み筋を再掲しやすく、何を気にすべきかを拾いやすい
  • 寡黙 / 暗黒: 再掲が少なく、描写から読む比重が高い
  • 寛容 / 適正: 少し調子に乗れる軽快さが強め。いわゆる「また、俺何かやっちゃいました？」寄りの読み味になりやすい
  • 峻烈 / 悪辣: 「何で俺がこんな目に……」というぼやき、皮肉、冷笑、やせ我慢が少し強め。危ない場面ほど苦みのある軽口になりやすい

自分の好みがあるなら、あとからでも変えてかまいません。

6. ゲームの進み方

実際の進行では、局面を積み重ねて1本のシナリオを終え、それを重ねてキャンペーンを進めます。
PBMでは、基本的に次の順で進みます。

1) GMが局面を投稿する

GMが状況、期限、争点、既知情報、難度を示します。
選択肢例も、ふつうはここで示されます。福音 / 啓示の初心者卓では必ず添えられ、寡黙では短く、暗黒では出ないこともあります。

ただし、峻烈や悪辣では、局面の問いが「勝つ」「脱出する」そのものではなく、「戦う」「出口へ向かう」「橋を渡る」 のように一段浅く置かれることがあります。成功が保証されるのは、その局面で実際に問われた内容です。

複数PC卓では、GMは局面や結果を 共通部分 と 個別部分 に分けて提示してよいです。共通部分は全員が知っていてよい事実、個別部分はそのPCだけが知っている事実です。

2) プレイヤーが宣言を投稿する

あなたは、その局面でPCがどう動くかを書きます。
局面を一つ進めるたびに、シナリオの事件が少しずつ解決へ向かいます。

複数PC卓では、各プレイヤーが自分のPCについて独立に1つの宣言を出します。宣言前に相談してもよいですが、全員の合意を待つ必要はありません。まだ共有されていない仲間の行動や情報を前提にした連携は、保証されません。

PBMの複数PC卓では、GMが投稿期限を決めることがあります。期限までに宣言を出せないPCは、卓の開始時に共有された既定行動（待機、安全寄り行動、撤退優先など）で処理されることがあります。

3) GMが結果を投稿する

GMは、主目的の達成、クリア条件の進み具合、課題タグ、課題タグ更新、状況の変化を書きます。
その結果が、次の局面の土台になります。結果返答でも、差分だけではなく、更新後のPC状態・世界状態・タグ・履歴が毎回再掲されます。

複数PC卓でも、GMは複数のソロ卓を並べて処理するのではなく、1つの共有世界・1つの卓として裁定します。共通する出来事はまとめて確定し、そのうえで各PCの達成、代償、余波、個別情報を返します。プレイヤー同士の食い違いは、明示的な妨害がない限り、空欄や未定義と同じように扱われます。

4) 次の局面に進む

問題が残っていれば次の局面へ、答えが出たならシナリオ終了やエピローグへ進みます。
ただし、ひとつの局面で「見つけた」「奪った」「救い出した」からといって、そのままシナリオクリアとは限りません。帰還、退出、引き渡し、生還が残っているなら、そこまで処理して初めてクリアです。

シナリオが終わったら、キャンペーン全体では野望にどこまで近づいたかが見直され、必要なら次のシナリオが始まります。

7. 宣言の書き方

このゲームでは、宣言は次の書式で書きます。
慣れてきたら、補足や演出、具体的な手順や台詞 などを自由に書き足してかまいません（書式は崩れても大丈夫です）。

補助ツール: 宣言を組み立てるときは、プレイヤー用の補助ツールとして 宣言ツール も使えます。GMの局面文の整理、PC情報の参照、宣言テンプレートや宣言前確認の下書きを1画面で扱えます。

書式（基本形）

  • 主目的: 今回いちばん達成したいこと
  • 最低限ライン: ここまでは達成したいこと
  • 手段: 何をどう使うか。3つくらいまで
  • 守りたいもの: できれば傷つけたくないもの
  • 捨ててもよいもの: 代償として差し出してもよいもの
  • 退路 / 先に確保するもの (任意): 逃げ道、支援、証拠、治療役など、先に押さえる足がかり
  • 共有ボトルネックで担当するもの (任意): 同じ退路、同じ証人、同じ窓口、同じ鍵など、全員に効く一つの結果の担当
  • 伏線から読んだ危険への対策 (任意): 崩れそうな橋へのロープ、アンデッド対策の聖水、別出口の確認など、問題が顕在化する前に打つ備え
  • 担保 (任意): 自分から提案する代償や余波の候補
  • 最終局面での整理 (任意): 今回の主目的 / 同時に取りたい別利益 / 今回は捨ててもよいもの
  • 補足 (任意): 上級者はここに自由に書き足してよい（作戦の細部、使いたいNPC、狙い、台詞など）

全部を毎回埋める必要はありません。
ただし、空欄はGM一任 であり、どう読まれるかは主に 裁定厳格度 で決まります。少なくとも、空欄で得をすることはできません。

主目的はその局面でいちばん進めたいこと、最低限ラインは全部は無理でもここだけは守りたい線です。後始末や共有ボトルネックまで一言添えると、非ファンタジー題材でも宣言が立てやすくなります。

それぞれの意味

主目的

その局面で、いちばん判定してほしいことです。
1つだけ に絞ると遊びやすくなります。

最低限ライン

理想通りでなくても、ここまでは達成したい、という線です。
PBMではこれがあると、GMが結果を返しやすくなります。

例:

  • 「犯人の確保が理想だが、最低でも正体だけは確定したい」
  • 「全員救出が理想だが、最低でも子どもだけは避難させたい」

手段

使う方法です。
手段から自然に出る代償は受け入れやすくなります。

守りたいもの

今の局面で、とくに守りたいものです。
大切なものと重なっても構いません。

捨ててもよいもの

代償として差し出してもよいものです。
金、時間、体力、荷物、評判、借りなどをここに書けます。

ただし、ここへ書けるのはそのPCが自分で処分できるものに限ります。他のPCの大切なものや、勝手に差し出せない共有資源は、明示的な合意なしには書けません。

担保

「このくらいの代償なら受ける」と自分から提案する欄です。
ただし、これは 免責 ではありません。
担保を出しても、世界の反応として余波が広がることはあります。

また、担保はGMに「ここまでなら払う」という意思を伝える材料であって、無条件で卓越になる切符ではありません。 局面の問いに触れ、既知情報と矛盾しない宣言であることは必要です。

宣言のコツ

  • 長く書いてもよいが、主目的は1つ に絞る
  • 手順を5つ並べても、保証される中心は主目的1つだと考える
  • 「守りたいもの」と「捨ててもよいもの」を分けるとPCらしさが出る
  • 連携を強く拾ってほしいなら、合流、合図、役割分担、誰が何を知っているか まで書く
  • PC本人だけでなく、仲間、NPC、組織、町の手順 をどう動かすかまで書いてよい
  • 上級以上では、局面で成功したあと誰に観測され、何を監査され、どこを逆用されると困るか まで書けると強い

空欄や矛盾を残さない

全部を完璧に書く必要はありませんが、書いていない手段や守りたいものは、特別には拾われません。

このゲームでは、世界のすべてが最初から決まっているとは限りません。そのため、既知情報、PC設定、直前の展開に反しない範囲なら、局面判断に不要な余白を補う書き方をしてかまいません。

ただし、プレイヤーが決めてよいのは 周辺の補足 までです。GMしか知らない秘密、謎の正答、敵の真の弱点、攻略条件、今後の展開を決める内容は、宣言だけで確定しません。

空欄や未定義をどう読むか は、主に 裁定厳格度 で決まります。寛容は善意に拾われやすく、適正は通常処理、峻烈は安易に有利補完されにくく、悪辣は不利な合理読みに寄りやすいです。指針提示度 は、GMがその不足をどこまで丁寧に言語化してくれるかの差です。

選択肢をそのまま選ぶだけ の短い宣言は、GMから見ると 無工夫宣言 になりやすいです。とくに 寡黙 / 暗黒 や 峻烈 / 悪辣 では、次のどれかを1つでも足すだけで読み味が大きく変わります。

  • 何を守りたいか
  • 何を捨ててもよいか
  • どの退路や証拠を先に確保するか
  • 伏線から読んだ危険にどう対策するか

長く書いてもかまいませんが、保証される中心は主目的1つ です。無傷、退路確保、副目標、証拠回収まで一度に狙うほど、支払いは重くなりやすいと考えてください。最終局面で 生還 / 持ち帰り / 追手処理 / タグ解除 までまとめて書くと、別々に請求が乗りやすくなります。

最終局面では、補足に 「今回の主目的」「同時に取りたい別利益」「今回は捨ててもよいもの」 を書き足すと役に立ちます。これは欲張ってはいけないという意味ではなく、どれが主目的で、どれが別利益なのか を見える形にするための整理です。

迷ったら、少なくとも 何をしたいか / どうやるか / 何を守りたいか の3つを書いてください。さらに余裕があれば、何を捨てられるか / 何を先に確保するか / どこから帰るか まで足すと、GMが読み違えにくくなります。

複数PC卓での宣言

複数PC卓では、あなたは自分のPCの宣言だけを書きます。仲間と相談してもよいですが、全員の最終合意を待つ必要はありません。

ただし、フィクション上でまだ共有されていない情報、合流していない仲間の動き、通信できていない相手の準備を前提にした連携成果は保証されません。連携を強く拾ってほしいなら、合流、合図、役割分担、誰が何を知っているか を書いてください。

他のプレイヤーとの食い違いがあっても、明示的な妨害や敵対を書いていない限り、GMはまず両立する部分を優先して裁定します。相談していないこと自体が、失敗や妨害の理由にはなりません。

逆に、同じNPC、同じ物、同じ機会を取り合う宣言では、世界の結果は一つしか確定しません。あなたの行動が結果へ寄与しても、他PCと排他的に競合した主目的そのものは達成されないことがあります。

複数PC卓では、PC が増えるぶん GM も 同時進行の問題 や 外部からの干渉 を増やします。全員で同じ前進だけを重ねれば自動で楽になる、とは限りません。誰が何を押さえ、誰が共有ボトルネックを担当し、誰が後始末や生還線を受け持つかの役割分担そのものがゲームの一部です。

GM は、同じ NPC、同じ物、同じ退路、同じ窓口などを 共有ボトルネック として置くことがあります。誰かがそこを押さえると別PCの宣言が通りやすくなり、放置すると全員のどこかに穴が残ります。複数PC卓を「並んだソロ卓」ではなく「一つの卓」として遊ぶための仕掛けだと考えてください。

ウィスパー・モード が有効な卓では、あなたのPCだけが知っている事情や、あなたのPCだけが強く望む着地点があるかもしれません。そのときも、まずは 自分のPCが何をしたいか を書けば十分です。秘密の中身を毎回全部明かす必要はありません。

ただし、まだ共有していない事実を他PCが当然知っている前提で連携を書くことはできません。逆に、相談していないこと自体が失敗や妨害に読み替えられることもありません。同じ対象の排他的な終着点 を取り合うときだけ、GM が競合として扱います。

宣言前確認

GMが選択肢例を出しているなら、まずその中から近いものを選んでも構いません。福音 / 啓示の初心者卓では、選択肢例が出るのが基本です。寡黙では短く、暗黒では無いことがあります。

細部だけ迷うときは、補足に「Aならこう、Bならこう」と条件分岐を書く ほうが向いています。

ただし、方針そのものが決められない ときに限り、単独卓では1局面につき1回、3点まで、複数PC卓では各PCが各局面につき1回、3点まで の「宣言前確認」をしてよいです。
質問は短い事実確認だけにしてください。導入局面では、GMが全PC分をまとめて返すこともあります。

【宣言前確認】
1.
2.
3.

GMの返答は、次の4種類です。

  • 【回答】: 説明不足だった前提です。今日の日付、距離、同行者、その場で見えているもの、期限の数え方など。
    この確認では時間もコストも発生しません。
  • 【既知】: PCなら知っていて当然の情報です。土地の常識、所属の慣習、職能でわかる基礎知識など。
    この確認では時間もコストも発生しません。
  • 【要宣言】: 今はまだわからない情報です。犯人、黒幕、未調査の事情、相手の本音など。
    GMは答えず、「調査・聞き込み・観察などの宣言で確かめてください」と返します。
  • 【却下】: 成功率、最適解、GMの予定、拒否された秘密の聞き直し、ゲーム中の「本当はどうすればよかったか」には答えません。

例:

  • 「今日は何日ですか」 → 【回答】
  • 「この国では、夜の外出許可証は一般常識ですか」 → 【既知】
  • 「今見つけた密室殺人の犯人は誰ですか」 → 【要宣言】
  • 「A案とB案のどちらが成功しやすいですか」 → 【却下】
  • 「あの時どうすればよかったのですか」 → 【却下】

同じ局面で確認を繰り返すのは原則できません。
善意の確認で不利益を受けることはありませんが、答えられないことは宣言で確かめてください。

宣言前確認は代償の値引き制度ではありません。
確認で軽くなるのは、距離、期限、観測条件、退路の有無などを読み違えた事故です。別利益の同時達成や、そのシナリオで元から要求されている支払いまでは軽くなりません。

複数PC卓では、各PCの確認回数は独立です。共通前提に関わる確認はGMが一括で共有し、個別事情に関わる確認はそのPCにだけ返してよいです。導入局面では、GMが全PC分をまとめて返すこともあります。他のプレイヤーの確認待ちだけを理由に、自分の宣言が必ず止まるわけではありません。

事実確認はしてよいですが、最適解相談はできません。 セッション終了後なら、GMが当時のメモを見せて振り返ってくれることがあります。

高難度卓では、質問の中に手順や勝ち筋を入れすぎると 【却下】 や 【要宣言】 になりやすいです。
「X に偽装すれば Y を通せますか」より、「現場に X はあるか」「Y は何を見て通すか」「ID 照合はいつ走るか」のように、事実へ分けて聞く ほうが通ります。

GM は事実を返しますが、「余地は十分ある」「それを逆用すれば抜けられる」のような半歩先の助言は、高難度卓では返しません。そこから先を組み立てるのが宣言です。

宣言前確認は答え合わせではなく、宣言を書くための観測です。 仕様、距離、可視情報、既知制度は聞いてよいですが、勝ち筋の優先順位は自分で組むものとして扱います。

良い確認:

  • いま見えている物理状態は？
  • この対象は可搬か据置か？
  • 距離、時間、監視周期は？
  • 現時点で公開済みの仕様は？

悪い確認:

  • どれが一番いいですか？
  • 何を先に片づけるべきですか？
  • 後半で詰まない方法は？
  • これで安全に生還できますか？

高難度卓では、「何を見れば判断できるか」を聞く方向へ分解すると通りやすくなります。

宣言の例

主目的: 吹雪の前に、峠の避難路を確保する
最低限ライン: 子どもと老人だけでも山小屋へ移せる状態にする
手段: 荷運び仲間を集め、荷車を分解して簡易そりにし、夜明け前に道しるべを立てる
守りたいもの: 弟の安全
捨ててもよいもの: 荷物と体力
担保: 【疲労｜短期: 次の局面では無理が利かない / 半日休めば軽減】 は受ける

許可・値引き・保険・変換で読む

能力タグや装備タグを見たときは、「強い / 弱い」よりも、このタグは何をしてくれるのか を4分類で読むと判断が速くなります。

  • 許可: ふつうは別工程になりやすいことを、一宣言で扱いやすくする
  • 値引き: 必要担保や、捨てる別利益を軽くする
  • 保険: 余波の形を軽くしたり、発火を遅らせたりする
  • 変換: 成果や代償を素材・手札・ドラマフラグへ変える

実戦では、「このタグがあるからできる」より「このタグがあるから安く済む」 のほうが頻繁に効きます。

Build根拠は二つまでと考える

強い宣言は、材料をたくさん並べることではなく、主処理に効く根拠を絞る ことです。
QBS の Builds はメニューではなく値段の差なので、1回の宣言で主に数える Build 根拠は 2つまで と考えると読みやすくなります。

三つ目以降は、演出や状況説明として書いてかまいません。ただし、値引きや保険の主根拠として強く数えるのは二つまで、という感覚で組むと、「何が本当に効いているのか」 が見えやすくなります。

誰の心をどう動かしたいかを書く

Stories を強く回したいときは、主目的を「説得する」「止める」で止めず、相手にどんな変化を起こしたいのか を書きます。

今回動かしたい相手:
その相手にどう感じてほしいか / どう決めてほしいか:
そのために見せるもの・差し出すもの:
守りたい関係:
壊れてもよいもの:

この書き方をすると、GMは結果を 関係タグ・感情タグ・ドラマフラグ として残しやすくなります。

シーン種別を使うと、白紙負荷が減る

GMが場面を 導入 / 親密 / 対立 / 行動 / 余韻 のどれとして出しているかが見えると、書き方の迷いが減ります。

  • 親密 なら、告白、相談、回復、距離の縮まりを書く
  • 対立 なら、言い分の衝突、秘密の火種、立場の揺らぎを書く
  • 行動 なら、主目的と最低限ラインを明快にする
  • 余韻 なら、残った痛みや、言えなかったことを次へ残す

選択肢メニューに戻る必要はありません。いま何を食べる場面なのか が見えるだけで十分です。

8. オブザーバー・モード

オブザーバー・モードでは、プレイヤーは 局面ごとの宣言を行いません。 代わりに、PC作成後またはシナリオ開始時に、その PC がどういう人物として動くかをまとめた 観測方針 を提出します。以後、GM はその観測方針・既存タグ・履歴・関係・世界状態に従って、その局面の 既定宣言 を作って裁定します。

これは自由宣言の放棄ではありません。「その場で宣言する」代わりに「どういう人物として動くかを先に宣言しておく」 モードです。

観測方針

観測者 PC は、通常の PC 記入欄に加えて、次を持ちます。

【観測方針】
最優先:
取りにいくもの:
平時の第一手:
危機時の第一手:
捨ててもよいもの:
禁止事項:
退路基準:
  • 最優先: 最後まで守ろうとするもの。1〜2個。
  • 取りにいくもの: 積極的に得ようとするもの。1〜2個。
  • 平時の第一手: 情報不足のとき、まず何をするか。
  • 危機時の第一手: 危険が見えたとき、まず何をするか。
  • 捨ててもよいもの: 代償にしてよいもの。1〜2個。
  • 禁止事項: どれだけ得でもしないこと。1個。
  • 退路基準: どの線を越えたら撤退・離脱・見切りを選ぶか。

基本処理

  1. GM は局面文を通常どおり提示します。
  2. 観測者 PC のプレイヤーは、局面ごとの宣言を返しません。
  3. GM は、観測方針・既存タグ・履歴・関係・現在の世界状態から、その局面でもっとも自然な 既定宣言 を1つ作ります。
  4. 既定宣言は、内部的には通常の宣言と同じく、主目的 / 最低限ライン / 手段 / 守りたいもの / 捨ててもよいもの を持つものとして扱います。
  5. GM はその既定宣言を裁定し、結果を返します。

GM の結果で見るところ

観測者 PC への返答では、GM は結果の中に次の 3 点を明記します。

1. 今回 PC が取った行動
2. その根拠になった観測方針・タグ・履歴
3. 生じた結果、タグ、履歴、関係の変化

プレイヤーが見るべきなのは、自分の PC がどう動いたか と、次に方針を書き換えるべき変化が起きたか の二つです。

GM がしてはいけないこと

GM は、その PC がまだ得ていない情報を使って 最適化 してはいけません。行動選択に使ってよいのは、その PC が知っていること / 持っているもの / 観測方針に書かれていること だけです。

複数の行動が同程度に自然なら、禁止事項に触れない → 退路基準を守る → 最優先を守る → 平時 / 危機時の第一手に従う → 不可逆損失がより小さい の順で選びます。

既定宣言の強さ

  • 観測方針だけ から生じた既定宣言は、原則として 妥当 までです。
  • 既出の準備、退路、支援、証拠確保、手札タグがそのまま噛み合うなら、周到 まで上げて構いません。
  • 卓越 は、事前に明文化された長期仕込みや準備が現在局面に強く噛み合ったときだけです。

観測更新

観測方針は、原則として シナリオ開始時と終了時 にだけ書き換えてよいものとします。長期キャンペーンでは、各シナリオの合間に更新して構いません。観測者でも、命運点、履歴、手札タグ、関係・通信タグ、失ったものは通常どおり変化します。

【オブザーバー quick ref】
観測方針を書き、局面ごとの宣言は GM が既定宣言化する
GM は未取得情報で最適化しない
既定宣言の質は 原則: 妥当 / 噛み合いで: 周到 / 長期仕込みで: 卓越
観測更新は シナリオ開始時・終了時（長期では幕間も可）

9. プロバビリティ・モード

このモードでは、自由宣言の読み取りに加えて、運の揺らぎ を判定に入れます。QBS の基本はあくまで宣言読解です。したがって、出目が良くても薄い宣言は伸びにくく、出目が悪くても深い宣言は壊れにくくします。

いつ振るか

主目的が局面の中心問題に触れていて、かつ 偶然・競合・不確実性 があるなら振ります。既知確認、自明な移動、安全な準備だけの行動、成否より記述だけが重要な小動作は、振らなくて構いません。原則は 1宣言につき1回 です。

達成値 = 2d6 + 宣言品質修正 + タグ修正 + その他修正

品質修正見るところ

有利な手札タグ・恒久タグ・準備・装備: 1つにつき +1（最大 +2）

品質による上限と下限

  • 無策 の宣言は、どれだけ出目が良くても、結果の上限を 成功 とします。鮮烈成功にはなりません。
  • 周到 または 卓越 の宣言は、どれだけ出目が悪くても、結果の下限を 失敗 とします。破綻失敗にはしません。

10. クオンティティ・モード

このモードでは、自然言語で書かれた PC 記述に加えて、一般的な RPG 風の 能力値 を持たせます。ただし、能力値は主役ではありません。タグ、履歴、関係、準備、世界状態で直接読めることを、能力値で上書きしてはいけません。

標準能力値

標準能力値は 筋力 / 器用 / 知力 / 魅力 / 意志 です。世界観に合わせて、体 / 技 / 知 / 社 / 志 へ言い換えて構いません。

  • 筋力: 力、耐久、踏ん張り、重作業、肉体的強行
  • 器用: 速さ、精密さ、潜入、操作、身のこなし
  • 知力: 分析、記憶、知識、工作、読解
  • 魅力: 説得、交渉、演技、信用、場の支配
  • 意志: 勇気、集中、信念、平静、誘惑や恐怖への抵抗

初期値

PC を通常どおり作成したあと、5 つの能力値に 4, 3, 3, 2, 2 を 1つずつ割り振ります。4 は明確な得意分野、2 は明確な不得意分野です。4 と 2 には、それぞれ根拠になる一文を添えます。

器用4: 屋根伝いの移動と潜入に慣れている
意志2: 仲間を切る判断が遅れる

基本処理

  1. まず タグ / 履歴 / 関係 / 準備 / 世界状態 を読みます。
  2. それだけで結果が十分に読めるなら、能力値は使いません。
  3. まだ余白があるときだけ、主能力を1つ決めて参照します。

プレイヤーは宣言末尾に 主能力: 器用 のように添えて構いません。ただし、それだけで有利になることはなく、最終的に何を参照するかは GM が決めます。

クオンティティだけを使うとき

  • 主能力が 4 なら、難度を 1段階軽く してよい
  • 主能力が 2 なら、難度を 1段階重く してよい
  • 主能力が 3 なら、変化なし

この補正は、タグや準備で直接読める部分には使いません。 あくまで GM がまだ迷う余白だけに使います。

プロバビリティと併用するとき

プロバビリティと併用するなら、能力値は達成値修正に読み替えます。

能力修正 = 能力値 - 3
4 → +1 / 3 → 0 / 2 → -1

優先順位

文章で書かれたタグ
> 履歴・関係・準備・世界状態
> 能力値

能力値だけで不可能を可能にはしません。逆に、能力値が低くても、十分な準備と筋の通った宣言があるなら通します。

能力値の変化

能力値は原則として 固定 です。QBS における成長の主役は、あくまでタグ、履歴、関係、拠点、仲間です。ただし、再訓練 / 長期療養や後遺症 / 身体改造 / 身分や立場の大きな変化 / 信念や覚悟の決定的変化 があったときは、シナリオ終了時に 1 点だけ入れ替えて構いません。

【クオンティティ quick ref】
能力値: 筋力 / 器用 / 知力 / 魅力 / 意志
初期値: 4,3,3,2,2 を1つずつ配る
単独運用: 4なら難度1段階軽減 / 2なら1段階増加 / 3なら変化なし
プロバビリティ併用: 4なら+1 / 3なら0 / 2なら-1
優先順位: タグ > 履歴・関係・準備 > 能力値

11. リラックス・モード

何をするモードか

標準のキャンペーンでは、シナリオの積み重ねが最終的に野望の終点や最終局面へ向かいます。リラックス・モードでは、その直線を弱めます。今回の事件を解き、今回の問いに答え、次はどこへ行くかをそのつど選ぶ 運用に寄せます。

  • 残すもの: シナリオの問い / クリア条件 / 失敗条件 / 期限 / 終了処理
  • 弱めるもの: 毎回野望度を動かす圧、最終決着へ一直線に進む圧
  • 増やすもの: 次回候補の選択、休養や仕込みの回、世界課題タグの細かな回収

「終わらない一つの問題」をだらだら引き延ばすのではありません。終わる単位を小さく保ったまま、次の行き先をプレイヤーが選び続ける のが核心です。

次回候補へどう希望を出すか

シナリオ終了時には、GM が次回候補を 2〜4本 出すのが基本です。あなたはその中から選んでもよいですし、同じ重さの別案 を提案してもかまいません。

このときは、次の三つを書くと希望が通りやすくなります。

  • 今いちばん触りたいこと: 人、場所、問題、勢力、後始末
  • 次に仕込みたいもの: 協力者、退路、補給線、工房、証拠、関係修復
  • 今回は避けたい重さ: 連戦、怨情の激化、致死級危険、政治抗争など

候補を「正解当てクイズ」と思う必要はありません。このPCなら今どこへ向かいたいか を短く出せば十分です。

次に行きたい方向:
- 触りたいもの:
- 仕込みたいもの:
- 今回は避けたい重さ:
- もし候補外なら、やってみたい別案:

野望度が動かない回の読み方

このモードでは、野望度が動かない回 = 無意味な回 ではありません。野望度は人生の節目に使い、日常の充実や小さな達成は別の場所で受け取ります。

その回で主に進むものよくある内容主な返り
Quiddities聞き込み、調査、後始末、土地勘づくり、世界課題の回収成長点、世界状態の改善、秘密段階の前進
Builds補給、工作、工房整備、拠点開設、協力者の固定素材、Build 素片、長期資産ノード
Stories和解、謝罪、再会、里帰り、同行、看病、祭礼絆の進展、感情の変質、ドラマフラグ、再登場線

成長点は地道な研鑽、野望度は人生の到達度 と切り分けて読むと、このモードでも停滞しません。

回復回でも何かを残す

休む回、遊ぶ回、取材する回、里帰りする回でも、次の宣言が少し楽になるもの を一つ残すと、リラックス・モード運用が締まります。

  • 安全な宿や隠れ家を押さえる
  • 道案内や仲介役を得る
  • 補給線や連絡網を整える
  • 謝罪や対話で絆の壊れ方を変える
  • あとで使える小さな証拠や素材を持ち帰る

逆に、何も変えずに全快だけする 回を続けると、QBS らしい「宣言が残る」感触が薄くなります。穏やかな回でも、何が残ったかは一行でよいので意識してください。

11A. ゼノス・モード

何をするモードか

ゼノス・モードでは、大きな野望の達成や差し迫った任務の成功よりも、未知の土地、異文化、奇習、祭り、風景、驚きに触れる旅 を前面へ出します。QBS の基本は変わらず、宣言は局面単位で行い、世界は反応し、代償や余波も起きます。ただし、重く見るのは 何を攻略したか より 何に出会い、誰と交流し、何を持ち帰ったか です。

リラックスとの違い

リラックス・モード は、野望は残したまま歩幅をゆるめるモードです。ゼノス・モード は、野望そのものが薄くても、異境への引力で旅を続けられるモードです。回復や後始末が中心ならリラックス、驚きや越境や交流が中心ならゼノス、と考えると読み分けやすくなります。

このモードでは、交流失敗、作法違い、祭りへの遅刻、珍品の買い損ねをそのままシナリオ失敗へ直結させません。誤解、借り、別の案内人、思わぬ招待、新しい寄り道 へ枝分かれさせるほうが、QBS らしい旅の感触が残ります。

旅の収穫の読み方

このモードでの収穫は、新事実 / 新しい縁 / 新しい手札 / 新しい余韻 のどれかです。野望度が毎回動かなくても、回復点 / 畏敬点 / Build 素片 / 関係の変化 / 持ち帰り品 / 世界の見え方の変化 が残るなら、その回は十分に前進しています。

野望も「成り上がり」だけではありません。世界のさまざまな場所を見て回る、平穏な日常を守る、困っている人々を助ける のようなものでも構いません。ゼノス・モードでは、まだ何者になりたいか決めきれていないが、まず世界を歩いて確かめたい という薄い野望から始めてもよいです。

12. ソーシャル・モード

ソーシャル・モードは、相手の内面を当てるゲーム ではありません。表層サインから、非開示情報と協力可能線を推論する モードです。QBS では、他 PC の内面を直接決めることはできませんし、「本音そのもの」を事実確認で聞くこともできません。したがって、このモードの成果は、情報開示 / 虚偽の揺らぎ / 条件付き協力 / 判断保留 / 一時的譲歩 / 関係タグや感情タグの変化 として返します。

局面で公開される情報

【局面主目的】

【表層サイン】
- 言い淀み
- 話題回避
- 過剰反応
- 矛盾
- 強調しすぎる点
- 誰の前では言い方が変わるか

【共有圧】
- 放置すると黙秘が固まる
- 面子コストが上がる
- 口裏合わせが進む
- 退席や通報が近づく
- 第三者が来る

【次防御予告】
- 論点ずらし
- 虚偽の補強
- 逆質問
- 退席
- 第三者を盾にする
- 条件闘争へ持ち込む

【残っている共有状態】
- 二人きり
- 安全保証
- 面子の退路
- 証拠突合せ
- 第三者退席
- 限定協力の土台

タクティカル・モードでの「誰がどこにいるか」に相当するのが、ソーシャル・モードでは 表層サイン です。心理そのものは見えないので、ここを出さないとプレイヤーが読む材料がありません。

宣言補足の型

既存の 「今回動かしたい相手 / どう感じてほしいか・どう決めてほしいか / 見せるもの / 守りたい関係 / 壊れてもよいもの」 を、そのまま社会局面用に使えます。ソーシャル・モードでは、次の形にすると読みやすくなります。

【ソーシャル補足】
今回動かしたい相手:
引き出したい非開示情報 / 変えたい判断:
そのために見せるもの・差し出すもの:
どの共有圧を引き受けるか:
目指す落としどころ:

返りとして狙うもの

  • 情報の開示: 全面自白ではなくても、輪郭や条件や当事者性が見える
  • 虚偽の揺らぎ: 嘘の補強に無理が出る、言い分が変わる、次防御が一段重くなる
  • 条件付き協力: 全面協力ではなくても、その場限りの提案が届く
  • 判断の保留 / 一時的譲歩: 即時拒否が消え、別条件の交渉窓が生える
  • 関係変化: 絆タグ / 感情タグ / ドラマフラグが動く

つまり、このモードで狙うのは「改心させる」ではなく、言い分を変えざるを得ない状態を作る、この場ではこちらに乗る理由を作る、本心を隠しきれない圧を作る です。

読みやすい共有状態の例

【二人きり｜許可: 建前処理と本音探りを一宣言で扱える / 露見すると【密談疑惑】】
【面子の退路｜値引き: 相手が前言撤回・方針転換するときの支払いを1段軽くする】
【証拠突合せ｜保険: 虚偽の補強や口裏合わせの顕在化を1段遅らせる】
【限定協力案｜変換: 全面自白を〔条件付き開示〕〔一時協力〕〔沈黙の合意〕へ変換できる】
【安全保証｜許可: 告白後の報復不安を抱えた相手にも、限定開示の提案が届く】

やってはいけないこと

内面そのものを確定させない のが、このモードの一番大事な制限です。「改心した」「恐怖した」「罪悪感に屈した」のような断定を主目的にしないでください。QBS らしい強さは、内面断定ではなく 共有状態を変えること にあります。

13. タクティカル・モード

タクティカル・モードは、戦闘や難所にだけ 位置取り、共有ボトルネック、ラウンド圧、敵や環境の次の手 を見える化する追加です。自由宣言をコマンド選択へ戻すためのルールではありません。

何が増えるか

  • 位置関係: 誰がどこにいて、どこへ行けるか
  • 共有圧: 放置すると全員のどこかへしわ寄せが出る問題
  • 敵意図 / 環境意図: 次に何が起こりそうか
  • 共有状態: 退路、遮蔽、時間稼ぎ、搬送線など、誰かが作ると味方も使える足場

増えるのは 読む材料 であって、選べる行動が固定されるわけではありません。

まず何を読むか

最初に読むべきなのは、どの圧を誰が受け持つか です。個人最適だけでなく、誰が共有ボトルネックを押さえると他PCが楽になるか を見ると、自由宣言の質が一段上がります。

次に、敵や環境が次に何をしようとしているか を見ます。勝つことそのものより、いま止めるべきものを止める ほうが強い局面が多いです。

宣言に一行添える

タクティカル・モードが有効な局面では、いつもの宣言に次の一行を足すだけでも十分です。

戦術補足:
- どの共有圧を受け持つか
- 誰の退路や生還線を支えるか
- 何を次ラウンドへ残したいか

これは書式の強制ではありません。連携の芯 を明文化するための補助です。

14. アンロック・モード

アンロック・モードは、高難度シナリオ設計の縮図を、1局面サイズへ圧縮したもの です。鍵開けに限らず、儀式停止、隔離解除、爆発物処理、手順解体、医療工程、設備再起動のように、構造そのもの を読む場面で使います。

このモードでは、「正解そのもの」を言い当てるより、何を根拠にその読みを採ったか が大事です。QBS の自由宣言を崩さずに、構造読解 を前面へ出すためのモードだと考えてください。

局面で公開される情報

【局面主目的】

【可視構成要素 / 現在状態】
- 触れられる部位
- 見えている接続
- 現在の作動状態
- 露出した層
- 欠けている情報

【既知の物理条件】
- これを動かすと何が連動するか
- これは外せるか据置か
- 何が可逆で何が不可逆か
- 時間 / 熱 / 圧 / 通電 / 封印状態

【構造圧】
- 放置で進行するもの
- 誤操作蓄積
- 加熱
- 再封印
- 起爆猶予減少
- 汚染進行

【次の構造反応】
- 再ロック
- 別系統の起動
- 隔壁閉鎖
- 警報
- 連動先への波及
- 即死級工程の露出

【残っている共有安全状態】
- 仮固定
- 一時停止
- 隔離
- 試験端子確保
- 別系統遮断
- 緊急排出路

宣言補足の型

【アンロック補足】
今回触る工程:
その判断根拠にした観測事実:
読みの前提:
先に作る安全状態:
切ってよいもの / 捨てる別利益:

「正解そのもの」ではなく、何を観測したのでそう読んだか を書くと、構造読解として筋が通ります。

宣言前確認の例外

アンロック・モードだけは、宣言前確認を少し厚くして構いません。ただし、それでも 答え合わせ や 値引き制度 ではなく、あくまで 観測 です。

したがって、このモードでは 同一工程・同一対象 に限り、1回の宣言前確認で複数の原子的事実を束ねて聞いてよい とします。目安は 現状態 / 連動先 / 既知の物理固定条件 の 3 点までです。

ただし、次のことは引き続き聞けません。

  • どの順が正解か
  • 何を先にすべきか
  • これで安全に生還できるか

結果の見方

アンロック・モードでは、読み違えたが前進はした を構造カウンターで返し、触ってはいけない失着を踏んだ を最速敗着線で返します。したがって、同じ「うまくいかなかった」でも、後から返るのか / その場で崩れるのか を見分けるのが重要です。

見やすい共有安全状態の例

【試験端子｜許可: 観測と仮説検証を一宣言で扱える / 誤接続で【構造圧+1】】
【仮固定具｜保険: 誤操作の即時致命化を1段遅らせる】
【対応表の断片｜値引き: 手順特定のための追加支払いを1段軽くする】
【バイパス回路｜変換: 完全解除を〔一時停止〕〔隔離〕〔手動安全化〕へ変換できる】
【爆風隔壁｜保険: 失着を即死から【局面崩壊】や【証拠損壊】へずらす】

15. バーサス・モード

何をするモードか

バーサス・モードは、同じ対象の同じ層の最終状態を、PCどうしや陣営どうしで奪い合うときに使います。たとえば、同じNPCを「捕まえたい / 逃がしたい」、同じ証拠を「確保したい / 隠したい」、同じ退路を「押さえたい / 通したい」といった場面です。

大事なのは、ここでもなお 自由宣言のゲームである ことです。コマンドを選ぶのではなく、主目的 と 手段 を文章で書き、その結果をGMが一つの共有世界へ畳みます。詳しい競合解決手順や中間臨界状態表は、GM側の運用として GMガイド にまとめられています。プレイヤーとしては、今回どの層を争うか、何を守り、何を捨てるか を明確に書ければ十分です。

主に争う層は四つに分ける

層何を争うか置きやすい中間状態
身体拘束、負傷、行動不能、とどめ体勢不利 / 拘束 / 行動不能 / 瀕死
位置接敵、遮断、離脱、包囲、退路確保捕捉 / 退路半壊 / 遮断 / 包囲
物品奪取、保持、破損、証拠確保所在露見 / 争奪中 / 破損 / 機能停止
社会疑惑、失脚、支持、評判、審問停止注目 / 疑惑 / 審問停止 / 失脚寸前

バーサスでは、強く争う層を1宣言につき1つだけに絞ると安定します。ほかの層に触れる内容は、演出、別利益、余波、次局面の争点へ回します。

宣言では何を書くか

主目的は1つ、強く争う層も1つが基本です。たとえば「相手を拘束しつつ、証拠を奪い、世論も落とす」は一文では書けても、標準価格では全部は通りません。

【バーサス宣言】
主目的:
対象:
争う層: 身体 / 位置 / 物品 / 社会 から1つ
手段:
守りたいもの:
捨ててもよいもの:
別利益(任意):
今回あえて取らないもの(任意):
使うタグ / 資産 / 共有状態(任意):

バーサス局面では、ふだんの宣言に 対象 と 争う層 を明示すると読みやすくなります。とくにPBMでは、同時提出 を前提にしたほうが投稿順の有利不利が減ります。

GM返答は三段で読む

GMの返答では、まず 共通成果 を見ます。追い詰めた、退路候補を一つ潰した、証拠の所在を露見させた、疑惑を公にした、といった部分です。ここは、競り負けても残ることがあります。

次に 排他的な最終状態 を見ます。拘束されたのか、離脱を許したのか、証拠を持ち去ったのか、失脚させたのか、という一つしか立たない結果です。

最後に、決着が持ち越されたなら 中間臨界状態 を見ます。ここが次局面の争点です。QBSのバーサスは、即断の勝敗よりも、まだどちらにも転びうる臨界点 を作ったほうが面白くなりやすいです。

やってはいけない書き方

  • 他PCの内面を直接確定する — 「恐怖した」「改心した」「恋に落ちた」は、宣言だけでは決まりません。
  • 合意のない恒久喪失へ一気に飛ぶ — 致死や大切なものの喪失は、卓宣誓で許可された範囲で扱います。
  • 一宣言で複数層を強く取り切る — 身体 / 位置 / 物品 / 社会 の全部を一気に主処理へ入れないでください。
  • 勝ったら無傷だと思う — 勝者にも、注目、悪評、消耗、監視、後始末は残ります。

16. ウィスパー・モード

ウィスパー・モードは、複数PC卓に 秘匿情報 と 個人別利益 を足す追加です。完全対立ではなく、共有主目的は同じだが、望む着地点が少しずつ違う 卓を作るために使います。

何をするモードか

このモードでずらすのは、何を守りやすいか と 何を捨ててもよいと思いやすいか です。行動メニューを配るのではありません。

既定は 初級 / 中級ではなし、上級 / 熟練ではあり です。ただし、シナリオの難しさよりも 参加者が遊びたい形 を優先してください。半協力をしたくない卓に無理に入れる必要はありません。

ウィスパー・シートは「優先順位カード」として読む

良いウィスパー・シートは、「Aをしろ / Bをしろ」と命令しません。代わりに、次の形で そのPCが動きやすい方向 を渡します。

【秘匿ウィスパー】
秘匿事実:
個人別利益:
この秘密ゆえに守りたいもの:
この秘密ゆえに捨ててもよいもの:
伏せたままでもよい線:
露見しうる経路:
公開化したときの更新候補:

個人別利益 は、共有主目的の代わりではありません。たとえば「次の被害を止める」は共有主目的で、「あのNPCだけは生かしたい」「証拠の出し方は自分の陣営に寄せたい」が個人別利益です。

宣言では何を書くか

秘匿があっても、宣言の骨格は変わりません。

主目的:
手段:
守りたいもの:
捨ててもよいもの:
いまは伏せる補足(任意):

秘密の中身を全部明かさなくても、守りたいもの / 捨ててもよいもの を書けば、かなりの情報が伝わります。ウィスパー・シートは、宣言そのものよりも 支払いの向き を変えるものだと考えると扱いやすいです。

秘密はいつ共有するか

個別情報は、合流 / 通信確立 / 公開描写 / 共有宣言 のどれかがあるまで、他PCの宣言根拠にはできません。逆に、まだ共有していないこと自体を理由に、GM が勝手に妨害や失敗へ読み替えることもありません。

秘密が露見したら、その事実は共通世界へ上がります。以後は他PCもその情報を使えますし、関係、世界課題タグ、履歴 のどれかへ変換されることが多いです。

競合はどこで起きるか

半協力卓で本当に競合になるのは、同じ対象の排他的な終着点 を取り合ったときです。たとえば「同じNPCを捕縛したい / 逃がしたい」「同じ証拠を提出したい / 隠したい」のような場合です。

相談していないこと、動機が少しずれていることだけでは競合になりません。QBS の基本どおり、GM はまず 両立する部分 を優先して裁定します。

成長点の返り

ウィスパー・モードでは、共有主目的を達成したうえで 個人別利益まで通したPCに、追加の +1 候補 がつくことがあります。ただしこれは新しい別通貨ではなく、既存の 追加で +1（最大2回まで） の中に入ります。

共有主目的が失敗した場合、個人別利益だけを通しても成長点ボーナスにはなりません。その代わり、関係の進展、感情の変質、秘密開示段階の前進 として返ることがあります。

17. 難度5段階の見方

実際の運用では、普通 と 困難 が中心です。
容易 はGMが与える息継ぎの局面、過酷 と 絶望 は山場だと考えるとわかりやすくなります。

難度 感触 ふつう起こること 卓越宣言のボーナス
容易 立て直しや整え直し 回復や整理はできるが、軽い余波が残ることがある 余波も起こらない
普通 基準の難しさ 主目的は達成。代償タグがつく 代償タグではなく軽い余波タグだけになる
困難 かなり厳しい 主目的は達成。代償タグ + 余波タグ 代償タグだけで済む
過酷 前進するが理想形ではない 意味のある達成はするが、結果の質や範囲が少し欠ける。代償タグ + 余波タグ 主目的が完全達成になる
絶望 前進するが、いちばん痛いところを取りに来る かなり重い代償タグ + 余波タグ。成果は残るが、嫌な支払いが来る 代償が「重いが妥当」な線まで下がる

容易

容易は、ふつう GMが与える回復局面 です。
プレイヤーが好きなタイミングで要求するものではありません。

ただし、回復は容易でしかできないわけではありません。
実際のプレイでは、普通や困難の宣言の中に休息、治療、手当て、引き継ぎ、負担の組み替えを組み込んで、少しずつタグを軽くしていく ことが重要です。

過酷

過酷は、結果の質や範囲が少し落ちる難度 です。
完全には届かないけれど、意味のある前進は残ります。

例:

  • 連れ去られた人は救えたが、黒幕の証拠は焼けた
  • 灯台の怪異は静まったが、港の操業はしばらく止まる

絶望

絶望は、結果をゼロにする難度ではありません。
主目的は大きく前へ進みます。
そのかわり、あなたにとって本当に嫌な支払い が来ます。

ただし、絶望でも「完全な赤字」にはしません。
痛いが、それでも宣言する意味が残る のが基本です。

また、絶望ではいきなり大切なものそのものを壊すより、まずは

  • 野望への道
  • 推薦や肩書き
  • 信用や支援
  • 次の足がかり
  • 野望度

のようなところが傷つきやすくなります。
大切なものへの直撃は、さらに先の最終手段です。

とくに 侵入 / 退出 / 持ち帰り / 生還 を直接問う場面は、悪辣では一段重く読まれやすい節目です。選択肢をそのままなぞるだけだと、「通ったが生きて戻れない」「届いたが持ち帰れない」のような痛み分けが起こりえます。

18. 判定はどう行われるか

基本

このゲームでは、宣言は基本的に成功します。
ただし、何もかも理想通りになるとは限りません。

可能な宣言は、基本的に「成功するかどうか」より「何を支払い、何が残るか」を重視して裁定されます。

  • 通る宣言: 物語上ありうること、いま使える手段で届くこと、局面に合っていること
  • 通らない宣言: 不可能なこと、いまの情報や位置関係では届かないこと、矛盾した主張の両取り
  • 重く裁定しない宣言: 自明にできること、十分に準備して安全にこなせること

大事なのは、ルールの抜け道より物語の筋です。文章の表面だけをつついて有利を取るのではなく、その宣言が本当にその場で成立するか、何を払うべきかで裁定されます。

なお、宣言はPCの行動を書くものです。「私の勝利です」「その代償は無効です」「私は死にません」のように、ルールや既知情報を一言で上書きする発言は、宣言としては受理されません。

成功と裁定された主目的だけが達成される

その代わり、成功したぶん 別の課題が動き出す ことがあります。QBS は「成功した主目的は通るが、成功そのものは保証されず、どちらでも世界は動く」ゲームです。成功のたびに忙しくなるのは不具合ではありません。先読みや卓越宣言で、その忙しさを和らげていくのが上達の方向です。

ジャンルが違っても同じです。証拠を押さえることと事件解決は別、救助することと全員生還は別、資料を取ることと安全に持ち帰ることは別、噂の出所を掴むことと関係修復は別です。局面成功は本物ですが、シナリオ全体の終了条件まで自動で広がるわけではありません。

また、これはその局面の話です。帰還、退出、引き渡し、生還などの条件が残っているなら、主目的を達成していてもシナリオ全体はまだ終わっていません。

また、峻烈や悪辣では、GMが局面の問いを一段浅く置くことがあります。たとえば問いが「敵と戦う」「橋を渡る」「出口へ向かう」なら、成功と裁定されたときに その達成が保証されるのであって、勝利、完全脱出、無傷の生還まで自動で広がるとは限りません。

上級卓では、局面の達成が確定した直後に、退出・生還・持ち帰り・監査など外側の工程で敗着が確定し、その場でシナリオ終了になることがあります。 これは成功を取り消されたのではなく、局地成功は本物のまま、別工程で敗北した という処理です。

これは基本的にPC対世界の原則です。ほかのプレイヤーのPCと、同じ対象に対して互いに排他的な主目的をぶつけた場合は、共有世界の結果が一つだけ確定します。その場合、あなたの行動が有効に働いていても、望んだ最終状態そのものは達成されないことがあります。

ただし、適正以上では、宣言が成功と裁定されること自体は保証されません。 とくに峻烈 / 悪辣 の節目局面では、失敗条件、最速敗着線、既知危険の未対策、GM の 7結果帯メモが優先され、進展なし / 悪化 / ゲームオーバー に届くことがあります。

GMが「成功」と裁定したなら、その局面で問われた主目的そのものは本当に達成されます。
「成功したが実質失敗」は、このゲームでは避けるべき結果です。

不可能なことはある

どれだけ自由でも、一度の宣言で

  • 王位を奪う
  • 手がかりゼロで黒幕を断定する
  • 離れた土地の出来事を一気に片づける

のようなことはできません。
ただしGMは、可能なら 今できる規模 を返してくれます。

成功したら、代償や余波がつく

結果には、たいてい何かしらのタグがつきます。
その場で重傷になるとは限らず、

代償タグ は成功のために直接支払った値段、余波タグ はそのあとに広がる世界の反応や二次被害です。

例読み方
橋を焼いて追っ手を止める代償: 橋の焼失 / 余波: 追及の激化
強引に聖剣を引き抜く代償: 重傷 / 余波: 封印の崩落
社内サーバーへ無理な侵入を通す代償: 痕跡を残す / 余波: 監査の強化

あなたが自分から代償を提案しても、それだけで自動的に十分とは限りません。軽すぎる提案なら、より重い代償や追加の余波がつきます。

複数PC卓では、直接の代償は原則として各PCが自分の宣言に応じて支払います。ただし、余波は共有世界の反応なので、他PCの行動があなたへ飛んでくることもあります。たとえば仲間の犯罪が発覚した結果、パーティ全体に疑いが向くことはあります。

  • 時間を失う
  • 誰かに借りを作る
  • 悪評が立つ
  • 立場が揺らぐ
  • 心が重くなる
  • 次の局面に響く疲れが残る

など、さまざまな形がありえます。

GMの返答で見るべきところ

GMの返答では、とくに次の 8つ を見ると遊びやすくなります。

  1. 達成 : 何がかなったか
  2. クリア条件の進み具合 : 今回満たしたものと、まだ残っているもの
  3. 失敗条件との接触 : 今回どこまで危ない線へ近づいたか
  4. 代償タグ と 余波タグ : 何を支払い、何が広がったか
  5. 課題タグ更新 : どの課題タグが増え、重くなり、軽くなり、消えたか
  6. この成功で新たに動いた問題 : 何が片づいた代わりに、何が立ち上がったか
  7. 共有ボトルネックと世界 / NPC反応 : どこが詰まり、誰や何が次に動くか
  8. 次に見えている問題と現在の状態 : 命運点、野望度、有効課題タグ、大切なもの の変化、手札タグ、履歴、次に踏むと危ない線

途中の結果では、今回満たしたクリア条件 と まだ残っているクリア条件 が示されます。結末 や 【シナリオ終了】 は、本当にシナリオが終わるときだけ出ると覚えてください。

複数PC卓では、結果が 共通部分 と 個別部分 に分かれることがあります。共通部分は全員が知っていてよい事実、個別部分はそのPCだけが知っている事実です。次の宣言では、まず共通部分を見て、自分の個別部分を重ねて考えてください。

19. 課題タグ・Build台帳・成長処理

成長点はいつ使うか

成長点を使うのは、原則としてシナリオ終了時の整理です。
GMが成長点・Build 素片・一時手札を確定したあと、プレイヤーが 固定 / 増設 / 昇格 / 再訓練 の希望を申告します。再訓練 は幕間のダウンタイムで扱います。

実戦では、シナリオ中に得たものを、その場で使い切って終わりにしない ことが Build 台帳の肝です。何を恒久化できるかを、終了時に必ず見直してください。

課題タグは期間つきで読む

課題タグには期間が書いてあります。
これで、どれくらい長く効く困りごとかがわかります。課題タグは飾りではなく、次の判断を変える実際の制約です。

  • 短期: すぐ対処できるが、放置すると悪化しやすい
  • 中期: その事件のあいだ効きやすい
  • 長期: シナリオをまたいで残ることがある

課題タグ更新とは何か

GMの返答では、課題タグが次のように動きます。

  • 新規: 新しい課題タグがつく
  • 強化: 既存の課題タグが重くなる、長くなる
  • 軽減: 既存の課題タグが弱くなる、短くなる
  • 解除: 課題タグが消える
  • 残存: そのまま残る

例:

  • 新規: 【疲労｜短期: 次の局面では無理が利かない / 半日休めば軽減】
  • 強化: 【疲労｜短期: ...】 → 【消耗｜中期: 無理を続けると危ない / しっかり休み医者の診断を受ければ軽減】
  • 軽減: 【疑念｜中期: 話を信じてもらいにくい / 証拠を示せば解除】 → 【警戒｜短期: まだ様子見されている / 次の成果で解除】
  • 解除: 【借り｜中期: 頼みを断りにくい / 約束を果たせば解除】 を解除
  • 残存: 【悪評｜長期: 街道筋で名が悪く広まっている / 別の土地で功績を立てれば軽減】 は残る

手札タグと履歴はこう使う

キャンペーンでは、困りごとだけでなく、有利な持ち越しや用途未定の事実も積み重なります。
それらをすべて課題タグとして扱うのではなく、次の3つに分けて考えると宣言しやすくなります。

種類どう使うか書き方
課題タグ放置すると困る。次の宣言で対処や軽減を考える【...】
手札タグ使えば有利になる。あなたが自発的に宣言で使う〔...〕
履歴事実として覚えておく。使い道は宣言の中で提案する一行の事実記録

手札タグに向く例:

  • 〔瞬間移動の巻物｜長期: 行ったことのある場所へ一瞬で行ける / 使用・破損・紛失で失う〕
  • 〔パトロンの口利き｜長期: 一度だけ後ろ盾として名を使える / 恩返し・関係悪化で失う〕
  • 〔救国の名誉｜長期: 第一印象が良く、面会や事情聴取が通りやすい / 大失態・失脚で弱まる〕

履歴に向く例:

  • 村人Aを助けた
  • 村近くの洞窟のゴブリンを退治した
  • 旧水路の抜け道を見つけた

履歴は、使い道を先に固定しないのが大事です。
あとで思いついた使い方を宣言し、GMが妥当だと判断したら、その局面で有効な一時的有利や継続する手札タグとして扱われます。

同じ危険に同じまま踏み込むと悪化する

同じ危険に、対策なしでまた踏み込むと、課題タグは本当に悪化します。名前だけ重くなって影響が変わらない、という処理が基本ではありません。

  • 【軽傷】のまま正面戦闘に行く → 【重傷】になる
  • 【重傷】のままさらに同じ無理を押す → 命運点が減ることがある
  • 命運点が 0 になる → そのPCは退場する

だから、GMの返答でついたタグは、次の宣言に必ず織り込んでください。GMが危険予告を書いてきたら、それは重要情報です。

ふつうの局面の中で回復する

この節でいう「回復」は、主に 課題タグの軽減・解除 のことです。
普通や困難の宣言の中でどう立て直しを織り込むかが大事です。

たとえば、

  • 無理を押す前に半日休む
  • 治療役を手配してから夜に動く
  • 自分で全部抱えず、他人に引き継ぐ
  • 荷を捨てて負担を減らす
  • 説得の前に誤解を解く材料を集める

といった工夫で、主目的を進めながらタグを軽減・解除できます。

回復を織り込む宣言の例

主目的はあくまで「今の問題の解決」に置き、手段や最低限ラインの中で回復のプロセスを挟み込みます。

  • 主目的: 追っ手の足取りを掴む
  • 最低限ライン: 足取りは掴めずとも、これ以上の疲労は防ぐ
  • 手段: まずは半日しっかり休んで体力を戻し、その後に聞き込みに向かう
  • 担保: 【時間経過｜短期: 追っ手との距離が開く】は受ける

このように、「休むこと」自体を主目的にするのではなく、「休んでから行動する」と書くことで、タグを軽くしながら前へ進むことができます。

これは命運点の回復ではありません。 命運点を戻したいなら、「命運点の回復」そのものを主目的とした別の宣言が必要です。

容易はGMが与える息継ぎ

容易の局面は、GMが「ここでいったん立て直しを入れてよい」と判断したときに来ます。
プレイヤー側が自由に選ぶものではありません。

とくに単発の峻烈 / 悪辣では、容易が一度も来ない ことは普通にあります。立て直しは、自分で勝ち取る前提だと考えてください。

シナリオ終了時の処理

シナリオが終わったら、ふつうは次を行います。

  • 短期課題タグを解除する
  • 中期課題タグを整理し、残るなら長期課題タグへ移す
  • 長期課題タグはそのまま残す
  • 新しく得た手札タグを記録する
  • 用途未定の事実は履歴へ追記する
  • GMが適切だと判断すれば、野望度を 1 上げる

また、シナリオが終わるときは、ふつう今回の結末をまとめたエンディング投稿が入ります。次のシナリオへ進む場合も、導入は原則として別投稿です。

ウィスパー・モード を使う卓では、共有主目的を達成したうえで 個人別利益まで通したPCに、追加の +1 候補 がつくことがあります。ただしこれは新しい別枠ではなく、上の 追加で +1（最大2回まで） の中に入ります。共有主目的が失敗した場合、個人別利益だけを通しても成長点ボーナスにはなりません。

シナリオ終了後のチェックリスト

  1. 今回もらった 成長点 と Build 素片 を確認する
  2. 今回得た 一時手札・臨時協力・素材 を確認する
  3. 固定 / 増設 / 昇格 / 再訓練 のどれで残すか決める
  4. 更新後の 長期資産ノード台帳 と 恒久タグ を書き直す

基本は GMが進行し、プレイヤーが申告する 形です。「今回の顔役とのコネを固定したい」「隠れ家をセーフハウスへ育てたい」と、終了時に言葉にして決めるとぶれません。

Build台帳の書き方

実戦では、Build を「装備欄」だけで見ないほうが管理しやすいです。
装備4枠は残したうえで、それ以外の Builds は 長期資産ノード台帳 にまとめます。

絆を参照する / 点火する / 切る

絆タグは、強さに応じて3段階で使い分けると扱いやすいです。

  • 参照する: 根拠として言及するだけです。ふつうの手札タグのように使います。
  • 点火する: 「今はすぐ壊さないが、あとで効かせたい」ときに、ドラマフラグ を作ります。
  • 切る: その絆に大きく頼って、強い成功を取ります。代わりに、次回までに必ず文言が変質します。

たとえば、〔妹への信頼〕 を切って「妹にだけは本音を見せる」ことはできます。
ただしそのあとで、それは 〔妹への負い目〕 や 〔妹を守りすぎる癖〕 に変わるかもしれません。成功と変質をセットで取る のが Stories の肝です。

ドラマフラグの使いどころ

ドラマフラグは、抱え込むよりも 「次の名場面を呼ぶ」 つもりで使うと回しやすいです。代表的な使い道は4つです。

  1. 親密 / 対立シーンを強く起こす
  2. 大切なものや重要NPCを、いまの場面に引き込む
  3. 秘密の開示段階を1つ進める
  4. 感情の中心にある主目的だけを、強く通す

どれに使っても、そのあとに 関係・感情・秘密 のどれかが動くのが基本です。「使ったのに何も変わらない」は避けたほうが、きれいに物語になります。

20. 命運点、野望度、大切なもの

命運点の初期値と増加

命運点の初期最大値は、GMが採用した大切なものの数 + 1 です。

途中で新しい大切なものが正式に増えたなら、最大値や現在値が増えることがあります。

命運点は何を表しているか

命運点は、単なるHPではありません。1つしかない命に残った余力を、何段階に分けて扱うかをまとめて表しています。

  • 曖昧な損耗や、直接タグでは言い切れない削られ方
  • 同種の危険を踏み抜き続けたときの最後のしわ寄せ
  • 「助かったが、もう余裕はない」という最終手段
イメージ
命運点 1 なら、その命をほぼ100%賭けて無理をしている状態です。命運点 2 や 3 なら、その同じ命を 1/2 や 1/3 ずつ削るように危険を受けられます。

課題タグのほうが上位です。危険が具体的に課題タグで示されているなら、まずはその課題タグへの対応が求められます。命運点は、課題タグ処理を置き換える万能保険ではありません。

命運点を失うとき

命運点は、課題タグでは足りないほど直接に命を削るときに使われます。
また、同じ危険に同じまま踏み込み続けて、課題タグの強化だけでは足りなくなったときにも使われます。

  • 増水した川へ飛び込む
  • 呪いに真正面から触れる
  • 重傷のままさらに無理を押す

命運点を減らす裁定は、原則としてGMが本当に限界へ達した代償として提示します。最終手段ですが、必要になったのに避けるものではありません。

ただし、プレイヤー側から「課題タグでは足りないほどの無理を通したい」という強い意思表示として、宣言の「担保」に命運点の減少を提案することもできます。
このとき、命運点の減少を担保として差し出した宣言は、卓越になりやすくはなりますが、無条件卓越ではありません。局面の問いに触れていること、既知情報と矛盾しないこと、論理的に可能な規模であることが前提です。命そのものを削っても、不可能は可能になりません。

重要
命運点を払う提案をしても、致死級タグが消えるわけではありません。
致死級タグのラベルは一本化されており、課題タグ側で死が予告されているなら、まずその課題タグへの対処が必要です。

命運点が0になったら

命運点が 0 になったPCは、死亡するか、冒険の主役としては立ち上がれない状態になります。
単独卓では事実上のゲームオーバーです。複数PC卓では、そのPCは退場し、残りのPCでシナリオが続くこともあります。

これは演出ではなく、実際の処理です。【さらに重傷】のように名前だけが重くなって、普通に行動できる という扱いにはなりません。

死亡許容が高い卓では、致死級タグを放置したままの撤退、退路のない場所からの未脱出、瘴気や崩落など致死環境からの未生還も、そのまま 0 に届く処理になりえます。主目的を達成していても、生還条件が満たせていなければ安心はできません。

致死級タグと生還

このゲームでは、死や退場は命運点だけで決まりません。
致死級タグへの対応と、卓の最初に共有された厳しさも関わります。

通常の卓では、重い危険はまず 軽傷 → 重傷 → 致死級タグ のように課題タグで示されます。ここで大事なのは、ラベル名ではなく、タグ本文の期限句が処理を決めることです。

たとえば 【致死級タグ｜この局面の終わりまでに緊急治療が必要 / 未処置なら退場】 なら、その局面の終わりが締切です。【致死級タグ｜次の局面に進む前に脱出が必要 / 未処置なら死亡】 なら、次へ進む前が締切です。同じラベルでも、本文の期限句が違えば扱いは違います。

さらに、【致死級タグ｜次の局面に進む前に解毒が必要 / 未処置なら死亡】 のように、解除条件を満たさないまま次へ進めば死亡・退場が確定する課題タグが出ることもあります。この場合も、命運点が残っているだけでは助かりません。

致死級タグ は、どの卓でも勝手に出るものではありません。GMは卓の開始時点で、致死級タグが出る卓かどうかを共有します。宣言されていない卓では、いきなり即死級の運用はされません。

また、救出や撃破の 主目的 を達成していても、未処置の致死級タグ、未脱出の崩落現場、酸素切れの隔壁、進行しきった呪いなどが残っているなら、生還条件は別問題として処理されることがあります。

ジャンル例
ファンタジー致死級タグ: 竜の炎で全身に深い火傷を負う、崩落する奈落橋に取り残される、石化の呪いが完成する。
現代致死級タグ: 大量出血、焼傷、圧壊、炎上ビルの倒壊圏に残る、気密室で酸素が尽きる。
SF / ホラー致死級タグ: 致死性寄生体への侵食、隔壁破損による深刻な損傷、宇宙空間へ放り出される、完全同化装置が作動する。

命運点の回復

減少した命運点は、シナリオの終了や単なる休息では自動的に回復しません。
回復するためには、長期の療養や劇的な気力の回復など、「命運点の回復」そのものを主目的とした宣言を行う必要があります。

また、物語の中で何らかの方法で新しく「大切なもの」を得た場合、命運点の最大値が 1 増え、同時に現在の命運点も 1 回復します。

単発の峻烈 / 悪辣では、命運点回復の機会が 0 回でも正常 です。高難度単発では、命運点は切れたら終わる資源として読んでください。

大切なものを失ったとき

大切なものを 永久に失った ときは、命運点の最大値が 1 減り、命運点も 1 減ります（下限 0）。
この減少は、命運点が「最後の余力」であると同時に、大切なものがPCの支えでもあることを表します。

大切なものはすぐには壊れない

大切なものは、PCにとって本当に痛い場所です。
だからこそ、ふつうは低い難度でいきなり永久喪失にはなりません。

まずは

  • そこへ近づきつつある危険
  • 失いかけている気配
  • その意味を揺らす出来事

がタグで示されます。

例:

  • 【弟の疑念｜中期: 弟はあなたを怖がっている / 弟の前で約束を守り切れば解除】
  • 【継承の傷｜長期: 師の継承者としての資格を疑われている / 研究を守り抜けば軽減】

大切なものが多いPCについて

大切なもの が多い PC は、命運点 の面では厚くなることがあります。
ただしそのぶん、物語のなかで自然に触れられる接点 も増えます。GMは、既知情報・タグ・NPC の立場・局面の流れから自然に届く範囲で、それらを揺らしたり危険にさらしたりできます。

大切なものが多ければ多いほど必ず得、ではありません。それは保険であると同時に、守るべき弱点でもあります。

野望度が上がるとき

野望度は、シナリオを終えれば自動で上がるわけではありません。野望そのものへ一歩進み、足場や支持も残った とGMが判断したときに上がります。

目安は次の通りです。

  • +1: クリーンな前進か、高価でも確かな前進をした
  • 0: 痛み分けで終わった、局面は抜けたが野望そのものは進んでいない
  • 0 か -1: 前へ進んだように見えて、推薦、資格、支援、証拠、立場などの足場を失った

とくに、選択肢をそのままなぞっただけで局面を抜けた、生還や持ち帰りが未処理のまま終わった、代償のせいで野望の足場が崩れた ときは、+1 にならないことがあります。

局地成功だけでは、自動で +1 にはなりません。 たとえば資料は取れたが汚染を広げて社会的に終わった、のように外側で大きな敗着が立ったときは、足場が残るなら 0、推薦・資格・信用など野望への足場を失ったなら 0 か -1 です。

つまり、主目的を達成したことと、野望に前進したことは同じではありません。 局地成功の直後に外側の敗着が立ち、その時点でシナリオが終わることもあります。その場合も局地成功は消えませんが、野望への足場が残らなければ +1 ではなく 0 か -1 になります。

野望度が下がるとき

野望度の低下は重い結果です。単に痛い目にあっただけでは、ふつう下がりません。

たとえば、シナリオの目的そのものに失敗したとしても、野望に向けて全力を尽くした結果であれば野望度は下がりません。野望度が下がるのは、次のように自ら野望に背く行動をとったり、野望への足がかりを失ったりしたときに限られます。

  • 野望への足がかりを自分で壊した
  • 推薦、信用、資格、支援を失い、野望への道が目に見えて遠のいた
  • 絶望の局面で、野望に関わる大事なものを差し出した

逆に、局地では勝っていても、野望への足がかりを失えば野望度は下がりえます。 信用、資格、推薦、支援、立場などが崩れ、今後その野望を進めにくくなったなら、失敗条件とは別に重い結果として扱われます。

野望度が0になったら

その野望はいったん折れます。
PCは次のどちらかを選びます。

  • 引退し、その物語を終える
  • 新しい野望を決め、野望度1で再出発する

再出発を選ぶ場合、これまでの「大切なもの」や「命運点の最大値」、残っている「長期課題タグ」は原則としてそのまま引き継ぎます。ただし、新しい野望に向かうにあたり、GMと相談の上で「大切なもの」を1つだけ現在の状況に合うものへ書き換えても構いません。

キャンペーンでは、同じPCでの再出発だけでなく、別PCがその野望線を継ぐ こともあります。新PCは先代の命運点や全部の手札を丸ごともらうのではなく、履歴・関係・残された課題 を数点だけ受け継いで入るのが普通です。

また、再起不能だが生きている PC への愛着が強いなら、その PC は現キャンペーンからはいったん退き、別の野望を立てた別キャンペーン の主人公にして構いません。現キャンペーンの主役枠と混ぜない、という点だけ守れば自然です。詳しい手順は キャンペーン設計ガイド を見てください。

野望度が終点に達したら

そのPCは野望をかなえて、エンディングへ進みます。
終点の目安は 5 ですが、開始時にGMがもっと高くしていることがあります。

21. 卓越宣言

卓越宣言は、頭のいい人しか使えない特別技 ではありません。
このゲームで見えている情報を使って、一手先に備える 宣言なら十分に卓越になりえます。

上級卓では、目の前の局面に対する卓越 と、その後の持ち帰り・生還・後始末まで含めた卓越 は別です。前者だけでも十分に価値があります。局面では完璧でも、外側の問題へ無対策なら、局地はうまくいっても後でしっぺ返しを受けることがあります。
「目の前の局面には勝った」が「そのシナリオに勝った」とは限りません。 退出・生還・持ち帰り・引き渡し・後始末が別工程なら、局地成功の直後に外側の敗着が確定することがあります。その場合も局地成功そのものは取り消されません。ただし、野望への足場が残らなければ野望度は +1 にならず、0 か -1 になることがあります。

卓越宣言になりやすいもの

  • 追っ手が来る前に避難路を確保しておく
  • 説得の前に、証人や証拠を先に押さえる
  • 夜の探索の前に、休息や治療を組み込んでタグを軽くしておく
  • アンデッドらしい兆候を読んで、先に聖水や聖別の段取りを整える
  • 崩れそうな橋だと読んで、先にロープや代替ルートを用意しておく
  • 自分だけで抱えず、適切なNPCへ役割を振る

卓越の本体は、顕在化する前の備えです。 以前の情報、手札タグ、履歴、立場、人脈を使って「起きそうな詰まり」へ先に手を打てたなら、それは十分に卓越の候補になります。

派手でも卓越とは限らないもの

  • ただ危ない場所へ単独で突っ込む
  • 手がかりがないのに、根拠なく一発逆転を狙う
  • 情報も準備もなく、大声で押し切る
  • 危険が見えてから、都合よく専用の脱出アイテムや万能札があったことにする

卓越宣言の考え方

「名案を思いつかなかった」と感じたら、次のどれかを考えるだけで十分です。

  • 先に休めないか
  • 先に証拠を取れないか
  • 先に味方を増やせないか
  • 先に逃げ道を作れないか
  • 先に一番怖い代償を避けられないか

これだけでも、立派に卓越宣言の入口です。

考え方の目安は3つです。設定補完（立場上自然な持ち物や同行者）は通っても卓越そのものではありません。仕込み回収（既出情報や以前の準備を回収する宣言）は卓越の本命です。突破資源（いま見えている失敗条件を専用にひっくり返す新出の切り札）は、通っても卓越になりにくいと覚えてください。

なお、命運点の減少を担保に出すと卓越になりやすくはなりますが、それだけで自動的に卓越や自動成功になるわけではありません。見えている情報を使い、今の局面に触れる書き方はやはり必要です。

リプレイ例: 卓越宣言の雰囲気を実際に見たいなら、聖人サックの慈善活動 を読んでみてください。悪辣GMでも、先回りや段取りで局面の構造を崩していく流れの参考になります。

22. うまく遊ぶためのコツ

主目的は1つに絞る

保証される中心は主目的1つです。
欲張りすぎると、GMも返しづらくなります。

とくに最終局面では、受理 / 生還 / 経歴洗浄 / 追手処理 / 後始末 を全部まとめて主目的にしないでください。どれか一つを主目的にし、残りは 別利益 として書くと、GM も重さをぶらさず裁定できます。

最低限ラインを書く

理想通りでなくても、どこまで達成したいかを書いておくと、PBMで話が止まりにくくなります。

ただし、悪辣や峻烈では、最低限ラインだけを書いた短い直球宣言は守りになりません。手段 と 守りたいもの も添えて、どう切り抜けるつもりかを見せてください。

NPCや組織も動かしてよい

このゲームでは、PC本人の一挙手一投足だけでなく、

  • 誰に声をかけるか
  • 誰へ何を任せるか
  • 町や集団の手順をどう変えるか

まで宣言して構いません。

同じように、GM も NPC や組織や世論や環境を動かします。複数PC卓では、PCだけで世界が閉じず、外部主体が割って入ってくるのが普通です。上級者向けの卓ほど、成功した直後に別の窓口、第三勢力、追手、期限が動いて忙しくなります。

さらに、まだ決まっていない余白 については、世界観と既知情報に矛盾しない範囲 で、プレイヤー側から提案して構いません。これが 文脈補完スロット です。

  • 「この修道院には、夜間見回り用の細い保守通路があるはずだ」
  • 「この港町では、嵐の前日は荷役順が変わる慣習があるのでは」
  • 「この組織なら、非常時だけ使う紙の承認手順が残っていそうだ」

提案してよい のは、まだ未定義の余白 です。すでに否定された事実、GMが秘密として固定した核心、ただちに完璧解へ飛ぶだけの都合のよすぎる追加設定 は、そのまま通るとは限りません。

書き出し例:
「この土地の慣習として〜ではありませんか」
「保守用の導線が残っているなら、それを使いたいです」
「非常時の例外手順がある前提で、そこへ働きかけます」

うまく通る提案は、世界観に合っている、既知情報とぶつからない、何かしらの支払い・段取り・働きかけを伴う の三つを満たしています。つまり、「設定を生やす」こと自体が抜け道ではなく、宣言の材料 になります。

抜け道より、筋の通った支払いを考える

自分から代償を提案するのは良いことです。
ただし、なるべく都合のよい無敵の支払いを作るより、このPCなら何を差し出すか を考えたほうが面白くなります。

課題タグ・手札タグ・履歴を見て次の宣言を考える

今の課題タグは、次に何をすべきかのヒントです。
どの課題タグを軽くしながら前へ出るかを考えると、宣言が作りやすくなります。

同時に、使えそうな手札タグや、あとで意味を持ちそうな履歴も見てください。
「いま何を止めるか」だけでなく、「何を使えるか」「どの事実を持ち出せるか」まで見ると、宣言の幅が広がります。

困ったときの読み方

  • 同じ手段の連打 は、基本的にそのままでは通りません。対策されたり、余波タグが悪化したりします。
  • 宣言の穴 は有利に読まれません。書いていない部分が都合よく埋まるとは限りません。
  • 自分で出した軽い代償 は免罪符ではありません。GMはより自然で重い裁定を選べます。
  • 追加の大切なものや都合のよい切り札 は、物語上の獲得かGM承認が必要です。危険を見てから後出しでは増やせません。
  • 卓越 は、後から完璧な答えを生やすことではなく、前から見えていた詰まりに先に手を打つことです。
  • 主目的が通った直後に別の問題が増えるのは普通です。成功を否定されたのではなく、世界が反応しています。共有ボトルネックと外部主体の動きを見てください。

例1: 雪山の救難行

GMの局面

  • 現況: 峠道の先で隊商が足止めされ、二人の子どもが吹雪の中に取り残されている
  • 期限: 夜明けまで
  • 争点: 子どもの救出、村の避難路、自分の体力
  • 既知情報: 山小屋までは半日、ふだんの道は雪で埋まりかけている
  • 難度: 普通

プレイヤーの宣言

主目的: 夜明けまでに子どもたちを山小屋へ移す
最低限ライン: せめて吹雪の直撃を避けられる場所まで運ぶ
手段: 荷車を解体して簡易そりを作り、荷運び仲間を二人集め、交代で道しるべを立てながら進む
守りたいもの: 弟の安全
捨ててもよいもの: 荷物と体力
担保: 【疲労｜短期: 次の局面では無理が利かない / 半日休めば軽減】 は受ける

ありうる結果

  • 達成: 子どもたちは山小屋まで運び込めた
  • 代償タグ: 【疲労｜短期: 次の局面では無理が利かない / 半日休めば軽減】
  • 余波タグ: なし
  • 課題タグ更新: 新規 1件
  • 状況の変化: 避難路の目印も残り、次に人を動かしやすくなった

例2: 呪われた灯台

GMの局面

  • 現況: 港の外れの灯台に青い火がともり、夜ごと船が礁に引き寄せられている
  • 期限: 次の満潮まで
  • 争点: 港の安全、灯台守の行方、怪異の鎮静
  • 既知情報: 旧記録には、火を消すには海難者の名を呼び供物を返す必要があるとある
  • 難度: 困難

プレイヤー宣言

主目的: 次の満潮までに灯台の怪異を静める
最低限ライン: せめて今夜の船の入港だけは安全にする
手段: 古記録を持って灯台に入り、行方不明の灯台守を探しつつ、海難者の供物を海へ返す儀式を行う
守りたいもの: 港の操業
捨ててもよいもの: 自分の睡眠と評判
担保: 【寝不足｜短期: 次の朝は集中しづらい / 半日眠れば解除】 は受ける

ありうる結果

  • 達成: 今夜の灯は静まり、船は入港できる
  • 代償タグ: 【寝不足｜短期: 次の朝は集中しづらい / 半日眠れば解除】
  • 余波タグ: 【港のざわめき｜中期: 怪異を見た者が噂を広げている / 事情を説明し成果を示せば解除】
  • 課題タグ更新: 新規 2件
  • 状況の変化: 港は救われたが、怪異の原因や灯台守の件はまだ尾を引く

例3: 宮廷の偽書状

GMの局面

  • 現況: 亡き師が反逆者と通じていたように見える書状が見つかり、三日後に審問が開かれる
  • 期限: 三日後の審問まで
  • 争点: 師の名誉、自分の立場、真相解明
  • 既知情報: 書状そのものは本物に見えるが、保管記録に不審な空白がある
  • 難度: 絶望

プレイヤーの宣言

主目的: 審問をいったん延期させ、真相調査の時間を作る
最低限ライン: 師の家族だけは即時拘束から守る
手段: 書庫の記録を洗い、旧知の書記を証人として確保し、審問官の前で保管記録の不備を突く
守りたいもの: 師の名誉
捨ててもよいもの: 自分の昇進の見込み
担保: 【書記長への借り｜中期: 一つ頼みを返すまで逆らいにくい / 頼みを果たせば解除】 は受ける

ありうる結果

  • 達成: 審問は延期され、師の家族もいったん拘束を免れた
  • 代償タグ: 【昇進停止｜中期: 自分の出世は当面止まる / 次の功績を立てれば軽減】
  • 余波タグ: 【書庫の監視｜中期: あなたの動きを複数の派閥が見ている / 真相を一つでも示せば軽減】
  • 課題タグ更新: 新規 2件
  • 状況の変化: 時間は稼いだが、野望への足がかりは傷ついた

旅情 / 人情 / 激情 / 怨情の読み方

劇的展開度は別軸ですが、既定では 初級 / 寛容 = 旅情、 中級 / 適正 = 人情、 上級 / 峻烈 = 激情、 熟練 / 悪辣 = 怨情 に寄せて運用されます。

  • 旅情: 景色、休息、発見、再生、穏やかな達成を取りに行く
  • 人情: 義理、信頼、日常、すれ違いと和解を取りに行く
  • 激情: 告白、対立、犠牲、秘密、反転を取りに行く
  • 怨情: 裏切り、執着、嫉妬、破局、激しい感情の衝突を取りに行く

悪辣だけど旅情 のような卓ももちろん成立します。その場合は「景色や発見は美しいが、生還や手放しの痛みは重い」卓になります。

再訓練は「ビルドの否定」ではなく「今の物語への寄せ直し」

成長点 1 点と妥当なダウンタイムがあれば、恒久タグや装備枠は 再訓練 できます。
おすすめは、完全に別人へ作り変えるより、一語だけ残して寄せ直す やり方です。

例:

  • 〔街の顔役との貸し〕 → 〔街道宿の顔なじみ〕
  • 〔急ごしらえの封印具〕 → 〔携帯用の結界具〕
  • 〔逃げ足の癖〕 → 〔退路管理の型〕

そうすると、過去の履歴と今のビルドが自然につながります。

GMが選択肢例を出したとき

選択肢は、宣言の足がかりです。
ただし、説明が厚いこと と その選択肢が安く通ること は別です。福音は安全策の保証ではなく、あくまで見せ方が厚いという意味です。

  • そのまま1つ選んでよい
  • 少し言い換えてよい
  • 複数を組み合わせてよい
  • 選択肢にない行動を書いてもよい

GMは公開側では、ふつう 正面案 / 仕込み案 / 歪み案 のような形で出します。けれども、その案が安いか高いかは 裁定厳格度 で決まります。悪辣では、その3案に妥当解が混ざらないこともあります。

指針提示度選択肢の見え方
福音案の意図、必要担保、読み取ってほしい伏線、起こりうる代償の気配まで書かれやすいです。
啓示案そのものは具体的に見えますが、結果予想や安全のための注釈はあまり付きません。
寡黙案は短い動詞句か周辺目的までに圧縮され、原則として1手だけが書かれます。成果予告、複数工程、安全のための手段は書かれにくいです。
暗黒選択肢例が出ないことがあります。期限、争点、危険予告などの必須情報は別に示されます。

福音 + 悪辣 なら、筋がよさそうに読める3案が厚い説明つきで並び、しかも全部悪い、ということがあります。逆に 暗黒 + 寛容 なら、選択肢例はなくても、読み取れた宣言は比較的軽く通ることがあります。

また、峻烈 / 悪辣 では、前面に出る選択肢が 結果 ではなく 一手や周辺目的 になっていることがあります。「脱出する」ではなく「出口へ向かう」、「敵に勝つ」ではなく「敵と戦う」 という形です。その場合、選択肢をそのまま選んだだけでは、本当の最終目的までは保証されません。選択肢文に補助手順や結果の気配が書いてあっても、それ全部が保証されるとは限りません。

助かりたいなら、選択肢をそのままなぞるだけでなく、退路 / 守りたいもの / 捨ててもよいもの / 先に確保するもの / 伏線から読んだ危険への対策 を書き足してください。

そのまま選ぶだけでは、ふつう卓越宣言にはなりません。迷ったら、最低でも 主目的 / 手段 / 守りたいもの の3つを書いてください。

悪辣でそのまま選ぶとき

たった一言でも、書き足すとGMが拾える材料が増えます。

  • 先にやる準備
  • 守りたいもの
  • 捨ててもよいもの
  • 退路や証拠の確保
  • 誰を先に止めるか
  • どのタグだけは避けたいか
  • 伏線から読んだ危険への対策

悪辣で生き残りたいなら、次のどれかを1つだけでも足してください。

そのぶん支払いは重くなりやすく、無傷、退路の維持、副目標の同時達成、きれいな後味は期待しないほうが安全です。とくに 侵入 / 退出 / 持ち帰り / 生還 を問う節目では、悪辣は安く通しません。

成功と裁定された場合でも、通るのは その局面で問われた内容 だけです。それが「出口へ向かう」「橋を渡る」「敵と戦う」のような一段浅い問いなら、そこだけが通るのであって、完全脱出や勝利や生還までは別です。選択肢文に「時間を稼いでから」「確実に」「無傷で」のような言葉が含まれていても、書き足しなしではそこまで保証されません。

適正以上では、無工夫宣言に成功保証はありません。 寛容では通りやすくても、適正以上では 進展なし / 悪化 に落ちます。とくに峻烈 / 悪辣 で、既知の危険に未対策のまま節目を踏む と、達成なしのままゲームオーバー まであります。

悪辣でも、GMの選択肢をそのまま選んではいけないわけではありません。
ただし、何も書き足さずに選ぶだけだと、GMはそれを 無工夫宣言 として扱うことがあります。

選択肢をそのまま選ぶだけで足りる卓と、足りない卓

選択肢の説明が厚いこと と そのまま選んで安く通ること は別です。指針提示度 は読みやすさ、裁定厳格度 は支払いの重さだと分けて考えてください。

  • 福音 + 寛容: 案も厚く、値段も軽いので、そのままでも前へ進みやすいです。
  • 福音 + 悪辣: 案は厚いのに、公開された3案はどれも悪い、ということがあります。
  • 暗黒 + 寛容: 案は出ませんが、読み取れた宣言は比較的軽く通ります。

そのため、同じ「選択肢を1つ選ぶ」でも、卓の契約によって意味が変わります。

  • 読み筋が見えていて、請求も軽い卓: まず動き出すための足がかり
  • 読み筋は見えるが、請求は基準以上の卓: 局面の問いだけを通す最低ライン
  • 読み筋の再掲が少なく、請求も重い卓: 高い請求を払ってでも前へ出る意思表示

とくに悪辣では、公開3案のどれかが「実は安全な正解」である保証はありません。 助かりたいなら、第三の案や事前対策を自分で足す前提で考えてください。

また、伏線の置き場所も違います。福音 / 啓示 では、書ける範囲の核心情報が既知情報や危険予告にも出やすく、寡黙 では既知情報まで、暗黒 では小説パートや描写だけに置かれることがあります。ただし、再掲してよい範囲の上限は裁定厳格度と運用強度で決まります。 峻烈では初級相当まで、悪辣では中級相当までが親切に再掲されないことがあります。暗黒では、選択肢だけでなく描写も読み、先回りの材料にしてください。

とくに悪辣では、
「その一手は通る」
と
「その先までは保証されず、しかも高くつく」
は両立します。

  • 現代オカルト: 「現場へ向かう」は、被害現場に着けることの保証であって、一般人退避や証拠保全までの保証ではありません。
  • SF閉鎖環境: 「隔壁の向こうへ行く」は、気密維持や搬送成功までの保証ではありません。
  • 学園もの: 「本人に会う」は、誤解解消や噂停止までの保証ではありません。

  • GMの選択肢から1つ選び、主目的・手段・守りたいものだけ足す
  • GMの選択肢にない第三の案を1つ書く
  • まずは何を止めたいか
  • まずは誰を守りたいか

24. よくある疑問

Q. GMの選択肢が、どれもいやなものしかありません

ありえます。とくに悪辣では、公開された3案が全部 状況悪化か即死級悪化 ということがあります。福音でも例外ではありません。 その3案の中に、安全策や妥当解が置かれていないこともあります。

その場合でも、選択肢にない第三の案 を書いて構いません。
選択肢は例であって、可能な行動の一覧ではありません。

また、悪辣では選択肢が最終結果ではなく一手になっていることがあります。たとえば「出口へ向かう」は「脱出できる」の約束ではありません。描写や既知情報を見て、第三の案や事前対策を書き足す余地を探してください。

いまその場でGMに「本当はどうすればよかったのか」と聞くことはできません。事実確認 はできますが、最適解相談 は却下されます。セッション終了後なら、GMが当時のメモを見せて振り返ってくれることがあります。

Q. 代償が怖いので、何も行動しないほうが得では?

いいえ。何もしないと、シナリオの期限や脅威はそのまま進みます。
「寝る」「様子を見る」も1回の宣言として処理されるので、何もしないことで損だけを避ける のは難しいです。

Q. 代償タグが多くなったら不利すぎませんか?

だからこそ、普通や困難の宣言の中で、うまく休息、治療、引き継ぎ、根回しを組み込むことが大事です。
タグはただ苦しむための印ではなく、次に何をすべきかを教える印 でもあります。

Q. 同じやり方を繰り返せば、いつか通りますか?

基本的には、いいえ。
状況が変わらないまま同じ手段を繰り返しても、効果は増えません。同じ危険に同じまま踏み込めば、むしろタグが悪化し、命運点の減少に進むことがあります。

とくに峻烈や悪辣では、同じ争点に工夫なく直球で付き合ってもらえる回数は少なく、1〜2回で失敗条件に近づくことがあります。

一度通った理屈は、次から世界に学習されることがあります。監視、監査、受取基準のどれかがあなたのやり方を観測し、同じ理屈ほど後半で不利になる ことがあります。

Q. 容易の局面にしてほしいと頼めますか?

単発の峻烈 / 悪辣では、容易が 0 回、命運点回復の機会が 0 回でも普通 です。高難度単発では、休める前提で動くより、休めない前提で先回りしたほうが安全です。

容易は、基本的にGMが与える立て直しの局面です。
ここでいう「普通や困難の宣言の中に回復を組み込む」で軽くできるのは、主に 課題タグ です。命運点の回復は別で、主目的として扱う必要があります。

Q. 悪辣って、GMの言い方がきついことですか?

いいえ。悪辣は 口調 ではなく 裁定厳格度 です。必要な情報は示したまま、無工夫宣言を安く通さず、別利益を高く請求し、隙を見せたときに重い代償を積み上げるのが悪辣です。高難度の単発では、優れた宣言をしていないのにシナリオクリアになる前提ではなく、ゲームオーバーが正常に起こります。

また、悪辣では前面に出された選択肢がそのまま最善手とは限りません。公開3案に救いが混ざるとは限らず、勝ち筋は第三の案や第四案として掘ることがあります。どれを選んでも重い請求が来うるので、助かりたいなら第三の案や退路の確保まで書くのが大切です。

なお、どれだけ詳しく案や注意書きが示されるか は 指針提示度 の話です。公開3案に救いを混ぜるか、悪い道だけを前面に出すか、空欄を善意に読むか不利に読むか は 裁定厳格度 の話です。福音 + 悪辣 なら、厚い説明つきで3案が並んでいても、その3案がどれも悪い、ということがあります。

Q. 大切なものは3つ以上書いてもいいですか?

はい。継続PCや特別な単発では、GM確認のうえで 3つ以上 にしてかまいません。
ただし、明確な上限がないかわりに、命運点 に数えられるかどうかはGMが確認して確定します。重複・抽象・量産は統合や設定扱いになることがあります。

Q. 致死級タグは、どの卓でも出ますか?

いいえ。重い危険そのものは多くの卓にありますが、致死級タグがどの程度出る卓か は、卓の最初に共有されます。
宣言されていない卓では、いきなり即死級の運用になる前提ではありません。

QBS では、この種の重い危険のラベルは 致死級タグ に一本化します。身体損傷でも呪いでも飢餓でも、本文の期限句と解除条件を読んでください。

Q. 命運点ばかり減らされませんか?

命運点は最終手段です。
ふつうは先に課題タグで危険が示されます。

ただし、同じ危険に同じまま踏み込んで、課題タグの悪化だけでは足りなくなったときは命運点が減ります。だから、GMの返答でついた課題タグや危険予告は、次の宣言に必ず反映してください。

また、致死級タグが出る卓かどうか は卓の最初に共有されます。危険予告や生還条件の説明が出ているなら、そこまで含めて次の宣言を考えてください。

【次のターンで死ぬ】 のような課題タグが出ているなら、命運点が残っていても、その課題タグを解除しないまま次へ進めば死にます。
このゲームでは、課題タグ処理が主で、命運点は補助です。

死亡許容が高い卓では、致死環境から生きて戻る手順まで書いていないと、主目的を達成していても命運点処理や退場処理へ進むことがあります。

Q. 自分の提案した代償に、さらに余波がつくのは不公平では?

不公平ではありません。
あなたの提案した代償は、成功の値段 として尊重されます。
ただし、その値段を払った結果、世界がどう反応するかまでは別です。
それが余波です。

Q. 卓越宣言は、頭のいい人しか使えませんか?

いいえ。
難しい名案より、一手先の備え ができていれば十分です。

  • 先に休む
  • 先に証人を連れていく
  • 先に逃げ道を作る
  • 先に手当て役を確保する
  • 先に誤解をほどく材料を集める
  • 伏線を見て、先に対策を用意する

こうした宣言でも、十分に卓越になりえます。

なお、命運点を担保に差し出せば自動的に卓越になるわけではありません。準備や見通しがあるほど、卓越として扱われやすくなります。逆に、危険が見えてから「実は完璧な脱出アイテムがあった」とするだけでは、卓越にはなりにくいと考えてください。

Q. NPCや町や組織を動かす宣言をしてもいいですか?

はい。むしろこのゲームでは、そこが強みになりやすいです。
PC本人が何をするかだけでなく、誰に何を任せ、どんな手順を作るか まで宣言して構いません。

また、慣習、保守用通路、例外手順、地元で当然の顔つなぎ のような まだ未定義の余白 も、矛盾しない範囲なら提案してかまいません。これが通るときは、GM がその場で「その前提ならこういう支払い・こういう制約が付く」と返してくれます。都合のよい万能設定を後出しで確定する権利 ではなく、世界の余白を使って勝ち筋を作る権利 だと考えると使いやすいです。

Q. 大切なものを書いたら、そこばかり狙われませんか?

大切なものは「ここが本当に痛い」という目印ですが、低い難度で即座に壊される前提ではありません。
まずは、その周辺が揺らぐタグとして現れるのが基本です。

また、大切なもの が多いほど、純粋に得になるわけでもありません。守れる幅は広がりますが、そのぶん物語のなかで自然に触れられる接点も増えます。

ただし、それは恣意的に嫌がらせしてよいという意味ではありません。GMは、既知情報、タグ、NPCの立場、局面の流れから自然に届く範囲でしかそこへ触れられません。

Q. キャンペーンでPCが再起不能になったら、終わりですか?

単発なら、そのまま終わることがあります。キャンペーンでは、そのPCの物語が現キャンペーンで一区切りになり、別PCが野望線を継ぐ 形が基本です。

大切なのは、敗北を値引きしてだらだら延命しない ことです。再起不能や大局的敗北が確定したなら、継承に使う履歴や縁を整理して、新しい主人公で次へ入るほうが QBS らしく進みます。

どうしても生きている元PCに愛着があるなら、その人物は現キャンペーンでは離脱 / 引退として残し、後日、別の野望を立てた別キャンペーンの主役にできます。現行キャンペーンでは、支援者や留守番として距離を取って扱います。

Q. Buildとは何ですか? ジョブやスキルと何が違いますか?

A. Build は、自由宣言に 許可・値引き・保険・変換 を与える 長期資産 です。
固定コマンドの一覧ではなく、「この行動しかできない」ではなく「この行動を安く安全に通しやすい」を作るものだと考えてください。

Q. ビルドが増えると、自由宣言が減りませんか?

A. 減りません。QBS のビルドは、「この行動しかできない」ではなく、どの宣言をどんな値段で通せるか を変えるものです。自由宣言の射程を広げる方向へ働かせてください。

Q. 絆を使うと、関係が壊れるだけになりませんか?

A. いいえ。壊れることもあれば、深まることも、ねじれることもあります。大事なのは、何も変わらないままの強い成功 を続けないことです。

Q. 成長点はいつ使えますか?

A. 原則として シナリオ終了時の成長処理 で使います。GMが成長点と Build 素片を確定したあと、プレイヤーが「固定したい」「増設したい」「昇格したい」と申告します。再訓練 は幕間で行います。

Q. 成長点は、戦闘や大事件だけでしかもらえませんか?

A. いいえ。QBS では、調査成功、工作、世界課題の解決、お使い、秘密開示、和解や告白 でも増えます。地道に進めて自分だけのビルドを作る遊びを、正式に支えるためです。

Q. リラックス・モードだと、野望が進まないのでは?

A. 進み方が細くなるだけで、止まるわけではありません。普段は成長点、長期資産、関係、秘密で積み、節目の回だけ野望度を動かす と読むと安定します。むしろ、毎回無理に野望度を動かさないぶん、節目の重みがはっきりします。

Q. リラックス・モードから、普通のキャンペーンに戻せますか?

A. 戻せます。大きな敵や価値選択が見えてきた節目で、GM に「ここからは終点へ向かいたい」と伝えれば十分です。モードはキャンペーンの終点を消すものではなく、急がない ためのものです。

25. 最後に

QBS では、宣言は基本的に実ります。
だからこそ、そのたびに 何を支払うか が物語になります。

うまく遊ぶコツは、完璧を目指しすぎないことです。
どこまで進めば十分か、何なら差し出せるか、どのタグを軽くしながら前へ出るか を考えると、このゲームはぐっと遊びやすくなります。

基本を確認したいときは プレイヤー基本ルール へ戻り、GMとして既存シナリオを回す準備や裁定まで知りたいときは GMガイド、自分で単発シナリオを設計したいときは シナリオ設計ガイド、長いキャンペーンを設計したいときは キャンペーン設計ガイド へ進んでください。
